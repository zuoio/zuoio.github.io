1.1 MyBatis 是什么 —— ORM 框架定位与设计哲学

MyBatis 是一款优秀的半自动 ORM（Object-Relational Mapping）框架，它将 Java 对象与 SQL 语句之间建立映射关系，让开发者在保留对 SQL 完全控制权的同时，免去 JDBC 中大量重复的样板代码。

核心定位

  • 不是全自动 ORM：MyBatis 不会自动生成 SQL，你需要自己编写每一条 SQL 语句
  • SQL 与 Java 代码解耦：SQL 写在 XML 或注解中，与业务逻辑分离
  • 灵活的结果映射：支持简单类型到复杂嵌套对象的各种映射场景

设计哲学

MyBatis 的设计哲学可以概括为一句话：SQL 是开发者最好的朋友，框架不应该试图隐藏它，而应该让它更容易管理。

对比原生 JDBC 代码，可以直观感受 MyBatis 解决了什么问题：

// 原生 JDBC —— 充斥大量样板代码
public User findById(Long id) {
    Connection conn = null;
    PreparedStatement ps = null;
    ResultSet rs = null;
    try {
        conn = dataSource.getConnection();
        ps = conn.prepareStatement("SELECT id, username, email FROM user WHERE id = ?");
        ps.setLong(1, id);
        rs = ps.executeQuery();
        if (rs.next()) {
            User user = new User();
            user.setId(rs.getLong("id"));
            user.setUsername(rs.getString("username"));
            user.setEmail(rs.getString("email"));
            return user;
        }
        return null;
    } catch (SQLException e) {
        throw new RuntimeException(e);
    } finally {
        // 还需要手动关闭 rs、ps、conn...
    }
}

// MyBatis —— 只需关注 SQL 本身
public interface UserMapper {
    @Select("SELECT id, username, email FROM user WHERE id = #{id}")
    User findById(Long id);
}

MyBatis 帮你做了什么？ 连接管理、参数设置、结果集到对象的映射、异常转换 —— 这些 JDBC 中最繁琐的部分全部由框架接管。

1.2 MyBatis vs Hibernate vs JPA —— 选型对比

对比维度 MyBatis Hibernate Spring Data JPA
ORM 程度 半自动（手写 SQL） 全自动（自动生成 SQL） 全自动（基于 Hibernate）
学习曲线 低（会 SQL 就能上手） 高（HQL、Session 状态、级联策略） 中（约定优于配置，但底层是 Hibernate）
SQL 控制力 完全控制 有限（可用 Native Query） 有限（可用 @Query）
复杂查询 擅长（动态 SQL 强大） 较弱（复杂查询需退化为原生 SQL） 较弱（同 Hibernate）
数据库移植性 低（SQL 方言耦合） 高（HQL 屏蔽方言差异） 高（JPQL 屏蔽方言差异）
缓存 一级 + 二级缓存 一级 + 二级 + 查询缓存 继承 Hibernate 缓存体系
代码量 中等（需写 SQL + 映射） 少（单表 CRUD 零 SQL） 最少（接口方法命名即查询）
国内使用率 极高（互联网公司主流） 较低 中等（外企、传统企业较多）
典型场景 复杂业务查询、报表、DBA 强管控 领域模型驱动、快速原型 Spring 生态全家桶项目

选型建议：国内互联网项目绝大多数使用 MyBatis（或 MyBatis-Plus），原因很现实 —— 业务 SQL 复杂度高、DBA 需要审核 SQL、团队对 SQL 掌控力要求强。如果你的项目是简单 CRUD 且追求开发效率，JPA 也是不错的选择。

1.3 核心架构总览 —— SqlSessionFactory、SqlSession、Mapper 三层关系

MyBatis 的核心 API 由三个关键角色构成，它们的关系是一个从全局到会话到操作的层级结构：

SqlSessionFactory（工厂 —— 全局唯一）

  • 职责：持有全局配置信息（Configuration 对象），负责创建 SqlSession 实例
  • 生命周期：应用级单例，随应用启动而创建，随应用关闭而销毁
  • 类比：相当于数据库连接池的管理者
// 通过配置文件构建 SqlSessionFactory（整个应用只需执行一次）
String resource = "mybatis-config.xml";
InputStream inputStream = Resources.getResourceAsStream(resource);
SqlSessionFactory sqlSessionFactory = new SqlSessionFactoryBuilder().build(inputStream);

SqlSession（会话 —— 每次请求创建）

  • 职责：执行 SQL 命令、获取 Mapper 实例、管理事务
  • 生命周期：方法级或请求级，用完必须关闭（类似 JDBC 的 Connection）
  • 线程安全性：不是线程安全的，不能作为类的成员变量共享
// 每次数据库操作创建一个 SqlSession
try (SqlSession session = sqlSessionFactory.openSession()) {
    UserMapper mapper = session.getMapper(UserMapper.class);
    User user = mapper.findById(1L);
    // ...
} // try-with-resources 自动关闭

Mapper（映射器 —— 绑定到 SqlSession）

  • 职责：定义数据库操作的接口，每个方法对应一条 SQL 语句
  • 生命周期：从属于 SqlSession，随 SqlSession 关闭而失效
  • 本质：Mapper 接口没有实现类，MyBatis 通过 JDK 动态代理 在运行时生成实现
public interface UserMapper {
    User findById(Long id);
    List<User> findAll();
    int insert(User user);
    int update(User user);
    int deleteById(Long id);
}

三者关系总结

SqlSessionFactory（全局单例）
    │
    │  openSession()
    ▼
SqlSession（一次会话，非线程安全）
    │
    │  getMapper(UserMapper.class)
    ▼
UserMapper（代理对象，绑定到当前 SqlSession）
    │
    │  findById(1L) → 查找对应 SQL → 执行并映射结果
    ▼
数据库

记住这个职责链：SqlSessionFactory 管配置和创建会话，SqlSession 管事务和获取 Mapper，Mapper 管具体的数据库操作。三者各司其职，层层委托。

1.4 环境搭建

1.4.1 Maven 依赖配置

一个纯 MyBatis 项目只需要两个核心依赖：MyBatis 本身 + 数据库驱动。

<dependencies>
    <!-- MyBatis 核心 -->
    <dependency>
        <groupId>org.mybatis</groupId>
        <artifactId>mybatis</artifactId>
        <version>3.5.16</version>
    </dependency>

    <!-- MySQL 驱动（根据你的数据库选择对应驱动） -->
    <dependency>
        <groupId>com.mysql</groupId>
        <artifactId>mysql-connector-j</artifactId>
        <version>8.3.0</version>
    </dependency>

    <!-- 用于运行测试 -->
    <dependency>
        <groupId>junit</groupId>
        <artifactId>junit</artifactId>
        <version>4.13.2</version>
        <scope>test</scope>
    </dependency>
</dependencies>

提示：如果不想安装 MySQL，可以使用 H2 内存数据库进行学习，将驱动替换为 com.h2database:h2，连接 URL 改为 jdbc:h2:mem:test 即可。

2. 实体类 User.java

package com.example.entity;

public class User {
    private Long id;
    private String username;
    private String email;
    private java.time.LocalDateTime createTime;

    // Getter & Setter
    public Long getId() { return id; }
    public void setId(Long id) { this.id = id; }

    public String getUsername() { return username; }
    public void setUsername(String username) { this.username = username; }

    public String getEmail() { return email; }
    public void setEmail(String email) { this.email = email; }

    public java.time.LocalDateTime getCreateTime() { return createTime; }
    public void setCreateTime(java.time.LocalDateTime createTime) { this.createTime = createTime; }

    @Override
    public String toString() {
        return "User{id=" + id + ", username='" + username + "', email='" + email + "', createTime=" + createTime + "}";
    }
}

4. Mapper XML 映射文件 UserMapper.xml

<?xml version="1.0" encoding="UTF-8" ?>
<!DOCTYPE mapper
    PUBLIC "-//mybatis.org//DTD Mapper 3.0//EN"
    "http://mybatis.org/dtd/mybatis-3-mapper.dtd">

<mapper namespace="com.example.mapper.UserMapper">

    <select id="findById" parameterType="long" resultType="com.example.entity.User">
        SELECT id, username, email, create_time AS createTime
        FROM user
        WHERE id = #{id}
    </select>

    <select id="findAll" resultType="com.example.entity.User">
        SELECT id, username, email, create_time AS createTime
        FROM user
    </select>

    <insert id="insert" parameterType="com.example.entity.User" useGeneratedKeys="true" keyProperty="id">
        INSERT INTO user (username, email)
        VALUES (#{username}, #{email})
    </insert>

    <update id="update" parameterType="com.example.entity.User">
        UPDATE user
        SET username = #{username}, email = #{email}
        WHERE id = #{id}
    </update>

    <delete id="deleteById" parameterType="long">
        DELETE FROM user WHERE id = #{id}
    </delete>

</mapper>

5. MyBatis 全局配置文件 mybatis-config.xml

<?xml version="1.0" encoding="UTF-8" ?>
<!DOCTYPE configuration
    PUBLIC "-//mybatis.org//DTD Config 3.0//EN"
    "http://mybatis.org/dtd/mybatis-3-config.dtd">

<configuration>

    <settings>
        <!-- 开启驼峰命名自动映射，如 create_time -> createTime -->
        <setting name="mapUnderscoreToCamelCase" value="true"/>
        <!-- 控制台打印 SQL 日志（开发阶段使用） -->
        <setting name="logImpl" value="STDOUT_LOGGING"/>
    </settings>

    <environments default="development">
        <environment id="development">
            <transactionManager type="JDBC"/>
            <dataSource type="POOLED">
                <property name="driver" value="com.mysql.cj.jdbc.Driver"/>
                <property name="url" value="jdbc:mysql://localhost:3306/mybatis_demo?useSSL=false&amp;serverTimezone=Asia/Shanghai&amp;characterEncoding=utf-8"/>
                <property name="username" value="root"/>
                <property name="password" value="123456"/>
            </dataSource>
        </environment>
    </environments>

    <mappers>
        <mapper resource="mapper/UserMapper.xml"/>
    </mappers>

</configuration>

注意：mybatis-config.xml 中子标签的顺序是固定的，必须按照 properties → settings → typeAliases → typeHandlers → objectFactory → objectWrapperFactory → reflectorFactory → plugins → environments → databaseIdProvider → mappers 的顺序排列，否则会报 XML 校验错误。

6. 测试类 MyBatisTest.java

package com.example;

import com.example.entity.User;
import com.example.mapper.UserMapper;
import org.apache.ibatis.io.Resources;
import org.apache.ibatis.session.SqlSession;
import org.apache.ibatis.session.SqlSessionFactory;
import org.apache.ibatis.session.SqlSessionFactoryBuilder;
import org.junit.BeforeClass;
import org.junit.Test;

import java.io.InputStream;
import java.util.List;

public class MyBatisTest {

    private static SqlSessionFactory sqlSessionFactory;

    @BeforeClass
    public static void init() throws Exception {
        String resource = "mybatis-config.xml";
        InputStream inputStream = Resources.getResourceAsStream(resource);
        sqlSessionFactory = new SqlSessionFactoryBuilder().build(inputStream);
    }

    @Test
    public void testFindById() {
        try (SqlSession session = sqlSessionFactory.openSession()) {
            UserMapper mapper = session.getMapper(UserMapper.class);
            User user = mapper.findById(1L);
            System.out.println(user);
        }
    }

    @Test
    public void testFindAll() {
        try (SqlSession session = sqlSessionFactory.openSession()) {
            UserMapper mapper = session.getMapper(UserMapper.class);
            List<User> users = mapper.findAll();
            users.forEach(System.out::println);
        }
    }

    @Test
    public void testInsert() {
        try (SqlSession session = sqlSessionFactory.openSession()) {
            UserMapper mapper = session.getMapper(UserMapper.class);
            User user = new User();
            user.setUsername("王五");
            user.setEmail("wangwu@example.com");
            int rows = mapper.insert(user);
            session.commit(); // 手动提交事务
            System.out.println("插入 " + rows + " 行，生成的主键：" + user.getId());
        }
    }

    @Test
    public void testUpdate() {
        try (SqlSession session = sqlSessionFactory.openSession()) {
            UserMapper mapper = session.getMapper(UserMapper.class);
            User user = mapper.findById(1L);
            user.setEmail("zhangsan_new@example.com");
            int rows = mapper.update(user);
            session.commit();
            System.out.println("更新 " + rows + " 行");
        }
    }

    @Test
    public void testDelete() {
        try (SqlSession session = sqlSessionFactory.openSession()) {
            UserMapper mapper = session.getMapper(UserMapper.class);
            int rows = mapper.deleteById(3L);
            session.commit();
            System.out.println("删除 " + rows + " 行");
        }
    }
}

关键点：纯 MyBatis 环境下，默认不会自动提交事务。执行增删改操作后必须手动调用 session.commit()，否则数据不会持久化。如果需要自动提交，可以通过 sqlSessionFactory.openSession(true) 开启。

1.5 MyBatis 执行流程概览 —— 一条 SQL 从 Mapper 接口到数据库的完整链路

当你调用 mapper.findById(1L) 时，MyBatis 内部经历了以下完整链路：

执行步骤

  1. 调用 Mapper 接口方法

    • 调用 userMapper.findById(1L)，但 UserMapper 是一个接口，没有实现类

  2. 动态代理拦截

    • MyBatis 在启动时为每个 Mapper 接口生成了 JDK 动态代理对象（MapperProxy）
    • 代理对象拦截方法调用，根据接口全限定名 + 方法名定位到对应的 SQL 语句
    • 即：com.example.mapper.UserMapper.findById → 匹配 XML 中 namespace + id

  3. 构建 MappedStatement

    • MyBatis 从全局 Configuration 对象中找到对应的 MappedStatement
    • MappedStatement 包含了 SQL 文本、参数类型、返回类型、缓存配置等所有元信息

  4. Executor 执行

    • SqlSession 将执行委托给 Executor（默认为 SimpleExecutor）
    • Executor 先检查二级缓存（如果开启），再检查一级缓存
    • 缓存未命中则继续向下执行

  5. StatementHandler 创建 Statement

    • StatementHandler 根据 SQL 类型创建 JDBC 的 PreparedStatement
    • 将 SQL 模板中的 #{id} 替换为 ? 占位符

  6. ParameterHandler 设置参数

    • ParameterHandler 通过 TypeHandler 将 Java 参数值设置到 PreparedStatement 的对应占位符
    • 例：ps.setLong(1, 1L)

  7. 执行 SQL

    • 调用 JDBC PreparedStatement.execute() 向数据库发送 SQL 并获取 ResultSet

  8. ResultSetHandler 映射结果

    • ResultSetHandler 遍历 ResultSet，根据 resultType 或 resultMap 的配置
    • 通过 TypeHandler 将数据库字段值逐个转换为 Java 类型
    • 构造并填充目标 Java 对象（如 User）

  9. 返回结果

    • 映射后的结果对象逐层返回：ResultSetHandler → Executor（写入一级缓存）→ SqlSession → MapperProxy → 调用方

流程简图

mapper.findById(1L)
    │
    ▼
MapperProxy（动态代理） ─── 根据 namespace + id 定位 SQL
    │
    ▼
SqlSession → Executor ─── 查缓存（一级/二级）
    │                          │
    │ 缓存未命中                │ 缓存命中 → 直接返回
    ▼                          
StatementHandler ─── 创建 PreparedStatement，设置 SQL
    │
    ▼
ParameterHandler ─── 通过 TypeHandler 设置参数值
    │
    ▼
JDBC 执行 ─── 发送 SQL 到数据库，获取 ResultSet
    │
    ▼
ResultSetHandler ─── 通过 TypeHandler 将结果映射为 Java 对象
    │
    ▼
返回 User 对象（同时写入一级缓存）

面试高频点：MyBatis 的四大核心组件 —— Executor、StatementHandler、ParameterHandler、ResultSetHandler，它们也是插件（拦截器）可以介入的四个扩展点，详见第八章。
