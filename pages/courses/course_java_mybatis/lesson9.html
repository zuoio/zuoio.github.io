第九章 TypeHandler 与类型体系 📖

处理特殊类型映射的利器。MyBatis 在设置 PreparedStatement 参数和从 ResultSet 取值时，都依赖 TypeHandler 完成 Java 类型与 JDBC 类型之间的转换。掌握 TypeHandler 机制，能让你轻松应对枚举、JSON、数组等非标准类型映射场景。

9.1 内置 TypeHandler 一览

MyBatis 内置了大量 TypeHandler，覆盖了绝大多数 Java 基本类型和常用类型。以下是常用的内置 TypeHandler 汇总：

Java 类型 JDBC 类型 TypeHandler

byte / Byte TINYINT ByteTypeHandler
short / Short SMALLINT ShortTypeHandler
int / Integer INTEGER IntegerTypeHandler
long / Long BIGINT LongTypeHandler
float / Float FLOAT FloatTypeHandler
double / Double DOUBLE DoubleTypeHandler
String VARCHAR StringTypeHandler
String CHAR StringTypeHandler
String CLOB ClobTypeHandler
String NVARCHAR NStringTypeHandler
String NCHAR NStringTypeHandler
String NCLOB NClobTypeHandler
byte[] BLOB BlobTypeHandler
byte[] LONGVARBINARY BlobInputStreamTypeHandler
BigDecimal NUMERIC / DECIMAL BigDecimalTypeHandler
BigInteger BIGINT BigIntegerTypeHandler
Date (java.util) TIMESTAMP DateTypeHandler
Date (java.util) DATE DateOnlyTypeHandler
Date (java.util) TIME TimeOnlyTypeHandler
Date (java.sql) DATE SqlDateTypeHandler
Time (java.sql) TIME SqlTimeTypeHandler
Timestamp (java.sql) TIMESTAMP SqlTimestampTypeHandler
LocalDate DATE LocalDateTypeHandler
LocalTime TIME LocalTimeTypeHandler
LocalDateTime TIMESTAMP LocalDateTimeTypeHandler
Instant TIMESTAMP InstantTypeHandler
OffsetDateTime TIMESTAMP_WITH_TIMEZONE OffsetDateTimeTypeHandler

说明： MyBatis 3.4+ 对 Java 8 日期时间 API（java.time.*）提供了开箱即用的支持，无需额外配置。

MyBatis 在启动时会将这些 TypeHandler 注册到 TypeHandlerRegistry 中。当你的 Java 属性类型与数据库列类型匹配上述任一组合时，MyBatis 会自动选择对应的 TypeHandler，无需手动指定。

9.2 自定义 TypeHandler 实战

当内置 TypeHandler 无法满足需求时（如枚举映射策略、JSON 字段、数据库数组类型），就需要编写自定义 TypeHandler。

自定义 TypeHandler 需要实现 org.apache.ibatis.type.TypeHandler<T> 接口，或继承更方便的 BaseTypeHandler<T> 抽象类，重写以下四个方法：

public abstract class BaseTypeHandler<T> implements TypeHandler<T> {
    // 设置参数：Java → JDBC
    public abstract void setNonNullParameter(PreparedStatement ps, int i, T parameter, JdbcType jdbcType) throws SQLException;

    // 获取结果：JDBC → Java（按列名）
    public abstract T getNullableResult(ResultSet rs, String columnName) throws SQLException;

    // 获取结果：JDBC → Java（按列索引）
    public abstract T getNullableResult(ResultSet rs, int columnIndex) throws SQLException;

    // 获取结果：JDBC → Java（存储过程）
    public abstract T getNullableResult(CallableStatement cs, int columnIndex) throws SQLException;
}

9.2.1 枚举类型处理 —— EnumTypeHandler vs EnumOrdinalTypeHandler

MyBatis 对枚举类型提供了两种内置 TypeHandler：

特性 EnumTypeHandler（默认） EnumOrdinalTypeHandler
存储内容 枚举的 name() 值（字符串） 枚举的 ordinal() 值（序号）
数据库列类型 VARCHAR INTEGER
可读性 高，数据库中直接可读 低，需要对照枚举定义
安全性 重命名枚举会导致数据不一致 调整枚举顺序会导致数据不一致
适用场景 枚举值稳定且数量较少 不推荐在生产环境使用

示例枚举：

public enum OrderStatus {
    PENDING,    // ordinal = 0
    PAID,       // ordinal = 1
    SHIPPED,    // ordinal = 2
    COMPLETED,  // ordinal = 3
    CANCELLED;  // ordinal = 4
}

使用 EnumTypeHandler 时，数据库中存储的是 "PENDING"、"PAID" 等字符串；使用 EnumOrdinalTypeHandler 时，存储的是 0、1 等整数。

两者的问题： 在实际项目中，枚举通常带有一个业务编码（如 status = 10 表示待支付），既不等于枚举名称也不等于声明顺序。此时就需要自定义 TypeHandler。

自定义枚举 TypeHandler —— 按业务编码存储

首先定义一个通用接口，让所有需要持久化的枚举实现它：

/**
 * 所有需要自定义数据库存储值的枚举实现此接口
 */
public interface BaseEnum<T> {
    T getCode();
}

改造枚举类：

public enum OrderStatus implements BaseEnum<Integer> {
    PENDING(10, "待支付"),
    PAID(20, "已支付"),
    SHIPPED(30, "已发货"),
    COMPLETED(40, "已完成"),
    CANCELLED(50, "已取消");

    private final int code;
    private final String desc;

    OrderStatus(int code, String desc) {
        this.code = code;
        this.desc = desc;
    }

    @Override
    public Integer getCode() {
        return code;
    }

    public String getDesc() {
        return desc;
    }
}

编写通用的枚举 TypeHandler：

import org.apache.ibatis.type.BaseTypeHandler;
import org.apache.ibatis.type.JdbcType;
import org.apache.ibatis.type.MappedJdbcTypes;
import org.apache.ibatis.type.MappedTypes;

import java.sql.*;
import java.util.Arrays;
import java.util.Map;
import java.util.function.Function;
import java.util.stream.Collectors;

public class BaseEnumTypeHandler<E extends Enum<E> & BaseEnum<Integer>> extends BaseTypeHandler<E> {

    private final Map<Integer, E> enumMap;

    public BaseEnumTypeHandler(Class<E> type) {
        E[] enums = type.getEnumConstants();
        if (enums == null) {
            throw new IllegalArgumentException(type.getSimpleName() + " 不是枚举类型");
        }
        this.enumMap = Arrays.stream(enums)
                .collect(Collectors.toMap(BaseEnum::getCode, Function.identity()));
    }

    @Override
    public void setNonNullParameter(PreparedStatement ps, int i, E parameter, JdbcType jdbcType) throws SQLException {
        ps.setInt(i, parameter.getCode());
    }

    @Override
    public E getNullableResult(ResultSet rs, String columnName) throws SQLException {
        int code = rs.getInt(columnName);
        return rs.wasNull() ? null : enumMap.get(code);
    }

    @Override
    public E getNullableResult(ResultSet rs, int columnIndex) throws SQLException {
        int code = rs.getInt(columnIndex);
        return rs.wasNull() ? null : enumMap.get(code);
    }

    @Override
    public E getNullableResult(CallableStatement cs, int columnIndex) throws SQLException {
        int code = cs.getInt(columnIndex);
        return cs.wasNull() ? null : enumMap.get(code);
    }
}

要点： 构造器接收枚举 Class，启动时将所有枚举常量按 code 索引到 Map 中，查询时直接通过 code 反查枚举实例，时间复杂度 O(1)。

9.2.2 JSON 字段 ↔ Java 对象

许多项目会在数据库中用 VARCHAR / TEXT / JSON 列存储 JSON 字符串，而在 Java 端希望直接映射为对象。以下是基于 Jackson 的通用 JSON TypeHandler 实现。

实体类

@Data
public class User {
    private Long id;
    private String name;
    private UserProfile profile; // 数据库中以 JSON 字符串存储
}

@Data
public class UserProfile {
    private String avatar;
    private String bio;
    private List<String> tags;
}

Jackson 版本

import com.fasterxml.jackson.core.JsonProcessingException;
import com.fasterxml.jackson.databind.ObjectMapper;
import org.apache.ibatis.type.BaseTypeHandler;
import org.apache.ibatis.type.JdbcType;
import org.apache.ibatis.type.MappedJdbcTypes;
import org.apache.ibatis.type.MappedTypes;

import java.sql.*;

@MappedJdbcTypes(JdbcType.VARCHAR)
public class JacksonTypeHandler<T> extends BaseTypeHandler<T> {

    private static final ObjectMapper OBJECT_MAPPER = new ObjectMapper();
    private final Class<T> clazz;

    public JacksonTypeHandler(Class<T> clazz) {
        if (clazz == null) {
            throw new IllegalArgumentException("Type argument cannot be null");
        }
        this.clazz = clazz;
    }

    @Override
    public void setNonNullParameter(PreparedStatement ps, int i, T parameter, JdbcType jdbcType) throws SQLException {
        try {
            ps.setString(i, OBJECT_MAPPER.writeValueAsString(parameter));
        } catch (JsonProcessingException e) {
            throw new SQLException("Error converting object to JSON string", e);
        }
    }

    @Override
    public T getNullableResult(ResultSet rs, String columnName) throws SQLException {
        return parseJson(rs.getString(columnName));
    }

    @Override
    public T getNullableResult(ResultSet rs, int columnIndex) throws SQLException {
        return parseJson(rs.getString(columnIndex));
    }

    @Override
    public T getNullableResult(CallableStatement cs, int columnIndex) throws SQLException {
        return parseJson(cs.getString(columnIndex));
    }

    private T parseJson(String json) throws SQLException {
        if (json == null || json.isEmpty()) {
            return null;
        }
        try {
            return OBJECT_MAPPER.readValue(json, clazz);
        } catch (JsonProcessingException e) {
            throw new SQLException("Error parsing JSON string to " + clazz.getSimpleName(), e);
        }
    }
}

Gson 版本

import com.google.gson.Gson;
import org.apache.ibatis.type.BaseTypeHandler;
import org.apache.ibatis.type.JdbcType;

import java.sql.*;

public class GsonTypeHandler<T> extends BaseTypeHandler<T> {

    private static final Gson GSON = new Gson();
    private final Class<T> clazz;

    public GsonTypeHandler(Class<T> clazz) {
        if (clazz == null) {
            throw new IllegalArgumentException("Type argument cannot be null");
        }
        this.clazz = clazz;
    }

    @Override
    public void setNonNullParameter(PreparedStatement ps, int i, T parameter, JdbcType jdbcType) throws SQLException {
        ps.setString(i, GSON.toJson(parameter));
    }

    @Override
    public T getNullableResult(ResultSet rs, String columnName) throws SQLException {
        return parseJson(rs.getString(columnName));
    }

    @Override
    public T getNullableResult(ResultSet rs, int columnIndex) throws SQLException {
        return parseJson(rs.getString(columnIndex));
    }

    @Override
    public T getNullableResult(CallableStatement cs, int columnIndex) throws SQLException {
        return parseJson(cs.getString(columnIndex));
    }

    private T parseJson(String json) {
        if (json == null || json.isEmpty()) {
            return null;
        }
        return GSON.fromJson(json, clazz);
    }
}

提示： ObjectMapper 和 Gson 都是线程安全的，可以声明为 static 共享实例，避免重复创建带来的性能开销。

9.2.3 数据库 Array 类型处理

PostgreSQL 等数据库支持原生 Array 列类型（如 INTEGER[]、TEXT[]），MyBatis 默认无法直接映射为 Java 的 List 或数组。需要编写专门的 TypeHandler 来处理。

import org.apache.ibatis.type.BaseTypeHandler;
import org.apache.ibatis.type.JdbcType;
import org.apache.ibatis.type.MappedJdbcTypes;

import java.sql.*;
import java.util.Arrays;
import java.util.List;

@MappedJdbcTypes(JdbcType.ARRAY)
public class StringArrayTypeHandler extends BaseTypeHandler<List<String>> {

    @Override
    public void setNonNullParameter(PreparedStatement ps, int i, List<String> parameter, JdbcType jdbcType) throws SQLException {
        Connection conn = ps.getConnection();
        Array array = conn.createArrayOf("VARCHAR", parameter.toArray());
        ps.setArray(i, array);
    }

    @Override
    public List<String> getNullableResult(ResultSet rs, String columnName) throws SQLException {
        return toList(rs.getArray(columnName));
    }

    @Override
    public List<String> getNullableResult(ResultSet rs, int columnIndex) throws SQLException {
        return toList(rs.getArray(columnIndex));
    }

    @Override
    public List<String> getNullableResult(CallableStatement cs, int columnIndex) throws SQLException {
        return toList(cs.getArray(columnIndex));
    }

    @SuppressWarnings("unchecked")
    private List<String> toList(Array array) throws SQLException {
        if (array == null) {
            return null;
        }
        String[] result = (String[]) array.getArray();
        return Arrays.asList(result);
    }
}

使用注意：

  • createArrayOf 的第一个参数是数据库类型名称，需要与实际列类型一致（如 PostgreSQL 的 VARCHAR、INTEGER）
  • MySQL 不支持原生 Array 类型，若需要存储数组结构，通常使用 JSON 字符串方案（参见 9.2.2）
  • 对于 Integer[]、Long[] 等其他类型的数组，按相同模式编写对应的 TypeHandler 即可

9.3 全局注册 vs 局部指定

TypeHandler 有两种注册方式：全局注册让所有映射自动生效，局部指定则精确控制单个字段。

全局注册

方式一：mybatis-config.xml（纯 MyBatis 项目）

<configuration>
    <typeHandlers>
        <!-- 注册单个 TypeHandler -->
        <typeHandler handler="com.example.handler.BaseEnumTypeHandler" />

        <!-- 按包名批量扫描注册 -->
        <package name="com.example.handler" />
    </typeHandlers>
</configuration>

配合 @MappedTypes 和 @MappedJdbcTypes 注解，MyBatis 可以自动推断 TypeHandler 处理的类型：

@MappedTypes(OrderStatus.class)
@MappedJdbcTypes(JdbcType.INTEGER)
public class OrderStatusTypeHandler extends BaseEnumTypeHandler<OrderStatus> {
    public OrderStatusTypeHandler() {
        super(OrderStatus.class);
    }
}

方式二：Spring Boot 配置（推荐）

在 application.yml 中指定 TypeHandler 包路径，Spring Boot 自动扫描并注册：

mybatis:
  type-handlers-package: com.example.handler

或者在 application.properties 中：

mybatis.type-handlers-package=com.example.handler

也可以通过 Java Config 手动注册：

@Configuration
public class MyBatisConfig {

    @Bean
    public ConfigurationCustomizer configurationCustomizer() {
        return configuration -> {
            configuration.getTypeHandlerRegistry()
                    .register(OrderStatus.class, new BaseEnumTypeHandler<>(OrderStatus.class));
        };
    }
}

局部指定

当你不想全局注册、或者同一个 Java 类型在不同字段需要不同的 TypeHandler 时，可以在 resultMap 或参数中局部指定。

在 resultMap 中指定

<resultMap id="userMap" type="com.example.entity.User">
    <id column="id" property="id" />
    <result column="name" property="name" />
    <result column="profile" property="profile"
            typeHandler="com.example.handler.JacksonTypeHandler" />
    <result column="status" property="status"
            typeHandler="com.example.handler.BaseEnumTypeHandler" />
</resultMap>

在 SQL 参数中指定

<insert id="insertUser">
    INSERT INTO user (name, profile, status) VALUES (
        #{name},
        #{profile, typeHandler=com.example.handler.JacksonTypeHandler},
        #{status, typeHandler=com.example.handler.BaseEnumTypeHandler}
    )
</insert>

全局注册 vs 局部指定的选择建议

方式 优点 缺点 适用场景
全局注册 一次配置，处处生效；XML 更简洁 粒度较粗，同类型只能对应一个 Handler 项目中同一 Java 类型的映射策略统一
局部指定 精确控制每个字段 每处使用都要写，容易遗漏 同一类型在不同字段有不同映射需求

最佳实践： 通用的 TypeHandler（如所有枚举统一按 code 存储）走全局注册；特殊字段（如某个字段需要 JSON 处理）走局部指定。两者可以搭配使用——全局注册的是默认行为，局部指定的优先级更高，会覆盖全局配置。

9.4 TypeHandler 在 MyBatis-Plus 中的使用

MyBatis-Plus 对 TypeHandler 的使用做了进一步简化，可以直接通过注解在实体类上指定，无需手写 resultMap。

使用 @TableField 指定 TypeHandler

import com.baomidou.mybatisplus.annotation.TableField;
import com.baomidou.mybatisplus.annotation.TableName;

@Data
@TableName(autoResultMap = true)  // 必须开启，否则查询时 TypeHandler 不生效
public class User {

    private Long id;
    private String name;

    @TableField(typeHandler = JacksonTypeHandler.class)
    private UserProfile profile;

    @TableField(typeHandler = BaseEnumTypeHandler.class)
    private OrderStatus status;
}

关键点：autoResultMap = true

@TableName(autoResultMap = true) 告诉 MyBatis-Plus 自动为该实体生成包含 TypeHandler 的 resultMap。如果不加这个注解，TypeHandler 只在写入（INSERT/UPDATE）时生效，查询（SELECT）时不会生效，这是一个非常常见的坑。

枚举处理：@EnumValue 注解

MyBatis-Plus 提供了 @EnumValue 注解，可以直接标注枚举中用于存储的字段，无需编写 TypeHandler：

public enum OrderStatus {
    PENDING(10, "待支付"),
    PAID(20, "已支付"),
    SHIPPED(30, "已发货"),
    COMPLETED(40, "已完成"),
    CANCELLED(50, "已取消");

    @EnumValue  // 标记数据库存储的值
    private final int code;
    private final String desc;

    OrderStatus(int code, String desc) {
        this.code = code;
        this.desc = desc;
    }
}

配合全局配置（application.yml）：

mybatis-plus:
  configuration:
    default-enum-type-handler: com.baomidou.mybatisplus.core.handlers.MybatisEnumTypeHandler
  type-handlers-package: com.example.handler

配置 MybatisEnumTypeHandler 作为默认枚举处理器后，所有使用了 @EnumValue 的枚举都会自动按标记字段的值存取，实体类中无需再为枚举字段单独指定 @TableField(typeHandler = ...)。

JSON 处理

MyBatis-Plus 从 3.5.x 版本起内置了 JacksonTypeHandler 和 GsonTypeHandler（位于 com.baomidou.mybatisplus.extension.handlers 包），可以直接使用：

import com.baomidou.mybatisplus.extension.handlers.JacksonTypeHandler;

@Data
@TableName(autoResultMap = true)
public class User {
    private Long id;
    private String name;

    @TableField(typeHandler = JacksonTypeHandler.class)
    private UserProfile profile;
}

使用 MyBatis-Plus 内置的 JacksonTypeHandler 时，确保项目中已引入 Jackson 依赖（Spring Boot Web Starter 已自带）。

在自定义 XML 中使用

即使使用了 MyBatis-Plus，如果你在自定义 XML 中编写了 SQL，仍需要在 resultMap 中手动指定 TypeHandler：

<!-- 自定义 XML 中 TypeHandler 不会自动生效，需要手动指定 -->
<resultMap id="customUserMap" type="com.example.entity.User">
    <id column="id" property="id" />
    <result column="name" property="name" />
    <result column="profile" property="profile"
            typeHandler="com.baomidou.mybatisplus.extension.handlers.JacksonTypeHandler" />
</resultMap>

<select id="selectCustom" resultMap="customUserMap">
    SELECT id, name, profile FROM user WHERE id = #{id}
</select>

MyBatis-Plus TypeHandler 使用清单

场景 方案 备注
枚举按业务编码存取 @EnumValue + 全局配置 MybatisEnumTypeHandler 最简洁，推荐
JSON 字段映射 @TableField(typeHandler = JacksonTypeHandler.class) + @TableName(autoResultMap = true) MP 内置，开箱即用
其他自定义类型 @TableField(typeHandler = XxxHandler.class) + @TableName(autoResultMap = true) 需自行编写 Handler
自定义 XML 查询 在 <resultMap> 中手动指定 typeHandler 注解方式在自定义 XML 中不自动生效

本章小结

  • 内置 TypeHandler 覆盖了绝大部分基本类型，日常开发无需关注
  • 自定义 TypeHandler 继承 BaseTypeHandler<T> 并实现四个方法即可，适用于枚举、JSON、数组等特殊场景
  • 枚举映射 推荐按业务编码存储，避免使用 EnumOrdinalTypeHandler
  • JSON 映射 通过泛型 TypeHandler 实现通用的 JSON ↔ 对象转换
  • 全局注册 适合统一策略，局部指定 适合精细控制，两者可搭配使用
  • MyBatis-Plus 通过 @TableField、@EnumValue 和 autoResultMap 进一步简化了 TypeHandler 的使用
