第六章 注解开发 📖

简单场景可用注解，复杂 SQL 仍推荐 XML。
本章系统介绍 MyBatis 注解开发方式，帮助你在适当场景下用更少的文件完成映射配置。

6.1 @Select / @Insert / @Update / @Delete —— 基础注解

MyBatis 允许直接在 Mapper 接口方法上用注解编写 SQL，省去 XML 文件。四个基础注解分别对应 CRUD 操作。

完整 CRUD 示例

假设数据表 user 结构如下：

CREATE TABLE user (
    id          BIGINT PRIMARY KEY AUTO_INCREMENT,
    username    VARCHAR(50)  NOT NULL,
    email       VARCHAR(100),
    age         INT,
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP
);

对应实体类：

public class User {
    private Long id;
    private String username;
    private String email;
    private Integer age;
    private LocalDateTime createTime;

    // getter / setter 省略
}

Mapper 接口：

public interface UserMapper {

    // ---------- 查询 ----------

    @Select("SELECT * FROM user WHERE id = #{id}")
    User selectById(Long id);

    @Select("SELECT * FROM user")
    List<User> selectAll();

    @Select("SELECT * FROM user WHERE username LIKE CONCAT('%', #{keyword}, '%')")
    List<User> searchByUsername(String keyword);

    // ---------- 插入 ----------

    @Insert("INSERT INTO user(username, email, age) VALUES(#{username}, #{email}, #{age})")
    @Options(useGeneratedKeys = true, keyProperty = "id")
    int insert(User user);

    // ---------- 更新 ----------

    @Update("UPDATE user SET username = #{username}, email = #{email}, age = #{age} WHERE id = #{id}")
    int update(User user);

    // ---------- 删除 ----------

    @Delete("DELETE FROM user WHERE id = #{id}")
    int deleteById(Long id);
}

要点说明：

  • @Options(useGeneratedKeys = true, keyProperty = "id")：插入后自动将生成的主键回填到对象的 id 属性，等价于 XML 中 <insert useGeneratedKeys="true" keyProperty="id">。
  • 参数绑定规则与 XML 一致：单参数直接用 #{参数名}；多参数需配合 @Param。

多参数示例

@Select("SELECT * FROM user WHERE username = #{username} AND age = #{age}")
User selectByNameAndAge(@Param("username") String username, @Param("age") Integer age);

6.2 @Results / @Result —— 注解版 resultMap

当数据库字段与 Java 属性名不一致，或需要做复杂映射时，使用 @Results + @Result 替代 XML 的 <resultMap>。

基础映射

@Select("SELECT id, username, email, age, create_time FROM user WHERE id = #{id}")
@Results(id = "userResultMap", value = {
    @Result(property = "id",         column = "id",          id = true),
    @Result(property = "username",   column = "username"),
    @Result(property = "email",      column = "email"),
    @Result(property = "age",        column = "age"),
    @Result(property = "createTime", column = "create_time")
})
User selectByIdWithMapping(Long id);
  • id = "userResultMap"：为这个映射命名，其他方法可通过 @ResultMap("userResultMap") 复用。
  • id = true：标记主键列，等价于 XML 中 <id>。

复用已定义的 @Results

@Select("SELECT id, username, email, age, create_time FROM user")
@ResultMap("userResultMap")
List<User> selectAllWithMapping();

映射复杂属性（嵌套对象）

假设 User 内嵌一个 Address 对象：

public class Address {
    private String province;
    private String city;
    private String street;
    // getter / setter 省略
}

public class UserDetail {
    private Long id;
    private String username;
    private Address address;
    // getter / setter 省略
}

数据库字段为 province、city、street 直接平铺在 user_detail 表中时，可以这样映射：

@Select("SELECT id, username, province, city, street FROM user_detail WHERE id = #{id}")
@Results({
    @Result(property = "id",               column = "id", id = true),
    @Result(property = "username",         column = "username"),
    @Result(property = "address.province", column = "province"),
    @Result(property = "address.city",     column = "city"),
    @Result(property = "address.street",   column = "street")
})
UserDetail selectUserDetail(Long id);

提示： property = "address.province" 利用了 MyBatis 对嵌套属性的自动赋值能力（需要 Address 有无参构造器）。

6.3 @One / @Many —— 注解版关联查询

@One 和 @Many 分别对应 XML 中 <association> 和 <collection> 的嵌套查询（Nested Select）方式。

实体类准备

public class Department {
    private Long id;
    private String deptName;
    private List<Employee> employees;  // 一对多
    // getter / setter 省略
}

public class Employee {
    private Long id;
    private String empName;
    private Long deptId;
    private Department department;     // 一对一（多对一）
    // getter / setter 省略
}

@One —— 一对一（多对一）

查询员工时，关联查出其所属部门：

public interface EmployeeMapper {

    @Select("SELECT id, emp_name, dept_id FROM employee WHERE id = #{id}")
    @Results({
        @Result(property = "id",      column = "id", id = true),
        @Result(property = "empName", column = "emp_name"),
        @Result(property = "deptId",  column = "dept_id"),
        @Result(property = "department", column = "dept_id",
                one = @One(select = "com.example.mapper.DepartmentMapper.selectById",
                           fetchType = FetchType.LAZY))
    })
    Employee selectWithDepartment(Long id);
}

public interface DepartmentMapper {

    @Select("SELECT id, dept_name FROM department WHERE id = #{id}")
    @Results({
        @Result(property = "id",       column = "id", id = true),
        @Result(property = "deptName", column = "dept_name")
    })
    Department selectById(Long id);
}

说明：

  • column = "dept_id"：将当前结果集中 dept_id 的值作为参数传给关联查询。
  • one = @One(select = "...")：指定关联查询的完整方法路径（namespace + 方法名）。
  • fetchType = FetchType.LAZY：启用懒加载，只在真正访问 department 属性时才执行子查询。可选值还有 FetchType.EAGER（立即加载）和 FetchType.DEFAULT（跟随全局配置）。

@Many —— 一对多

查询部门时，关联查出该部门下所有员工：

public interface DepartmentMapper {

    @Select("SELECT id, dept_name FROM department WHERE id = #{id}")
    @Results({
        @Result(property = "id",       column = "id", id = true),
        @Result(property = "deptName", column = "dept_name"),
        @Result(property = "employees", column = "id",
                many = @Many(select = "com.example.mapper.EmployeeMapper.selectByDeptId",
                             fetchType = FetchType.LAZY))
    })
    Department selectWithEmployees(Long id);
}

public interface EmployeeMapper {

    @Select("SELECT id, emp_name, dept_id FROM employee WHERE dept_id = #{deptId}")
    @Results({
        @Result(property = "id",      column = "id", id = true),
        @Result(property = "empName", column = "emp_name"),
        @Result(property = "deptId",  column = "dept_id")
    })
    List<Employee> selectByDeptId(Long deptId);
}

注意： @One / @Many 只支持嵌套查询方式（分步查询），不支持嵌套结果映射（JOIN 映射）。如果需要 JOIN 映射复杂结果，建议使用 XML 的 <resultMap> + <association> / <collection>。

6.4 @SelectProvider / @InsertProvider 等 —— 动态 SQL Provider

当 SQL 需要动态拼接但又不想写 XML 时，可以用 Provider 注解将 SQL 构建逻辑委托给一个 Java 方法。

四个 Provider 注解：

注解 对应操作
@SelectProvider 查询
@InsertProvider 插入
@UpdateProvider 更新
@DeleteProvider 删除

使用 SQL 构建器（new SQL()）

MyBatis 内置了 org.apache.ibatis.jdbc.SQL 构建器，提供链式 API 来生成 SQL 字符串。

Provider 类

import org.apache.ibatis.jdbc.SQL;

public class UserSqlProvider {

    /**
     * 根据条件动态构建查询 SQL
     */
    public String selectByCondition(final User condition) {
        return new SQL() {{
            SELECT("id, username, email, age, create_time");
            FROM("user");

            if (condition.getUsername() != null) {
                WHERE("username LIKE CONCAT('%', #{username}, '%')");
            }
            if (condition.getEmail() != null) {
                WHERE("email = #{email}");
            }
            if (condition.getAge() != null) {
                WHERE("age = #{age}");
            }

            ORDER_BY("id DESC");
        }}.toString();
    }

    /**
     * 动态插入：只插入非空字段
     */
    public String insertSelective(final User user) {
        return new SQL() {{
            INSERT_INTO("user");

            if (user.getUsername() != null) {
                VALUES("username", "#{username}");
            }
            if (user.getEmail() != null) {
                VALUES("email", "#{email}");
            }
            if (user.getAge() != null) {
                VALUES("age", "#{age}");
            }
        }}.toString();
    }

    /**
     * 动态更新：只更新非空字段
     */
    public String updateSelective(final User user) {
        return new SQL() {{
            UPDATE("user");

            if (user.getUsername() != null) {
                SET("username = #{username}");
            }
            if (user.getEmail() != null) {
                SET("email = #{email}");
            }
            if (user.getAge() != null) {
                SET("age = #{age}");
            }

            WHERE("id = #{id}");
        }}.toString();
    }

    /**
     * 根据 ID 删除
     */
    public String deleteById(final Long id) {
        return new SQL() {{
            DELETE_FROM("user");
            WHERE("id = #{id}");
        }}.toString();
    }
}

Mapper 接口

public interface UserMapper {

    @SelectProvider(type = UserSqlProvider.class, method = "selectByCondition")
    List<User> selectByCondition(User condition);

    @InsertProvider(type = UserSqlProvider.class, method = "insertSelective")
    @Options(useGeneratedKeys = true, keyProperty = "id")
    int insertSelective(User user);

    @UpdateProvider(type = UserSqlProvider.class, method = "updateSelective")
    int updateSelective(User user);

    @DeleteProvider(type = UserSqlProvider.class, method = "deleteById")
    int deleteById(Long id);
}

SQL 构建器常用方法速查

方法 说明
SELECT(String...) 添加 SELECT 列
FROM(String...) 添加 FROM 表
WHERE(String...) 添加 WHERE 条件（多个自动用 AND 连接）
OR() 在 WHERE 条件之间插入 OR
SET(String...) 添加 SET 子句
INSERT_INTO(String) 指定插入表名
VALUES(String, String) 添加 VALUES 列和值
DELETE_FROM(String) 指定删除表名
JOIN(String...) INNER JOIN
LEFT_OUTER_JOIN(String...) LEFT JOIN
RIGHT_OUTER_JOIN(String...) RIGHT JOIN
GROUP_BY(String...) GROUP BY
HAVING(String...) HAVING
ORDER_BY(String...) ORDER BY
LIMIT(int) LIMIT（MyBatis 3.5.2+）
OFFSET(int) OFFSET（MyBatis 3.5.2+）

带 @Param 的多参数场景

当 Mapper 方法有多个 @Param 参数时，Provider 方法可以接收一个 Map：

// Mapper 接口
@SelectProvider(type = UserSqlProvider.class, method = "selectByNameAndAge")
List<User> selectByNameAndAge(@Param("username") String username, @Param("age") Integer age);

// Provider 方法 —— 参数以 Map 形式接收
public String selectByNameAndAge(Map<String, Object> params) {
    return new SQL() {{
        SELECT("*");
        FROM("user");
        if (params.get("username") != null) {
            WHERE("username = #{username}");
        }
        if (params.get("age") != null) {
            WHERE("age = #{age}");
        }
    }}.toString();
}

6.5 注解 vs XML 的适用场景对比

维度 注解方式 XML 方式
代码位置 SQL 写在 Java 接口中，接口即文档 SQL 写在独立 XML 文件中，Java 与 SQL 分离
可读性 简单 SQL 直观；复杂 SQL 因字符串拼接变得混乱 标签结构化，复杂 SQL 可读性更好
动态 SQL 需借助 Provider + SQL 构建器，灵活度有限 <if> / <choose> / <foreach> 等标签，表达力强
结果映射 @Results 只支持嵌套查询，不支持嵌套结果映射 <resultMap> 支持全部映射方式
复用性 @ResultMap 可引用已命名映射，但不如 XML 灵活 <sql> + <include> 可复用 SQL 片段
重构友好 IDE 可直接跳转到方法，重命名同步 需手动同步 XML namespace 与方法名
维护成本 修改 SQL 需重新编译 Java 文件 修改 XML 后某些场景可热加载
适用场景 单表 CRUD、简单查询、原型快速开发 多表 JOIN、动态 SQL 复杂、需要精细 resultMap
团队协作 适合小型项目、微服务中的简单模块 适合中大型项目、SQL 审查需求高的团队

选型建议

  • 选注解： 单表增删改查、条件简单的查询、快速原型开发、团队倾向"代码即配置"。
  • 选 XML： 涉及多表 JOIN、复杂动态 SQL（多层嵌套条件）、需要嵌套结果映射、SQL 需要独立审查或由 DBA 维护。
  • 实际项目中： 两者混合使用最为常见——简单操作用注解、复杂逻辑用 XML。

6.6 混合使用：注解 + XML 的最佳实践

MyBatis 完全支持同一个 Mapper 接口中，部分方法用注解、部分方法用 XML 映射。两者可以共存，但同一个方法不能同时在注解和 XML 中定义。

配置要点

1. 确保 Mapper XML 能被扫描到

Spring Boot 中配置：

mybatis:
  mapper-locations: classpath:mapper/**/*.xml
  configuration:
    map-underscore-to-camel-case: true

2. Mapper 接口正常注册

通过 @MapperScan 或 @Mapper 注解注册即可，MyBatis 会同时收集注解和 XML 中的映射信息。

@MapperScan("com.example.mapper")
@SpringBootApplication
public class Application {
    public static void main(String[] args) {
        SpringApplication.run(Application.class, args);
    }
}

混合使用示例

假设 UserMapper 中，简单的单表操作用注解，复杂的多表查询用 XML。

Mapper 接口：

public interface UserMapper {

    // ===== 注解方式：简单 CRUD =====

    @Select("SELECT * FROM user WHERE id = #{id}")
    User selectById(Long id);

    @Insert("INSERT INTO user(username, email, age) VALUES(#{username}, #{email}, #{age})")
    @Options(useGeneratedKeys = true, keyProperty = "id")
    int insert(User user);

    @Update("UPDATE user SET username = #{username}, email = #{email} WHERE id = #{id}")
    int update(User user);

    @Delete("DELETE FROM user WHERE id = #{id}")
    int deleteById(Long id);

    // ===== XML 方式：复杂查询（方法声明在接口中，SQL 写在 XML 文件里）=====

    List<User> selectByCondition(UserQueryParam param);

    UserDetailVO selectUserWithOrders(Long userId);
}

XML 文件（mapper/UserMapper.xml）：

<?xml version="1.0" encoding="UTF-8" ?>
<!DOCTYPE mapper PUBLIC "-//mybatis.org//DTD Mapper 3.0//EN"
        "http://mybatis.org/dtd/mybatis-3-mapper.dtd">
<mapper namespace="com.example.mapper.UserMapper">

    <!-- 复杂动态查询 -->
    <select id="selectByCondition" parameterType="com.example.param.UserQueryParam"
            resultType="com.example.entity.User">
        SELECT id, username, email, age, create_time
        FROM user
        <where>
            <if test="username != null and username != ''">
                AND username LIKE CONCAT('%', #{username}, '%')
            </if>
            <if test="email != null">
                AND email = #{email}
            </if>
            <if test="minAge != null">
                AND age &gt;= #{minAge}
            </if>
            <if test="maxAge != null">
                AND age &lt;= #{maxAge}
            </if>
        </where>
        ORDER BY id DESC
    </select>

    <!-- 多表关联查询 + 嵌套结果映射 -->
    <resultMap id="userWithOrdersMap" type="com.example.vo.UserDetailVO">
        <id property="id" column="user_id" />
        <result property="username" column="username" />
        <result property="email" column="email" />
        <collection property="orders" ofType="com.example.vo.OrderVO">
            <id property="orderId" column="order_id" />
            <result property="orderNo" column="order_no" />
            <result property="amount" column="amount" />
            <result property="orderTime" column="order_time" />
        </collection>
    </resultMap>

    <select id="selectUserWithOrders" resultMap="userWithOrdersMap">
        SELECT
            u.id AS user_id,
            u.username,
            u.email,
            o.id AS order_id,
            o.order_no,
            o.amount,
            o.order_time
        FROM user u
        LEFT JOIN orders o ON u.id = o.user_id
        WHERE u.id = #{userId}
    </select>

</mapper>

混合使用的注意事项

  1. 同一方法不能同时定义：如果 selectById 在接口上有 @Select 注解，XML 里就不能再有 <select id="selectById">，否则启动时会报冲突。
  2. namespace 必须一致：XML 的 namespace 必须是 Mapper 接口的全限定类名。
  3. 注解中引用 XML 的 resultMap：注解方法可以通过 @ResultMap("userWithOrdersMap") 引用 XML 中定义的 <resultMap>，这是一种非常实用的混合技巧。
@Select("SELECT u.id AS user_id, u.username, u.email, " +
        "o.id AS order_id, o.order_no, o.amount, o.order_time " +
        "FROM user u LEFT JOIN orders o ON u.id = o.user_id " +
        "WHERE u.id = #{userId}")
@ResultMap("userWithOrdersMap")  // 引用 XML 中定义的 resultMap
UserDetailVO selectUserWithOrdersByAnnotation(Long userId);
  1. 分工原则：团队内建议约定清晰的分工标准，例如"单表操作一律用注解，涉及 JOIN 或动态条件超过 3 个的用 XML"，避免同一项目中风格混乱。

本章小结

知识点 核心内容
基础注解 @Select / @Insert / @Update / @Delete + @Options
结果映射 @Results / @Result / @ResultMap
关联查询 @One（一对一）/ @Many（一对多），仅支持嵌套查询方式
动态 SQL @SelectProvider 等 + SQL 构建器，适合中等复杂度的动态 SQL
选型建议 简单场景用注解提升效率，复杂场景用 XML 保障可维护性
混合使用 同一 Mapper 接口可同时拥有注解方法和 XML 方法，灵活组合
