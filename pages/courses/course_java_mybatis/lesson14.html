A.1 mybatis-config.xml settings 完整速查

<settings> 是 MyBatis 中最重要的全局调优标签，以下按功能分组列出所有常用配置项。

映射与命名

setting 名称 说明 可选值 默认值
mapUnderscoreToCamelCase 是否开启下划线命名到驼峰命名的自动映射（如 user_name → userName） true / false false
autoMappingBehavior 自动映射策略：NONE 不启用；PARTIAL 仅映射无嵌套的结果；FULL 映射所有结果（含嵌套） NONE / PARTIAL / FULL PARTIAL
autoMappingUnknownColumnBehavior 当发现自动映射目标中有未知列时的行为 NONE / WARNING / FAILING NONE
argNameBasedConstructorAutoMapping 构造器自动映射时是否按参数名匹配而非按顺序匹配 true / false false

缓存

setting 名称 说明 可选值 默认值
cacheEnabled 全局二级缓存开关（仍需在 Mapper XML 中配置 <cache> 才生效） true / false true
localCacheScope 一级缓存作用域：SESSION 为同一 SqlSession 内共享；STATEMENT 为每次查询后立即清除 SESSION / STATEMENT SESSION

延迟加载

setting 名称 说明 可选值 默认值
lazyLoadingEnabled 延迟加载全局开关，开启后所有 association / collection 默认延迟加载 true / false false
aggressiveLazyLoading 开启后任意属性的访问都会触发所有延迟属性的加载；3.4.1+ 默认关闭 true / false false
lazyLoadTriggerMethods 触发延迟加载的对象方法列表 逗号分隔的方法名 equals,clone,hashCode,toString
proxyFactory 延迟加载的代理工厂实现 CGLIB / JAVASSIST JAVASSIST

执行器与语句

setting 名称 说明 可选值 默认值
defaultExecutorType 默认执行器类型：SIMPLE 普通；REUSE 复用 PreparedStatement；BATCH 批量执行 SIMPLE / REUSE / BATCH SIMPLE
defaultStatementTimeout 设置所有 SQL 语句的默认超时时间（秒） 正整数 / null null（无限制）
defaultFetchSize 设置 JDBC fetchSize 的默认值，控制每次从数据库获取的行数 正整数 / null null（使用驱动默认）
defaultResultSetType 默认的 ResultSet 类型 FORWARD_ONLY / SCROLL_SENSITIVE / SCROLL_INSENSITIVE / DEFAULT null（使用驱动默认）
useGeneratedKeys 全局开启自动生成主键，可在 <insert> 标签中单独覆盖 true / false false

行为与安全

setting 名称 说明 可选值 默认值
safeRowBoundsEnabled 是否允许在嵌套查询中使用 RowBounds（分页），false 表示允许 true / false false
safeResultHandlerEnabled 是否允许在嵌套查询中使用 ResultHandler，false 表示允许 true / false true
multipleResultSetsEnabled 是否允许单个语句返回多个结果集（需驱动支持） true / false true
useColumnLabel 使用列标签而非列名来获取结果 true / false true
callSettersOnNulls 当结果集中的值为 null 时是否调用 setter / put，对 Map 返回值有影响 true / false false
returnInstanceForEmptyRow 当结果行所有列都为 null 时是否返回空实例而非 null true / false false
shrinkWhitespacesInSql 移除 SQL 中多余的空白字符，仅保留单个空格（便于日志阅读） true / false false
nullableOnForEach 当 <foreach> 的 collection 参数为 null 时是否不抛异常 true / false false

日志与调试

setting 名称 说明 可选值 默认值
logImpl 指定日志实现，不设置时 MyBatis 按优先级自动查找 SLF4J / LOG4J / LOG4J2 / JDK_LOGGING / COMMONS_LOGGING / STDOUT_LOGGING / NO_LOGGING 未设置（自动检测）
logPrefix 为所有 MyBatis 日志名添加统一前缀 任意字符串 未设置

其他

setting 名称 说明 可选值 默认值
jdbcTypeForNull 当参数值为 null 且没有指定 jdbcType 时使用的 JDBC 类型 任何 JdbcType 枚举值 OTHER
configurationFactory 指定一个 Configuration 工厂类，用于提供 Configuration 实例 类全限定名 未设置
vfsImpl 自定义 VFS（虚拟文件系统）实现类 类全限定名（逗号分隔） 未设置
defaultEnumTypeHandler 默认枚举类型处理器 类全限定名 org.apache.ibatis.type.EnumTypeHandler

A.3 配置示例

mybatis-config.xml 方式：

<configuration>
    <settings>
        <setting name="mapUnderscoreToCamelCase" value="true"/>
        <setting name="lazyLoadingEnabled" value="true"/>
        <setting name="cacheEnabled" value="true"/>
        <setting name="defaultExecutorType" value="SIMPLE"/>
        <setting name="logImpl" value="SLF4J"/>
        <setting name="localCacheScope" value="SESSION"/>
        <setting name="jdbcTypeForNull" value="NULL"/>
        <setting name="callSettersOnNulls" value="true"/>
    </settings>
</configuration>

Spring Boot application.yml 方式：

mybatis:
  mapper-locations: classpath:mapper/**/*.xml
  type-aliases-package: com.example.entity
  configuration:
    map-underscore-to-camel-case: true
    lazy-loading-enabled: true
    cache-enabled: true
    default-executor-type: simple
    log-impl: org.apache.ibatis.logging.slf4j.Slf4jImpl
    local-cache-scope: session
    jdbc-type-for-null: null
    call-setters-on-nulls: true

附录 B：MyBatis-Plus 条件构造器方法速查

以下所有方法均适用于 QueryWrapper、LambdaQueryWrapper、UpdateWrapper、LambdaUpdateWrapper（除特别注明外）。 Lambda 写法将字符串列名替换为方法引用，如 eq("name", "张三") → eq(User::getName, "张三")。 所有条件方法均支持 condition 前缀参数控制是否拼接：eq(age != null, "age", age)。

B.1 比较类

方法 对应 SQL 示例代码 说明
eq(column, val) column = val eq("name", "张三") 等于
ne(column, val) column <> val ne("status", 0) 不等于
gt(column, val) column > val gt("age", 18) 大于
ge(column, val) column >= val ge("age", 18) 大于等于
lt(column, val) column < val lt("price", 100) 小于
le(column, val) column <= val le("price", 100) 小于等于

B.2 模糊查询类

方法 对应 SQL 示例代码 说明
like(column, val) column LIKE '%val%' like("name", "张") 两边模糊
notLike(column, val) column NOT LIKE '%val%' notLike("name", "测试") NOT LIKE 两边模糊
likeLeft(column, val) column LIKE '%val' likeLeft("name", "三") 左模糊（以 val 结尾）
likeRight(column, val) column LIKE 'val%' likeRight("name", "张") 右模糊（以 val 开头）
notLikeLeft(column, val) column NOT LIKE '%val' notLikeLeft("name", "三") NOT LIKE 左模糊
notLikeRight(column, val) column NOT LIKE 'val%' notLikeRight("name", "张") NOT LIKE 右模糊

B.3 范围类

方法 对应 SQL 示例代码 说明
between(column, val1, val2) column BETWEEN val1 AND val2 between("age", 18, 30) 闭区间范围
notBetween(column, val1, val2) column NOT BETWEEN val1 AND val2 notBetween("age", 18, 30) 不在范围内
in(column, collection) column IN (v1, v2, ...) in("id", Arrays.asList(1, 2, 3)) IN 查询
in(column, values...) column IN (v1, v2, ...) in("id", 1, 2, 3) IN 查询（可变参数）
notIn(column, collection) column NOT IN (v1, v2, ...) notIn("status", Arrays.asList(0, -1)) NOT IN 查询
inSql(column, sqlValue) column IN (sql) inSql("id", "SELECT id FROM t WHERE status = 1") IN 子查询
notInSql(column, sqlValue) column NOT IN (sql) notInSql("id", "SELECT id FROM t WHERE deleted = 1") NOT IN 子查询

B.4 空值判断类

方法 对应 SQL 示例代码 说明
isNull(column) column IS NULL isNull("email") 字段为 NULL
isNotNull(column) column IS NOT NULL isNotNull("phone") 字段不为 NULL

B.5 分组、排序、Having

方法 对应 SQL 示例代码 说明
groupBy(columns...) GROUP BY column1, column2 groupBy("department", "status") 分组
orderByAsc(columns...) ORDER BY column ASC orderByAsc("create_time") 升序排序
orderByDesc(columns...) ORDER BY column DESC orderByDesc("create_time") 降序排序
orderBy(condition, isAsc, columns...) ORDER BY column ASC/DESC orderBy(true, true, "age") 通用排序，isAsc 控制方向
having(sqlHaving, params...) HAVING ... having("count(id) > {0}", 10) HAVING 子句

B.6 拼接类

方法 对应 SQL 示例代码 说明
apply(sqlSegment, params...) 直接拼接到 WHERE 后 apply("date_format(create_time,'%Y-%m-%d') = {0}", "2024-01-01") 拼接自定义 SQL 片段，{0} 占位防注入
last(lastSql) 拼接到 SQL 末尾 last("LIMIT 1") 慎用：无防注入保护，只能调用一次
exists(existsSql) EXISTS (sql) exists("SELECT 1 FROM orders WHERE orders.user_id = users.id") EXISTS 子查询
notExists(notExistsSql) NOT EXISTS (sql) notExists("SELECT 1 FROM blacklist WHERE blacklist.uid = users.id") NOT EXISTS 子查询
comment(comment) /* comment */ comment("查询活跃用户") 在 SQL 头部添加注释

B.7 逻辑连接类

方法 对应 SQL 示例代码 说明
and(consumer) AND (...) and(w -> w.eq("status", 1).ne("type", 0)) AND 嵌套，consumer 内的条件被括号包裹
or() OR eq("name", "张三").or().eq("name", "李四") 下一个条件用 OR 连接（默认是 AND）
or(consumer) OR (...) or(w -> w.eq("role", "admin").eq("level", 5)) OR 嵌套，consumer 内的条件被括号包裹
nested(consumer) (...) nested(w -> w.eq("a", 1).ne("b", 2)) 纯嵌套，不带 AND/OR 前缀
not(consumer) NOT (...) not(w -> w.eq("status", 0)) NOT 嵌套

B.8 字段选择（仅 QueryWrapper）

方法 对应 SQL 示例代码 说明
select(columns...) SELECT column1, column2 select("id", "name", "age") 指定查询字段
select(predicate) SELECT ... select(i -> i.getProperty().startsWith("test")) Lambda 过滤查询字段（仅 LambdaQueryWrapper）

B.9 字段设置（仅 UpdateWrapper）

方法 对应 SQL 示例代码 说明
set(column, val) SET column = val set("name", "新名字") 设置更新字段值
setSql(setSql) SET ... setSql("balance = balance - 100") 拼接自定义 SET SQL

B.10 综合使用示例

基础查询：

// 查询年龄 18~30 之间、名字包含"张"、按创建时间降序排列的用户
List<User> users = userMapper.selectList(
    new QueryWrapper<User>()
        .between("age", 18, 30)
        .like("name", "张")
        .orderByDesc("create_time")
);

Lambda 写法：

List<User> users = userMapper.selectList(
    new LambdaQueryWrapper<User>()
        .between(User::getAge, 18, 30)
        .like(User::getName, "张")
        .orderByDesc(User::getCreateTime)
);

复杂 OR 嵌套条件：

// WHERE (status = 1 AND type = 'vip') OR (level >= 10)
List<User> users = userMapper.selectList(
    new LambdaQueryWrapper<User>()
        .and(w -> w.eq(User::getStatus, 1).eq(User::getType, "vip"))
        .or(w -> w.ge(User::getLevel, 10))
);

条件动态拼接：

// 参数为 null 时自动跳过该条件
LambdaQueryWrapper<User> wrapper = new LambdaQueryWrapper<User>()
    .eq(StringUtils.isNotBlank(name), User::getName, name)
    .ge(minAge != null, User::getAge, minAge)
    .le(maxAge != null, User::getAge, maxAge);

UpdateWrapper 更新指定字段：

userMapper.update(null,
    new LambdaUpdateWrapper<User>()
        .eq(User::getId, 1L)
        .set(User::getName, "新名字")
        .set(User::getStatus, 1)
);

MyBatis Generator (MBG) 代码生成 官方代码生成器，根据数据库表自动生成实体类、Mapper 接口、XML 文件
MyBatis-Plus Generator 代码生成 MP 自带的代码生成器，支持自定义模板，生成 Entity/Mapper/Service/Controller 全套代码
p6spy SQL 监控 JDBC 代理驱动，记录完整 SQL 及执行耗时，适合开发和测试环境使用
Druid 连接池/监控 阿里巴巴开源的数据库连接池，内置 SQL 监控、慢查询统计、防 SQL 注入等功能
DBeaver / DataGrip 数据库客户端 免费/商业的数据库管理工具，方便执行和调试 SQL
Arthas JVM 诊断 阿里开源的 Java 诊断工具，可在线上环境动态查看 MyBatis 生成的 SQL 和执行情况
