本课是导航系列的第二课，承接第 10 课 Scaffold。在上一课中，我们学习了如何使用 BottomNavigationBar 实现底部导航切换，但在实际应用中，"顶部标签栏"同样是一种非常常见的导航模式——比如新闻应用的分类标签、社交应用的动态/关注/推荐切换等。Flutter 提供了 TabBar + TabBarView 这套组合来实现标签页导航，同时还提供了更底层的 PageView 来实现任意的可滑动页面效果。本课将全面讲解这些组件的使用方法和内部原理。

1. DefaultTabController 与 TabController

在使用 TabBar 和 TabBarView 之前，必须先提供一个 TabController 来协调标签栏与页面内容之间的联动关系。Flutter 提供了两种方式来创建 TabController：

1.1 DefaultTabController：简单快速的方式

DefaultTabController 是 Flutter 提供的一个便捷 Widget，它会在内部自动创建并管理一个 TabController，并通过 InheritedWidget 机制将其传递给子树中的 TabBar 和 TabBarView。

适用场景：

  • 标签数量固定不变
  • 不需要在代码中手动控制标签切换（如监听切换事件、编程式跳转到某个标签）
  • 快速原型开发或简单场景
import 'package:flutter/material.dart';

void main() => runApp(const MyApp());

class MyApp extends StatelessWidget {
  const MyApp({super.key});

  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      title: 'DefaultTabController 示例',
      theme: ThemeData(
        colorSchemeSeed: Colors.blue,
        useMaterial3: true,
      ),
      home: const DefaultTabControllerDemo(),
    );
  }
}

class DefaultTabControllerDemo extends StatelessWidget {
  const DefaultTabControllerDemo({super.key});

  @override
  Widget build(BuildContext context) {
    // DefaultTabController 包裹在 Scaffold 外层
    // length 表示标签的数量，必须与 TabBar 和 TabBarView 的子项数量一致
    return DefaultTabController(
      length: 3,
      child: Scaffold(
        appBar: AppBar(
          title: const Text('DefaultTabController 示例'),
          // TabBar 放在 AppBar 的 bottom 位置
          bottom: const TabBar(
            tabs: [
              Tab(icon: Icon(Icons.home), text: '首页'),
              Tab(icon: Icon(Icons.star), text: '收藏'),
              Tab(icon: Icon(Icons.person), text: '我的'),
            ],
          ),
        ),
        body: const TabBarView(
          children: [
            Center(child: Text('首页内容', style: TextStyle(fontSize: 24))),
            Center(child: Text('收藏内容', style: TextStyle(fontSize: 24))),
            Center(child: Text('我的内容', style: TextStyle(fontSize: 24))),
          ],
        ),
      ),
    );
  }
}

关键要点：

  • DefaultTabController 必须放在 TabBar 和 TabBarView 的共同祖先位置
  • length 参数必须与 TabBar 的 tabs 数量和 TabBarView 的 children 数量严格一致，否则会抛出断言错误
  • 子树中的 TabBar 和 TabBarView 会自动通过 DefaultTabController.of(context) 获取控制器

注意事项：DefaultTabController 内部使用的是 StatefulWidget，它自己管理了 TabController 的创建和销毁。如果你需要在 DefaultTabController 的子树中获取当前选中的标签索引，可以使用 DefaultTabController.of(context).index。

1.2 TabController：完全掌控的方式

当你需要更精细地控制标签行为时——比如监听标签切换事件、在代码中编程式地跳转到某个标签、或者在标签切换时执行额外逻辑——就需要手动创建 TabController。

适用场景：

  • 需要监听标签切换事件（addListener）
  • 需要编程式地跳转到某个标签（animateTo）
  • 标签数量可能动态变化
  • 需要在标签切换时执行网络请求、数据加载等逻辑
  • 需要阻止某些标签的切换（如未登录时跳转登录页）

使用 TabController 有一个重要前提：宿主 State 必须混入 TickerProviderStateMixin（如果只有一个 AnimationController / TabController 也可以使用 SingleTickerProviderStateMixin）。这是因为 TabController 内部使用了动画来驱动标签切换时的过渡效果，而动画需要一个 TickerProvider 来提供时钟脉冲。

import 'package:flutter/material.dart';

void main() => runApp(const MyApp());

class MyApp extends StatelessWidget {
  const MyApp({super.key});

  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      title: 'TabController 示例',
      theme: ThemeData(
        colorSchemeSeed: Colors.blue,
        useMaterial3: true,
      ),
      home: const TabControllerDemo(),
    );
  }
}

class TabControllerDemo extends StatefulWidget {
  const TabControllerDemo({super.key});

  @override
  State<TabControllerDemo> createState() => _TabControllerDemoState();
}

// 必须混入 TickerProviderStateMixin 以提供 Ticker
class _TabControllerDemoState extends State<TabControllerDemo>
    with TickerProviderStateMixin {
  late final TabController _tabController;

  final List<String> _tabTitles = ['推荐', '热门', '最新', '关注'];

  @override
  void initState() {
    super.initState();
    // 创建 TabController，vsync 传入 this（即当前 State，它实现了 TickerProvider）
    _tabController = TabController(
      length: _tabTitles.length,
      vsync: this,
      initialIndex: 0, // 初始选中第一个标签（默认值也是 0）
    );

    // 监听标签切换事件
    _tabController.addListener(_onTabChanged);
  }

  void _onTabChanged() {
    // 注意：TabController 的 listener 在动画过程中会被多次调用
    // 使用 indexIsChanging 可以只在切换确认时执行一次
    if (_tabController.indexIsChanging) {
      debugPrint('标签切换中：${_tabController.previousIndex} -> ${_tabController.index}');
    }
  }

  @override
  void dispose() {
    // 务必在 dispose 中移除监听并销毁 TabController，避免内存泄漏
    _tabController.removeListener(_onTabChanged);
    _tabController.dispose();
    super.dispose();
  }

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(
        title: const Text('TabController 示例'),
        bottom: TabBar(
          controller: _tabController, // 显式传入 TabController
          tabs: _tabTitles.map((title) => Tab(text: title)).toList(),
        ),
      ),
      body: TabBarView(
        controller: _tabController, // TabBar 和 TabBarView 必须使用同一个 controller
        children: _tabTitles.map((title) {
          return Center(
            child: Text(
              '$title 的内容',
              style: const TextStyle(fontSize: 24),
            ),
          );
        }).toList(),
      ),
      // 演示编程式跳转：通过 FAB 跳转到最后一个标签
      floatingActionButton: FloatingActionButton(
        onPressed: () {
          // animateTo 会带有平滑的动画效果
          _tabController.animateTo(_tabTitles.length - 1);
        },
        child: const Icon(Icons.last_page),
      ),
    );
  }
}

1.3 两种方式详细对比

特性 DefaultTabController TabController
使用复杂度 简单，只需用 Widget 包裹 较复杂，需要 StatefulWidget + Mixin
TickerProviderStateMixin 不需要（内部自行管理） 必须混入
监听切换事件 需通过 DefaultTabController.of(context) 获取后监听 直接通过 _tabController.addListener()
编程式跳转 需先获取控制器引用 直接调用 _tabController.animateTo(index)
动态标签数量 修改 length 后会重建整个控制器 需要重新创建 TabController
适合场景 简单的标签页、固定标签数 复杂交互、需要精细控制
生命周期管理 自动管理 需要手动 dispose()
Widget 类型 可在 StatelessWidget 中使用 必须在 StatefulWidget 中使用

最佳实践：

  • 大多数简单的标签页场景，优先使用 DefaultTabController，代码更简洁。
  • 当你发现需要调用 animateTo() 或 addListener() 时，切换为手动 TabController。
  • 使用 TabController 时，dispose() 中的清理工作不可遗漏，否则会导致内存泄漏和"A TabController was used after being disposed"错误。
  • 当页面中只有一个 TabController 时，可以使用 SingleTickerProviderStateMixin 替代 TickerProviderStateMixin，前者性能略优，因为它不需要维护一个 Ticker 列表。

1.4 TickerProviderStateMixin 详解

Ticker 是 Flutter 动画系统的核心组件之一，它在每一帧回调时触发，驱动动画前进。TickerProviderStateMixin 的作用是让 State 对象成为一个能够提供 Ticker 的"工厂"。

// SingleTickerProviderStateMixin：只有一个动画控制器时使用
class _MyState extends State<MyWidget> with SingleTickerProviderStateMixin {
  late final TabController _tabController;

  @override
  void initState() {
    super.initState();
    _tabController = TabController(length: 3, vsync: this);
  }
  // ...
}

// TickerProviderStateMixin：有多个动画控制器时使用
class _MyState2 extends State<MyWidget> with TickerProviderStateMixin {
  late final TabController _tabController;
  late final AnimationController _animationController;

  @override
  void initState() {
    super.initState();
    _tabController = TabController(length: 3, vsync: this);
    _animationController = AnimationController(
      vsync: this,
      duration: const Duration(milliseconds: 300),
    );
  }
  // ...
}

为什么需要 vsync？因为动画需要与屏幕刷新率同步。如果当前页面不可见（如被另一个页面覆盖），TickerProviderStateMixin 会自动暂停 Ticker，从而避免在不可见时浪费 CPU 资源执行动画计算。

2. TabBar：标签栏样式定制

TabBar 是 Material Design 规范中的顶部标签栏组件，通常放置在 AppBar 的 bottom 属性中。它提供了丰富的样式定制选项。

2.1 基础结构与 Tab Widget

TabBar 的 tabs 属性接收一个 Widget 列表，通常使用 Tab Widget 来构建每个标签项。Tab 支持以下内容组合：

属性 说明
text 标签文字
icon 标签图标（显示在文字上方）
child 完全自定义的标签内容（与 text/icon 互斥）
height 标签高度（默认 72，仅有文字或仅有图标时为 46） 
// 仅文字
Tab(text: '首页')

// 仅图标
Tab(icon: Icon(Icons.home))

// 文字 + 图标
Tab(icon: Icon(Icons.home), text: '首页')

// 完全自定义
Tab(
  child: Row(
    mainAxisSize: MainAxisSize.min,
    children: [
      Icon(Icons.home),
      SizedBox(width: 4),
      Text('首页'),
      SizedBox(width: 4),
      // 显示未读数角标
      Container(
        padding: EdgeInsets.all(4),
        decoration: BoxDecoration(
          color: Colors.red,
          shape: BoxShape.circle,
        ),
        child: Text('3', style: TextStyle(color: Colors.white, fontSize: 10)),
      ),
    ],
  ),
)

2.2 指示器样式定制

TabBar 底部的指示器（indicator）是用户感知当前选中标签的关键视觉元素，Flutter 提供了多个属性来定制它的外观。

属性 类型 说明
indicator Decoration 完全自定义指示器装饰（如 BoxDecoration），设置后其他 indicator 属性会被忽略
indicatorColor Color 指示器颜色
indicatorWeight double 指示器厚度（默认 2.0）
indicatorSize TabBarIndicatorSize 指示器宽度模式：.tab（与标签同宽）或 .label（与文字同宽）
indicatorPadding EdgeInsets 指示器的内边距 
import 'package:flutter/material.dart';

void main() => runApp(const MyApp());

class MyApp extends StatelessWidget {
  const MyApp({super.key});

  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      title: 'TabBar 指示器样式',
      theme: ThemeData(
        colorSchemeSeed: Colors.blue,
        useMaterial3: true,
      ),
      home: const IndicatorStyleDemo(),
    );
  }
}

class IndicatorStyleDemo extends StatelessWidget {
  const IndicatorStyleDemo({super.key});

  @override
  Widget build(BuildContext context) {
    return DefaultTabController(
      length: 4,
      child: Scaffold(
        appBar: AppBar(
          title: const Text('指示器样式'),
          bottom: TabBar(
            // 指示器颜色
            indicatorColor: Colors.orange,
            // 指示器厚度
            indicatorWeight: 4.0,
            // 指示器宽度模式：与文字标签同宽（而非与整个 Tab 同宽）
            indicatorSize: TabBarIndicatorSize.label,
            tabs: const [
              Tab(text: '推荐'),
              Tab(text: '热门视频'),
              Tab(text: '直播'),
              Tab(text: '图片'),
            ],
          ),
        ),
        body: const TabBarView(
          children: [
            Center(child: Text('推荐内容')),
            Center(child: Text('热门视频内容')),
            Center(child: Text('直播内容')),
            Center(child: Text('图片内容')),
          ],
        ),
      ),
    );
  }
}

使用 indicator 属性可以创建更复杂的指示器效果，比如圆角矩形背景：

class MyApp extends StatelessWidget {

              Tab(text: '未读'),

        body: const TabBarView(
          children: [
            Center(child: Text('全部消息')),

    );
  }

2.4 可滚动标签栏与对齐方式

当标签数量较多时，TabBar 默认会将所有标签平均分布在可用空间内，这可能导致标签文字过于拥挤。此时可以使用 isScrollable 属性让标签栏支持水平滚动。

属性 类型 说明
isScrollable bool 是否可滚动（默认 false，标签平均分布）
tabAlignment TabAlignment 标签的对齐方式（仅在 isScrollable: true 时有意义）

TabAlignment 的可选值：

值 说明
TabAlignment.fill 标签均匀填满可用空间（默认值，不可滚动时）
TabAlignment.center 标签居中排列
TabAlignment.start 标签靠左排列
TabAlignment.startOffset 标签靠左排列但带有一段起始偏移 
import 'package:flutter/material.dart';

void main() => runApp(const MyApp());

class MyApp extends StatelessWidget {
  const MyApp({super.key});

  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      title: '可滚动标签栏',
      theme: ThemeData(
        colorSchemeSeed: Colors.blue,
        useMaterial3: true,
      ),
      home: const ScrollableTabBarDemo(),
    );
  }
}

class ScrollableTabBarDemo extends StatelessWidget {
  const ScrollableTabBarDemo({super.key});

  @override
  Widget build(BuildContext context) {
    // 模拟大量标签
    final tabs = [
      '推荐', '热门', '科技', '体育', '娱乐', '财经',
      '教育', '健康', '旅游', '美食', '时尚', '游戏',
    ];

    return DefaultTabController(
      length: tabs.length,
      child: Scaffold(
        appBar: AppBar(
          title: const Text('可滚动标签栏'),
          bottom: TabBar(
            // 启用滚动
            isScrollable: true,
            // 标签靠左排列
            tabAlignment: TabAlignment.start,
            tabs: tabs.map((t) => Tab(text: t)).toList(),
          ),
        ),
        body: TabBarView(
          children: tabs.map((t) {
            return Center(
              child: Text('$t 频道', style: const TextStyle(fontSize: 24)),
            );
          }).toList(),
        ),
      ),
    );
  }
}

注意事项：

  • isScrollable: false（默认）时，所有标签会平均分配整个 TabBar 的宽度。如果标签文字长度差异很大，短文字标签可能看起来有大量空白。
  • isScrollable: true 时，每个标签只占用自身需要的宽度，多余的标签可以通过水平滑动查看。
  • Material 3 中 isScrollable: true 的默认对齐方式是 TabAlignment.startOffset。如果你希望靠左没有偏移，显式设置 tabAlignment: TabAlignment.start。

2.5 分割线颜色

dividerColor 属性控制 TabBar 底部分割线的颜色。在 Material 3 主题下，TabBar 默认会显示一条底部分割线。

TabBar(
  // 设置分割线颜色
  dividerColor: Colors.grey.shade300,
  // 如果不想显示分割线，设为透明
  // dividerColor: Colors.transparent,
  tabs: const [
    Tab(text: '标签1'),
    Tab(text: '标签2'),
    Tab(text: '标签3'),
  ],
)

2.6 TabBar 不在 AppBar 中的用法

TabBar 不一定必须放在 AppBar.bottom 中，它可以放在页面的任何位置。这在某些设计中非常有用，比如内容区域中间有一组标签切换。

import 'package:flutter/material.dart';

void main() => runApp(const MyApp());

class MyApp extends StatelessWidget {
  const MyApp({super.key});

  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      title: 'TabBar 独立使用',
      theme: ThemeData(
        colorSchemeSeed: Colors.blue,
        useMaterial3: true,
      ),
      home: const TabBarOutsideAppBarDemo(),
    );
  }
}

class TabBarOutsideAppBarDemo extends StatelessWidget {
  const TabBarOutsideAppBarDemo({super.key});

  @override
  Widget build(BuildContext context) {
    return DefaultTabController(
      length: 3,
      child: Scaffold(
        appBar: AppBar(title: const Text('TabBar 独立使用')),
        body: Column(
          children: [
            // 顶部的固定内容
            Container(
              padding: const EdgeInsets.all(16),
              color: Colors.blue.shade50,
              width: double.infinity,
              child: const Text(
                '这里是页面顶部的固定内容区域',
                style: TextStyle(fontSize: 16),
                textAlign: TextAlign.center,
              ),
            ),
            // TabBar 放在 body 中间
            const TabBar(
              labelColor: Colors.blue,
              unselectedLabelColor: Colors.grey,
              tabs: [
                Tab(text: '全部'),
                Tab(text: '进行中'),
                Tab(text: '已完成'),
              ],
            ),
            // TabBarView 需要用 Expanded 包裹，因为它内部使用 PageView，需要确定的高度
            const Expanded(
              child: TabBarView(
                children: [
                  Center(child: Text('全部任务列表')),
                  Center(child: Text('进行中的任务')),
                  Center(child: Text('已完成的任务')),
                ],
              ),
            ),
          ],
        ),
      ),
    );
  }
}

注意事项：当 TabBarView 不在 Scaffold.body 中直接使用，而是嵌套在 Column 等布局组件中时，必须给它一个确定的高度约束。最常见的做法是用 Expanded 包裹，让它占满剩余空间。否则 TabBarView 内部的 PageView 会因为没有高度约束而报错。

3. TabBarView：标签页内容联动

TabBarView 是与 TabBar 配对使用的页面内容容器。它本质上是一个内置了手势识别的 PageView，能够通过左右滑动切换页面内容，并与 TabBar 的选中状态自动保持同步。

3.1 核心属性

属性 类型 说明
controller TabController? 标签控制器（不传则使用 DefaultTabController）
children List<Widget> 每个标签对应的页面内容
physics ScrollPhysics? 滑动物理效果
dragStartBehavior DragStartBehavior 拖拽开始行为
viewportFraction double 每个页面占据视口的比例（默认 1.0）

3.2 同步滚动原理

TabBar 和 TabBarView 之间的联动是通过共享同一个 TabController 实现的：

  1. 用户点击 TabBar 中的标签 → TabController 的 index 被更新 → TabBarView 监听到变化后滑动到对应页面
  2. 用户在 TabBarView 中左右滑动 → TabBarView 更新 TabController 的 index → TabBar 监听到变化后移动指示器

这种双向绑定让标签栏和内容区域始终保持同步。

3.3 禁用滑动切换

某些场景下（如页面内容本身有水平滚动，或者希望用户只能通过点击标签来切换），可以禁用 TabBarView 的滑动手势：

TabBarView(
  controller: _tabController,
  // 使用 NeverScrollableScrollPhysics 禁用手势滑动
  physics: const NeverScrollableScrollPhysics(),
  children: const [
    Center(child: Text('页面 1')),
    Center(child: Text('页面 2')),
    Center(child: Text('页面 3')),
  ],
)

最佳实践：

  • TabBar 和 TabBarView 的子项数量必须与 TabController.length 完全一致，否则会触发断言错误。在开发过程中如果遇到 "controller's length" 相关的报错，首先检查这三者的数量是否匹配。
  • 如果某个标签页的内容非常复杂（如含有长列表），建议将每个标签页提取为独立的 Widget，而非在 children 列表中内联构建，以保持代码可读性。
  • TabBarView 默认在切换页面时会销毁不可见的页面。如果需要保持页面状态，请参考本课后面的"KeepAlive"章节。

4.2 PageController 详解

PageController 是控制 PageView 行为的核心对象，它提供了页面跳转、初始页面、视口比例等功能。

属性/方法 说明
initialPage 初始显示的页面索引（默认 0）
viewportFraction 每个页面占视口宽度的比例（默认 1.0，设为小于 1 可实现卡片轮播效果）
keepPage 是否保存页面位置用于恢复（默认 true）
page 当前页面索引（double 类型，滑动过程中会有小数值）
jumpToPage(int) 无动画跳转到指定页面
animateToPage(int, ...) 带动画跳转到指定页面
nextPage(...) 跳转到下一页
previousPage(...) 跳转到上一页 
import 'package:flutter/material.dart';

void main() => runApp(const MyApp());

class MyApp extends StatelessWidget {
  const MyApp({super.key});

  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      title: 'PageController 示例',

class PageControllerDemo extends StatefulWidget {
  const PageControllerDemo({super.key});

  @override
  State<PageControllerDemo> createState() => _PageControllerDemoState();
}

class _PageControllerDemoState extends State<PageControllerDemo> {
  late final PageController _pageController;
  int _currentPage = 0;

  final List<Map<String, dynamic>> _pages = [
    {'title': '欢迎使用', 'color': Colors.blue, 'icon': Icons.waving_hand},
    {'title': '发现功能', 'color': Colors.green, 'icon': Icons.explore},
    {'title': '个性设置', 'color': Colors.orange, 'icon': Icons.settings},
    {'title': '开始体验', 'color': Colors.purple, 'icon': Icons.rocket_launch},
  ];

  @override
  void initState() {
    super.initState();
    _pageController = PageController(
      initialPage: 0,
      // viewportFraction 小于 1 可以让相邻页面露出一部分，实现卡片轮播效果
      // viewportFraction: 0.85,
    );

    // 监听页面滚动
    _pageController.addListener(() {
      // page 属性是 double 类型，滑动过程中会有小数值
      // 比如从第 0 页滑到第 1 页的中间位置时，page 约为 0.5
      final page = _pageController.page;
      if (page != null) {
        final newPage = page.round();
        if (newPage != _currentPage) {
          setState(() => _currentPage = newPage);
        }
      }
    });
  }

  @override
  void dispose() {
    _pageController.dispose();
    super.dispose();
  }

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(title: const Text('引导页示例')),
      body: Column(
        children: [
          Expanded(
            child: PageView(
              controller: _pageController,
              children: _pages.map((page) {
                return Container(
                  color: (page['color'] as Color).withValues(alpha: 0.15),
                  child: Center(
                    child: Column(
                      mainAxisAlignment: MainAxisAlignment.center,
                      children: [
                        Icon(
                          page['icon'] as IconData,
                          size: 80,
                          color: page['color'] as Color,
                        ),
                        const SizedBox(height: 24),
                        Text(
                          page['title'] as String,
                          style: TextStyle(
                            fontSize: 28,
                            fontWeight: FontWeight.bold,
                            color: page['color'] as Color,
                          ),
                        ),
                      ],
                    ),
                  ),
                );
              }).toList(),
            ),
          ),
          // 页面指示器（小圆点）
          Padding(
            padding: const EdgeInsets.all(16),
            child: Row(
              mainAxisAlignment: MainAxisAlignment.center,
              children: List.generate(_pages.length, (index) {
                return AnimatedContainer(
                  duration: const Duration(milliseconds: 300),
                  margin: const EdgeInsets.symmetric(horizontal: 4),
                  width: _currentPage == index ? 24 : 8,
                  height: 8,
                  decoration: BoxDecoration(
                    borderRadius: BorderRadius.circular(4),
                    color: _currentPage == index
                        ? _pages[index]['color'] as Color
                        : Colors.grey.shade300,
                  ),
                );
              }),
            ),
          ),
          // 导航按钮
          Padding(
            padding: const EdgeInsets.only(left: 16, right: 16, bottom: 32),
            child: Row(
              mainAxisAlignment: MainAxisAlignment.spaceBetween,
              children: [
                // 上一页按钮
                TextButton(
                  onPressed: _currentPage > 0
                      ? () {
                          _pageController.previousPage(
                            duration: const Duration(milliseconds: 300),
                            curve: Curves.easeInOut,
                          );
                        }
                      : null,
                  child: const Text('上一页'),
                ),
                // 下一页 / 完成按钮
                FilledButton(
                  onPressed: () {
                    if (_currentPage < _pages.length - 1) {
                      _pageController.nextPage(
                        duration: const Duration(milliseconds: 300),
                        curve: Curves.easeInOut,
                      );
                    } else {
                      ScaffoldMessenger.of(context).showSnackBar(
                        const SnackBar(content: Text('引导完成！')),
                      );
                    }
                  },
                  child: Text(
                    _currentPage < _pages.length - 1 ? '下一页' : '开始使用',
                  ),
                ),
              ],
            ),
          ),
        ],
      ),
    );
  }

4.4 viewportFraction 实现卡片轮播

将 PageController 的 viewportFraction 设为小于 1 的值，可以让相邻页面露出一部分，形成常见的卡片轮播效果：

import 'package:flutter/material.dart';

void main() => runApp(const MyApp());

class MyApp extends StatelessWidget {
  const MyApp({super.key});

  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      title: '卡片轮播',
      theme: ThemeData(
        colorSchemeSeed: Colors.purple,
        useMaterial3: true,
      ),
      home: const CarouselDemo(),
    );
  }
}

class CarouselDemo extends StatefulWidget {
  const CarouselDemo({super.key});

  @override
  State<CarouselDemo> createState() => _CarouselDemoState();
}

class _CarouselDemoState extends State<CarouselDemo> {
  // viewportFraction 为 0.85，每张卡片只占视口 85% 宽度
  // 左右两侧会露出相邻卡片的边缘
  final PageController _controller = PageController(viewportFraction: 0.85);

  final List<Color> _colors = [
    Colors.red,
    Colors.blue,
    Colors.green,
    Colors.orange,
    Colors.purple,
  ];

  @override
  void dispose() {
    _controller.dispose();
    super.dispose();
  }

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(title: const Text('卡片轮播效果')),
      body: Center(
        child: SizedBox(
          height: 250,
          child: PageView.builder(
            controller: _controller,
            itemCount: _colors.length,
            itemBuilder: (context, index) {
              return AnimatedBuilder(
                animation: _controller,
                builder: (context, child) {
                  // 根据当前滚动位置计算缩放比例
                  double value = 1.0;
                  if (_controller.position.haveDimensions) {
                    value = (_controller.page ?? 0) - index;
                    value = (1 - (value.abs() * 0.3)).clamp(0.0, 1.0);
                  }
                  return Center(
                    child: SizedBox(
                      height: Curves.easeOut.transform(value) * 250,
                      child: child,
                    ),
                  );
                },
                child: Container(
                  margin: const EdgeInsets.symmetric(horizontal: 8),
                  decoration: BoxDecoration(
                    color: _colors[index].shade200,
                    borderRadius: BorderRadius.circular(16),
                    boxShadow: [
                      BoxShadow(
                        color: _colors[index].withValues(alpha: 0.3),
                        blurRadius: 8,
                        offset: const Offset(0, 4),
                      ),
                    ],
                  ),
                  child: Center(
                    child: Text(
                      '卡片 ${index + 1}',
                      style: const TextStyle(
                        fontSize: 28,
                        fontWeight: FontWeight.bold,
                        color: Colors.white,
                      ),
                    ),
                  ),
                ),
              );
            },
          ),
        ),
      ),
    );
  }
}

注意事项：

  • PageController 和 ScrollController 类似，使用完毕后必须在 dispose() 中销毁。
  • PageController.page 属性在 PageView 尚未完成布局时可能为 null，使用时应做空值检查。
  • jumpToPage() 是立即跳转（无动画），animateToPage() 是带动画的跳转。在引导页场景中，通常使用 animateToPage() 以获得更好的用户体验。

5. PageView.builder：懒加载大量页面

5.1 为什么需要 builder

PageView 的默认构造函数会一次性创建所有子页面的 Widget，当页面数量很多时，这会造成不必要的内存占用和性能开销。PageView.builder 采用"按需构建"的策略，只有当页面即将显示时才会创建对应的 Widget，这与 ListView.builder 的设计思想一致。

构造方式 页面创建时机 适合场景
PageView(children: [...]) 一次性创建所有子 Widget 页面数量少（3~5 个）
PageView.builder(itemBuilder: ...) 按需创建，仅构建可见页面及邻近页面 页面数量多或不确定

5.2 基础用法

import 'package:flutter/material.dart';

void main() => runApp(const MyApp());

class MyApp extends StatelessWidget {
  const MyApp({super.key});

  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      title: 'PageView.builder',
      theme: ThemeData(
        colorSchemeSeed: Colors.indigo,
        useMaterial3: true,
      ),
      home: const PageViewBuilderDemo(),
    );
  }
}

class PageViewBuilderDemo extends StatelessWidget {
  const PageViewBuilderDemo({super.key});

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(title: const Text('PageView.builder 懒加载')),
      body: PageView.builder(
        // itemCount 指定总页面数量
        // 如果不设置 itemCount，则页面可以无限滚动
        itemCount: 100,
        itemBuilder: (context, index) {
          debugPrint('构建页面: $index'); // 观察哪些页面被实际构建
          return Container(
            color: Colors.primaries[index % Colors.primaries.length].shade100,
            child: Center(
              child: Column(
                mainAxisAlignment: MainAxisAlignment.center,
                children: [
                  Text(
                    '第 ${index + 1} 页',
                    style: const TextStyle(fontSize: 32),
                  ),
                  const SizedBox(height: 8),
                  Text(
                    '共 100 页',
                    style: TextStyle(
                      fontSize: 16,
                      color: Colors.grey.shade600,
                    ),
                  ),
                ],
              ),
            ),
          );
        },
      ),
    );
  }
}

运行上面的代码并观察控制台输出，你会发现 debugPrint 只在页面即将显示时被调用，而非一开始就构建所有 100 个页面。

5.3 无限滚动的 PageView

如果不设置 itemCount，PageView.builder 会变成一个可以无限滚动的页面容器。这在某些场景（如无限轮播图）中非常有用：

import 'package:flutter/material.dart';

void main() => runApp(const MyApp());

class MyApp extends StatelessWidget {
  const MyApp({super.key});

  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      title: '无限轮播',
      theme: ThemeData(
        colorSchemeSeed: Colors.indigo,
        useMaterial3: true,
      ),
      home: const InfinitePageViewDemo(),

class InfinitePageViewDemo extends StatelessWidget {
  const InfinitePageViewDemo({super.key});

  @override
  Widget build(BuildContext context) {
    // 实际的图片/内容数据
    final items = ['风景', '美食', '城市', '动物', '科技'];

    return Scaffold(
      appBar: AppBar(title: const Text('无限轮播')),
      body: Center(
        child: SizedBox(
          height: 300,
          child: PageView.builder(
            // 不设置 itemCount，实现无限滚动
            itemBuilder: (context, index) {
              // 通过取模运算循环使用数据
              final realIndex = index % items.length;
              return Container(
                margin: const EdgeInsets.all(16),
                decoration: BoxDecoration(
                  color: Colors
                      .primaries[realIndex % Colors.primaries.length].shade200,
                  borderRadius: BorderRadius.circular(16),

                    style: const TextStyle(fontSize: 28),

              );
            },
          ),
        ),
      ),
    );

}

最佳实践：

  • 当页面数量超过 5 个时，优先使用 PageView.builder 而非默认构造函数。
  • 无限滚动的 PageView（不设 itemCount）要注意 index 会不断增长，用 index % dataLength 取模来映射实际数据。
  • 如果每个页面需要加载网络数据，在 itemBuilder 中结合 FutureBuilder 或状态管理方案来实现按需加载。

6. KeepAlive 与 AutomaticKeepAliveClientMixin

6.1 问题：页面状态丢失

默认情况下，TabBarView 和 PageView 为了节省内存，会在页面不可见时销毁对应的 Widget（调用 dispose()），当页面再次可见时重新创建（调用 initState() 和 build()）。

这意味着用户在一个标签页中的操作状态（如滚动位置、输入的文字、选中的选项等）会在切换到其他标签后丢失。

import 'package:flutter/material.dart';

void main() => runApp(const MyApp());

class MyApp extends StatelessWidget {
  const MyApp({super.key});

  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      title: '状态丢失演示',
      theme: ThemeData(
        colorSchemeSeed: Colors.red,
        useMaterial3: true,
      ),
      home: const StateLossDemo(),
    );
  }
}

class StateLossDemo extends StatelessWidget {
  const StateLossDemo({super.key});

  @override
  Widget build(BuildContext context) {
    return DefaultTabController(
      length: 3,
      child: Scaffold(
        appBar: AppBar(
          title: const Text('状态丢失演示'),
          bottom: const TabBar(
            tabs: [
              Tab(text: '标签 A'),
              Tab(text: '标签 B'),
              Tab(text: '标签 C'),
            ],
          ),
        ),
        body: const TabBarView(
          children: [
            _CounterPage(name: 'A'),
            _CounterPage(name: 'B'),
            _CounterPage(name: 'C'),
          ],
        ),
      ),
    );
  }
}

// 没有使用 KeepAlive 的计数器页面
class _CounterPage extends StatefulWidget {

  const _CounterPage({required this.name});

    super.dispose();
  }

  @override
  Widget build(BuildContext context) {

      child: Column(
        mainAxisAlignment: MainAxisAlignment.center,
        children: [

          const SizedBox(height: 16),

          const SizedBox(height: 16),

            onPressed: () => setState(() => _count++),
            child: const Text('点击 +1'),

            '增加计数后切换标签，再切回来观察计数是否保留',

运行这个示例：在标签 A 中点击几次增加计数，然后切换到标签 C，再切回标签 A，你会发现计数器被重置为 0，控制台也会打印 dispose 和 initState 的日志。

6.2 解决方案：AutomaticKeepAliveClientMixin

AutomaticKeepAliveClientMixin 是 Flutter 提供的一个 Mixin，用于告诉父级滚动容器（如 TabBarView、PageView、ListView）："请不要销毁我，即使我不在屏幕上"。

使用步骤：

  1. 在 State 类上混入 AutomaticKeepAliveClientMixin
  2. 重写 wantKeepAlive 属性，返回 true
  3. 在 build 方法开头调用 super.build(context)
import 'package:flutter/material.dart';

void main() => runApp(const MyApp());

class MyApp extends StatelessWidget {
  const MyApp({super.key});

  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      title: 'KeepAlive 演示',
      theme: ThemeData(
        colorSchemeSeed: Colors.green,
        useMaterial3: true,
      ),
      home: const KeepAliveDemo(),
    );
  }
}

class KeepAliveDemo extends StatelessWidget {
  const KeepAliveDemo({super.key});

  @override
  Widget build(BuildContext context) {
    return DefaultTabController(
      length: 3,
      child: Scaffold(
        appBar: AppBar(
          title: const Text('KeepAlive 演示'),
          bottom: const TabBar(
            tabs: [
              Tab(text: '标签 A'),
              Tab(text: '标签 B'),
              Tab(text: '标签 C'),
            ],
          ),
        ),
        body: const TabBarView(
          children: [
            _KeepAliveCounterPage(name: 'A'),
            _KeepAliveCounterPage(name: 'B'),
            _KeepAliveCounterPage(name: 'C'),
          ],
        ),
      ),
    );
  }
}

class _KeepAliveCounterPage extends StatefulWidget {
  final String name;
  const _KeepAliveCounterPage({required this.name});

  @override
  State<_KeepAliveCounterPage> createState() => _KeepAliveCounterPageState();
}

// 第一步：混入 AutomaticKeepAliveClientMixin
class _KeepAliveCounterPageState extends State<_KeepAliveCounterPage>
    with AutomaticKeepAliveClientMixin {
  int _count = 0;

  // 第二步：重写 wantKeepAlive，返回 true 表示希望保持存活
  @override
  bool get wantKeepAlive => true;

  @override
  void initState() {
    super.initState();
    debugPrint('${widget.name} - initState（仅创建一次）');
  }

  @override
  void dispose() {
    debugPrint('${widget.name} - dispose（只在整个 TabBarView 被销毁时调用）');
    super.dispose();
  }

  @override
  Widget build(BuildContext context) {
    // 第三步：必须调用 super.build(context)
    // 这一步很容易遗漏，但它是触发 KeepAlive 机制的关键
    super.build(context);

    return Center(
      child: Column(
        mainAxisAlignment: MainAxisAlignment.center,
        children: [
          Text('标签 ${widget.name}', style: const TextStyle(fontSize: 24)),
          const SizedBox(height: 8),
          const Text(
            '(使用了 KeepAlive)',
            style: TextStyle(color: Colors.green, fontWeight: FontWeight.bold),
          ),
          const SizedBox(height: 16),
          Text('计数: $_count', style: const TextStyle(fontSize: 48)),
          const SizedBox(height: 16),
          ElevatedButton(
            onPressed: () => setState(() => _count++),
            child: const Text('点击 +1'),
          ),
          const SizedBox(height: 8),
          const Text(
            '增加计数后切换标签，再切回来——计数会被保留！',
            style: TextStyle(color: Colors.grey),
          ),
        ],
      ),
    );
  }
}

现在再运行：在标签 A 中增加计数，切换到标签 C，再切回标签 A，计数值会被保留。控制台也不会打印 dispose 日志（除非整个页面被销毁）。

6.3 wantKeepAlive 的动态控制

wantKeepAlive 不一定总是返回 true。你可以根据业务逻辑动态决定是否需要保持存活。当 wantKeepAlive 的返回值发生变化时，需要调用 updateKeepAlive() 方法来通知框架。

class _SmartKeepAlivePageState extends State<SmartKeepAlivePage>
    with AutomaticKeepAliveClientMixin {
  bool _hasUnsavedData = false;

  // 只在有未保存的数据时才保持存活
  @override
  bool get wantKeepAlive => _hasUnsavedData;

  void _onDataChanged() {
    setState(() {
      _hasUnsavedData = true;
      // 当 wantKeepAlive 返回值改变时，必须调用 updateKeepAlive()
      updateKeepAlive();
    });
  }

  void _onDataSaved() {
    setState(() {
      _hasUnsavedData = false;
      updateKeepAlive(); // 数据已保存，允许被销毁以释放内存
    });
  }

  @override
  Widget build(BuildContext context) {
    super.build(context);
    return const Placeholder(); // 实际界面内容
  }
}

6.4 KeepAlive 的工作原理

理解 AutomaticKeepAliveClientMixin 的内部机制有助于在合适的场景中正确使用它：

  1. 当 wantKeepAlive 返回 true 时，Mixin 会在 Widget 树中插入一个 KeepAlive 父组件（这是 ParentDataWidget）
  2. KeepAlive 通知其父级 SliverMultiBoxAdaptorElement（如 PageView/TabBarView 内部使用的）保留该子组件的 Element 和 State，即使它离开了可见区域
  3. super.build(context) 是触发这个通知的关键调用，遗漏它会导致 KeepAlive 不生效

最佳实践：

  • 不要对所有页面都无脑使用 KeepAlive。保持存活意味着页面占用的内存不会被释放，如果有大量页面（如几十个 Tab），全部 KeepAlive 会显著增加内存占用。
  • 推荐只对"用户有交互状态需要保留的页面"使用 KeepAlive，如：有输入内容的表单页、有滚动位置的长列表页、有已加载数据的网络请求页。
  • 对于纯展示性的页面（无用户交互状态），让它自然销毁和重建是更好的选择。
  • super.build(context) 这行代码绝对不能遗漏，否则 KeepAlive 机制不会生效。这是使用这个 Mixin 最容易犯的错误。
  • 如果使用了 AutomaticKeepAliveClientMixin，IDE（如 VS Code / Android Studio）可能会提示 "build 的返回值未使用"的警告，这是正常的，可以忽略。

7. 综合实战：新闻应用标签页

将本课所学的知识整合，构建一个模拟新闻应用的标签页界面：

import 'package:flutter/material.dart';

void main() => runApp(const NewsApp());

class NewsApp extends StatelessWidget {
  const NewsApp({super.key});

  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      title: '新闻 Tab 应用',
      theme: ThemeData(
        colorSchemeSeed: Colors.blue,
        useMaterial3: true,
      ),
      home: const NewsHomePage(),
    );
  }
}

class NewsHomePage extends StatefulWidget {
  const NewsHomePage({super.key});

  @override
  State<NewsHomePage> createState() => _NewsHomePageState();
}

class _NewsHomePageState extends State<NewsHomePage>
    with SingleTickerProviderStateMixin {
  late final TabController _tabController;

  final List<String> _categories = [
    '推荐', '热点', '科技', '体育', '娱乐', '财经', '教育', '健康',
  ];

  @override
  void initState() {
    super.initState();
    _tabController = TabController(
      length: _categories.length,
      vsync: this,
    );
  }

  @override
  void dispose() {
    _tabController.dispose();
    super.dispose();
  }

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(
        title: const Text('每日新闻'),
        actions: [
          IconButton(
            icon: const Icon(Icons.search),
            onPressed: () {},
          ),
        ],
        bottom: TabBar(
          controller: _tabController,
          isScrollable: true,
          tabAlignment: TabAlignment.start,
          labelStyle: const TextStyle(
            fontSize: 15,
            fontWeight: FontWeight.bold,
          ),
          unselectedLabelStyle: const TextStyle(fontSize: 14),
          indicatorSize: TabBarIndicatorSize.label,
          indicatorWeight: 3,
          dividerColor: Colors.transparent,
          tabs: _categories.map((c) => Tab(text: c)).toList(),
        ),
      ),
      body: TabBarView(
        controller: _tabController,
        children: _categories.map((category) {
          return _NewsCategoryPage(category: category);
        }).toList(),
      ),
    );
  }
}

// 每个分类的新闻列表页面，使用 KeepAlive 保持状态
class _NewsCategoryPage extends StatefulWidget {
  final String category;
  const _NewsCategoryPage({required this.category});

  @override
  State<_NewsCategoryPage> createState() => _NewsCategoryPageState();
}

class _NewsCategoryPageState extends State<_NewsCategoryPage>
    with AutomaticKeepAliveClientMixin {
  // 保持页面状态，避免切换标签时重新加载
  @override
  bool get wantKeepAlive => true;

  // 模拟新闻数据
  late final List<Map<String, String>> _news;

  @override
  void initState() {
    super.initState();
    // 模拟不同分类的新闻数据
    _news = List.generate(20, (index) {
      return {
        'title': '${widget.category}新闻标题 ${index + 1}',
        'summary': '这是关于${widget.category}的第 ${index + 1} 条新闻摘要内容，'
            '包含了详细的描述信息。',
        'time': '${(index + 1) * 5} 分钟前',
      };
    });
    debugPrint('${widget.category} 分类 - 数据加载完成');
  }

  @override
  Widget build(BuildContext context) {
    super.build(context); // KeepAlive 必须调用

    return ListView.separated(
      padding: const EdgeInsets.all(12),
      itemCount: _news.length,
      separatorBuilder: (_, __) => const Divider(height: 1),
      itemBuilder: (context, index) {
        final item = _news[index];
        return ListTile(
          contentPadding: const EdgeInsets.symmetric(
            vertical: 8,
            horizontal: 4,
          ),
          title: Text(
            item['title']!,
            style: const TextStyle(fontWeight: FontWeight.w600),
          ),
          subtitle: Padding(
            padding: const EdgeInsets.only(top: 4),
            child: Column(
              crossAxisAlignment: CrossAxisAlignment.start,
              children: [
                Text(
                  item['summary']!,
                  maxLines: 2,
                  overflow: TextOverflow.ellipsis,
                ),
                const SizedBox(height: 4),
                Text(
                  item['time']!,
                  style: TextStyle(
                    fontSize: 12,
                    color: Colors.grey.shade500,
                  ),
                ),
              ],
            ),
          ),
          trailing: ClipRRect(
            borderRadius: BorderRadius.circular(8),
            child: Container(
              width: 80,
              height: 60,
              color: Colors.primaries[index % Colors.primaries.length].shade100,
              child: const Icon(Icons.image, color: Colors.grey),
            ),
          ),
          onTap: () {
            ScaffoldMessenger.of(context).showSnackBar(
              SnackBar(content: Text('点击了: ${item['title']}')),
            );
          },
        );
      },
    );
  }
}

8. 本课小结

本课学习了 Flutter 中 Tab 导航与页面切换的完整知识体系：

知识点 核心内容
DefaultTabController 便捷的标签控制器，适合简单场景，自动管理生命周期
TabController 手动创建的标签控制器，需要 TickerProviderStateMixin，适合需要精细控制的场景
TabBar 标签栏组件，支持丰富的样式定制（指示器、文字样式、颜色、可滚动、对齐方式等）
TabBarView 标签页内容容器，与 TabBar 共享 TabController 实现双向联动
PageView 可滑动页面容器，通过 PageController 控制页面行为，physics 控制滑动效果
PageView.builder 按需构建的 PageView，适合大量页面的懒加载
AutomaticKeepAliveClientMixin 保持页面状态不被销毁的 Mixin，需要重写 wantKeepAlive 并调用 super.build

关键原则：

  1. TabBar + TabBarView 必须共享同一个 TabController，且三者的数量（length / tabs / children）必须一致
  2. TabController 需要 TickerProviderStateMixin 提供动画驱动
  3. PageView.builder 用于大量页面的懒加载，避免一次性创建所有页面
  4. KeepAlive 应有选择地使用，不要对所有页面都开启，以平衡用户体验与内存占用
  5. 别忘了在 dispose() 中清理 TabController 和 PageController

9. 练习题

练习 1：基础练习 — 简单标签页应用

使用 DefaultTabController 创建一个 3 个标签的应用，要求：

  • 三个标签分别是"聊天"、"联系人"、"动态"
  • 每个标签带有对应的图标（Icons.chat、Icons.contacts、Icons.dynamic_feed）
  • 自定义指示器样式：指示器颜色为绿色，宽度与文字标签同宽，厚度为 3
  • 选中标签文字加粗，未选中标签文字为灰色

练习 2：中级练习 — 带状态保持的标签页

使用手动 TabController 创建一个 4 个标签的应用（推荐、视频、直播、专栏），要求：

  • 使用 isScrollable: true 和 tabAlignment: TabAlignment.start
  • 每个标签页包含一个带有 TextField 的表单和一个计数器
  • 使用 AutomaticKeepAliveClientMixin 保持所有标签页的状态
  • 在标签切换时在控制台打印日志（使用 addListener）
  • 通过一个 FloatingActionButton 可以编程式地跳转到第一个标签

练习 3：中级练习 — 引导页

使用 PageView 和 PageController 实现一个 App 引导页，要求：

  • 包含 4 个引导页面，每页有图标、标题和描述文字
  • 底部有页面指示器（小圆点），当前页面对应的圆点高亮且更大
  • 有"跳过"按钮（跳转到最后一页）和"下一步"/"完成"按钮
  • 到达最后一页后，"下一步"按钮变为"开始使用"
  • 使用 viewportFraction: 0.9 让页面带有卡片间距效果

练习 4：进阶练习 — 新闻客户端综合页面

构建一个完整的新闻客户端页面，要求：

  • 顶部使用可滚动的 TabBar 显示至少 8 个新闻分类
  • 每个分类标签页中使用 ListView.builder 显示新闻列表（模拟 20 条数据）
  • 使用 AutomaticKeepAliveClientMixin 保持各分类的滚动位置
  • 顶部新闻列表的第一项使用 PageView.builder 实现自动轮播的头条新闻（提示：使用 Timer.periodic 配合 PageController.animateToPage 实现自动翻页）
  • 支持下拉刷新（RefreshIndicator），刷新时模拟 1 秒延迟
  • 自定义 TabBar 样式：选中标签加粗放大、使用自定义圆角指示器
