1. Flutter 手势体系概览

在 Flutter 中，用户的每一次触摸、滑动、缩放都需要经过一套完整的事件处理流水线。理解这条链路是掌握手势交互的前提。

事件处理流水线（从底到高）：

层级 核心类 职责
原始指针层 Listener / RawGestureDetector 直接接收操作系统传来的触摸坐标、按下/抬起等原始事件
手势识别层 GestureDetector 将原始指针事件组合识别为语义化手势（点击、拖动、缩放等）
Material 反馈层 InkWell / InkResponse 在手势基础上叠加 Material Design 的水波纹视觉反馈

Flutter 采用**手势竞技场（GestureArena）**机制来解决同一指针事件被多个手势识别器争抢的问题——这是 Flutter 手势系统最精妙也最容易踩坑的部分，我们将在最后一节深入讨论。

2. GestureDetector：手势识别的核心组件

GestureDetector 是 Flutter 中最常用的手势识别组件。它本身不产生任何视觉效果，而是在内部创建多个 GestureRecognizer，将子组件的指针事件翻译为语义化的手势回调。

2.1 点击手势

点击是最基础的手势，包含完整的按下→抬起生命周期。

import 'package:flutter/material.dart';

void main() => runApp(const MaterialApp(home: TapDemoPage()));

class TapDemoPage extends StatefulWidget {
  const TapDemoPage({super.key});

  @override
  State<TapDemoPage> createState() => _TapDemoPageState();
}

class _TapDemoPageState extends State<TapDemoPage> {
  String _info = '请点击下方区域';

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(title: const Text('点击手势演示')),
      body: Center(
        child: Column(
          mainAxisAlignment: MainAxisAlignment.center,
          children: [
            Text(_info, style: const TextStyle(fontSize: 16)),
            const SizedBox(height: 24),
            GestureDetector(
              // 手指按下（还未抬起）
              onTapDown: (TapDownDetails details) {
                setState(() {
                  _info = '按下位置: ${details.localPosition}';
                });
              },
              // 手指抬起，完成一次点击
              onTapUp: (TapUpDetails details) {
                setState(() {
                  _info = '抬起位置: ${details.localPosition}';
                });
              },
              // 完整的点击事件（onTapDown + onTapUp 之后触发）
              onTap: () {
                setState(() {
                  _info = '单击完成！';
                });
              },
              // 点击被取消（如手指滑出范围）
              onTapCancel: () {
                setState(() {
                  _info = '点击被取消';
                });
              },
              child: Container(
                width: 200,
                height: 200,
                decoration: BoxDecoration(
                  color: Colors.blue.shade100,
                  borderRadius: BorderRadius.circular(16),
                ),
                alignment: Alignment.center,
                child: const Text('点击我', style: TextStyle(fontSize: 20)),
              ),
            ),
          ],
        ),
      ),
    );
  }
}

点击手势回调触发顺序：onTapDown → onTapUp → onTap。如果手指在按下后滑出了组件区域，则触发 onTapCancel，而不会触发 onTapUp 和 onTap。

2.2 双击手势

GestureDetector(
  onDoubleTap: () {
    setState(() {
      _info = '双击触发！';
    });
  },
  child: Container(
    width: 200,
    height: 200,
    color: Colors.green.shade100,
    alignment: Alignment.center,
    child: const Text('双击我'),
  ),
)

注意事项：当同时注册了 onTap 和 onDoubleTap 时，第一次点击后 Flutter 会等待约 200ms 来判断是否为双击。这意味着 onTap 的响应会有延迟。如果你的场景不需要双击，就不要注册 onDoubleTap，以获得更快的单击响应。

长按手势提供了完整的生命周期：开始、移动、结束。

2.4 拖动手势

拖动是构建可交互 UI 的重要手势。Flutter 提供三种拖动回调：

回调 方向限制
onPanXxx 任意方向拖动（水平 + 垂直）
onHorizontalDragXxx 仅水平方向
onVerticalDragXxx 仅垂直方向

注意事项：onPanXxx 与 onHorizontalDragXxx / onVerticalDragXxx 互斥，不能同时注册。原因是它们使用的手势识别器会在竞技场中冲突。

拖动画布上移动组件 —— 完整示例

这是一个经典的可拖动色块示例，在画布上自由拖动多个组件：

import 'package:flutter/material.dart';

void main() => runApp(const MaterialApp(home: DragCanvasPage()));

class DragCanvasPage extends StatefulWidget {
  const DragCanvasPage({super.key});

  @override
  State<DragCanvasPage> createState() => _DragCanvasPageState();
}

/// 画布上的可拖动块数据
class DraggableBlock {
  Offset position;
  final Color color;
  final String label;

  DraggableBlock({
    required this.position,
    required this.color,
    required this.label,
  });
}

class _DragCanvasPageState extends State<DragCanvasPage> {
  final List<DraggableBlock> _blocks = [
    DraggableBlock(position: const Offset(50, 100), color: Colors.red, label: 'A'),
    DraggableBlock(position: const Offset(200, 200), color: Colors.blue, label: 'B'),
    DraggableBlock(position: const Offset(100, 350), color: Colors.green, label: 'C'),
  ];

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(title: const Text('拖动画布示例')),
      body: Stack(
        children: [
          // 背景网格提示
          Positioned.fill(
            child: CustomPaint(painter: _GridPainter()),
          ),
          // 为每个色块生成可拖动组件
          for (int i = 0; i < _blocks.length; i++)
            Positioned(
              left: _blocks[i].position.dx,
              top: _blocks[i].position.dy,
              child: GestureDetector(
                onPanDown: (_) {
                  // 将当前拖动的块移到最上层（列表末尾）
                  setState(() {
                    final block = _blocks.removeAt(i);
                    _blocks.add(block);
                  });
                },
                onPanUpdate: (DragUpdateDetails details) {
                  setState(() {
                    // 最后一个元素就是当前拖动的块
                    _blocks.last.position += details.delta;
                  });
                },
                child: Container(
                  width: 80,
                  height: 80,
                  decoration: BoxDecoration(
                    color: _blocks[i].color.withOpacity(0.8),
                    borderRadius: BorderRadius.circular(12),
                    boxShadow: [
                      BoxShadow(
                        color: Colors.black.withOpacity(0.2),
                        blurRadius: 8,
                        offset: const Offset(2, 4),
                      ),
                    ],
                  ),
                  alignment: Alignment.center,
                  child: Text(
                    _blocks[i].label,
                    style: const TextStyle(
                      color: Colors.white,
                      fontSize: 24,
                      fontWeight: FontWeight.bold,
                    ),
                  ),
                ),
              ),
            ),
        ],
      ),
    );
  }
}

/// 绘制背景网格
class _GridPainter extends CustomPainter {
  @override
  void paint(Canvas canvas, Size size) {
    final paint = Paint()
      ..color = Colors.grey.shade200
      ..strokeWidth = 0.5;

    // 绘制竖线
    for (double x = 0; x < size.width; x += 20) {
      canvas.drawLine(Offset(x, 0), Offset(x, size.height), paint);
    }
    // 绘制横线
    for (double y = 0; y < size.height; y += 20) {
      canvas.drawLine(Offset(0, y), Offset(size.width, y), paint);
    }
  }

  @override
  bool shouldRepaint(covariant CustomPainter oldDelegate) => false;
}

水平 / 垂直方向限制拖动

GestureDetector(
  // 仅水平方向拖动
  onHorizontalDragUpdate: (DragUpdateDetails details) {
    setState(() {
      _offsetX += details.delta.dx;
    });
  },
  // 仅垂直方向拖动
  onVerticalDragUpdate: (DragUpdateDetails details) {
    setState(() {
      _offsetY += details.delta.dy;
    });
  },
  child: Transform.translate(
    offset: Offset(_offsetX, _offsetY),
    child: const FlutterLogo(size: 80),
  ),
)

2.5 缩放手势

缩放手势（scale）用于检测双指捏合/张开操作，常见于图片查看器。实际上 scale 手势也能同时检测旋转和平移。

import 'package:flutter/material.dart';

void main() => runApp(const MaterialApp(home: ScaleGesturePage()));

class ScaleGesturePage extends StatefulWidget {
  const ScaleGesturePage({super.key});

  @override
  State<ScaleGesturePage> createState() => _ScaleGesturePageState();
}

class _ScaleGesturePageState extends State<ScaleGesturePage> {
  double _scale = 1.0;
  double _previousScale = 1.0;

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(title: const Text('缩放手势演示')),
      body: Center(
        child: GestureDetector(
          onScaleStart: (ScaleStartDetails details) {
            _previousScale = _scale;
          },
          onScaleUpdate: (ScaleUpdateDetails details) {
            setState(() {
              // 在上一次缩放基础上叠加新的缩放值
              _scale = (_previousScale * details.scale).clamp(0.5, 4.0);
            });
          },
          onScaleEnd: (ScaleEndDetails details) {
            // 缩放结束，可在此做动画回弹等逻辑
          },
          child: Transform.scale(
            scale: _scale,
            child: Image.network(
              'https://flutter.dev/assets/images/shared/brand/flutter/logo/flutter-lockup.png',
              width: 250,
            ),
          ),
        ),
      ),
    );
  }
}

注意事项：onScaleXxx 与 onPanXxx 互斥。如果你同时需要平移和缩放，请使用 onScaleXxx —— 它的 ScaleUpdateDetails 同时提供了 scale（缩放比）、rotation（旋转角度）和 focalPointDelta（焦点偏移/平移量）。

3. InkWell 与 InkResponse：带水波纹的触摸反馈

在 Material Design 中，用户触摸控件时应当有水波纹（ripple）反馈。InkWell 和 InkResponse 就是为此而生的组件。

3.1 InkWell

InkWell 产生矩形范围的水波纹效果，是最常用的带反馈触摸组件。

import 'package:flutter/material.dart';

void main() => runApp(const MaterialApp(home: InkWellDemoPage()));

class InkWellDemoPage extends StatelessWidget {
  const InkWellDemoPage({super.key});

    return Scaffold(

      body: Center(
        child: Column(
          mainAxisAlignment: MainAxisAlignment.center,
          children: [
            // 基础用法
            InkWell(
              onTap: () => debugPrint('点击'),
              onLongPress: () => debugPrint('长按'),
              // 水波纹颜色
              splashColor: Colors.blue.withOpacity(0.3),
              // 按下时的高亮色
              highlightColor: Colors.blue.withOpacity(0.1),
              // 圆角（让水波纹也跟着圆角裁剪）

}

3.2 InkResponse

InkResponse 是 InkWell 的父类，提供更灵活的水波纹形状控制。与 InkWell 的主要区别：

特性 InkWell InkResponse
水波纹形状 默认矩形（可设置 borderRadius） 默认从触摸点向外扩散圆形
裁剪行为 裁剪到组件矩形边界内 水波纹可以超出组件边界
适用场景 列表项、卡片等矩形区域 图标按钮、圆形头像等非矩形区域 
InkResponse(
  onTap: () => debugPrint('InkResponse 点击'),
  // 是否将水波纹裁剪在组件范围内
  containedInkWell: false,
  // 高亮形状
  highlightShape: BoxShape.circle,
  // 水波纹半径
  radius: 40,
  splashColor: Colors.amber.withOpacity(0.4),
  child: const Padding(
    padding: EdgeInsets.all(16),
    child: Icon(Icons.favorite, size: 32, color: Colors.red),
  ),
)

最佳实践：

  • 如果你的触摸区域是矩形或圆角矩形，优先使用 InkWell
  • 如果你需要圆形水波纹或水波纹超出组件范围的效果，使用 InkResponse
  • InkWell / InkResponse 需要 Material 祖先才能正确显示水波纹。如果在没有 Material 祖先的地方使用，水波纹不会显示。Scaffold 的 body 自带 Material，通常不需要额外包裹。

4. Listener：原始指针事件监听

Listener 直接监听操作系统传来的原始指针（pointer）事件，不经过手势识别器处理，也不参与手势竞技场竞争。

4.1 Listener 与 GestureDetector 的区别

特性 Listener GestureDetector
事件类型 原始指针事件（PointerEvent） 语义化手势（tap、drag、scale 等）
竞技场参与 不参与，总能收到事件 参与竞技场，可能输给其他手势识别器
手势识别 不做任何手势识别 内置各种 GestureRecognizer
适用场景 需要不被竞技场干扰的原始事件 常规的手势交互

4.2 基础用法

import 'package:flutter/material.dart';

void main() => runApp(const MaterialApp(home: ListenerDemoPage()));

class ListenerDemoPage extends StatefulWidget {
  const ListenerDemoPage({super.key});

  @override
  State<ListenerDemoPage> createState() => _ListenerDemoPageState();
}

class _ListenerDemoPageState extends State<ListenerDemoPage> {
  String _eventInfo = '触摸下方区域';
  Offset _pointerPosition = Offset.zero;

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(title: const Text('Listener 演示')),
      body: Center(
        child: Column(
          mainAxisAlignment: MainAxisAlignment.center,
          children: [
            Text(_eventInfo, style: const TextStyle(fontSize: 14)),
            const SizedBox(height: 8),
            Text('坐标: $_pointerPosition', style: const TextStyle(fontSize: 14)),
            const SizedBox(height: 24),
            Listener(
              // 指针按下
              onPointerDown: (PointerDownEvent event) {
                setState(() {
                  _eventInfo = '指针按下 (按钮: ${event.buttons})';
                  _pointerPosition = event.localPosition;
                });
              },
              // 指针移动
              onPointerMove: (PointerMoveEvent event) {
                setState(() {
                  _eventInfo = '指针移动 (压力: ${event.pressure.toStringAsFixed(2)})';
                  _pointerPosition = event.localPosition;
                });
              },
              // 指针抬起
              onPointerUp: (PointerUpEvent event) {
                setState(() {
                  _eventInfo = '指针抬起';
                  _pointerPosition = event.localPosition;
                });
              },
              // 指针取消（如系统弹窗中断）
              onPointerCancel: (PointerCancelEvent event) {
                setState(() {
                  _eventInfo = '指针取消';
                });
              },
              child: Container(
                width: 300,
                height: 300,
                decoration: BoxDecoration(
                  color: Colors.grey.shade100,
                  border: Border.all(color: Colors.grey),
                  borderRadius: BorderRadius.circular(12),
                ),
                alignment: Alignment.center,
                child: const Text('触摸追踪区域', style: TextStyle(fontSize: 16)),
              ),
            ),
          ],
        ),
      ),
    );
  }
}

最佳实践：当你需要不受手势竞技场影响地获取触摸坐标（如自定义绘制中追踪手指轨迹），使用 Listener 比 GestureDetector 更可靠。

5. AbsorbPointer 与 IgnorePointer：手势拦截与穿透

这两个组件用于控制子树是否能响应手势事件，常用于实现组件禁用、加载遮罩等场景。

5.1 AbsorbPointer —— 吸收指针事件

AbsorbPointer 像一堵墙：阻止子组件接收事件，但自身仍然参与命中测试（即自身仍是"可触摸的"，事件传播到此就被"吸收"了）。

import 'package:flutter/material.dart';

void main() => runApp(const MaterialApp(home: AbsorbIgnoreDemoPage()));

class AbsorbIgnoreDemoPage extends StatefulWidget {
  const AbsorbIgnoreDemoPage({super.key});

  @override
  State<AbsorbIgnoreDemoPage> createState() => _AbsorbIgnoreDemoPageState();
}

class _AbsorbIgnoreDemoPageState extends State<AbsorbIgnoreDemoPage> {
  bool _absorbing = false;
  bool _ignoring = false;
  int _tapCount = 0;

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(title: const Text('AbsorbPointer 与 IgnorePointer')),
      body: Padding(
        padding: const EdgeInsets.all(24),
        child: Column(
          children: [
            // 控制面板
            SwitchListTile(
              title: const Text('AbsorbPointer（吸收事件）'),
              value: _absorbing,
              onChanged: (v) => setState(() => _absorbing = v),
            ),
            SwitchListTile(
              title: const Text('IgnorePointer（忽略事件）'),
              value: _ignoring,
              onChanged: (v) => setState(() => _ignoring = v),
            ),
            const SizedBox(height: 24),
            Text('按钮点击次数: $_tapCount', style: const TextStyle(fontSize: 18)),
            const SizedBox(height: 16),

            // AbsorbPointer 示例
            const Text('AbsorbPointer 包裹的按钮：'),
            const SizedBox(height: 8),
            AbsorbPointer(
              absorbing: _absorbing,
              child: ElevatedButton(
                onPressed: () => setState(() => _tapCount++),
                child: const Text('点击我（Absorb）'),
              ),
            ),
            const SizedBox(height: 24),

            // IgnorePointer 示例
            const Text('IgnorePointer 包裹的按钮：'),
            const SizedBox(height: 8),
            IgnorePointer(
              ignoring: _ignoring,
              child: ElevatedButton(
                onPressed: () => setState(() => _tapCount++),
                child: const Text('点击我（Ignore）'),
              ),
            ),
          ],
        ),
      ),
    );
  }
}

5.2 两者的关键区别

特性 AbsorbPointer IgnorePointer
子组件是否收到事件 ❌ 不收到 ❌ 不收到
自身是否参与命中测试 ✅ 参与（事件到此为止） ❌ 不参与（事件穿透到下层）
典型用途 禁用某区域，但阻止事件穿透到下方 让事件穿透到层叠布局中的下层组件

形象比喻：

  • AbsorbPointer = 一块不透明的玻璃：你点不到它后面的东西，点击会被玻璃挡住
  • IgnorePointer = 一块透明的玻璃：不仅你点不到它包裹的内容，点击还会直接穿透到它后面的组件
// 经典场景：加载遮罩 —— 用 AbsorbPointer 阻止加载中的误操作
Stack(
  children: [
    // 底层：可交互的表单
    MyForm(),
    // 顶层：加载遮罩（吸收所有触摸事件）
    if (_isLoading)
      AbsorbPointer(
        child: Container(
          color: Colors.black26,
          alignment: Alignment.center,
          child: const CircularProgressIndicator(),
        ),
      ),
  ],
)

6. Dismissible：滑动删除

Dismissible 是实现列表项侧滑删除的标准组件，包含完整的滑动动画和回调。

6.1 完整的滑动删除列表示例

import 'package:flutter/material.dart';

void main() => runApp(const MaterialApp(home: DismissibleDemoPage()));

class DismissibleDemoPage extends StatefulWidget {
  const DismissibleDemoPage({super.key});

  @override
  State<DismissibleDemoPage> createState() => _DismissibleDemoPageState();
}

class _DismissibleDemoPageState extends State<DismissibleDemoPage> {
  final List<String> _items = List.generate(20, (i) => '列表项 ${i + 1}');

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(
        title: const Text('滑动删除示例'),
        actions: [
          // 重置按钮
          IconButton(
            onPressed: () {
              setState(() {
                _items
                  ..clear()
                  ..addAll(List.generate(20, (i) => '列表项 ${i + 1}'));
              });
            },
            icon: const Icon(Icons.refresh),
          ),
        ],
      ),
      body: ListView.builder(
        itemCount: _items.length,
        itemBuilder: (context, index) {
          final item = _items[index];
          return Dismissible(
            // key 是必须的，用于唯一标识每个 Dismissible
            key: ValueKey(item),
            // 滑动方向（默认双向）
            direction: DismissDirection.horizontal,
            // 向右滑时的背景（startToEnd）
            background: Container(
              color: Colors.green,
              alignment: Alignment.centerLeft,
              padding: const EdgeInsets.only(left: 24),
              child: const Icon(Icons.archive, color: Colors.white),
            ),
            // 向左滑时的背景（endToStart）
            secondaryBackground: Container(
              color: Colors.red,
              alignment: Alignment.centerRight,
              padding: const EdgeInsets.only(right: 24),
              child: const Icon(Icons.delete, color: Colors.white),
            ),
            // 确认是否删除（返回 false 则取消删除）
            confirmDismiss: (direction) async {
              if (direction == DismissDirection.endToStart) {
                // 向左滑 → 弹出确认对话框
                return await showDialog<bool>(
                  context: context,
                  builder: (context) => AlertDialog(
                    title: const Text('确认删除'),
                    content: Text('确定要删除 "$item" 吗？'),
                    actions: [
                      TextButton(
                        onPressed: () => Navigator.pop(context, false),
                        child: const Text('取消'),
                      ),
                      TextButton(
                        onPressed: () => Navigator.pop(context, true),
                        child: const Text('删除'),
                      ),
                    ],
                  ),
                ) ?? false;
              }
              // 向右滑 → 直接归档
              return true;
            },
            // 删除完成后的回调
            onDismissed: (direction) {
              setState(() {
                _items.removeAt(index);
              });
              ScaffoldMessenger.of(context).showSnackBar(
                SnackBar(
                  content: Text(
                    direction == DismissDirection.endToStart
                        ? '"$item" 已删除'
                        : '"$item" 已归档',
                  ),
                  action: SnackBarAction(
                    label: '撤销',
                    onPressed: () {
                      setState(() {
                        _items.insert(index, item);
                      });
                    },
                  ),
                ),
              );
            },
            child: ListTile(
              leading: CircleAvatar(child: Text('${index + 1}')),
              title: Text(item),
              subtitle: const Text('左滑删除，右滑归档'),
            ),
          );
        },
      ),
    );
  }
}

注意事项：

  1. key 参数是必须的，Dismissible 靠它来追踪每个列表项。建议使用 ValueKey 并传入唯一标识（如 id）。
  2. 在 onDismissed 中一定要从数据源移除对应项，否则会出现 "A dismissed Dismissible widget is still part of the tree" 错误。
  3. 使用 confirmDismiss 可以在滑动完成前询问用户确认，返回 false 则自动回弹。

7. Draggable 与 DragTarget：拖拽交互

Draggable 和 DragTarget 是一对配合使用的拖拽组件。Draggable 生成可拖动的元素，DragTarget 定义接收拖放的目标区域。它们通过泛型 T 传递数据。

7.1 基础拖放示例

import 'package:flutter/material.dart';

void main() => runApp(const MaterialApp(home: DraggableDemoPage()));

class DraggableDemoPage extends StatefulWidget {
  const DraggableDemoPage({super.key});

  @override
  State<DraggableDemoPage> createState() => _DraggableDemoPageState();
}

class _DraggableDemoPageState extends State<DraggableDemoPage> {
  Color _targetColor = Colors.grey.shade300;
  String _targetText = '将色块拖到这里';

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(title: const Text('Draggable 与 DragTarget')),
      body: Column(
        mainAxisAlignment: MainAxisAlignment.spaceEvenly,
        children: [
          // 可拖动的色块行
          Row(
            mainAxisAlignment: MainAxisAlignment.spaceEvenly,
            children: [
              _buildDraggableColor(Colors.red, '红色'),
              _buildDraggableColor(Colors.green, '绿色'),
              _buildDraggableColor(Colors.blue, '蓝色'),
            ],
          ),
          // 拖放目标区域
          DragTarget<MapEntry<Color, String>>(
            builder: (context, candidateData, rejectedData) {
              // candidateData 包含当前悬停在目标区域上方的数据
              final isHovering = candidateData.isNotEmpty;
              return AnimatedContainer(
                duration: const Duration(milliseconds: 200),
                width: 200,
                height: 200,
                decoration: BoxDecoration(
                  color: _targetColor,
                  borderRadius: BorderRadius.circular(16),
                  border: Border.all(
                    color: isHovering ? Colors.amber : Colors.grey,
                    width: isHovering ? 3 : 1,
                  ),
                ),
                alignment: Alignment.center,
                child: Text(
                  _targetText,
                  style: TextStyle(
                    fontSize: 16,
                    color: _targetColor.computeLuminance() > 0.5
                        ? Colors.black
                        : Colors.white,
                  ),
                ),
              );
            },
            // 判断是否接受拖入的数据
            onWillAcceptWithDetails: (details) {
              return true; // 接受所有颜色
            },
            // 数据放入目标后触发
            onAcceptWithDetails: (details) {
              setState(() {
                _targetColor = details.data.key;
                _targetText = '已接收: ${details.data.value}';
              });
            },
          ),
        ],
      ),
    );
  }

  Widget _buildDraggableColor(Color color, String label) {
    return Draggable<MapEntry<Color, String>>(
      // 拖动时传递的数据
      data: MapEntry(color, label),
      // 拖动时跟随手指的反馈组件
      feedback: Material(
        elevation: 8,
        borderRadius: BorderRadius.circular(12),
        child: Container(
          width: 80,
          height: 80,
          decoration: BoxDecoration(
            color: color.withOpacity(0.8),
            borderRadius: BorderRadius.circular(12),
          ),
          alignment: Alignment.center,
          child: Text(label, style: const TextStyle(color: Colors.white)),
        ),
      ),
      // 拖动期间原位置显示的占位组件
      childWhenDragging: Container(
        width: 80,
        height: 80,
        decoration: BoxDecoration(
          color: color.withOpacity(0.2),
          borderRadius: BorderRadius.circular(12),
          border: Border.all(color: color, style: BorderStyle.solid),
        ),
      ),
      // 拖动开始
      onDragStarted: () => debugPrint('开始拖动 $label'),
      // 拖动结束（无论是否放到目标上）
      onDragEnd: (DraggableDetails details) {
        debugPrint('拖动结束 $label，是否被接受: ${details.wasAccepted}');
      },
      // 正常状态下显示的子组件
      child: Container(
        width: 80,
        height: 80,
        decoration: BoxDecoration(
          color: color,
          borderRadius: BorderRadius.circular(12),
        ),
        alignment: Alignment.center,
        child: Text(label, style: const TextStyle(color: Colors.white)),
      ),
    );
  }
}

7.2 Draggable 核心属性

属性 类型 说明
data T 拖动时携带的数据（泛型）
child Widget 正常状态显示的组件
feedback Widget 拖动中跟随手指的组件
childWhenDragging Widget? 拖动期间原位置显示的组件（为 null 则仍显示 child）
onDragStarted VoidCallback? 拖动开始回调
onDragEnd Function(DraggableDetails)? 拖动结束回调
onDragCompleted VoidCallback? 数据被 DragTarget 接受时触发
onDraggableCanceled Function(Velocity, Offset)? 拖动取消（未被接受）时触发

7.3 DragTarget 核心属性

属性 类型 说明
builder Function(context, candidateData, rejectedData) 构建目标区域 UI，candidateData 为悬停数据
onWillAcceptWithDetails Function(DragTargetDetails<T>)? 判断是否接受数据（返回 bool）
onAcceptWithDetails Function(DragTargetDetails<T>)? 数据被放入并接受时触发
onLeave Function(T?)? 拖动元素离开目标区域时触发

8. LongPressDraggable：长按拖拽

LongPressDraggable 继承自 Draggable，唯一区别是：需要长按后才能开始拖动，而非直接拖动。

8.1 与 Draggable 的区别

特性 Draggable LongPressDraggable
触发方式 直接拖动 长按后才能拖动
适用场景 专门用于拖放的元素 列表重排序等场景（不影响正常滚动）
与滚动的关系 可能与 ListView 滚动冲突 长按触发避免了滚动冲突

8.2 示例：长按拖拽排序

import 'package:flutter/material.dart';

void main() => runApp(const MaterialApp(home: LongPressDraggablePage()));

class LongPressDraggablePage extends StatefulWidget {
  const LongPressDraggablePage({super.key});

  @override
  State<LongPressDraggablePage> createState() => _LongPressDraggablePageState();
}

class _LongPressDraggablePageState extends State<LongPressDraggablePage> {
  final List<String> _items = ['苹果', '香蕉', '橙子', '葡萄', '西瓜'];

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(title: const Text('长按拖拽排序')),
      body: ListView.builder(
        itemCount: _items.length,
        itemBuilder: (context, index) {
          final item = _items[index];
          return DragTarget<int>(
            onWillAcceptWithDetails: (details) => details.data != index,
            onAcceptWithDetails: (details) {
              setState(() {
                final draggedItem = _items.removeAt(details.data);
                _items.insert(index, draggedItem);
              });
            },
            builder: (context, candidateData, rejectedData) {
              return LongPressDraggable<int>(
                data: index,
                // 长按延迟时间
                delay: const Duration(milliseconds: 300),
                feedback: Material(
                  elevation: 8,
                  child: Container(
                    width: MediaQuery.of(context).size.width,
                    padding: const EdgeInsets.all(16),
                    color: Colors.blue.shade50,
                    child: Text(item, style: const TextStyle(fontSize: 18)),
                  ),
                ),
                childWhenDragging: Container(
                  color: Colors.grey.shade200,
                  height: 56,
                ),
                child: Container(
                  decoration: BoxDecoration(
                    color: candidateData.isNotEmpty
                        ? Colors.blue.shade50
                        : Colors.white,
                    border: const Border(
                      bottom: BorderSide(color: Colors.grey, width: 0.5),
                    ),
                  ),
                  child: ListTile(
                    leading: const Icon(Icons.drag_handle),
                    title: Text(item),
                  ),
                ),
              );
            },
          );
        },
      ),
    );
  }
}

最佳实践：在列表中使用拖拽排序时，优先考虑 LongPressDraggable 而非 Draggable，因为直接拖动会与列表的滚动手势冲突。另外，Flutter 内置的 ReorderableListView 已经封装了长按拖拽排序的完整逻辑，简单场景可以直接使用。

9.2 使用 TransformationController 编程控制

TransformationController 允许你以编程方式获取和设置当前的变换矩阵，实现"重置缩放"、"缩放到指定位置"等功能。

import 'package:flutter/material.dart';

void main() => runApp(const MaterialApp(home: InteractiveViewerControllerPage()));

class InteractiveViewerControllerPage extends StatefulWidget {
  const InteractiveViewerControllerPage({super.key});

  @override
  State<InteractiveViewerControllerPage> createState() =>
      _InteractiveViewerControllerPageState();
}

class _InteractiveViewerControllerPageState
    extends State<InteractiveViewerControllerPage>
    with SingleTickerProviderStateMixin {
  final TransformationController _controller = TransformationController();
  late AnimationController _animationController;
  Animation<Matrix4>? _animation;

  @override
  void initState() {
    super.initState();
    _animationController = AnimationController(
      vsync: this,
      duration: const Duration(milliseconds: 300),
    );
  }

  @override
  void dispose() {
    _controller.dispose();
    _animationController.dispose();
    super.dispose();
  }

  /// 带动画的重置缩放
  void _resetTransform() {
    _animation = Matrix4Tween(
      begin: _controller.value,
      end: Matrix4.identity(),
    ).animate(CurvedAnimation(
      parent: _animationController,
      curve: Curves.easeOut,
    ));

    _animation!.addListener(() {
      _controller.value = _animation!.value;
    });

    _animationController
      ..reset()
      ..forward();
  }

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(title: const Text('TransformationController')),
      body: InteractiveViewer(
        transformationController: _controller,
        boundaryMargin: const EdgeInsets.all(100),
        minScale: 0.5,
        maxScale: 5.0,
        child: Container(
          width: 400,
          height: 400,
          decoration: BoxDecoration(
            border: Border.all(color: Colors.blue, width: 2),
          ),
          child: GridView.builder(
            physics: const NeverScrollableScrollPhysics(),
            gridDelegate: const SliverGridDelegateWithFixedCrossAxisCount(
              crossAxisCount: 4,
            ),
            itemCount: 16,
            itemBuilder: (context, index) {
              return Container(
                margin: const EdgeInsets.all(4),
                color: Colors.primaries[index % Colors.primaries.length],
                alignment: Alignment.center,
                child: Text(
                  '${index + 1}',
                  style: const TextStyle(
                    color: Colors.white,
                    fontWeight: FontWeight.bold,
                  ),
                ),
              );
            },
          ),
        ),
      ),
      floatingActionButton: FloatingActionButton(
        onPressed: _resetTransform,
        child: const Icon(Icons.refresh),
      ),
    );
  }
}

10. 手势冲突与手势竞技场机制（GestureArena）

这是 Flutter 手势系统中最重要也最容易被忽视的部分。当多个手势识别器同时监听同一区域时，Flutter 用**手势竞技场（GestureArena）**来决定"谁赢"。

10.1 什么是手势竞技场

想象一个拳击擂台赛：

  1. 报名：当用户手指按下（pointer down）时，命中区域内所有注册了手势识别器的组件都会将各自的识别器"报名"到竞技场
  2. 比赛：随着后续指针事件（移动、抬起）的到来，各识别器根据自己的规则判断"这是不是我要识别的手势"
  3. 宣布胜者：
    • 某个识别器确信当前事件匹配自己的手势 → 宣布胜出（win）
    • 某个识别器确信不匹配 → 主动退出（reject）
    • 只剩最后一个识别器 → 自动胜出
  4. 结果：胜出的识别器接管后续所有事件，其他识别器被清除

10.2 常见冲突场景

场景1：外层 GestureDetector 与内层 GestureDetector 的 onTap 冲突

import 'package:flutter/material.dart';

void main() => runApp(const MaterialApp(home: GestureConflictPage()));

class GestureConflictPage extends StatefulWidget {
  const GestureConflictPage({super.key});

  @override
  State<GestureConflictPage> createState() => _GestureConflictPageState();
}

class _GestureConflictPageState extends State<GestureConflictPage> {
  String _log = '';

  void _addLog(String msg) {
    setState(() {
      _log = '$msg\n$_log';
    });
  }

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(title: const Text('手势冲突演示')),
      body: Padding(
        padding: const EdgeInsets.all(16),
        child: Column(
          children: [
            // 外层 GestureDetector
            GestureDetector(
              onTap: () => _addLog('外层 onTap'),
              child: Container(
                color: Colors.amber.shade50,
                padding: const EdgeInsets.all(40),
                child: GestureDetector(
                  // 内层 GestureDetector
                  onTap: () => _addLog('内层 onTap'),
                  child: Container(
                    width: 150,
                    height: 150,
                    color: Colors.amber.shade200,
                    alignment: Alignment.center,
                    child: const Text('点击我'),
                  ),
                ),
              ),
            ),
            const SizedBox(height: 16),
            const Divider(),
            const Text('事件日志（新事件在上）：',
                style: TextStyle(fontWeight: FontWeight.bold)),
            const SizedBox(height: 8),
            Expanded(
              child: SingleChildScrollView(
                child: Text(_log),
              ),
            ),
          ],
        ),
      ),
    );
  }
}

点击内层容器时，你会发现只有内层的 onTap 被触发。这是因为在竞技场中，内层的 TapGestureRecognizer 和外层的 TapGestureRecognizer 同时参赛，而内层先"报名"（命中测试从叶子节点开始），最终内层胜出。

场景2：onTap 与 onVerticalDrag 冲突

当 GestureDetector 同时注册 onTap 和 onVerticalDragXxx 时，Flutter 的竞技场需要等更多事件来判断——用户到底是想点击还是想拖动：

  • 如果手指移动距离超过阈值 → 拖动识别器胜出
  • 如果手指在小范围内抬起 → 点击识别器胜出

10.3 手势消歧规则

Flutter 手势竞技场的核心消歧规则：

  1. 最具体者胜出：在嵌套组件中，子组件的手势识别器优先于父组件
  2. 先确认者胜出：谁先有把握确认当前手势匹配，谁就胜出
  3. 最后存活者胜出：其他识别器都 reject 后，最后一个自动 win
  4. 同类互斥：同一指针上不能有两个相同类型的手势同时胜出

10.4 使用 RawGestureDetector 解决冲突

当默认的竞技场机制不满足需求时，可以使用 RawGestureDetector 直接控制手势识别器的行为。

import 'package:flutter/gestures.dart';
import 'package:flutter/material.dart';

void main() => runApp(const MaterialApp(home: RawGestureDemoPage()));

class RawGestureDemoPage extends StatefulWidget {
  const RawGestureDemoPage({super.key});

  @override
  State<RawGestureDemoPage> createState() => _RawGestureDemoPageState();
}

class _RawGestureDemoPageState extends State<RawGestureDemoPage> {
  String _outerLog = '';
  String _innerLog = '';

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(title: const Text('RawGestureDetector 示例')),
      body: Padding(
        padding: const EdgeInsets.all(16),
        child: Column(
          children: [
            // 外层使用 RawGestureDetector 自定义竞技场策略
            RawGestureDetector(
              gestures: <Type, GestureRecognizerFactory>{
                // 自定义一个"总是胜出"的 Tap 识别器
                _AlwaysWinTapRecognizer:
                    GestureRecognizerFactoryWithHandlers<_AlwaysWinTapRecognizer>(
                  () => _AlwaysWinTapRecognizer(),
                  (_AlwaysWinTapRecognizer recognizer) {
                    recognizer.onTap = () {
                      setState(() => _outerLog = '外层被点击（始终胜出）');
                    };
                  },
                ),
              },
              child: Container(
                color: Colors.blue.shade50,
                padding: const EdgeInsets.all(40),
                child: GestureDetector(
                  onTap: () {
                    setState(() => _innerLog = '内层被点击');
                  },
                  child: Container(
                    width: 150,
                    height: 150,
                    color: Colors.blue.shade200,
                    alignment: Alignment.center,
                    child: const Text('点击内层'),
                  ),
                ),
              ),
            ),
            const SizedBox(height: 24),
            Text('外层日志: $_outerLog'),
            Text('内层日志: $_innerLog'),
          ],
        ),
      ),
    );
  }
}

/// 自定义 TapGestureRecognizer：在竞技场中始终主动胜出
class _AlwaysWinTapRecognizer extends TapGestureRecognizer {
  @override
  void rejectGesture(int pointer) {
    // 拒绝被拒绝 —— 即使竞技场想让我退出，我也强行接受
    acceptGesture(pointer);
  }
}

10.5 实用技巧：让父子手势同时响应

默认情况下父子的相同手势只有一个能胜出。如果你需要父子都响应，可以使用上面的 RawGestureDetector 技巧，或者使用 Listener 作为其中一方（因为 Listener 不参与竞技场）：

// 方案：外层用 Listener 监听原始指针事件（不参与竞技场）
Listener(
  onPointerDown: (event) {
    debugPrint('外层感知到按下');
  },
  child: GestureDetector(
    onTap: () {
      debugPrint('内层正常识别点击');
    },
    child: Container(
      width: 200,
      height: 200,
      color: Colors.teal.shade100,
      alignment: Alignment.center,
      child: const Text('两层都能响应'),
    ),
  ),
)

最佳实践：

  1. 大多数场景不需要手动干预竞技场，Flutter 的默认规则能很好地工作
  2. 如果遇到手势冲突，先检查是否注册了互斥的手势组合（如 onPan + onHorizontalDrag）
  3. 需要"穿透"手势时，考虑用 Listener 替代外层的 GestureDetector
  4. 只有在真正需要自定义竞技场策略时才使用 RawGestureDetector
  5. 调试手势问题时，可以用 debugPrintGestureArenaDiagnostics = true 打印竞技场详细日志

11. 本课小结

本课系统讲解了 Flutter 手势与交互体系的九大知识点：

  1. GestureDetector：最核心的手势识别组件，支持点击、双击、长按、拖动、缩放五大类手势。通过 onPanUpdate + Positioned / Transform.translate 可以实现画布自由拖动。

  2. InkWell 与 InkResponse：在手势基础上叠加 Material Design 水波纹反馈。InkWell 适合矩形区域，InkResponse 适合自定义形状。需要 Material 祖先才能正常工作。

  3. Listener：直接监听原始指针事件，不参与手势竞技场。适用于需要"总是能收到事件"的场景，如自定义绘制的触摸追踪。

  4. AbsorbPointer 与 IgnorePointer：前者吸收事件（子组件无法接收，事件不穿透），后者忽略事件（子组件无法接收，事件穿透到下层）。常用于加载遮罩和组件禁用。

  5. Dismissible：标准的滑动删除组件，key 是必须参数。confirmDismiss 可在删除前弹出确认，onDismissed 中必须从数据源移除对应项。

  6. Draggable 与 DragTarget：通过泛型传递数据的拖放交互组合。feedback 定义拖动时的视觉效果，DragTarget 的 onAcceptWithDetails 处理放入逻辑。

  7. LongPressDraggable：长按后才可拖动的变体，解决了直接拖动与列表滚动冲突的问题。

  8. InteractiveViewer：开箱即用的缩放平移组件，通过 TransformationController 可以编程控制变换状态。

  9. 手势竞技场（GestureArena）：Flutter 解决手势冲突的核心机制。理解"报名→比赛→胜出/淘汰"的流程，是排查手势问题的关键。RawGestureDetector 允许自定义竞技场策略。

核心原则：Flutter 的手势处理遵循"指针事件向上冒泡，手势识别通过竞技场消歧"的设计。组件层级越深（越靠近叶子节点），在竞技场中优先级越高。
