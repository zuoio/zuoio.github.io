第04课 图片与图标组件

图片和图标是移动应用中最常见的视觉元素。Flutter 提供了丰富的组件来加载和展示不同来源的图片，同时内置了强大的 Material Icons 图标系统。本课将系统讲解 Image、FadeInImage、Icon、ImageIcon 和 CircleAvatar 等核心组件的使用方法与最佳实践。

1. Image 组件概述

Image 是 Flutter 中用于展示图片的核心组件。它支持多种图片来源：本地资源（asset）、网络（network）、文件系统（file）和内存字节（memory）。每种来源都有对应的命名构造函数。

Image 组件的常用通用属性：

属性 类型 说明
width double? 图片显示宽度
height double? 图片显示高度
fit BoxFit? 图片填充模式（详见第3节）
alignment Alignment 图片在容器内的对齐方式
repeat ImageRepeat 图片平铺重复方式
color Color? 混合颜色
colorBlendMode BlendMode? 颜色混合模式
filterQuality FilterQuality 图片渲染质量
semanticLabel String? 无障碍语义标签

2. Image.asset — 加载本地资源图片

2.1 什么是 Asset 图片

Asset（资源）图片是打包在应用安装包内的图片文件。它们在编译时就已确定，不依赖网络，加载速度快、可靠性高。适用于应用图标、背景图、引导页插图等固定素材。

2.2 在 pubspec.yaml 中配置 assets

要使用 asset 图片，首先需要将图片文件放到项目目录中（通常放在 assets/images/ 下），然后在 pubspec.yaml 中声明：

flutter:
  assets:
    # 声明单个文件
    - assets/images/logo.png
    # 声明整个目录（注意结尾的斜杠）
    - assets/images/

推荐的项目图片目录结构：

my_app/
├── assets/
│   └── images/
│       ├── logo.png
│       ├── 2.0x/
│       │   └── logo.png
│       └── 3.0x/
│           └── logo.png
├── lib/
└── pubspec.yaml

Flutter 支持分辨率适配：根目录下的图片为 1x 分辨率，2.0x/ 和 3.0x/ 子目录中放置对应高分辨率图片。运行时 Flutter 会根据设备的 devicePixelRatio 自动选择最合适的图片。

2.3 使用 Image.asset

import 'package:flutter/material.dart';

void main() => runApp(const MyApp());

class MyApp extends StatelessWidget {
  const MyApp({super.key});

  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      home: Scaffold(
        appBar: AppBar(title: const Text('Image.asset 示例')),
        body: Center(
          child: Column(
            mainAxisAlignment: MainAxisAlignment.center,
            children: [
              // 基础用法：加载 assets/images/logo.png
              Image.asset(
                'assets/images/logo.png',
                width: 200,
                height: 200,
              ),
              const SizedBox(height: 20),
              // 指定适配模式和圆角裁剪
              ClipRRect(
                borderRadius: BorderRadius.circular(16),
                child: Image.asset(
                  'assets/images/logo.png',
                  width: 150,
                  height: 150,
                  fit: BoxFit.cover,
                ),
              ),
            ],
          ),
        ),
      ),
    );
  }
}

2.4 注意事项

注意： 修改 pubspec.yaml 中的 assets 配置后，需要停止应用并重新运行（热重载不会生效），否则可能出现 Unable to load asset 错误。

最佳实践： 将 asset 路径定义为常量，避免在多处硬编码字符串，方便统一管理和修改。

// 推荐：集中管理 asset 路径
class AppAssets {
  AppAssets._();
  static const String logo = 'assets/images/logo.png';
  static const String background = 'assets/images/background.jpg';
  static const String placeholder = 'assets/images/placeholder.png';
}

// 使用时
Image.asset(AppAssets.logo, width: 100, height: 100);

3. Image.network — 网络图片加载

3.1 基本用法

Image.network 用于从 URL 加载并显示网络图片。这是最常见的图片加载方式之一，适用于展示用户上传的头像、服务器返回的商品图等动态内容。

import 'package:flutter/material.dart';

void main() => runApp(const MyApp());

class MyApp extends StatelessWidget {
  const MyApp({super.key});

  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      home: Scaffold(
        appBar: AppBar(title: const Text('Image.network 示例')),
        body: Center(
          child: Image.network(
            'https://picsum.photos/300/200',
            width: 300,
            height: 200,
            fit: BoxFit.cover,
          ),
        ),
      ),
    );
  }
}

3.2 loadingBuilder — 加载进度占位

网络图片加载需要时间，用户等待期间应该展示加载指示器，提升用户体验。loadingBuilder 会在图片加载过程中被反复调用，你可以根据加载进度显示不同的 UI。

import 'package:flutter/material.dart';

void main() => runApp(const MyApp());

class MyApp extends StatelessWidget {
  const MyApp({super.key});

  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      home: Scaffold(
        appBar: AppBar(title: const Text('loadingBuilder 示例')),
        body: Center(
          child: Image.network(
            'https://picsum.photos/600/400',
            width: 300,
            height: 200,
            fit: BoxFit.cover,
            // loadingBuilder 在图片加载过程中被调用
            // child 是最终的图片组件
            // loadingProgress 包含加载进度信息，加载完成时为 null
            loadingBuilder: (
              BuildContext context,
              Widget child,
              ImageChunkEvent? loadingProgress,
            ) {
              if (loadingProgress == null) {
                // 加载完成，返回图片
                return child;
              }
              // 加载中，显示进度指示器
              return SizedBox(
                width: 300,
                height: 200,
                child: Center(
                  child: CircularProgressIndicator(
                    // 计算加载百分比
                    value: loadingProgress.expectedTotalBytes != null
                        ? loadingProgress.cumulativeBytesLoaded /
                            loadingProgress.expectedTotalBytes!
                        : null,
                  ),
                ),
              );
            },
          ),
        ),
      ),
    );
  }
}

3.3 errorBuilder — 加载失败处理

网络图片可能因为 URL 无效、网络异常等原因加载失败。使用 errorBuilder 可以在加载失败时展示友好的替代 UI，避免显示破碎的图片图标。

import 'package:flutter/material.dart';

void main() => runApp(const MyApp());

class MyApp extends StatelessWidget {
  const MyApp({super.key});

  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      home: Scaffold(
        appBar: AppBar(title: const Text('errorBuilder 示例')),
        body: Center(
          child: Image.network(
            // 故意使用一个无效的 URL 来触发错误
            'https://invalid-url-example.com/not-found.jpg',
            width: 300,
            height: 200,
            fit: BoxFit.cover,
            // 加载失败时的回调
            errorBuilder: (
              BuildContext context,
              Object error,
              StackTrace? stackTrace,
            ) {
              return Container(
                width: 300,
                height: 200,
                decoration: BoxDecoration(
                  color: Colors.grey[200],
                  borderRadius: BorderRadius.circular(8),
                ),
                child: const Column(
                  mainAxisAlignment: MainAxisAlignment.center,
                  children: [
                    Icon(Icons.broken_image, size: 48, color: Colors.grey),
                    SizedBox(height: 8),
                    Text(
                      '图片加载失败',
                      style: TextStyle(color: Colors.grey),
                    ),
                  ],
                ),
              );
            },
          ),
        ),
      ),
    );
  }
}

3.4 综合示例：loadingBuilder + errorBuilder

实际项目中，建议同时使用 loadingBuilder 和 errorBuilder，构建完善的图片加载体验：

import 'package:flutter/material.dart';

void main() => runApp(const MyApp());

class MyApp extends StatelessWidget {
  const MyApp({super.key});

  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      home: Scaffold(
        appBar: AppBar(title: const Text('网络图片完整示例')),
        body: Center(
          child: NetworkImageWithFeedback(
            url: 'https://picsum.photos/400/300',
            width: 300,
            height: 200,
          ),
        ),
      ),
    );
  }
}

/// 封装了加载状态和错误处理的网络图片组件
class NetworkImageWithFeedback extends StatelessWidget {
  final String url;
  final double width;
  final double height;

  const NetworkImageWithFeedback({
    super.key,
    required this.url,
    required this.width,
    required this.height,
  });

  @override
  Widget build(BuildContext context) {
    return ClipRRect(
      borderRadius: BorderRadius.circular(12),
      child: Image.network(
        url,
        width: width,
        height: height,
        fit: BoxFit.cover,
        loadingBuilder: (context, child, loadingProgress) {
          if (loadingProgress == null) return child;
          return Container(
            width: width,
            height: height,
            color: Colors.grey[100],
            child: Center(
              child: CircularProgressIndicator(
                value: loadingProgress.expectedTotalBytes != null
                    ? loadingProgress.cumulativeBytesLoaded /
                        loadingProgress.expectedTotalBytes!
                    : null,
              ),
            ),
          );
        },
        errorBuilder: (context, error, stackTrace) {
          return Container(
            width: width,
            height: height,
            color: Colors.grey[200],
            child: const Icon(
              Icons.error_outline,
              color: Colors.red,
              size: 40,
            ),
          );
        },
      ),
    );
  }
}

3.5 注意事项

注意： Android 9（API 28）及以上默认禁止 HTTP 明文请求，只允许 HTTPS。如果需要加载 HTTP 图片，需要在 android/app/src/main/AndroidManifest.xml 的 <application> 标签中添加 android:usesCleartextTraffic="true"。

最佳实践： Flutter 内置的 Image.network 没有磁盘缓存能力，每次重建 Widget 都可能重新请求。生产项目中推荐使用 cached_network_image 包，它提供了磁盘缓存、占位图、错误图等完整功能。

// 使用 cached_network_image 的示例（需先添加依赖）
// pubspec.yaml:
//   dependencies:
//     cached_network_image: ^3.3.0

import 'package:cached_network_image/cached_network_image.dart';

CachedNetworkImage(
  imageUrl: 'https://picsum.photos/400/300',
  placeholder: (context, url) => const CircularProgressIndicator(),
  errorWidget: (context, url, error) => const Icon(Icons.error),
  fit: BoxFit.cover,
);

4. Image.file 与 Image.memory — 从文件和内存加载

4.1 Image.file — 从设备文件系统加载

Image.file 用于加载设备本地文件系统中的图片，比如用户拍照后保存在相册中的图片、从网络下载缓存到本地的图片等。它接收一个 dart:io 的 File 对象。

典型使用场景：

  • 显示用户通过相机拍摄的照片
  • 显示从图片选择器选取的本地图片
  • 显示下载并缓存到本地的图片文件
import 'dart:io';
import 'package:flutter/material.dart';

void main() => runApp(const MyApp());

class MyApp extends StatelessWidget {
  const MyApp({super.key});

  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      home: Scaffold(
        appBar: AppBar(title: const Text('Image.file 示例')),
        body: const Center(
          child: FileImageDemo(),
        ),
      ),
    );
  }
}

class FileImageDemo extends StatefulWidget {
  const FileImageDemo({super.key});

  @override
  State<FileImageDemo> createState() => _FileImageDemoState();
}

class _FileImageDemoState extends State<FileImageDemo> {
  File? _imageFile;

  void _loadImage() {
    // 实际项目中，通常通过 image_picker 等插件获取文件路径
    // 这里用一个假设路径作为演示
    setState(() {
      _imageFile = File('/storage/emulated/0/DCIM/photo.jpg');
    });
  }

  @override
  Widget build(BuildContext context) {
    return Column(
      mainAxisAlignment: MainAxisAlignment.center,
      children: [
        if (_imageFile != null)
          Image.file(
            _imageFile!,
            width: 300,
            height: 300,
            fit: BoxFit.cover,
            // 同样支持 errorBuilder
            errorBuilder: (context, error, stackTrace) {
              return Container(
                width: 300,
                height: 300,
                color: Colors.grey[200],
                child: const Center(child: Text('无法加载图片')),
              );
            },
          )
        else
          Container(
            width: 300,
            height: 300,
            color: Colors.grey[200],
            child: const Center(
              child: Text('尚未选择图片', style: TextStyle(color: Colors.grey)),
            ),
          ),
        const SizedBox(height: 20),
        ElevatedButton(
          onPressed: _loadImage,
          child: const Text('选择图片'),
        ),
      ],
    );
  }
}

4.2 Image.memory — 从内存字节加载

Image.memory 接收一个 Uint8List（字节数组）来显示图片。适用于以下场景：

  • 从数据库（如 SQLite）中读取的 BLOB 图片数据
  • 通过 API 返回的 Base64 编码图片
  • 动态生成的图片数据
import 'dart:convert';
import 'dart:typed_data';
import 'package:flutter/material.dart';

void main() => runApp(const MyApp());

class MyApp extends StatelessWidget {
  const MyApp({super.key});

  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      home: Scaffold(
        appBar: AppBar(title: const Text('Image.memory 示例')),
        body: const Center(
          child: MemoryImageDemo(),
        ),
      ),
    );
  }
}

class MemoryImageDemo extends StatelessWidget {
  const MemoryImageDemo({super.key});

  @override
  Widget build(BuildContext context) {
    // 模拟一个 Base64 编码的图片数据
    // 实际项目中，这些数据通常来自 API 响应或数据库
    const String base64Image =
        'iVBORw0KGgoAAAANSUhEUgAAAAEAAAABCAYAAAAfFcSJAAAADUlEQVR4'
        'nGNgYPgPAAEDAQAIicLsAAAAASUVORK5CYII=';

    // 将 Base64 字符串解码为 Uint8List
    final Uint8List imageBytes = base64Decode(base64Image);

        const Text(
          '上方图片由 Base64 数据解码而来',
          style: TextStyle(color: Colors.grey),
        ),
      ],
    );
  }
}

4.3 注意事项

注意： Image.file 依赖 dart:io，因此在 Web 平台上不可用。如果需要跨平台兼容，请使用条件导入或直接使用 Image.network。

最佳实践（内存管理）： Image.memory 会将整个图片数据保持在内存中。对于大尺寸图片，应该考虑使用 cacheWidth 和 cacheHeight 参数来降低解码后的图片尺寸，从而减少内存占用：

Image.memory(
  largeImageBytes,
  cacheWidth: 300,  // 解码后缓存宽度限制为 300 像素
  fit: BoxFit.cover,
);

5. BoxFit 详解

BoxFit 是 Flutter 中控制子内容（不仅限于图片）如何适配父容器尺寸的枚举类型。它在 Image、FittedBox、DecorationImage 等多个组件中广泛使用。理解每种模式的行为差异对于正确展示图片至关重要。

5.1 七种 BoxFit 模式说明

BoxFit 值 行为 可能裁剪 可能留白 保持比例
fill 拉伸填满整个容器 否 否 否
contain 等比缩放，完整显示，可能留白 否 是 是
cover 等比缩放，填满容器，可能裁剪 是 否 是
fitWidth 等比缩放，宽度撑满 可能 可能 是
fitHeight 等比缩放，高度撑满 可能 可能 是
none 不缩放，原始尺寸居中显示 可能 可能 是
scaleDown 仅缩小不放大（类似 contain 但不放大） 否 是 是

详细解释每种模式：

  • BoxFit.fill：不管原始图片的宽高比，强制拉伸（或压缩）图片以完全填充容器。图片不会被裁剪，也不会留白，但宽高比可能被破坏，导致图片变形。适用于对比例要求不严格的纯色或渐变背景。

  • BoxFit.contain：在保持宽高比的前提下，尽可能放大图片，使其完整地显示在容器内。图片的某个方向会刚好填满容器，另一个方向则可能留白。适用于需要完整展示图片内容的场景（如查看详情图）。

  • BoxFit.cover：在保持宽高比的前提下，尽可能缩放图片，使其完全覆盖容器（不留白）。超出容器的部分会被裁剪。这是最常用的模式，适用于头像、封面图、卡片背景等场景。

  • BoxFit.fitWidth：在保持宽高比的前提下，缩放图片使其宽度刚好等于容器宽度。如果图片高度超出容器，则上下裁剪；如果不足，则上下留白。

  • BoxFit.fitHeight：在保持宽高比的前提下，缩放图片使其高度刚好等于容器高度。如果图片宽度超出容器，则左右裁剪；如果不足，则左右留白。

  • BoxFit.none：不进行任何缩放，以图片的原始尺寸在容器中央显示。如果图片大于容器，超出部分被裁剪；如果小于容器，周围留白。

  • BoxFit.scaleDown：行为类似 contain，但有一个关键区别——它只会缩小，不会放大。如果图片原始尺寸已经小于容器，则保持原始尺寸（与 none 效果相同）；如果图片大于容器，则缩小到完全显示在容器内（与 contain 效果相同）。

5.2 统一对比示例

以下示例在一个页面中展示所有七种 BoxFit 模式的效果，便于直观对比：

import 'package:flutter/material.dart';

void main() => runApp(const MyApp());

class MyApp extends StatelessWidget {
  const MyApp({super.key});

  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      home: Scaffold(
        appBar: AppBar(title: const Text('BoxFit 模式对比')),
        body: const BoxFitComparison(),
      ),
    );
  }
}

class BoxFitComparison extends StatelessWidget {
  const BoxFitComparison({super.key});

  @override
  Widget build(BuildContext context) {
    // 所有 BoxFit 模式及其中文说明
    const Map<BoxFit, String> fitModes = {
      BoxFit.fill: 'fill\n拉伸填满，可能变形',
      BoxFit.contain: 'contain\n完整显示，可能留白',
      BoxFit.cover: 'cover\n填满容器，可能裁剪',
      BoxFit.fitWidth: 'fitWidth\n宽度撑满',
      BoxFit.fitHeight: 'fitHeight\n高度撑满',
      BoxFit.none: 'none\n原始尺寸，不缩放',
      BoxFit.scaleDown: 'scaleDown\n仅缩小，不放大',
    };

    return SingleChildScrollView(
      padding: const EdgeInsets.all(16),
      child: Column(
        crossAxisAlignment: CrossAxisAlignment.start,
        children: [
          const Text(
            '下方每个方框大小为 200×150，图片原始比例不同于方框比例，\n'
            '可以观察不同 BoxFit 模式下图片的显示差异。',
            style: TextStyle(fontSize: 14, color: Colors.black54),
          ),
          const SizedBox(height: 16),
          Wrap(
            spacing: 16,
            runSpacing: 16,
            children: fitModes.entries.map((entry) {
              return Column(
                children: [
                  // 外部容器，用边框标识大小
                  Container(
                    width: 200,
                    height: 150,
                    decoration: BoxDecoration(
                      border: Border.all(color: Colors.blue, width: 2),
                    ),
                    child: Image.network(
                      'https://picsum.photos/seed/flutter/400/600',
                      fit: entry.key,
                    ),
                  ),
                  const SizedBox(height: 4),
                  Text(
                    entry.value,
                    textAlign: TextAlign.center,
                    style: const TextStyle(fontSize: 12),
                  ),
                ],
              );
            }).toList(),
          ),
        ],
      ),
    );
  }
}

5.3 如何选择合适的 BoxFit

使用场景 推荐 BoxFit
头像、封面图、卡片背景 cover
图片详情查看、需完整显示 contain
缩略图列表 cover
固定尺寸的装饰性背景 fill（对变形不敏感时）
不确定图片是否比容器大 scaleDown
横幅 Banner fitWidth

最佳实践： 在大多数需要展示图片的场景中，BoxFit.cover 是最安全的选择。它既能保证容器被完全填充（不留白），又能保持图片的宽高比（不变形）。但要注意，被裁剪的部分可能包含重要内容，此时可以配合 alignment 参数控制裁剪区域的偏移。

// 确保人脸（通常在上方）不被裁剪
Image.network(
  'https://example.com/portrait.jpg',
  fit: BoxFit.cover,
  alignment: Alignment.topCenter,  // 从顶部开始显示，裁剪底部
);

6. FadeInImage — 渐入效果加载图片

6.1 为什么需要 FadeInImage

当网络图片加载时，直接从空白跳到图片显示会显得生硬。FadeInImage 提供了从占位图到目标图片的平滑渐入过渡效果，让用户体验更加流畅自然。

6.2 配合 AssetImage 作为占位图

import 'package:flutter/material.dart';

void main() => runApp(const MyApp());

class MyApp extends StatelessWidget {
  const MyApp({super.key});

  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      home: Scaffold(
        appBar: AppBar(title: const Text('FadeInImage 示例')),
        body: Center(
          child: ClipRRect(
            borderRadius: BorderRadius.circular(12),
            child: FadeInImage(
              // 占位图：使用本地 asset 图片
              placeholder: const AssetImage('assets/images/placeholder.png'),
              // 目标图：网络图片
              image: const NetworkImage('https://picsum.photos/400/300'),
              width: 300,
              height: 200,
              fit: BoxFit.cover,
              // 渐入动画持续时间
              fadeInDuration: const Duration(milliseconds: 300),
              // 占位图渐出动画持续时间
              fadeOutDuration: const Duration(milliseconds: 100),
            ),
          ),
        ),
      ),
    );
  }
}

6.3 使用透明占位图（无需 asset 文件）

如果不想准备 asset 占位图，可以使用 transparent_image 包提供的透明 GIF，或者使用 FadeInImage.memoryNetwork：

6.4 使用 assetNetwork 便捷构造

FadeInImage.assetNetwork 是一个简化的命名构造函数，直接接收 asset 路径和网络 URL 字符串：

import 'package:flutter/material.dart';

6.5 注意事项

最佳实践： 占位图建议使用体积非常小的图片（如低分辨率缩略图、纯色图或加载动画 GIF），这样占位图本身可以瞬间加载。如果占位图也是网络图片，那就失去了"占位"的意义。

注意： FadeInImage 不支持 errorBuilder。如果需要同时处理渐入效果和错误状态，建议使用 cached_network_image 包，或自行封装 Image.network 配合 AnimatedSwitcher。

7. Icon 与 ImageIcon — 图标组件

7.1 Icon — Material Icons 使用

Icon 是 Flutter 中用于显示字体图标的组件。Flutter 内置了完整的 Material Icons 图标库，包含数千个常用图标，无需额外下载或配置即可使用。

字体图标相比图片图标有以下优势：

  • 矢量化：任意缩放不会模糊
  • 体积小：所有图标共用一个字体文件
  • 易着色：通过 color 属性即可改变颜色
  • 无障碍支持：可通过 semanticLabel 添加语义描述
import 'package:flutter/material.dart';

void main() => runApp(const MyApp());

class MyApp extends StatelessWidget {
  const MyApp({super.key});

  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      home: Scaffold(
        appBar: AppBar(title: const Text('Icon 示例')),
        body: Center(
          child: Column(
            mainAxisAlignment: MainAxisAlignment.center,
            children: [
              // 基础用法
              const Icon(Icons.home, size: 48),
              const SizedBox(height: 16),

              // 指定颜色和大小
              const Icon(
                Icons.favorite,
                size: 48,
                color: Colors.red,
                semanticLabel: '收藏',  // 无障碍标签
              ),
              const SizedBox(height: 16),

              // 不同风格的图标
              const Row(
                mainAxisAlignment: MainAxisAlignment.center,
                children: [
                  // 填充样式（默认）
                  Icon(Icons.bookmark, size: 40, color: Colors.blue),
                  SizedBox(width: 16),
                  // 轮廓样式
                  Icon(Icons.bookmark_border, size: 40, color: Colors.blue),
                  SizedBox(width: 16),
                  // 圆角样式
                  Icon(Icons.bookmark_rounded, size: 40, color: Colors.blue),
                  SizedBox(width: 16),
                  // 尖角样式
                  Icon(Icons.bookmark_sharp, size: 40, color: Colors.blue),
                ],
              ),
              const SizedBox(height: 24),

              // 常用 Material Icons 展示
              Wrap(
                spacing: 20,
                runSpacing: 20,
                children: const [
                  _LabeledIcon(icon: Icons.search, label: 'search'),
                  _LabeledIcon(icon: Icons.settings, label: 'settings'),
                  _LabeledIcon(icon: Icons.person, label: 'person'),
                  _LabeledIcon(icon: Icons.notifications, label: 'notifications'),

                  _LabeledIcon(icon: Icons.cloud_upload, label: 'cloud_upload'),

            ],

class _LabeledIcon extends StatelessWidget {
  final IconData icon;

}

提示： 完整的 Material Icons 列表可以在 fonts.google.com/icons 上浏览和搜索。在 Flutter 代码中通过 Icons.xxx 引用即可。

7.2 ImageIcon — 使用图片作为图标

ImageIcon 用于将图片（通常是 PNG）作为图标显示。它会将图片作为模板（template），用指定的颜色填充非透明区域。适用于使用自定义设计的图标图片场景。

import 'package:flutter/material.dart';

void main() => runApp(const MyApp());

class MyApp extends StatelessWidget {
  const MyApp({super.key});

  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      home: Scaffold(
        appBar: AppBar(title: const Text('ImageIcon 示例')),
        body: Center(
          child: Column(
            mainAxisAlignment: MainAxisAlignment.center,
            children: [
              // 使用本地 asset 图片作为图标
              const ImageIcon(
                AssetImage('assets/icons/custom_icon.png'),
                size: 48,
                color: Colors.blue,
              ),
              const SizedBox(height: 20),
              // 使用网络图片作为图标
              const ImageIcon(
                NetworkImage('https://example.com/icon.png'),
                size: 48,
                color: Colors.green,
              ),
              const SizedBox(height: 20),
              // 不同颜色展示同一个图标
              Row(
                mainAxisAlignment: MainAxisAlignment.center,
                children: [
                  Colors.red,
                  Colors.orange,
                  Colors.green,
                  Colors.blue,
                  Colors.purple,
                ].map((color) {
                  return Padding(
                    padding: const EdgeInsets.symmetric(horizontal: 8),
                    child: ImageIcon(
                      const AssetImage('assets/icons/custom_icon.png'),
                      size: 36,
                      color: color,
                    ),
                  );
                }).toList(),
              ),
            ],
          ),
        ),
      ),
    );
  }
}

7.3 自定义图标字体

除了使用 Material Icons 和图片图标，还可以使用自定义图标字体（如 Iconfont 或 FlutterIcon）。步骤如下：

第一步：下载字体文件，放入项目目录

my_app/
├── assets/
│   └── fonts/
│       └── my_icons.ttf

第二步：在 pubspec.yaml 中声明字体

flutter:
  fonts:
    - family: MyIcons
      fonts:
        - asset: assets/fonts/my_icons.ttf

第三步：定义 IconData 并使用

import 'package:flutter/material.dart';

/// 自定义图标数据类
/// 每个图标的 codePoint 值从图标字体网站导出时可以查看
class MyIcons {
  MyIcons._();

  static const String _fontFamily = 'MyIcons';

  // Unicode 码点值来自图标字体导出配置
  static const IconData wechat = IconData(0xe600, fontFamily: _fontFamily);
  static const IconData alipay = IconData(0xe601, fontFamily: _fontFamily);
  static const IconData taobao = IconData(0xe602, fontFamily: _fontFamily);
}

void main() => runApp(const MyApp());

class MyApp extends StatelessWidget {
  const MyApp({super.key});

  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      home: Scaffold(
        appBar: AppBar(title: const Text('自定义图标字体')),
        body: const Center(
          child: Row(
            mainAxisAlignment: MainAxisAlignment.center,
            children: [
              Icon(MyIcons.wechat, size: 48, color: Colors.green),
              SizedBox(width: 24),
              Icon(MyIcons.alipay, size: 48, color: Colors.blue),
              SizedBox(width: 24),
              Icon(MyIcons.taobao, size: 48, color: Colors.orange),
            ],
          ),
        ),
      ),
    );
  }
}

7.4 注意事项

Icon vs ImageIcon 选择建议：

  • 优先使用 Icon + Material Icons，因为它们是矢量图标，体积小，渲染快
  • 当需要品牌图标或 Material Icons 中没有的图标时，考虑自定义图标字体
  • 只有当图标是复杂的多色图标且无法用字体表示时，才考虑使用 ImageIcon 或直接用 Image

最佳实践： 使用 IconTheme 或 Theme 统一管理图标的默认大小和颜色，避免在每个 Icon 上重复设置：

IconTheme(
  data: const IconThemeData(
    size: 28,
    color: Colors.blueGrey,
  ),
  child: Row(
    children: const [
      Icon(Icons.home),      // 自动使用 28px 和 blueGrey 颜色
      Icon(Icons.search),
      Icon(Icons.person),
    ],
  ),
);

8. CircleAvatar — 圆形头像组件

8.1 基本介绍

CircleAvatar 是 Material Design 中用于展示圆形头像的专用组件。它不仅仅是一个圆形裁剪，还内置了背景色、文字首字母缩写等功能，是展示用户头像的首选组件。

常用属性：

属性 类型 说明
backgroundImage ImageProvider? 背景图片（头像图片）
backgroundColor Color? 背景颜色
foregroundColor Color? 前景颜色（文字颜色）
child Widget? 子组件（通常是文字或图标）
radius double? 圆的半径
minRadius double? 最小半径
maxRadius double? 最大半径
foregroundImage ImageProvider? 前景图片（覆盖在 child 上方）
onBackgroundImageError ImageErrorListener? 背景图加载失败回调
onForegroundImageError ImageErrorListener? 前景图加载失败回调

8.4 注意事项

注意： CircleAvatar 的 backgroundImage 属性接收的是 ImageProvider（如 NetworkImage、AssetImage），而不是 Image Widget。这是初学者常犯的错误。

// 正确 ✅
CircleAvatar(backgroundImage: NetworkImage('https://...'));
CircleAvatar(backgroundImage: AssetImage('assets/avatar.png'));

// 错误 ❌
CircleAvatar(backgroundImage: Image.network('https://...'));  // 编译报错

最佳实践： 始终为 CircleAvatar 设置 backgroundColor 和一个 child（如文字或图标），当 backgroundImage 尚未加载完成或加载失败时，用户至少能看到一个有意义的占位内容，而不是空白。

9. 本课小结

本课系统学习了 Flutter 中图片与图标相关的核心组件：

  1. Image.asset：加载打包在应用内的本地资源图片，需要在 pubspec.yaml 中声明 assets，支持多分辨率适配。

  2. Image.network：加载网络图片，配合 loadingBuilder 显示加载进度，配合 errorBuilder 处理加载失败，是最常用的图片加载方式。

  3. Image.file / Image.memory：分别从文件系统和内存字节数组加载图片，适用于相机拍照、数据库存储等场景。

  4. BoxFit：控制图片在容器中的适配方式，七种模式各有适用场景。cover 是最常用的模式，contain 适合需要完整展示的场景，fill 会破坏宽高比。

  5. FadeInImage：为网络图片加载提供从占位图到目标图的平滑渐入过渡效果。

  6. Icon / ImageIcon：Icon 用于显示 Material Icons 字体图标，矢量渲染、体积小；ImageIcon 用于将图片作为单色图标使用；还可以通过自定义字体引入第三方图标。

  7. CircleAvatar：Material Design 规范的圆形头像组件，支持背景图片、文字、图标等多种内容形式。

关键要点回顾：

  • 生产环境推荐使用 cached_network_image 包进行网络图片加载，具备磁盘缓存能力
  • 使用 cacheWidth / cacheHeight 控制图片解码尺寸，减少内存占用
  • 优先使用字体图标（Icon）而非图片图标（ImageIcon）
  • 始终为网络图片提供加载占位和错误处理
