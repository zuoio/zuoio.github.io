第21课 显式动画组件

课程系列：Flutter 常用组件系统学习
适用人群：有 Dart 语言基础，了解 Flutter 基本概念的开发者
课程目标：掌握 Flutter 显式动画体系，能够使用 AnimationController、Tween、Transition 组件以及交错动画、Hero 动画等高级技巧，实现对动画的完全控制

本课是动画系列的第二课。在第20课中，我们学习了隐式动画——你只需要声明目标状态，Flutter 自动完成过渡。但在实际开发中，很多场景需要更精细的控制：循环播放的加载动画、手势驱动的交互动画、多个动画按时序编排的复杂效果……这些都是隐式动画力不从心的地方。显式动画（Explicit Animation）让你能完全掌控动画的播放、暂停、倒放、循环、联动等一切行为。

1. 隐式动画 vs 显式动画：何时升级

在深入显式动画之前，先明确两者的定位和选择策略。

对比维度 隐式动画（第20课） 显式动画（本课）
核心思想 声明目标值，自动过渡 手动控制动画的每一帧
AnimationController 内部自动管理，开发者不可见 开发者手动创建和管理
控制能力 只能"从当前值到目标值" 播放、暂停、倒放、循环、跳转到任意位置
多动画编排 不支持 一个 Controller 驱动多个 Tween + Interval
手势联动 不支持 controller.value 可与手势位移绑定
性能优化 自动处理 可通过 AnimatedBuilder 的 child 参数手动优化
代码量 少 多（需要 mixin、dispose 等）
适用场景 状态变化触发的简单过渡 循环动画、交错动画、手势驱动、页面转场

选择原则：

  • 如果动画只是"属性 A 变到属性 B"的一次性过渡 → 用隐式动画
  • 如果动画需要循环、暂停、倒放、与手势联动、多动画编排 → 用显式动画
  • 如果你不确定 → 先试隐式动画，不够用了再升级到显式动画

2. AnimationController 与 TickerProvider

AnimationController 是显式动画的核心引擎，它产生一个在指定时间内从 lowerBound 到 upperBound 线性变化的值（默认 0.0 → 1.0）。

2.1 vsync 与 TickerProviderStateMixin

AnimationController 需要一个 TickerProvider 来驱动每一帧的回调。Ticker 的作用是在每个垂直同步信号（vsync）到来时通知动画更新，这确保了动画与屏幕刷新率同步（通常 60fps 或 120fps），避免不必要的计算。

  • SingleTickerProviderStateMixin：当 State 中只有一个 AnimationController 时使用
  • TickerProviderStateMixin：当 State 中有多个 AnimationController 时使用

注意：vsync 参数传入 this，前提是当前 State 类混入了上述 mixin。如果组件不在屏幕上显示（比如被切到后台），Ticker 会自动停止，节省资源。

2.2 创建与配置 AnimationController

import 'package:flutter/material.dart';

void main() => runApp(const MyApp());

class MyApp extends StatelessWidget {
  const MyApp({super.key});

  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      home: const ControllerBasicDemo(),
    );
  }
}

class ControllerBasicDemo extends StatefulWidget {
  const ControllerBasicDemo({super.key});

  @override
  State<ControllerBasicDemo> createState() => _ControllerBasicDemoState();
}

/// 使用 SingleTickerProviderStateMixin，因为只有一个 AnimationController
class _ControllerBasicDemoState extends State<ControllerBasicDemo>
    with SingleTickerProviderStateMixin {
  late AnimationController _controller;

  @override
  void initState() {
    super.initState();
    _controller = AnimationController(
      // 动画时长
      duration: const Duration(seconds: 2),
      // 值的下界，默认 0.0
      lowerBound: 0.0,
      // 值的上界，默认 1.0
      upperBound: 1.0,
      // vsync 绑定当前 State 的 Ticker
      vsync: this,
    );

    // 监听动画值变化
    _controller.addListener(() {
      setState(() {});
    });

    // 监听动画状态变化
    _controller.addStatusListener((status) {
      debugPrint('动画状态: $status');
      // AnimationStatus.dismissed  — 动画停在起点
      // AnimationStatus.forward    — 正在正向播放
      // AnimationStatus.reverse    — 正在反向播放
      // AnimationStatus.completed  — 动画停在终点
    });
  }

  @override
  void dispose() {
    // 必须释放 AnimationController，否则会内存泄漏！
    _controller.dispose();
    super.dispose();
  }

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(title: const Text('AnimationController 基础')),
      body: Center(
        child: Column(
          mainAxisAlignment: MainAxisAlignment.center,
          children: [
            // 用 controller 的值控制容器的大小
            Container(
              width: 50 + 150 * _controller.value,
              height: 50 + 150 * _controller.value,
              decoration: BoxDecoration(
                color: Colors.blue,
                borderRadius: BorderRadius.circular(8),
              ),
            ),
            const SizedBox(height: 20),
            Text(
              '当前值: ${_controller.value.toStringAsFixed(2)}',
              style: const TextStyle(fontSize: 18),
            ),
          ],
        ),
      ),
      // 控制按钮区域
      bottomNavigationBar: Padding(
        padding: const EdgeInsets.all(16),
        child: Wrap(
          spacing: 8,
          alignment: WrapAlignment.center,
          children: [
            ElevatedButton(
              onPressed: () => _controller.forward(),
              child: const Text('forward'),
            ),
            ElevatedButton(
              onPressed: () => _controller.reverse(),
              child: const Text('reverse'),
            ),
            ElevatedButton(
              onPressed: () => _controller.repeat(),
              child: const Text('repeat'),
            ),
            ElevatedButton(
              onPressed: () => _controller.repeat(reverse: true),
              child: const Text('repeat 往返'),
            ),
            ElevatedButton(
              onPressed: () => _controller.stop(),
              child: const Text('stop'),
            ),
            ElevatedButton(
              onPressed: () => _controller.reset(),
              child: const Text('reset'),
            ),
            ElevatedButton(
              onPressed: () => _controller.animateTo(0.5),
              child: const Text('animateTo(0.5)'),
            ),
            ElevatedButton(
              onPressed: () => _controller.fling(),
              child: const Text('fling'),
            ),
          ],
        ),
      ),
    );
  }
}

2.3 AnimationController 常用方法总结

方法 说明
forward({from}) 从当前位置（或 from）正向播放到 upperBound
reverse({from}) 从当前位置（或 from）反向播放到 lowerBound
repeat({reverse, min, max, period}) 循环播放；reverse: true 表示往返循环
stop({canceled}) 停止动画，保持在当前位置
reset() 重置到 lowerBound，不触发动画
animateTo(target, {duration, curve}) 动画过渡到指定值，可自定义时长和曲线
animateBack(target, {duration, curve}) 动画过渡回指定值
fling({velocity}) 以物理弹性方式快速弹到边界
dispose() 释放资源，必须在 State.dispose 中调用

2.4 dispose 释放资源

⚠️ 最佳实践：每个 AnimationController 都必须在 dispose() 方法中释放。这是显式动画最容易出错的地方——忘记 dispose 会导致内存泄漏和 Ticker 异常。

@override
void dispose() {
  _controller.dispose();
  // 如果有多个控制器，每个都要 dispose
  // _controller2.dispose();
  // _controller3.dispose();
  super.dispose();
}

如果在 dispose 之后还试图使用 controller，会抛出如下错误：

FlutterError: AnimationController.forward() called after AnimationController.dispose()

3. Tween 与 CurvedAnimation

AnimationController 只产生 0.0 → 1.0 的线性值。要将它映射到实际需要的值范围（如颜色、大小、偏移量），需要 Tween；要改变动画的节奏（缓入缓出等），需要 CurvedAnimation。

3.1 Tween<T> 值映射

Tween<T> 将 0.0 → 1.0 的输入映射到 begin → end 的输出。

// 将 0.0~1.0 映射到 50.0~200.0
final sizeTween = Tween<double>(begin: 50.0, end: 200.0);

// 获取中间值：当 controller.value = 0.5 时
// sizeTween.evaluate(controller) = 125.0

Flutter 提供了多种内置 Tween 子类：

Tween 类 说明 示例
Tween<double> 双精度值映射 Tween(begin: 0.0, end: 1.0)
ColorTween 颜色插值 ColorTween(begin: Colors.red, end: Colors.blue)
IntTween 整数映射 IntTween(begin: 0, end: 255)
SizeTween Size 插值 SizeTween(begin: Size(50,50), end: Size(200,200))
RectTween Rect 插值 RectTween(begin: Rect.zero, end: Rect.fromLTWH(...))
AlignmentTween 对齐插值 AlignmentTween(begin: Alignment.topLeft, end: Alignment.bottomRight)
BorderRadiusTween 圆角插值 BorderRadiusTween(begin: BorderRadius.zero, end: BorderRadius.circular(20))
DecorationTween Decoration 插值 DecorationTween(begin: BoxDecoration(...), end: BoxDecoration(...))
EdgeInsetsTween 内边距插值 EdgeInsetsTween(begin: EdgeInsets.zero, end: EdgeInsets.all(20))
TextStyleTween 文字样式插值 TextStyleTween(begin: TextStyle(...), end: TextStyle(...))

3.2 CurvedAnimation 动画曲线

CurvedAnimation 不改变值的范围，而是改变值随时间的变化节奏。它是 AnimationController 的包装器。

late AnimationController _controller;
late Animation<double> _curvedAnimation;

@override
void initState() {
  super.initState();
  _controller = AnimationController(
    duration: const Duration(seconds: 1),
    vsync: this,
  );

  // 用 CurvedAnimation 包装，正向使用 easeInOut，反向使用 easeIn
  _curvedAnimation = CurvedAnimation(
    parent: _controller,
    curve: Curves.easeInOut,
    reverseCurve: Curves.easeIn,
  );
}

常用曲线（Curves 类）：

曲线 效果
Curves.linear 匀速
Curves.easeIn 缓入（开始慢，结束快）
Curves.easeOut 缓出（开始快，结束慢）
Curves.easeInOut 缓入缓出
Curves.bounceOut 弹跳效果
Curves.elasticIn 弹性效果
Curves.decelerate 减速
Curves.fastOutSlowIn Material Design 标准曲线

3.3 链式写法：tween.animate(controller)

这是显式动画中最常用的写法——将 Tween 和 CurvedAnimation（或 controller）链接在一起，生成一个可直接使用的 Animation<T> 对象。

import 'package:flutter/material.dart';

void main() => runApp(const MyApp());

class MyApp extends StatelessWidget {
  const MyApp({super.key});

  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      home: const TweenChainDemo(),
    );
  }
}

class TweenChainDemo extends StatefulWidget {
  const TweenChainDemo({super.key});

  @override
  State<TweenChainDemo> createState() => _TweenChainDemoState();
}

class _TweenChainDemoState extends State<TweenChainDemo>
    with SingleTickerProviderStateMixin {
  late AnimationController _controller;
  late Animation<double> _sizeAnimation;
  late Animation<Color?> _colorAnimation;
  late Animation<BorderRadius?> _borderRadiusAnimation;

  @override
  void initState() {
    super.initState();
    _controller = AnimationController(
      duration: const Duration(seconds: 1),
      vsync: this,
    );

    // 使用 CurvedAnimation 包装曲线
    final curvedAnimation = CurvedAnimation(
      parent: _controller,
      curve: Curves.easeInOut,
    );

    // 链式写法：Tween.animate(curvedAnimation)
    _sizeAnimation = Tween<double>(begin: 80, end: 200).animate(curvedAnimation);
    _colorAnimation = ColorTween(begin: Colors.blue, end: Colors.orange).animate(curvedAnimation);
    _borderRadiusAnimation = BorderRadiusTween(
      begin: BorderRadius.circular(8),
      end: BorderRadius.circular(100),
    ).animate(curvedAnimation);
  }

  @override
  void dispose() {
    _controller.dispose();
    super.dispose();
  }

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(title: const Text('Tween 链式写法')),
      body: AnimatedBuilder(
        animation: _controller,
        builder: (context, child) {
          return Center(
            child: Container(
              width: _sizeAnimation.value,
              height: _sizeAnimation.value,
              decoration: BoxDecoration(
                color: _colorAnimation.value,
                borderRadius: _borderRadiusAnimation.value,
              ),
            ),
          );
        },
      ),
      floatingActionButton: FloatingActionButton(
        onPressed: () {
          if (_controller.isCompleted) {
            _controller.reverse();
          } else {
            _controller.forward();
          }
        },
        child: const Icon(Icons.play_arrow),
      ),
    );
  }
}

注意：tween.animate(parent) 返回的是一个 Animation<T> 对象，它本身不产生值，而是在 parent（controller 或 curvedAnimation）变化时计算映射值。你不需要手动 dispose 这个 Animation 对象，只需 dispose controller 即可。

4. AnimatedBuilder

AnimatedBuilder（Flutter 3.x 中是 AnimatedBuilder 的正式名称，历史版本中也写作 AnimatedBuilder）是显式动画中最推荐的构建方式。它的核心优势是将动画逻辑与 UI 构建分离，并通过 child 参数优化性能。

4.1 基本用法

import 'package:flutter/material.dart';

void main() => runApp(const MyApp());

class MyApp extends StatelessWidget {
  const MyApp({super.key});

  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      home: const AnimatedBuilderDemo(),
    );
  }
}

class AnimatedBuilderDemo extends StatefulWidget {
  const AnimatedBuilderDemo({super.key});

  @override
  State<AnimatedBuilderDemo> createState() => _AnimatedBuilderDemoState();
}

class _AnimatedBuilderDemoState extends State<AnimatedBuilderDemo>
    with SingleTickerProviderStateMixin {
  late AnimationController _controller;
  late Animation<double> _rotationAnimation;

  @override
  void initState() {
    super.initState();
    _controller = AnimationController(
      duration: const Duration(seconds: 3),
      vsync: this,
    )..repeat(); // 无限循环旋转

    _rotationAnimation = Tween<double>(begin: 0, end: 2 * 3.14159).animate(_controller);
  }

  @override
  void dispose() {
    _controller.dispose();
    super.dispose();
  }

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(title: const Text('AnimatedBuilder 示例')),
      body: Center(
        // AnimatedBuilder 监听 _controller，每帧调用 builder 重建
        child: AnimatedBuilder(
          animation: _controller,
          // child 参数传入不需要动画的子树，避免每帧重建
          child: Container(
            width: 100,
            height: 100,
            color: Colors.blue,
            child: const Center(
              child: Text(
                'Flutter',
                style: TextStyle(color: Colors.white, fontWeight: FontWeight.bold),
              ),
            ),
          ),
          builder: (context, child) {
            // builder 每帧都会调用，但 child 不会重建
            return Transform.rotate(
              angle: _rotationAnimation.value,
              child: child, // 复用上面传入的 child
            );
          },
        ),
      ),
    );
  }
}

4.2 child 参数的性能优化

AnimatedBuilder 的 builder 回调在每一帧都会被调用。如果在 builder 中直接构建整个子树，那些不需要动画的部分也会被反复重建，造成不必要的性能开销。

优化方案：将不参与动画的子树通过 child 参数传入 AnimatedBuilder。这个 child 在父级 build 时创建一次，之后每帧复用。

// ❌ 错误写法：Text 每帧都重建
AnimatedBuilder(
  animation: _controller,
  builder: (context, child) {
    return Transform.scale(
      scale: _scaleAnimation.value,
      child: Container(
        width: 100,
        height: 100,
        color: Colors.blue,
        child: const Text('每帧都会重建我'), // 不必要的重建
      ),
    );
  },
)

// ✅ 正确写法：Text 只构建一次
AnimatedBuilder(
  animation: _controller,
  child: Container(
    width: 100,
    height: 100,
    color: Colors.blue,
    child: const Text('我只构建一次'),
  ),
  builder: (context, child) {
    return Transform.scale(
      scale: _scaleAnimation.value,
      child: child, // 复用
    );
  },
)

最佳实践：始终尽可能利用 child 参数。即使当前看不出性能差异，养成良好的习惯可以在复杂 UI 中避免掉帧。

5. AnimatedWidget 子类化

AnimatedWidget 提供了另一种封装动画的方式——将动画组件封装为独立的 Widget 类。适合创建可复用的动画组件。

5.1 基本原理

AnimatedWidget 内部会自动监听传入的 Listenable（通常是 AnimationController 或 Animation），当值变化时自动调用 build 方法重建。你不需要手动 addListener + setState。

5.2 自定义 AnimatedWidget

import 'package:flutter/material.dart';

void main() => runApp(const MyApp());

class MyApp extends StatelessWidget {
  const MyApp({super.key});

  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      home: const AnimatedWidgetDemo(),
    );
  }
}

/// 自定义 AnimatedWidget：脉冲缩放效果
class PulseWidget extends AnimatedWidget {
  final Widget child;

  const PulseWidget({
    super.key,
    required Animation<double> animation,
    required this.child,
  }) : super(listenable: animation);

  @override
  Widget build(BuildContext context) {
    final animation = listenable as Animation<double>;
    return Transform.scale(
      scale: animation.value,
      child: child,
    );
  }
}

class AnimatedWidgetDemo extends StatefulWidget {
  const AnimatedWidgetDemo({super.key});

  @override
  State<AnimatedWidgetDemo> createState() => _AnimatedWidgetDemoState();
}

class _AnimatedWidgetDemoState extends State<AnimatedWidgetDemo>
    with SingleTickerProviderStateMixin {
  late AnimationController _controller;
  late Animation<double> _scaleAnimation;

  @override
  void initState() {
    super.initState();
    _controller = AnimationController(
      duration: const Duration(milliseconds: 800),
      vsync: this,
    )..repeat(reverse: true);

    _scaleAnimation = Tween<double>(begin: 0.8, end: 1.2).animate(
      CurvedAnimation(parent: _controller, curve: Curves.easeInOut),
    );
  }

  @override
  void dispose() {
    _controller.dispose();
    super.dispose();
  }

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(title: const Text('自定义 AnimatedWidget')),
      body: Center(
        // 使用自定义的 PulseWidget，代码清爽
        child: PulseWidget(
          animation: _scaleAnimation,
          child: const Icon(Icons.favorite, color: Colors.red, size: 80),
        ),
      ),
    );
  }
}

5.3 AnimatedBuilder vs AnimatedWidget

特性 AnimatedBuilder AnimatedWidget 子类
使用方式 在 build 方法中内联使用 封装为独立类
child 优化 内置 child 参数 需手动实现
可复用性 较低，通常一次性使用 高，可在多处复用
代码组织 适合简单、一次性的动画 适合需要复用的动画组件
推荐场景 大多数场景 动画组件需要在多处使用

6. 内置 Transition 组件

Flutter 提供了一组开箱即用的 Transition 组件，它们都是 AnimatedWidget 的子类，分别对应常见的动画效果。使用它们可以避免手写 Transform/Opacity 等包装代码。

6.1 组件概览

组件 动画属性 参数名 说明
FadeTransition 透明度 opacity 淡入淡出
SlideTransition 偏移量 position 滑动进出
ScaleTransition 缩放比例 scale 缩放进出
RotationTransition 旋转圈数 turns 旋转效果
SizeTransition 尺寸比例 sizeFactor 尺寸展开/收起
DecoratedBoxTransition Decoration decoration 装饰过渡
PositionedTransition RelativeRect rect 在 Stack 中的位置过渡
RelativePositionedTransition Rect rect 相对位置过渡

6.2 完整示例：六大 Transition

import 'package:flutter/material.dart';

void main() => runApp(const MyApp());

class MyApp extends StatelessWidget {
  const MyApp({super.key});

  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      home: const TransitionShowcase(),
    );
  }
}

class TransitionShowcase extends StatefulWidget {
  const TransitionShowcase({super.key});

  @override
  State<TransitionShowcase> createState() => _TransitionShowcaseState();
}

class _TransitionShowcaseState extends State<TransitionShowcase>
    with SingleTickerProviderStateMixin {
  late AnimationController _controller;

  // 各种动画
  late Animation<double> _fadeAnimation;
  late Animation<Offset> _slideAnimation;
  late Animation<double> _scaleAnimation;
  late Animation<double> _rotationAnimation;
  late Animation<double> _sizeAnimation;

  @override
  void initState() {
    super.initState();
    _controller = AnimationController(
      duration: const Duration(seconds: 2),
      vsync: this,
    );

    final curved = CurvedAnimation(parent: _controller, curve: Curves.easeInOut);

    // 透明度：0 → 1
    _fadeAnimation = Tween<double>(begin: 0.0, end: 1.0).animate(curved);

    // 滑动偏移：从左侧滑入（-1 表示向左偏移一个自身宽度）
    _slideAnimation = Tween<Offset>(
      begin: const Offset(-1.0, 0.0),
      end: Offset.zero,
    ).animate(curved);

    // 缩放：从 0 到 1
    _scaleAnimation = Tween<double>(begin: 0.0, end: 1.0).animate(curved);

    // 旋转：旋转 1 圈（turns = 1.0 表示 360°）
    _rotationAnimation = Tween<double>(begin: 0.0, end: 1.0).animate(curved);

    // 尺寸：从 0 展开到完整尺寸
    _sizeAnimation = Tween<double>(begin: 0.0, end: 1.0).animate(curved);
  }

  @override
  void dispose() {
    _controller.dispose();
    super.dispose();
  }

  Widget _buildCard(String label, Widget child) {
    return Card(
      margin: const EdgeInsets.symmetric(vertical: 8, horizontal: 16),
      child: Padding(
        padding: const EdgeInsets.all(16),
        child: Row(
          children: [
            SizedBox(width: 140, child: Text(label, style: const TextStyle(fontSize: 14))),
            Expanded(child: Center(child: child)),
          ],
        ),
      ),
    );
  }

  @override
  Widget build(BuildContext context) {
    final box = Container(width: 50, height: 50, color: Colors.blue);

    return Scaffold(
      appBar: AppBar(title: const Text('内置 Transition 组件')),
      body: ListView(
        children: [
          _buildCard(
            'FadeTransition',
            FadeTransition(opacity: _fadeAnimation, child: box),
          ),
          _buildCard(
            'SlideTransition',
            SlideTransition(position: _slideAnimation, child: box),
          ),
          _buildCard(
            'ScaleTransition',
            ScaleTransition(scale: _scaleAnimation, child: box),
          ),
          _buildCard(
            'RotationTransition',
            RotationTransition(turns: _rotationAnimation, child: box),
          ),
          _buildCard(
            'SizeTransition',
            SizeTransition(
              sizeFactor: _sizeAnimation,
              axisAlignment: -1.0, // 从顶部展开
              child: box,
            ),
          ),
        ],
      ),
      floatingActionButton: FloatingActionButton(
        onPressed: () {
          if (_controller.isCompleted) {
            _controller.reverse();
          } else {
            _controller.forward();
          }
        },
        child: const Icon(Icons.play_arrow),
      ),
    );
  }
}

7. Staggered Animation（交错动画）

交错动画是显式动画最强大的特性之一——用一个 AnimationController 驱动多个 Tween，通过 Interval 控制每个 Tween 在总时间线中的播放区间，实现"此起彼伏"的编排效果。

7.1 Interval 的作用

Interval 是一种特殊的 Curve，它定义了动画在 controller 0.0~1.0 时间线中的激活区间。

// 这个动画只在 controller 进度 0%~40% 时活跃
CurvedAnimation(
  parent: controller,
  curve: const Interval(0.0, 0.4, curve: Curves.easeOut),
)

// 这个动画只在 controller 进度 40%~70% 时活跃
CurvedAnimation(
  parent: controller,
  curve: const Interval(0.4, 0.7, curve: Curves.easeInOut),
)

7.2 完整的交错动画示例

以下示例展示一个多步骤的卡片入场动画：先滑入，再展开，然后淡入文字，最后显示按钮。

import 'package:flutter/material.dart';

void main() => runApp(const MyApp());

class MyApp extends StatelessWidget {
  const MyApp({super.key});

  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      theme: ThemeData(useMaterial3: true),
      home: const StaggeredAnimationDemo(),
    );
  }
}

class StaggeredAnimationDemo extends StatefulWidget {
  const StaggeredAnimationDemo({super.key});

  @override
  State<StaggeredAnimationDemo> createState() => _StaggeredAnimationDemoState();
}

class _StaggeredAnimationDemoState extends State<StaggeredAnimationDemo>
    with SingleTickerProviderStateMixin {
  late AnimationController _controller;

  // 第1阶段（0%~30%）：卡片从下方滑入
  late Animation<Offset> _slideAnimation;
  // 第2阶段（20%~50%）：卡片透明度渐入
  late Animation<double> _fadeAnimation;
  // 第3阶段（40%~70%）：卡片宽度展开
  late Animation<double> _widthAnimation;
  // 第4阶段（60%~85%）：标题文字淡入
  late Animation<double> _titleFadeAnimation;
  // 第5阶段（75%~100%）：按钮缩放出现
  late Animation<double> _buttonScaleAnimation;

  @override
  void initState() {
    super.initState();
    _controller = AnimationController(
      duration: const Duration(milliseconds: 2500),
      vsync: this,
    );

    // 每个动画使用不同的 Interval，在总时间线的不同区间激活
    _slideAnimation = Tween<Offset>(
      begin: const Offset(0.0, 1.5),
      end: Offset.zero,
    ).animate(CurvedAnimation(
      parent: _controller,
      curve: const Interval(0.0, 0.3, curve: Curves.easeOut),
    ));

    _fadeAnimation = Tween<double>(begin: 0.0, end: 1.0).animate(
      CurvedAnimation(
        parent: _controller,
        curve: const Interval(0.2, 0.5, curve: Curves.easeIn),
      ),
    );

    _widthAnimation = Tween<double>(begin: 200.0, end: 340.0).animate(
      CurvedAnimation(
        parent: _controller,
        curve: const Interval(0.4, 0.7, curve: Curves.easeInOut),
      ),
    );

    _titleFadeAnimation = Tween<double>(begin: 0.0, end: 1.0).animate(
      CurvedAnimation(
        parent: _controller,
        curve: const Interval(0.6, 0.85, curve: Curves.easeIn),
      ),
    );

    _buttonScaleAnimation = Tween<double>(begin: 0.0, end: 1.0).animate(
      CurvedAnimation(
        parent: _controller,
        curve: const Interval(0.75, 1.0, curve: Curves.elasticOut),
      ),
    );
  }

  @override
  void dispose() {
    _controller.dispose();
    super.dispose();
  }

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(title: const Text('交错动画（Staggered Animation）')),
      body: Center(
        child: AnimatedBuilder(
          animation: _controller,
          builder: (context, child) {
            return SlideTransition(
              position: _slideAnimation,
              child: Opacity(
                opacity: _fadeAnimation.value,
                child: Container(
                  width: _widthAnimation.value,
                  padding: const EdgeInsets.all(24),
                  decoration: BoxDecoration(
                    color: Colors.white,
                    borderRadius: BorderRadius.circular(16),
                    boxShadow: [
                      BoxShadow(
                        color: Colors.black.withOpacity(0.1),
                        blurRadius: 20,
                        offset: const Offset(0, 10),
                      ),
                    ],
                  ),
                  child: Column(
                    mainAxisSize: MainAxisSize.min,
                    children: [
                      const Icon(Icons.check_circle, color: Colors.green, size: 60),
                      const SizedBox(height: 16),
                      // 标题淡入
                      Opacity(
                        opacity: _titleFadeAnimation.value,
                        child: const Column(
                          children: [
                            Text(
                              '支付成功！',
                              style: TextStyle(fontSize: 24, fontWeight: FontWeight.bold),
                            ),
                            SizedBox(height: 8),
                            Text(
                              '您的订单已确认，感谢您的购买。',
                              textAlign: TextAlign.center,
                              style: TextStyle(fontSize: 14, color: Colors.grey),
                            ),
                          ],
                        ),
                      ),
                      const SizedBox(height: 24),
                      // 按钮缩放出现
                      Transform.scale(
                        scale: _buttonScaleAnimation.value,
                        child: ElevatedButton(
                          onPressed: () {},
                          style: ElevatedButton.styleFrom(
                            padding: const EdgeInsets.symmetric(horizontal: 40, vertical: 14),
                          ),
                          child: const Text('查看订单'),
                        ),
                      ),
                    ],
                  ),
                ),
              ),
            );
          },
        ),
      ),
      backgroundColor: Colors.grey.shade200,
      floatingActionButton: FloatingActionButton(
        onPressed: () {
          if (_controller.isCompleted) {
            _controller.reverse();
          } else {
            _controller.forward(from: 0.0);
          }
        },
        child: const Icon(Icons.play_arrow),
      ),
    );
  }
}

8. Hero 动画

Hero 是 Flutter 内置的页面间共享元素过渡动画。当两个页面中存在相同 tag 的 Hero 组件时，在页面切换过程中 Flutter 会自动生成"飞行"过渡效果。

8.1 基本用法

要实现 Hero 动画只需两步：

  1. 在源页面和目标页面中分别用 Hero 包裹要过渡的组件
  2. 两个 Hero 的 tag 必须相同

8.2 完整的 Hero 动画示例

import 'package:flutter/material.dart';

void main() => runApp(const MyApp());

class MyApp extends StatelessWidget {
  const MyApp({super.key});

  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      theme: ThemeData(useMaterial3: true),
      home: const HeroListPage(),
    );
  }
}

/// 源页面：图片列表
class HeroListPage extends StatelessWidget {
  const HeroListPage({super.key});

  static const List<Map<String, String>> _items = [

    {'id': '4', 'title': '森林小径', 'url': 'https://picsum.photos/id/40/400/300'},
  ];

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(title: const Text('图片画廊')),
      body: GridView.builder(
        padding: const EdgeInsets.all(12),
        gridDelegate: const SliverGridDelegateWithFixedCrossAxisCount(
          crossAxisCount: 2,
          mainAxisSpacing: 12,
          crossAxisSpacing: 12,

8.3 自定义 Hero 飞行路径

默认的 Hero 飞行路径是直线。通过 createRectTween 可以自定义飞行轨迹：

Hero(
  tag: 'custom_hero',
  // 自定义飞行路径：弧线轨迹
  createRectTween: (begin, end) {
    return MaterialRectCenterArcTween(begin: begin, end: end);
  },
  child: Image.network('https://picsum.photos/200'),
)

8.4 flightShuttleBuilder 自定义飞行中的外观

flightShuttleBuilder 允许你自定义 Hero 在飞行过程中的外观（默认使用目标页面的子组件）：

Hero(
  tag: 'avatar_hero',
  // 在飞行过程中显示自定义的 Widget
  flightShuttleBuilder: (
    BuildContext flightContext,
    Animation<double> animation,
    HeroFlightDirection flightDirection,
    BuildContext fromHeroContext,
    BuildContext toHeroContext,
  ) {
    // 飞行过程中显示圆形裁剪的图片，并带有阴影
    return AnimatedBuilder(
      animation: animation,
      builder: (context, child) {
        return Material(
          elevation: 8 * animation.value,
          shape: const CircleBorder(),
          clipBehavior: Clip.antiAlias,
          child: Image.network('https://picsum.photos/200', fit: BoxFit.cover),
        );
      },
    );
  },
  child: const CircleAvatar(radius: 30, backgroundImage: NetworkImage('https://picsum.photos/200')),
)

注意事项：

  • Hero 的 tag 在同一页面中必须唯一，否则会抛出异常
  • Hero 只在 MaterialPageRoute 或 CupertinoPageRoute 等标准路由切换中生效
  • 如果源页面和目标页面的 Hero 子组件差异很大（如从小圆形变大矩形），考虑使用 flightShuttleBuilder 定义平滑的过渡外观
  • Hero 动画期间，子组件会被提升到 Overlay 层渲染，脱离原页面的布局

9. AnimatedList 与 AnimatedGrid

AnimatedList 和 AnimatedGrid 为列表/网格的增删操作提供了平滑的动画效果，是构建动态列表不可或缺的组件。

9.1 AnimatedList 基础

AnimatedList 的核心在于：

  • 使用 GlobalKey<AnimatedListState> 获取列表状态
  • 通过 insertItem() 和 removeItem() 触发增删动画
  • 增删时需同步操作数据源和 AnimatedList

9.2 完整的增删动画列表

import 'package:flutter/material.dart';

void main() => runApp(const MyApp());

class MyApp extends StatelessWidget {
  const MyApp({super.key});

  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      theme: ThemeData(useMaterial3: true),
      home: const AnimatedListDemo(),
    );
  }
}

class AnimatedListDemo extends StatefulWidget {
  const AnimatedListDemo({super.key});

  @override
  State<AnimatedListDemo> createState() => _AnimatedListDemoState();
}

class _AnimatedListDemoState extends State<AnimatedListDemo> {
  // 用 GlobalKey 获取 AnimatedListState
  final _listKey = GlobalKey<AnimatedListState>();
  // 数据源
  final List<String> _items = ['苹果', '香蕉', '橙子'];
  int _counter = 0;

  /// 插入新项目
  void _addItem() {
    _counter++;
    final index = _items.length;
    _items.insert(index, '新水果 $_counter');
    // 通知 AnimatedList 插入项目，触发入场动画
    _listKey.currentState?.insertItem(
      index,
      duration: const Duration(milliseconds: 500),
    );
  }

  /// 删除指定项目
  void _removeItem(int index) {
    final removedItem = _items[index];
    _items.removeAt(index);
    // 通知 AnimatedList 移除项目，提供退场动画的 builder
    _listKey.currentState?.removeItem(
      index,
      (context, animation) => _buildItem(removedItem, animation),
      duration: const Duration(milliseconds: 400),
    );
  }

  /// 构建列表项（入场和退场共用）
  Widget _buildItem(String item, Animation<double> animation) {
    return SizeTransition(
      sizeFactor: animation,
      child: FadeTransition(
        opacity: animation,
        child: Card(
          margin: const EdgeInsets.symmetric(horizontal: 16, vertical: 4),
          child: ListTile(
            leading: const CircleAvatar(child: Icon(Icons.fastfood)),
            title: Text(item),
            trailing: IconButton(
              icon: const Icon(Icons.delete_outline, color: Colors.red),
              onPressed: () {
                final currentIndex = _items.indexOf(item);
                if (currentIndex != -1) {
                  _removeItem(currentIndex);
                }
              },
            ),
          ),
        ),
      ),
    );
  }

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(
        title: const Text('AnimatedList 示例'),
        actions: [
          IconButton(
            onPressed: _addItem,
            icon: const Icon(Icons.add),
            tooltip: '添加',
          ),
        ],
      ),
      body: AnimatedList(
        key: _listKey,
        // 初始项目数量
        initialItemCount: _items.length,
        itemBuilder: (context, index, animation) {
          return _buildItem(_items[index], animation);
        },
      ),
    );
  }
}

9.3 AnimatedList 自定义入场/退场动画

你可以组合多种 Transition 来创建更丰富的入场/退场效果：

import 'package:flutter/material.dart';

void main() => runApp(const MyApp());

class MyApp extends StatelessWidget {
  const MyApp({super.key});

  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      theme: ThemeData(useMaterial3: true),
      home: const FancyAnimatedListDemo(),
    );
  }
}

class FancyAnimatedListDemo extends StatefulWidget {
  const FancyAnimatedListDemo({super.key});

  @override
  State<FancyAnimatedListDemo> createState() => _FancyAnimatedListDemoState();
}

class _FancyAnimatedListDemoState extends State<FancyAnimatedListDemo> {
  final _listKey = GlobalKey<AnimatedListState>();
  final List<String> _items = [];
  int _nextId = 1;

  void _addItem() {
    final index = _items.length;
    _items.add('任务 ${_nextId++}');
    _listKey.currentState?.insertItem(index, duration: const Duration(milliseconds: 600));
  }

  void _removeItem(int index) {
    final removed = _items.removeAt(index);
    _listKey.currentState?.removeItem(
      index,
      (context, animation) {
        // 退场动画：向右滑出 + 淡出
        return SlideTransition(
          position: Tween<Offset>(
            begin: Offset.zero,
            end: const Offset(1.0, 0.0),
          ).animate(CurvedAnimation(parent: animation, curve: Curves.easeIn)),
          child: FadeTransition(
            opacity: animation,
            child: _buildTile(removed, animation),
          ),
        );
      },
      duration: const Duration(milliseconds: 400),
    );
  }

  Widget _buildTile(String item, Animation<double> animation) {
    return Card(
      margin: const EdgeInsets.symmetric(horizontal: 16, vertical: 4),
      child: ListTile(
        leading: CircleAvatar(child: Text(item.split(' ').last)),
        title: Text(item),
      ),
    );
  }

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(title: const Text('自定义入场/退场动画')),
      body: AnimatedList(
        key: _listKey,
        initialItemCount: _items.length,
        itemBuilder: (context, index, animation) {
          // 入场动画：从左滑入 + 淡入
          return SlideTransition(
            position: Tween<Offset>(
              begin: const Offset(-1.0, 0.0),
              end: Offset.zero,
            ).animate(CurvedAnimation(parent: animation, curve: Curves.easeOut)),
            child: FadeTransition(
              opacity: animation,
              child: _buildTile(_items[index], animation),
            ),
          );
        },
      ),
      floatingActionButton: Column(
        mainAxisSize: MainAxisSize.min,
        children: [
          FloatingActionButton(
            heroTag: 'add',
            onPressed: _addItem,
            child: const Icon(Icons.add),
          ),
          const SizedBox(height: 8),
          FloatingActionButton(
            heroTag: 'remove',
            onPressed: () {
              if (_items.isNotEmpty) _removeItem(0);
            },
            backgroundColor: Colors.red,
            child: const Icon(Icons.remove),
          ),
        ],
      ),
    );
  }
}

9.4 AnimatedGrid

AnimatedGrid（Flutter 3.7+）与 AnimatedList 的 API 设计一致，只是布局方式变为网格。使用 GlobalKey<AnimatedGridState> 控制增删。

import 'package:flutter/material.dart';

void main() => runApp(const MyApp());

class MyApp extends StatelessWidget {
  const MyApp({super.key});

  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      theme: ThemeData(useMaterial3: true),
      home: const AnimatedGridDemo(),
    );
  }
}

class AnimatedGridDemo extends StatefulWidget {
  const AnimatedGridDemo({super.key});

  @override
  State<AnimatedGridDemo> createState() => _AnimatedGridDemoState();
}

class _AnimatedGridDemoState extends State<AnimatedGridDemo> {
  final _gridKey = GlobalKey<AnimatedGridState>();
  final List<int> _items = [1, 2, 3, 4, 5, 6];
  int _nextId = 7;

  void _addItem() {
    final index = _items.length;
    _items.add(_nextId++);
    _gridKey.currentState?.insertItem(index);
  }

  void _removeItem(int index) {
    final removed = _items.removeAt(index);
    _gridKey.currentState?.removeItem(
      index,
      (context, animation) => _buildGridItem(removed, animation),
    );
  }

  Widget _buildGridItem(int item, Animation<double> animation) {
    return ScaleTransition(
      scale: animation,
      child: FadeTransition(
        opacity: animation,
        child: Card(
          color: Colors.primaries[item % Colors.primaries.length].shade100,
          child: Center(
            child: Text(
              '$item',
              style: const TextStyle(fontSize: 24, fontWeight: FontWeight.bold),
            ),
          ),
        ),
      ),
    );
  }

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(
        title: const Text('AnimatedGrid 示例'),
        actions: [
          IconButton(onPressed: _addItem, icon: const Icon(Icons.add)),
          IconButton(
            onPressed: () {
              if (_items.isNotEmpty) _removeItem(_items.length - 1);
            },
            icon: const Icon(Icons.remove),
          ),
        ],
      ),
      body: AnimatedGrid(
        key: _gridKey,
        initialItemCount: _items.length,
        gridDelegate: const SliverGridDelegateWithFixedCrossAxisCount(
          crossAxisCount: 3,
          mainAxisSpacing: 8,
          crossAxisSpacing: 8,
        ),
        padding: const EdgeInsets.all(12),
        itemBuilder: (context, index, animation) {
          return _buildGridItem(_items[index], animation);
        },
      ),
    );
  }
}

注意事项：

  • 增删操作必须同时更新数据源和 AnimatedList/AnimatedGrid 状态，两者不能漏掉任何一个
  • removeItem 的 builder 接收的是退场动画的 Animation，此时数据源中该项已被移除，所以要提前保存被移除的数据
  • initialItemCount 必须与初始数据源的长度一致
  • 列表项上使用 GlobalKey 时要注意避免 key 冲突

10. 多个 AnimationController 的管理

当一个页面需要独立控制多个动画时（如一个循环旋转 + 一个点击触发的缩放），需要使用 TickerProviderStateMixin（注意没有 Single 前缀）。

import 'package:flutter/material.dart';

void main() => runApp(const MyApp());

class MyApp extends StatelessWidget {
  const MyApp({super.key});

  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      home: const MultiControllerDemo(),
    );
  }
}

class MultiControllerDemo extends StatefulWidget {
  const MultiControllerDemo({super.key});

  @override
  State<MultiControllerDemo> createState() => _MultiControllerDemoState();
}

/// 使用 TickerProviderStateMixin（不带 Single）以支持多个控制器
class _MultiControllerDemoState extends State<MultiControllerDemo>
    with TickerProviderStateMixin {
  // 控制器1：持续旋转
  late AnimationController _rotationController;
  // 控制器2：点击缩放
  late AnimationController _scaleController;

  late Animation<double> _scaleAnimation;

  @override
  void initState() {
    super.initState();

    _rotationController = AnimationController(
      duration: const Duration(seconds: 4),
      vsync: this,
    )..repeat();

    _scaleController = AnimationController(
      duration: const Duration(milliseconds: 300),
      vsync: this,
    );

    _scaleAnimation = Tween<double>(begin: 1.0, end: 1.5).animate(
      CurvedAnimation(parent: _scaleController, curve: Curves.easeOutBack),
    );
  }

  @override
  void dispose() {
    // 每个控制器都要 dispose
    _rotationController.dispose();
    _scaleController.dispose();
    super.dispose();
  }

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(title: const Text('多个 AnimationController')),
      body: Center(
        child: GestureDetector(
          onTapDown: (_) => _scaleController.forward(),
          onTapUp: (_) => _scaleController.reverse(),
          onTapCancel: () => _scaleController.reverse(),
          child: AnimatedBuilder(
            animation: Listenable.merge([_rotationController, _scaleController]),
            builder: (context, child) {
              return Transform.scale(
                scale: _scaleAnimation.value,
                child: Transform.rotate(
                  angle: _rotationController.value * 2 * 3.14159,
                  child: child,
                ),
              );
            },
            child: Container(
              width: 120,
              height: 120,
              decoration: BoxDecoration(
                gradient: const LinearGradient(
                  colors: [Colors.blue, Colors.purple],
                ),
                borderRadius: BorderRadius.circular(20),
              ),
              child: const Center(
                child: Icon(Icons.star, color: Colors.white, size: 50),
              ),
            ),
          ),
        ),
      ),
    );
  }
}

最佳实践：

  • 只有一个 controller 时用 SingleTickerProviderStateMixin，多个时用 TickerProviderStateMixin
  • 可以使用 Listenable.merge([...]) 将多个动画合并为一个 Listenable 传给 AnimatedBuilder
  • 每个 controller 都必须在 dispose() 中释放

11. 显式动画最佳实践总结

11.1 生命周期管理清单

✅ initState() 中创建 AnimationController
✅ initState() 中创建 Tween 和 CurvedAnimation
✅ dispose() 中释放每一个 AnimationController
✅ 使用正确的 TickerProviderStateMixin
❌ 不要在 build() 中创建 AnimationController
❌ 不要忘记 dispose()
❌ 不要在 dispose() 之后再调用 controller 的方法

11.2 性能优化要点

  1. 使用 AnimatedBuilder 的 child 参数：将不参与动画的子树缓存起来
  2. 避免在 addListener 回调中调用 setState：这会导致整个 State 重建。优先使用 AnimatedBuilder 或 AnimatedWidget
  3. 合理使用 RepaintBoundary：为动画密集的区域添加 RepaintBoundary，避免影响其他区域的重绘
  4. Ticker 的自动暂停：当组件不可见时（如页面被覆盖），Ticker 会自动暂停，你不需要手动处理

11.3 选择动画方案的决策树

需要动画效果
  │
  ├─ 属性 A → 属性 B 的简单过渡？
  │    ├─ 是 → 使用隐式动画（AnimatedContainer 等）
  │    └─ 否 ↓
  │
  ├─ 需要循环播放？
  │    ├─ 是 → AnimationController + repeat()
  │    └─ 否 ↓
  │
  ├─ 需要暂停/倒放控制？
  │    ├─ 是 → AnimationController + forward/reverse/stop
  │    └─ 否 ↓
  │
  ├─ 多个动画需要时序编排？
  │    ├─ 是 → Staggered Animation（Interval）
  │    └─ 否 ↓
  │
  ├─ 页面间共享元素过渡？
  │    ├─ 是 → Hero
  │    └─ 否 ↓
  │
  ├─ 列表增删动画？
  │    ├─ 是 → AnimatedList / AnimatedGrid
  │    └─ 否 ↓
  │
  └─ 需要完全自定义的物理/手势驱动动画？
       └─ 是 → AnimationController + 自定义逻辑

12. 本课小结

本课系统学习了 Flutter 显式动画体系的核心内容：

知识点 核心要点
AnimationController 动画引擎，产生 0.0~1.0 的值；需要 TickerProvider；必须 dispose
Tween 将 controller 的 0~1 映射到任意类型的值范围
CurvedAnimation 改变动画节奏（缓入缓出等）
AnimatedBuilder 推荐的动画构建方式，child 参数优化性能
AnimatedWidget 封装可复用的动画组件
Transition 系列 内置的常用动画组件（Fade、Slide、Scale、Rotation、Size 等）
Staggered Animation 一个 controller + 多个 Interval，实现多步骤动画编排
Hero 页面间共享元素的飞行过渡动画
AnimatedList/Grid 列表增删的动画效果

与第20课的关系：隐式动画适合"设置目标值自动过渡"的简单场景；显式动画适合需要精细控制（循环、暂停、编排、手势联动）的复杂场景。两者互补，实际项目中往往结合使用。

13. 练习题

基础题

  1. 呼吸灯效果：创建一个圆形组件，使用 AnimationController + repeat(reverse: true) 实现不断变大变小的呼吸效果（缩放在 0.8~1.2 之间循环）。

  2. 颜色渐变循环：创建一个方形容器，使用 ColorTween 和 AnimationController 实现颜色从红色 → 蓝色 → 红色的无限循环。

  3. Fade + Slide 组合入场：创建一个文本组件，同时使用 FadeTransition 和 SlideTransition，实现从下方淡入滑出的入场效果。

中级题

  1. 自定义加载动画：用 RotationTransition 实现一个旋转的加载图标，用 FadeTransition 实现加载文字的闪烁效果，两者共用一个 AnimationController。

  2. 卡片翻转动画：使用 AnimationController 和 Transform，实现点击卡片后绕 Y 轴翻转 180° 显示背面内容的效果。提示：前半段（00.5）显示正面，后半段（0.51.0）显示背面。

  3. 改进的 AnimatedList：在本课 AnimatedList 示例的基础上，增加以下功能：

    • 长按列表项触发排序动画
    • 滑动删除（结合 Dismissible）并带有 SnackBar 撤销功能

进阶题

  1. 多步骤表单动画：设计一个三步表单（个人信息 → 地址信息 → 确认），每一步的切换使用交错动画（旧步骤滑出 + 新步骤滑入 + 进度条更新），全部由一个 AnimationController 驱动。

  2. Hero + 自定义路由过渡：实现一个图片画廊，点击缩略图后使用 Hero 跳转到详情页，同时自定义 PageRouteBuilder 的 transitionsBuilder，让页面背景淡入的同时 Hero 飞行，并在详情页支持手势下拉关闭（结合 GestureDetector 和 Navigator.pop）。

  3. 弹幕动画系统：使用多个 AnimationController 实现弹幕效果——每条弹幕从右侧滑入到左侧消失，速度随机，垂直位置随机，支持动态添加新弹幕。提示：使用 TickerProviderStateMixin 管理多个控制器。
