课程系列：Flutter 常用组件系统学习
适用人群：有 Dart 语言基础，了解 Flutter 基本概念的开发者

主题系统是 Flutter 应用的"基础设施"。一个成熟的 App 不会在每个组件里硬编码颜色和字号，而是通过主题系统统一管理视觉风格。这样做有三个核心价值：一致性（全局统一的视觉语言）、可维护性（改一处主题，全局联动）、灵活性（轻松支持亮/暗色模式切换、品牌色定制等）。

本课将从 ThemeData 的全局配置出发，逐步深入 ColorScheme、TextTheme、组件主题、主题切换、主题继承，最终讲解高级的 ThemeExtension 自定义扩展，并以一个完整的实战项目串联所有知识点。

1. MaterialApp 与 ThemeData 全局主题配置

1.1 ThemeData 在架构中的位置

Flutter 的主题系统核心是 ThemeData，它通过 MaterialApp 注入到 Widget 树中，所有 Material 组件都会自动读取最近的 ThemeData 来决定自己的外观。

MaterialApp
  ├── theme: ThemeData(...)          ← 亮色主题
  ├── darkTheme: ThemeData(...)      ← 暗色主题
  ├── themeMode: ThemeMode.system    ← 跟随系统 / 亮色 / 暗色
  └── home: MyHomePage()
         └── Scaffold (自动读取主题中的 scaffoldBackgroundColor)
               └── AppBar (自动读取主题中的 AppBarTheme)
                     └── Text (自动读取主题中的 TextTheme)

1.2 基础主题配置

import 'package:flutter/material.dart';

void main() => runApp(const MyApp());

class MyApp extends StatelessWidget {
  const MyApp({super.key});

  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      title: '主题演示',
      // 亮色主题
      theme: ThemeData(
        useMaterial3: true, // 启用 Material 3 设计规范
        colorSchemeSeed: Colors.blue, // 从种子色自动生成 ColorScheme
        brightness: Brightness.light,
      ),
      // 暗色主题
      darkTheme: ThemeData(
        useMaterial3: true,
        colorSchemeSeed: Colors.blue,
        brightness: Brightness.dark,
      ),
      // 主题模式：跟随系统设置
      themeMode: ThemeMode.system,
      home: const HomePage(),
    );
  }
}

class HomePage extends StatelessWidget {
  const HomePage({super.key});

  @override
  Widget build(BuildContext context) {
    // 获取当前主题
    final theme = Theme.of(context);

    return Scaffold(
      appBar: AppBar(title: const Text('主题演示')),
      body: Center(
        child: Column(
          mainAxisAlignment: MainAxisAlignment.center,
          children: [
            Text(
              '当前主题亮度: ${theme.brightness}',
              style: theme.textTheme.headlineMedium,
            ),
            const SizedBox(height: 16),
            ElevatedButton(
              onPressed: () {},
              child: const Text('主题按钮'),
            ),
          ],
        ),
      ),
    );
  }
}

1.4 useMaterial3 的影响

Material 3（又称 Material You）是 Google 最新的设计规范，Flutter 3.16+ 默认启用。与 Material 2 相比，主要变化包括：

  • 组件视觉样式更新（圆角更大、色调映射不同）
  • 颜色系统从 primarySwatch 迁移到 ColorScheme
  • 新增 NavigationBar、SegmentedButton、SearchAnchor 等组件
  • Typography 更新为新版字体规模
// Material 3（推荐）
ThemeData(
  useMaterial3: true,
  colorSchemeSeed: Colors.deepPurple,
)

// Material 2（旧版，逐渐淘汰）
ThemeData(
  useMaterial3: false,
  primarySwatch: Colors.deepPurple,
)

最佳实践：新项目一律使用 useMaterial3: true。已有项目如果仍在使用 Material 2，建议制定迁移计划逐步切换。

2. ColorScheme：Material 3 颜色体系

2.1 理解 ColorScheme 的颜色角色

ColorScheme 是 Material 3 颜色系统的核心。它不是简单地定义"主色"和"强调色"，而是定义了一组颜色角色（Color Roles），每种角色都有明确的用途和语义。

颜色角色 用途说明 搭配关系
primary 主品牌色，用于关键 UI 元素 onPrimary（文字/图标在 primary 上的颜色）
primaryContainer 主色容器背景 onPrimaryContainer（容器内的文字/图标）
secondary 次要品牌色，辅助 UI 元素 onSecondary
secondaryContainer 次色容器背景 onSecondaryContainer
tertiary 第三品牌色，平衡和对比 onTertiary
tertiaryContainer 第三色容器背景 onTertiaryContainer
surface 表面色（Card、Dialog 背景） onSurface（表面上的文字）
surfaceContainerHighest 最高层级容器 —
error 错误提示色 onError
errorContainer 错误容器背景 onErrorContainer
outline 边框/分割线颜色 —
outlineVariant 次要边框颜色 —
shadow 阴影颜色 —
inverseSurface 反转表面色（用于 SnackBar 等） onInverseSurface
inversePrimary 反转主色 —

核心配对原则：xxx 和 onXxx 总是成对使用——xxx 做背景色时，onXxx 做前景色（文字/图标），以确保对比度和可读性。

2.2 ColorScheme.fromSeed 从种子色生成

这是 Material 3 推荐的创建 ColorScheme 的方式。只需提供一个种子颜色（Seed Color），算法会自动生成一套和谐的、符合无障碍对比度要求的完整配色方案。

import 'package:flutter/material.dart';

void main() => runApp(const ColorSchemeDemo());

class ColorSchemeDemo extends StatelessWidget {
  const ColorSchemeDemo({super.key});

  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      theme: ThemeData(
        useMaterial3: true,
        // 从种子色生成亮色方案
        colorScheme: ColorScheme.fromSeed(
          seedColor: Colors.teal,
          brightness: Brightness.light,
        ),
      ),
      darkTheme: ThemeData(
        useMaterial3: true,
        // 从相同的种子色生成暗色方案
        colorScheme: ColorScheme.fromSeed(
          seedColor: Colors.teal,
          brightness: Brightness.dark,
        ),
      ),
      home: const ColorSchemeShowcase(),
    );
  }
}

  const ColorSchemeShowcase({super.key});

2.3 ColorScheme.fromImageProvider 从图片提取配色

Material 3 还支持从图片中提取主色调来生成 ColorScheme，这在需要根据内容动态调整主题色时非常有用（例如音乐播放器根据封面变色）。

import 'package:flutter/material.dart';

class DynamicThemeFromImage extends StatefulWidget {
  const DynamicThemeFromImage({super.key});

  @override
  State<DynamicThemeFromImage> createState() => _DynamicThemeFromImageState();
}

class _DynamicThemeFromImageState extends State<DynamicThemeFromImage> {
  ColorScheme? _colorScheme;
  final String _imageUrl = 'https://picsum.photos/200';

  @override
  void initState() {
    super.initState();
    _extractColors();
  }

  Future<void> _extractColors() async {
    // 从网络图片提取配色方案
    final colorScheme = await ColorScheme.fromImageProvider(
      provider: NetworkImage(_imageUrl),
      brightness: Brightness.light,
    );
    setState(() => _colorScheme = colorScheme);
  }

  @override
  Widget build(BuildContext context) {
    final effectiveScheme = _colorScheme ??
        ColorScheme.fromSeed(seedColor: Colors.blue);

    return Theme(
      data: ThemeData(
        useMaterial3: true,
        colorScheme: effectiveScheme,
      ),
      child: Scaffold(
        appBar: AppBar(title: const Text('图片动态配色')),
        body: Center(
          child: Column(
            mainAxisAlignment: MainAxisAlignment.center,
            children: [
              ClipRRect(
                borderRadius: BorderRadius.circular(16),
                child: Image.network(_imageUrl, width: 200, height: 200, fit: BoxFit.cover),
              ),
              const SizedBox(height: 24),
              if (_colorScheme != null) ...[
                FilledButton(
                  onPressed: () {},
                  child: const Text('从图片提取的主色'),
                ),
                const SizedBox(height: 8),
                FilledButton.tonal(
                  onPressed: () {},
                  child: const Text('色调按钮'),
                ),
              ] else
                const CircularProgressIndicator(),
            ],
          ),
        ),
      ),
    );
  }
}

2.4 手动自定义 ColorScheme

如果设计师提供了精确的品牌色值，你可以手动构建 ColorScheme，也可以在 fromSeed 的基础上覆盖部分颜色：

// 方式一：在种子色基础上覆盖部分颜色
final lightScheme = ColorScheme.fromSeed(
  seedColor: const Color(0xFF6750A4),
  brightness: Brightness.light,
).copyWith(
  // 覆盖特定的颜色角色
  secondary: const Color(0xFF625B71),
  error: const Color(0xFFBA1A1A),
);

// 方式二：完全手动定义（适合有严格品牌规范的项目）
const brandScheme = ColorScheme(
  brightness: Brightness.light,
  primary: Color(0xFF1A73E8),
  onPrimary: Color(0xFFFFFFFF),
  primaryContainer: Color(0xFFD3E3FD),
  onPrimaryContainer: Color(0xFF001D36),
  secondary: Color(0xFF5F6368),
  onSecondary: Color(0xFFFFFFFF),
  secondaryContainer: Color(0xFFE8EAED),
  onSecondaryContainer: Color(0xFF1D1B20),
  tertiary: Color(0xFF188038),
  onTertiary: Color(0xFFFFFFFF),
  tertiaryContainer: Color(0xFFCEEAD6),
  onTertiaryContainer: Color(0xFF0D3B1A),
  error: Color(0xFFD93025),
  onError: Color(0xFFFFFFFF),
  errorContainer: Color(0xFFFCE8E6),
  onErrorContainer: Color(0xFF410E0B),
  surface: Color(0xFFFEF7FF),
  onSurface: Color(0xFF1D1B20),
  outline: Color(0xFF79747E),
  outlineVariant: Color(0xFFCAC4D0),
  shadow: Color(0xFF000000),
);

注意事项：

  1. 使用 colorSchemeSeed 时，ThemeData 会自动调用 ColorScheme.fromSeed，此时不要同时设置 colorScheme 参数，否则会冲突。
  2. fromSeed 生成的颜色经过无障碍对比度验证，手动定义时请自行确保前景/背景色的对比度比达到 4.5:1（AA 级）以上。

3. TextTheme：统一文本样式管理

3.1 Material 3 的 15 种文本样式

Material 3 定义了 5 个类别（Display、Headline、Title、Body、Label）× 3 种尺寸（Large、Medium、Small），共 15 种文本样式：

类别 Large Medium Small 典型用途
Display displayLarge displayMedium displaySmall 超大标题，用于英雄区域
Headline headlineLarge headlineMedium headlineSmall 标题，用于页面/区域标题
Title titleLarge titleMedium titleSmall 子标题，用于 AppBar、对话框标题
Body bodyLarge bodyMedium bodySmall 正文，最常用的文本样式
Label labelLarge labelMedium labelSmall 标签，用于按钮文字、表单标签

3.3 集成 Google Fonts

google_fonts 包可以让你轻松使用 Google Fonts 中的千余种字体，并与 TextTheme 无缝集成：

# pubspec.yaml
dependencies:
  google_fonts: ^6.1.0

import 'package:flutter/material.dart';
import 'package:google_fonts/google_fonts.dart';

class GoogleFontsThemeDemo extends StatelessWidget {
  const GoogleFontsThemeDemo({super.key});

  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      theme: ThemeData(
        useMaterial3: true,
        colorSchemeSeed: Colors.green,
        // 方式一：整体替换 TextTheme 的字体族
        textTheme: GoogleFonts.notoSansScTextTheme(
          Theme.of(context).textTheme,
        ),
      ),
      home: Scaffold(
        appBar: AppBar(title: const Text('Google Fonts 集成')),
        body: Center(
          child: Column(
            mainAxisAlignment: MainAxisAlignment.center,
            children: [
              // 方式二：在单个 TextStyle 上使用 Google Fonts
              Text(
                '自定义字体标题',
                style: GoogleFonts.notoSerifSc(
                  fontSize: 28,
                  fontWeight: FontWeight.bold,
                ),
              ),
              const SizedBox(height: 16),
              Text(
                '正文内容使用思源黑体',
                style: GoogleFonts.notoSansSc(fontSize: 16),
              ),
            ],
          ),
        ),
      ),
    );
  }
}

最佳实践：

  1. 中文项目推荐 Noto Sans SC（思源黑体）或 Noto Serif SC（思源宋体）作为基础字体。
  2. 在 ThemeData.textTheme 中统一设置字体，避免在各组件中分散指定。
  3. 生产环境建议将字体文件打包到 assets 中（GoogleFonts.config.allowRuntimeFetching = false），避免运行时下载。

4. 组件主题定制

4.1 组件主题的层级关系

Material 组件在决定自身样式时，按以下优先级从高到低查找：

组件自身的样式参数（如 ElevatedButton 的 style 参数）
     ↓ 未指定时
局部 Theme widget 中的组件主题
     ↓ 未指定时
MaterialApp.theme 中的全局组件主题
     ↓ 未指定时
ThemeData 内置的默认值

    // Divider 主题
    dividerTheme: DividerThemeData(
      color: colorScheme.outlineVariant,
      thickness: 1,
      space: 1,
    ),

    // FloatingActionButton 主题
    floatingActionButtonTheme: FloatingActionButtonThemeData(
      backgroundColor: colorScheme.primaryContainer,
      foregroundColor: colorScheme.onPrimaryContainer,
      elevation: 3,
      shape: RoundedRectangleBorder(
        borderRadius: BorderRadius.circular(16),
      ),
    ),

    // NavigationBar 主题（底部导航）
    navigationBarTheme: NavigationBarThemeData(
      indicatorColor: colorScheme.primaryContainer,
      labelTextStyle: WidgetStatePropertyAll(
        TextStyle(fontSize: 12, color: colorScheme.onSurface),
      ),
    ),
  );
}

最佳实践：

  1. 将主题配置抽取到独立文件（如 lib/theme/app_theme.dart），不要把大量主题代码堆在 main.dart 中。
  2. 使用 ColorScheme 中的语义颜色而非硬编码颜色值，这样切换亮/暗模式时组件颜色会自动适配。
  3. 先从全局组件主题入手，只在必要时才在单个组件上使用 style 参数做局部覆盖。

5.3 使用 ValueNotifier 管理主题状态（推荐模式）

在实际项目中，我们通常需要在应用的不同位置访问和修改主题状态。使用 ValueNotifier + ValueListenableBuilder 是一种轻量级解决方案：

import 'package:flutter/material.dart';

// 全局主题状态管理器
class ThemeNotifier extends ValueNotifier<ThemeMode> {
  ThemeNotifier() : super(ThemeMode.system);

  void setThemeMode(ThemeMode mode) => value = mode;

  void toggleBrightness(BuildContext context) {
    final brightness = Theme.of(context).brightness;
    value = brightness == Brightness.dark ? ThemeMode.light : ThemeMode.dark;
  }
}

// 在顶层创建实例
final themeNotifier = ThemeNotifier();

void main() => runApp(const MyApp());

class MyApp extends StatelessWidget {
  const MyApp({super.key});

  @override
  Widget build(BuildContext context) {
    return ValueListenableBuilder<ThemeMode>(
      valueListenable: themeNotifier,
      builder: (context, themeMode, _) {
        return MaterialApp(
          theme: ThemeData(
            useMaterial3: true,
            colorSchemeSeed: Colors.blue,
            brightness: Brightness.light,
          ),
          darkTheme: ThemeData(
            useMaterial3: true,
            colorSchemeSeed: Colors.blue,
            brightness: Brightness.dark,
          ),
          themeMode: themeMode,
          home: const HomePage(),
        );
      },
    );
  }
}

class HomePage extends StatelessWidget {
  const HomePage({super.key});

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(
        title: const Text('首页'),
        actions: [
          // 在任意位置都可以切换主题
          IconButton(
            icon: Icon(
              Theme.of(context).brightness == Brightness.dark
                  ? Icons.light_mode
                  : Icons.dark_mode,
            ),
            onPressed: () => themeNotifier.toggleBrightness(context),
          ),
        ],
      ),
      body: const Center(child: Text('使用 ValueNotifier 管理主题')),
    );
  }
}

注意事项：

  1. 如果项目中已经使用了状态管理方案（如 Riverpod、Bloc、Provider），直接用其管理 ThemeMode 即可，无需额外引入 ValueNotifier。
  2. 切换主题后，MaterialApp 会重建整个子树，所有 Material 组件会自动读取新主题——你不需要逐个组件更新。
  3. 用户的主题偏好应该持久化存储（如 SharedPreferences），在应用启动时恢复。

6.3 ThemeData.copyWith 的妙用

copyWith 允许你在现有主题基础上只修改部分属性，其余属性保持不变。这是局部覆盖和主题变体的关键方法。

// 创建一个"高对比度"变体
final highContrastTheme = Theme.of(context).copyWith(
  textTheme: Theme.of(context).textTheme.apply(
    fontSizeFactor: 1.2, // 所有字号放大 20%
    bodyColor: Colors.black, // 正文颜色加深
    displayColor: Colors.black, // 标题颜色加深
  ),
);

// 创建一个局部的"错误区域"主题
final errorZoneTheme = Theme.of(context).copyWith(
  colorScheme: Theme.of(context).colorScheme.copyWith(
    primary: Theme.of(context).colorScheme.error,
    primaryContainer: Theme.of(context).colorScheme.errorContainer,
  ),
);

注意事项：

  1. Theme.of(context) 依赖 BuildContext，必须在 Widget 的 build 方法或之后调用，不能在 initState 中使用。
  2. 局部 Theme widget 中使用 Theme.of(context) 时注意 context 的位置——如果 context 位于 Theme widget 之上，获取到的仍然是父主题。需要用 Builder 获取新的 context。

7. 自定义主题扩展 ThemeExtension<T>

7.1 为什么需要 ThemeExtension

ThemeData 内置了大量 Material 组件的主题配置，但在实际项目中，我们经常会有一些自定义的样式需求无法被现有属性覆盖。例如：

  • 自定义品牌渐变色
  • 特定业务组件的颜色（如"交易成功绿"、"交易失败红"）
  • 自定义间距体系
  • 自定义阴影配置

ThemeExtension<T> 就是为解决这一问题而设计的。它允许你定义任意结构的主题数据类，并将其挂载到 ThemeData 上，享受与原生主题一样的读取方式和亮/暗模式插值支持。

7.2 定义自定义主题数据类

import 'package:flutter/material.dart';

/// 自定义品牌颜色扩展
class BrandColors extends ThemeExtension<BrandColors> {
  const BrandColors({
    required this.success,
    required this.onSuccess,
    required this.warning,
    required this.onWarning,
    required this.info,
    required this.onInfo,
    required this.gradientStart,
    required this.gradientEnd,
  });

  /// 成功状态颜色
  final Color success;
  final Color onSuccess;

  /// 警告状态颜色
  final Color warning;
  final Color onWarning;

  /// 信息状态颜色
  final Color info;
  final Color onInfo;

  /// 品牌渐变色
  final Color gradientStart;
  final Color gradientEnd;

  /// copyWith：返回新实例，未提供的字段保持不变
  @override
  BrandColors copyWith({
    Color? success,
    Color? onSuccess,
    Color? warning,
    Color? onWarning,
    Color? info,
    Color? onInfo,
    Color? gradientStart,
    Color? gradientEnd,
  }) {
    return BrandColors(
      success: success ?? this.success,
      onSuccess: onSuccess ?? this.onSuccess,
      warning: warning ?? this.warning,
      onWarning: onWarning ?? this.onWarning,
      info: info ?? this.info,
      onInfo: onInfo ?? this.onInfo,
      gradientStart: gradientStart ?? this.gradientStart,
      gradientEnd: gradientEnd ?? this.gradientEnd,
    );
  }

  /// lerp：在两个主题之间进行线性插值（用于亮/暗模式切换动画）
  @override
  BrandColors lerp(covariant BrandColors? other, double t) {
    if (other == null) return this;
    return BrandColors(
      success: Color.lerp(success, other.success, t)!,
      onSuccess: Color.lerp(onSuccess, other.onSuccess, t)!,
      warning: Color.lerp(warning, other.warning, t)!,
      onWarning: Color.lerp(onWarning, other.onWarning, t)!,
      info: Color.lerp(info, other.info, t)!,
      onInfo: Color.lerp(onInfo, other.onInfo, t)!,
      gradientStart: Color.lerp(gradientStart, other.gradientStart, t)!,
      gradientEnd: Color.lerp(gradientEnd, other.gradientEnd, t)!,
    );
  }
}

最佳实践：

  1. 每个 ThemeExtension 子类负责一个独立的职责域（颜色、间距、阴影等），遵循单一职责原则。
  2. 一定要正确实现 lerp 方法——它是亮/暗主题切换动画平滑过渡的关键。对于 Color 类型用 Color.lerp，double 类型用 lerpDouble，BoxShadow 类型用 BoxShadow.lerp。
  3. 为 extension<T>() 的返回值做空安全处理：开发阶段可以用 ! 强制解包（配合断言检测），正式代码建议提供默认值兜底。
  4. 推荐为常用扩展添加 BuildContext 上的便捷扩展方法：
extension BrandColorsExtension on BuildContext {
  BrandColors get brandColors => Theme.of(this).extension<BrandColors>()!;
}

extension AppSpacingExtension on BuildContext {
  AppSpacing get spacing => Theme.of(this).extension<AppSpacing>()!;
}

// 使用更简洁
final colors = context.brandColors;
final spacing = context.spacing;

class BrandColors extends ThemeExtension<BrandColors> {
  const BrandColors({
    required this.success,
    required this.onSuccess,
    required this.successContainer,
    required this.onSuccessContainer,
    required this.warning,
    required this.onWarning,
    required this.warningContainer,
    required this.onWarningContainer,
    required this.info,
    required this.onInfo,
    required this.gradientColors,
    required this.shimmerBase,
    required this.shimmerHighlight,
  });

  final Color success;
  final Color onSuccess;
  final Color successContainer;
  final Color onSuccessContainer;
  final Color warning;
  final Color onWarning;
  final Color warningContainer;
  final Color onWarningContainer;
  final Color info;
  final Color onInfo;
  final List<Color> gradientColors;
  final Color shimmerBase;
  final Color shimmerHighlight;

  @override
  BrandColors copyWith({
    Color? success,
    Color? onSuccess,
    Color? successContainer,
    Color? onSuccessContainer,
    Color? warning,
    Color? onWarning,
    Color? warningContainer,
    Color? onWarningContainer,
    Color? info,
    Color? onInfo,
    List<Color>? gradientColors,
    Color? shimmerBase,
    Color? shimmerHighlight,
  }) {
    return BrandColors(
      success: success ?? this.success,
      onSuccess: onSuccess ?? this.onSuccess,
      successContainer: successContainer ?? this.successContainer,
      onSuccessContainer: onSuccessContainer ?? this.onSuccessContainer,
      warning: warning ?? this.warning,
      onWarning: onWarning ?? this.onWarning,
      warningContainer: warningContainer ?? this.warningContainer,
      onWarningContainer: onWarningContainer ?? this.onWarningContainer,
      info: info ?? this.info,
      onInfo: onInfo ?? this.onInfo,
      gradientColors: gradientColors ?? this.gradientColors,
      shimmerBase: shimmerBase ?? this.shimmerBase,
      shimmerHighlight: shimmerHighlight ?? this.shimmerHighlight,
    );
  }

  @override
  BrandColors lerp(covariant BrandColors? other, double t) {
    if (other == null) return this;
    return BrandColors(
      success: Color.lerp(success, other.success, t)!,
      onSuccess: Color.lerp(onSuccess, other.onSuccess, t)!,
      successContainer: Color.lerp(successContainer, other.successContainer, t)!,
      onSuccessContainer: Color.lerp(onSuccessContainer, other.onSuccessContainer, t)!,
      warning: Color.lerp(warning, other.warning, t)!,
      onWarning: Color.lerp(onWarning, other.onWarning, t)!,
      warningContainer: Color.lerp(warningContainer, other.warningContainer, t)!,
      onWarningContainer: Color.lerp(onWarningContainer, other.onWarningContainer, t)!,
      info: Color.lerp(info, other.info, t)!,
      onInfo: Color.lerp(onInfo, other.onInfo, t)!,
      gradientColors: _lerpColorList(gradientColors, other.gradientColors, t),
      shimmerBase: Color.lerp(shimmerBase, other.shimmerBase, t)!,
      shimmerHighlight: Color.lerp(shimmerHighlight, other.shimmerHighlight, t)!,
    );
  }

  static List<Color> _lerpColorList(List<Color> a, List<Color> b, double t) {
    final length = a.length < b.length ? a.length : b.length;
    return List.generate(length, (i) => Color.lerp(a[i], b[i], t)!);
  }
}

class AppSpacing extends ThemeExtension<AppSpacing> {
  const AppSpacing({
    this.xs = 4.0,
    this.sm = 8.0,
    this.md = 16.0,
    this.lg = 24.0,
    this.xl = 32.0,
    this.xxl = 48.0,
  });

  final double xs;
  final double sm;
  final double md;
  final double lg;
  final double xl;
  final double xxl;

  @override
  AppSpacing copyWith({
    double? xs, double? sm, double? md,
    double? lg, double? xl, double? xxl,
  }) {
    return AppSpacing(
      xs: xs ?? this.xs, sm: sm ?? this.sm, md: md ?? this.md,
      lg: lg ?? this.lg, xl: xl ?? this.xl, xxl: xxl ?? this.xxl,
    );
  }

  @override
  AppSpacing lerp(covariant AppSpacing? other, double t) {
    if (other == null) return this;
    return AppSpacing(
      xs: lerpDouble(xs, other.xs, t)!,
      sm: lerpDouble(sm, other.sm, t)!,
      md: lerpDouble(md, other.md, t)!,
      lg: lerpDouble(lg, other.lg, t)!,
      xl: lerpDouble(xl, other.xl, t)!,
      xxl: lerpDouble(xxl, other.xxl, t)!,
    );
  }
}

      ? _lightColorScheme

    brightness: brightness,
    colorScheme: colorScheme,

    // 组件主题
    appBarTheme: _appBarTheme(colorScheme),
    elevatedButtonTheme: _elevatedButtonTheme(colorScheme),
    outlinedButtonTheme: _outlinedButtonTheme(colorScheme),
    filledButtonTheme: _filledButtonTheme(colorScheme),

    navigationBarTheme: _navigationBarTheme(colorScheme),
    floatingActionButtonTheme: _fabTheme(colorScheme),
    dialogTheme: _dialogTheme(colorScheme),
    bottomSheetTheme: _bottomSheetTheme(colorScheme),
    chipTheme: _chipTheme(colorScheme),
    dividerTheme: _dividerTheme(colorScheme),

    // 自定义扩展
    extensions: <ThemeExtension>[
      brandColors,
      const AppSpacing(),
    ],
  );
}

  Widget _statusChip(IconData icon, String label, Color bg, Color fg) {
    return Container(
      padding: const EdgeInsets.symmetric(vertical: 12),
      decoration: BoxDecoration(
        color: bg,
        borderRadius: BorderRadius.circular(12),
      ),
      child: Column(
        children: [
          Icon(icon, color: fg, size: 28),
          const SizedBox(height: 4),
          Text(label, style: TextStyle(color: fg, fontWeight: FontWeight.w600)),
        ],
      ),
    );
  }
}

实战要点总结：

  1. buildAppTheme 工厂函数接受 Brightness 参数，一套代码生成亮/暗两套主题。
  2. 所有组件主题工厂函数都以 ColorScheme 作为参数，确保颜色来源统一。
  3. ThemeExtension 用于管理 Material 组件体系之外的品牌色和间距。
  4. BuildContext 扩展方法让自定义主题的读取和 Theme.of(context) 一样方便。
  5. 主题切换按钮在 system → light → dark 三种模式间循环。

9. 本课小结

核心概念回顾

概念 要点
ThemeData Flutter 主题系统的核心数据类，通过 MaterialApp 注入到 Widget 树
ColorScheme Material 3 颜色体系，推荐用 fromSeed 从种子色生成
TextTheme 15 种标准文本样式（5 类别 × 3 尺寸），统一管理全局文字风格
组件主题 每种 Material 组件都有对应的 Theme 类，用于全局定制默认样式
ThemeMode 控制使用亮色 / 暗色 / 跟随系统的主题
Theme.of(context) 从 Widget 树中读取最近的 ThemeData
Theme widget 在子树中局部覆盖主题
ThemeExtension<T> 扩展自定义主题数据，需实现 copyWith 和 lerp

架构决策清单

  • 新项目 → useMaterial3: true + ColorScheme.fromSeed
  • 颜色管理 → 所有颜色来自 ColorScheme 或 ThemeExtension，不硬编码
  • 文本管理 → 所有文本样式来自 TextTheme，通过 Theme.of(context).textTheme 获取
  • 组件样式 → 优先通过组件主题全局配置，局部覆盖用 style 参数
  • 亮/暗模式 → MaterialApp 同时提供 theme 和 darkTheme
  • 自定义数据 → 使用 ThemeExtension，不要用全局常量或静态变量
