1. TextField — 基础输入框

1.1 简介

TextField 是 Flutter 中最基本的文本输入组件，对应 Material Design 的文本输入框。用户可以通过键盘在 TextField 中输入文本，开发者可以通过各种属性控制其外观和行为。

1.2 基本用法

最简单的 TextField 不需要任何参数即可使用：

import 'package:flutter/material.dart';

void main() => runApp(const MyApp());

class MyApp extends StatelessWidget {
  const MyApp({super.key});

  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      home: Scaffold(
        appBar: AppBar(title: const Text('TextField 基础示例')),
        body: const Padding(
          padding: EdgeInsets.all(16.0),
          child: Column(
            children: [
              // 最简单的输入框
              TextField(),

              SizedBox(height: 16),

              // 带提示文本的输入框
              TextField(
                decoration: InputDecoration(
                  labelText: '用户名',
                  hintText: '请输入用户名',
                ),
              ),
            ],
          ),
        ),
      ),
    );
  }
}

1.4 onChanged 与 onSubmitted

import 'package:flutter/material.dart';

void main() => runApp(const MyApp());

class MyApp extends StatelessWidget {
  const MyApp({super.key});

  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      home: const TextFieldCallbackDemo(),
    );
  }
}

class TextFieldCallbackDemo extends StatefulWidget {
  const TextFieldCallbackDemo({super.key});

  @override
  State<TextFieldCallbackDemo> createState() => _TextFieldCallbackDemoState();
}

class _TextFieldCallbackDemoState extends State<TextFieldCallbackDemo> {
  String _currentText = '';
  String _submittedText = '';

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(title: const Text('回调示例')),
      body: Padding(
        padding: const EdgeInsets.all(16.0),
        child: Column(
          crossAxisAlignment: CrossAxisAlignment.start,
          children: [
            TextField(
              decoration: const InputDecoration(
                labelText: '搜索',
                hintText: '输入关键词后按回车搜索',
                prefixIcon: Icon(Icons.search),
              ),
              textInputAction: TextInputAction.search,
              // 每次文本变化都会触发
              onChanged: (value) {
                setState(() {
                  _currentText = value;
                });
              },
              // 按下键盘确认键时触发
              onSubmitted: (value) {
                setState(() {
                  _submittedText = value;
                });
              },
            ),
            const SizedBox(height: 16),
            Text('实时输入：$_currentText'),
            Text('已提交：$_submittedText'),
          ],
        ),
      ),
    );
  }
}

注意事项：

  • onChanged 在每次文本变化时都会触发，适合实时搜索、字数统计等场景。
  • onSubmitted 在用户按下键盘上的确认键（回车/搜索/完成等）时触发，适合提交操作。
  • 如果需要更精细地控制文本，推荐使用 TextEditingController 而非 onChanged。

2. InputDecoration 详解

InputDecoration 是 TextField 中最重要的属性之一，它控制输入框的所有装饰性外观，包括标签、提示、图标、边框、错误信息等。由于属性众多，我们按功能分类讲解。

2.1 文本类属性

这组属性控制输入框中显示的各种提示文本：

属性 说明 显示时机
labelText 标签文本，浮动在输入框上方 聚焦或有内容时浮动到上方，否则显示在输入区域
hintText 占位提示文本 输入框为空且获得焦点时显示
helperText 帮助文本 始终显示在输入框下方
errorText 错误提示文本 设置后显示在输入框下方，同时输入框变为错误样式
counterText 计数器文本 显示在输入框右下方 
import 'package:flutter/material.dart';

void main() => runApp(const MyApp());

class MyApp extends StatelessWidget {
  const MyApp({super.key});

  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      home: Scaffold(
        appBar: AppBar(title: const Text('文本类属性')),
        body: Padding(
          padding: const EdgeInsets.all(16.0),
          child: Column(
            children: [
              // labelText：标签文本
              // 未聚焦时显示在输入区域，聚焦时浮动到上方
              const TextField(
                decoration: InputDecoration(
                  labelText: '邮箱地址',
                ),
              ),
              const SizedBox(height: 24),

              // hintText：占位提示
              // 输入框为空且聚焦时显示
              const TextField(
                decoration: InputDecoration(
                  labelText: '手机号码',
                  hintText: '请输入11位手机号码',
                ),
              ),
              const SizedBox(height: 24),

              // helperText：帮助说明
              // 始终显示在输入框下方
              const TextField(
                decoration: InputDecoration(
                  labelText: '密码',
                  helperText: '密码长度至少8位，需包含字母和数字',
                ),
              ),
              const SizedBox(height: 24),

              // errorText：错误提示
              // 设置后输入框变为错误状态
              const TextField(
                decoration: InputDecoration(
                  labelText: '用户名',
                  errorText: '用户名已被占用',
                ),
              ),
              const SizedBox(height: 24),

              // counterText：计数器文本
              const TextField(
                decoration: InputDecoration(
                  labelText: '个人简介',
                  counterText: '0/200',
                ),
              ),
            ],
          ),
        ),
      ),
    );
  }
}

注意事项：

  • 当 errorText 不为 null 时，helperText 会被替换为错误文本显示。
  • labelText 和 hintText 可以同时使用，Flutter 会智能地管理它们的显示时机。
  • 如果同时设置了 maxLength，TextField 会自动显示字符计数器（如 "5/20"），此时 counterText 可用于自定义计数文本或设置为空字符串来隐藏默认计数器。

2.2 图标类属性

InputDecoration 提供了多种方式在输入框中添加图标或自定义组件：

属性 位置 说明
icon 输入框外部左侧 显示在整个输入框左侧
prefixIcon 输入框内部左侧 常用于搜索图标、手机图标等
suffixIcon 输入框内部右侧 常用于清除按钮、密码可见切换等
prefix 输入文字前方 自定义前缀 Widget，仅在聚焦时显示
suffix 输入文字后方 自定义后缀 Widget，仅在聚焦时显示
prefixText 输入文字前方 前缀文本，仅在聚焦时显示
suffixText 输入文字后方 后缀文本，仅在聚焦时显示 
import 'package:flutter/material.dart';

void main() => runApp(const MyApp());

class MyApp extends StatelessWidget {
  const MyApp({super.key});

  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      home: const IconDecorationDemo(),
    );
  }
}

class IconDecorationDemo extends StatefulWidget {
  const IconDecorationDemo({super.key});

  @override
  State<IconDecorationDemo> createState() => _IconDecorationDemoState();
}

class _IconDecorationDemoState extends State<IconDecorationDemo> {
  bool _obscureText = true;
  final _searchController = TextEditingController();

  @override
  void dispose() {
    _searchController.dispose();
    super.dispose();
  }

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(title: const Text('图标类属性')),
      body: Padding(
        padding: const EdgeInsets.all(16.0),
        child: Column(
          children: [
            // icon：输入框外部左侧图标
            const TextField(
              decoration: InputDecoration(
                icon: Icon(Icons.person),
                labelText: '用户名',
              ),
            ),
            const SizedBox(height: 24),

            // prefixIcon：输入框内部左侧图标
            // suffixIcon：输入框内部右侧图标（此处用作清除按钮）
            TextField(
              controller: _searchController,
              decoration: InputDecoration(
                prefixIcon: const Icon(Icons.search),
                suffixIcon: IconButton(
                  icon: const Icon(Icons.clear),
                  onPressed: () => _searchController.clear(),
                ),
                labelText: '搜索',
                hintText: '输入搜索关键词',
              ),
            ),
            const SizedBox(height: 24),

            // 密码输入框：suffixIcon 切换密码可见性
            TextField(
              obscureText: _obscureText,
              decoration: InputDecoration(
                prefixIcon: const Icon(Icons.lock),
                suffixIcon: IconButton(
                  icon: Icon(
                    _obscureText ? Icons.visibility_off : Icons.visibility,
                  ),
                  onPressed: () {
                    setState(() {
                      _obscureText = !_obscureText;
                    });
                  },
                ),
                labelText: '密码',
              ),
            ),
            const SizedBox(height: 24),

            // prefix 与 suffix：自定义前后缀 Widget
            const TextField(
              keyboardType: TextInputType.number,
              decoration: InputDecoration(
                labelText: '金额',
                prefixText: '¥ ',
                suffixText: '元',
              ),
            ),
            const SizedBox(height: 24),

            // prefix 使用自定义 Widget
            const TextField(
              decoration: InputDecoration(
                labelText: '网站地址',
                prefix: Text(
                  'https://',
                  style: TextStyle(color: Colors.grey),
                ),
              ),
            ),
          ],
        ),
      ),
    );
  }
}

最佳实践：

  • prefixIcon / suffixIcon 适合放置固定图标，它们始终可见。
  • prefix / suffix 适合放置与输入内容紧密相关的前后缀（如货币符号、单位），它们仅在输入框获得焦点或有内容时才显示。
  • 清除按钮是 suffixIcon 最常见的用途之一，记得在文本为空时隐藏它。

2.5 综合示例：自定义精美输入框

将上述所有装饰属性组合起来，创建一个视觉效果完善的输入框：

import 'package:flutter/material.dart';

void main() => runApp(const MyApp());

class MyApp extends StatelessWidget {
  const MyApp({super.key});

  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      theme: ThemeData(
        colorSchemeSeed: Colors.indigo,
        useMaterial3: true,
      ),
      home: const CustomInputDemo(),
    );
  }
}

class CustomInputDemo extends StatefulWidget {
  const CustomInputDemo({super.key});

  @override
  State<CustomInputDemo> createState() => _CustomInputDemoState();
}

class _CustomInputDemoState extends State<CustomInputDemo> {
  final _emailController = TextEditingController();
  String? _errorText;

  @override
  void dispose() {
    _emailController.dispose();
    super.dispose();
  }

  void _validateEmail(String value) {
    setState(() {
      if (value.isEmpty) {
        _errorText = null;
      } else if (!RegExp(r'^[\w-\.]+@([\w-]+\.)+[\w-]{2,4}$').hasMatch(value)) {
        _errorText = '请输入有效的邮箱地址';
      } else {
        _errorText = null;
      }
    });
  }

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(title: const Text('精美输入框')),
      body: Padding(
        padding: const EdgeInsets.all(24.0),
        child: TextField(
          controller: _emailController,
          keyboardType: TextInputType.emailAddress,
          onChanged: _validateEmail,
          decoration: InputDecoration(
            // 文本类
            labelText: '邮箱地址',
            hintText: 'example@email.com',
            helperText: '我们不会向第三方分享您的邮箱',
            errorText: _errorText,

            // 图标类
            prefixIcon: const Icon(Icons.email_outlined),
            suffixIcon: _emailController.text.isNotEmpty
                ? IconButton(
                    icon: const Icon(Icons.clear),
                    onPressed: () {
                      _emailController.clear();
                      _validateEmail('');
                    },
                  )
                : null,

            // 填充
            filled: true,
            fillColor: Theme.of(context).colorScheme.surfaceContainerHighest.withOpacity(0.3),

            // 边框
            border: OutlineInputBorder(
              borderRadius: BorderRadius.circular(12),
            ),
            enabledBorder: OutlineInputBorder(
              borderRadius: BorderRadius.circular(12),
              borderSide: BorderSide(
                color: Colors.grey.shade300,
              ),
            ),
            focusedBorder: OutlineInputBorder(
              borderRadius: BorderRadius.circular(12),
              borderSide: BorderSide(
                color: Theme.of(context).colorScheme.primary,
                width: 2,
              ),
            ),

            // 间距
            contentPadding: const EdgeInsets.symmetric(
              horizontal: 16,
              vertical: 16,
            ),
          ),
        ),
      ),
    );
  }
}

最佳实践：

  • 在实际项目中，建议通过 ThemeData 中的 InputDecorationTheme 统一配置输入框样式，避免在每个 TextField 中重复设置。
  • 使用 filled: true + borderSide: BorderSide.none 可以创建现代风格的无边框填充输入框。
  • contentPadding 对触控体验很重要——太小会导致用户难以点击，推荐最小高度为 48dp。

3. TextEditingController — 文本控制器

3.1 基本概念

TextEditingController 是管理 TextField 文本内容的控制器。相比 onChanged 回调，它提供了更强大的文本操作能力：

  • 获取和设置文本内容
  • 监听文本变化
  • 控制光标位置和文本选择

3.2 基本用法：获取与设置文本

import 'package:flutter/material.dart';

void main() => runApp(const MyApp());

class MyApp extends StatelessWidget {
  const MyApp({super.key});

  @override
  Widget build(BuildContext context) {
    return MaterialApp(home: const ControllerBasicDemo());
  }
}

class ControllerBasicDemo extends StatefulWidget {
  const ControllerBasicDemo({super.key});

  @override
  State<ControllerBasicDemo> createState() => _ControllerBasicDemoState();
}

class _ControllerBasicDemoState extends State<ControllerBasicDemo> {
  // 创建控制器，可传入初始文本
  final _controller = TextEditingController(text: '初始文本');

  @override
  void dispose() {
    // 必须在 dispose 中释放控制器，防止内存泄漏
    _controller.dispose();
    super.dispose();
  }

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(title: const Text('TextEditingController')),
      body: Padding(
        padding: const EdgeInsets.all(16.0),
        child: Column(
          children: [
            TextField(
              controller: _controller,
              decoration: const InputDecoration(
                labelText: '输入内容',
                border: OutlineInputBorder(),
              ),
            ),
            const SizedBox(height: 16),
            Wrap(
              spacing: 8,
              children: [
                // 获取文本
                ElevatedButton(
                  onPressed: () {
                    final text = _controller.text;
                    ScaffoldMessenger.of(context).showSnackBar(
                      SnackBar(content: Text('当前文本：$text')),
                    );
                  },
                  child: const Text('获取文本'),
                ),

                // 设置文本
                ElevatedButton(
                  onPressed: () {
                    _controller.text = 'Hello Flutter!';
                  },
                  child: const Text('设置文本'),
                ),

                // 清空文本
                ElevatedButton(
                  onPressed: () {
                    _controller.clear();
                  },
                  child: const Text('清空'),
                ),

                // 追加文本
                ElevatedButton(
                  onPressed: () {
                    _controller.text += ' 追加内容';
                  },
                  child: const Text('追加文本'),
                ),
              ],
            ),
          ],
        ),
      ),
    );
  }
}

3.3 监听文本变化：addListener

除了 onChanged 回调，还可以通过 TextEditingController 的 addListener 来监听变化：

import 'package:flutter/material.dart';

void main() => runApp(const MyApp());

class MyApp extends StatelessWidget {
  const MyApp({super.key});

  @override
  Widget build(BuildContext context) {

4. FocusNode — 焦点管理

4.1 基本概念

在表单应用中，管理焦点（即哪个输入框当前处于"激活"状态）非常重要。FocusNode 是 Flutter 焦点系统的核心类，允许开发者：

  • 编程式地请求或释放焦点
  • 监听焦点变化
  • 在多个输入框之间移动焦点

4.4 FocusScope — 焦点域管理

FocusScope 可以管理一组焦点节点，常用于在多个输入框之间移动焦点：

import 'package:flutter/material.dart';

void main() => runApp(const MyApp());

class MyApp extends StatelessWidget {
  const MyApp({super.key});

  @override
  Widget build(BuildContext context) {
    return MaterialApp(home: const FocusScopeDemo());
  }
}

class FocusScopeDemo extends StatefulWidget {
  const FocusScopeDemo({super.key});

  @override
  State<FocusScopeDemo> createState() => _FocusScopeDemoState();
}

class _FocusScopeDemoState extends State<FocusScopeDemo> {
  final _field1Focus = FocusNode();
  final _field2Focus = FocusNode();
  final _field3Focus = FocusNode();

  @override
  void dispose() {
    _field1Focus.dispose();
    _field2Focus.dispose();
    _field3Focus.dispose();
    super.dispose();
  }

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(title: const Text('焦点域示例')),
      body: Padding(
        padding: const EdgeInsets.all(16.0),
        child: Column(
          children: [
            // 点击键盘"下一步"自动跳转到下一个输入框
            TextField(
              focusNode: _field1Focus,
              textInputAction: TextInputAction.next,
              onSubmitted: (_) => _field2Focus.requestFocus(),
              decoration: const InputDecoration(
                labelText: '第一个字段',
                border: OutlineInputBorder(),
              ),
            ),
            const SizedBox(height: 16),
            TextField(
              focusNode: _field2Focus,
              textInputAction: TextInputAction.next,
              onSubmitted: (_) => _field3Focus.requestFocus(),
              decoration: const InputDecoration(
                labelText: '第二个字段',
                border: OutlineInputBorder(),
              ),
            ),
            const SizedBox(height: 16),
            TextField(
              focusNode: _field3Focus,
              textInputAction: TextInputAction.done,
              onSubmitted: (_) {
                _field3Focus.unfocus();
                ScaffoldMessenger.of(context).showSnackBar(
                  const SnackBar(content: Text('所有字段已填写完毕！')),
                );
              },
              decoration: const InputDecoration(
                labelText: '第三个字段',
                border: OutlineInputBorder(),
              ),
            ),
            const SizedBox(height: 16),

            // 使用 FocusScope 收起键盘
            ElevatedButton(
              onPressed: () {
                // 最简洁的收起键盘方式
                FocusScope.of(context).unfocus();
              },
              child: const Text('收起键盘'),
            ),
          ],
        ),
      ),
    );
  }
}

最佳实践：

  • 在多字段表单中，使用 textInputAction: TextInputAction.next 配合 onSubmitted 实现字段间的跳转，这是优秀的用户体验。
  • 最后一个字段使用 TextInputAction.done，按下后收起键盘或触发提交。
  • 收起键盘的推荐方式：FocusScope.of(context).unfocus()。
  • 记住在 dispose 中释放所有 FocusNode，否则会导致内存泄漏。

6.3 自定义 TextInputFormatter

当内置格式化器无法满足需求时，可以自定义 TextInputFormatter。以下示例实现一个手机号码格式化器，自动将 "13812345678" 格式化为 "138 1234 5678"：

import 'package:flutter/material.dart';
import 'package:flutter/services.dart';

void main() => runApp(const MyApp());

class MyApp extends StatelessWidget {
  const MyApp({super.key});

  @override
  Widget build(BuildContext context) {
    return MaterialApp(home: const CustomFormatterDemo());
  }
}

/// 手机号码格式化器：138 1234 5678
class PhoneNumberFormatter extends TextInputFormatter {
  @override
  TextEditingValue formatEditUpdate(
    TextEditingValue oldValue,
    TextEditingValue newValue,
  ) {
    // 移除所有非数字字符
    final digitsOnly = newValue.text.replaceAll(RegExp(r'\D'), '');

    // 限制最多 11 位数字
    final limited = digitsOnly.length > 11
        ? digitsOnly.substring(0, 11)
        : digitsOnly;

    // 按 3-4-4 格式化
    final buffer = StringBuffer();
    for (int i = 0; i < limited.length; i++) {
      if (i == 3 || i == 7) buffer.write(' ');
      buffer.write(limited[i]);
    }
    final formatted = buffer.toString();

    return TextEditingValue(
      text: formatted,
      selection: TextSelection.collapsed(offset: formatted.length),
    );
  }
}

/// 银行卡号格式化器：每4位加一个空格
class BankCardFormatter extends TextInputFormatter {
  @override
  TextEditingValue formatEditUpdate(
    TextEditingValue oldValue,
    TextEditingValue newValue,
  ) {
    final digitsOnly = newValue.text.replaceAll(RegExp(r'\D'), '');
    final limited = digitsOnly.length > 19
        ? digitsOnly.substring(0, 19)
        : digitsOnly;

    final buffer = StringBuffer();
    for (int i = 0; i < limited.length; i++) {
      if (i > 0 && i % 4 == 0) buffer.write(' ');
      buffer.write(limited[i]);
    }
    final formatted = buffer.toString();

    return TextEditingValue(
      text: formatted,
      selection: TextSelection.collapsed(offset: formatted.length),
    );
  }
}

              ),
            ),
          ],
        ),
      ),

最佳实践：

  • 自定义格式化器时，formatEditUpdate 方法接收旧值和新值，返回格式化后的值。需要同时处理好 text 和 selection（光标位置）。
  • 多个格式化器可以串联使用，会按列表顺序依次执行。
  • keyboardType 与 inputFormatters 应配合使用：前者提供合适的键盘布局，后者确保输入内容的合法性。

7.3 autovalidateMode — 自动验证模式

autovalidateMode 控制何时自动触发验证：

模式 说明
AutovalidateMode.disabled 禁用自动验证（默认），仅在调用 validate() 时触发
AutovalidateMode.always 始终自动验证（每次文本变化都验证）
AutovalidateMode.onUserInteraction 用户首次交互后自动验证（推荐） 
TextFormField(
  // 用户输入后自动验证，体验最佳
  autovalidateMode: AutovalidateMode.onUserInteraction,
  decoration: const InputDecoration(
    labelText: '邮箱',
    border: OutlineInputBorder(),
  ),
  validator: (value) {
    if (value == null || value.isEmpty) {
      return '请输入邮箱';
    }
    if (!value.contains('@')) {
      return '请输入有效的邮箱地址';
    }
    return null;
  },
)

最佳实践：

  • 推荐使用 AutovalidateMode.onUserInteraction，这样用户在首次输入前不会看到错误提示，避免页面一打开就满屏红字的糟糕体验。
  • validator 函数应保持纯函数特性——不要在其中产生副作用（如网络请求），仅做本地校验。

8. Form 与 GlobalKey<FormState>

8.1 Form 组件的作用

Form 是一个容器组件，它将多个 TextFormField 组织在一起，统一管理验证、保存和重置操作。

核心要素：

要素 说明
Form 表单容器，包裹所有表单字段
GlobalKey<FormState> 用于获取 FormState 的全局 Key
FormState.validate() 触发所有字段的 validator，返回是否全部通过
FormState.save() 触发所有字段的 onSaved
FormState.reset() 重置所有字段到初始值

注意事项：

  • GlobalKey<FormState> 必须在 State 的生命周期内保持不变（作为成员变量），不要在 build 方法中创建。
  • validate() 返回 bool：true 表示所有字段验证通过，false 表示存在验证失败的字段。
  • 调用顺序应为 validate() → save()，先验证通过再收集数据。
  • reset() 会清空所有字段的文本并移除错误提示。如果 TextFormField 设置了 initialValue，reset() 会恢复到初始值。

这个实战示例涵盖了以下核心技术点：

技术点 在示例中的应用
TextFormField + validator 每个字段都有完整的验证逻辑
TextEditingController 获取字段值，确认密码比对
FocusNode + textInputAction 字段间焦点跳转
InputDecoration labelText、hintText、prefixIcon、suffixIcon 等
obscureText 密码和确认密码的隐藏/显示切换
inputFormatters 用户名限制字符，密码禁止空格
Form + GlobalKey<FormState> 表单统一验证和提交
autovalidateMode 用户交互后自动验证
密码强度指示器 实时计算并展示密码安全等级
加载状态 提交时显示 loading，禁用按钮防重复提交
用户协议勾选 注册前的协议确认

最佳实践总结：

  1. 资源释放：所有 TextEditingController 和 FocusNode 都必须在 dispose() 中释放。
  2. 焦点跳转：使用 TextInputAction.next + onFieldSubmitted 实现字段间的流畅跳转。
  3. 自动验证：使用 AutovalidateMode.onUserInteraction，避免表单一打开就显示错误。
  4. 防重复提交：在网络请求期间禁用按钮并显示加载指示器。
  5. 密码确认：确认密码的 validator 直接与密码 controller 比对。
  6. mounted 检查：异步操作完成后调用 setState 前，先检查 mounted 避免在页面销毁后更新状态。
  7. 统一主题：通过 InputDecorationTheme 统一输入框样式，减少重复代码。

10. 本课小结

本课系统学习了 Flutter 中输入组件与表单的完整知识体系：

  1. TextField 是最基础的输入组件，通过 InputDecoration 可以实现丰富的视觉效果。

  2. InputDecoration 是重点中的重点，其属性按功能可分为四大类：

    • 文本类：labelText、hintText、helperText、errorText、counterText
    • 图标类：icon、prefixIcon、suffixIcon、prefix、suffix
    • 边框类：border、enabledBorder、focusedBorder、errorBorder
    • 填充类：filled、fillColor、contentPadding

  3. TextEditingController 提供了编程式的文本操控能力：获取/设置文本、监听变化、控制光标位置。

  4. FocusNode 用于焦点管理：请求焦点、释放焦点、监听焦点变化、字段间跳转。

  5. TextInputFormatter 可在输入时拦截和格式化文本，内置了数字过滤、正则过滤、长度限制等常用格式化器，也支持自定义。

  6. TextFormField 是为表单设计的增强版 TextField，增加了 validator、onSaved、autovalidateMode 等表单能力。

  7. Form + GlobalKey<FormState> 将多个表单字段统一管理，提供 validate()、save()、reset() 三大操作。

核心原则：在实际项目中，始终注意资源释放（dispose）、焦点管理（提升用户体验）、输入验证（安全性），以及通过 InputDecorationTheme 统一项目样式。
