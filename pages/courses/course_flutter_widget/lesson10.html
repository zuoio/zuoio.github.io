第10课 Scaffold 页面脚手架

本课是导航系列的第一课。前 9 课我们学习了各种基础组件和布局方式，从本课开始，我们将把这些组件组装到一个完整的页面结构中。Scaffold 就是 Flutter 为我们提供的"页面骨架"——它定义了一个标准 Material Design 页面应有的所有槽位（AppBar、Body、Drawer、FAB、BottomNavigationBar 等），让我们只需往这些槽位中填充内容即可快速构建专业的页面布局。

1. MaterialApp 与 Scaffold 的关系

在 Flutter 中，MaterialApp 和 Scaffold 是构建 Material Design 应用的两个核心组件，但它们承担的职责不同：

组件 职责 作用范围
MaterialApp 提供应用级别的配置：主题（ThemeData）、路由（routes / onGenerateRoute）、本地化、导航器（Navigator） 整个应用
Scaffold 提供单个页面的结构骨架：AppBar、Body、Drawer、FAB、BottomNavigationBar 等 单个页面

简单来说：MaterialApp 管全局，Scaffold 管页面。

一个典型的 Flutter 应用结构如下：

import 'package:flutter/material.dart';

void main() => runApp(const MyApp());

class MyApp extends StatelessWidget {
  const MyApp({super.key});

  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      title: 'Scaffold 示例',
      theme: ThemeData(
        colorSchemeSeed: Colors.blue,
        useMaterial3: true,
      ),
      home: const HomePage(),
    );
  }
}

class HomePage extends StatelessWidget {
  const HomePage({super.key});

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(title: const Text('首页')),
      body: const Center(child: Text('页面主体内容')),
    );
  }
}

注意事项：

  • 一个应用通常只有一个 MaterialApp（位于组件树顶层），但可以有多个 Scaffold（每个页面/路由一个）。
  • Scaffold 必须放在 MaterialApp 的子树中，否则无法获取主题、媒体查询等上下文信息。
  • Scaffold.of(context) 或 ScaffoldMessenger.of(context) 可以从子组件中访问最近的 Scaffold，常用于显示 SnackBar 等。

2. AppBar 应用栏

AppBar 是 Scaffold 最顶部的工具栏区域，用于展示页面标题、导航按钮和操作按钮。

2.1 核心属性一览

属性 类型 说明
title Widget? 标题组件，通常是 Text，也可以是搜索框等自定义组件
leading Widget? 左侧组件，通常是返回按钮或菜单按钮
automaticallyImplyLeading bool 是否自动推断 leading（默认 true，有 Drawer 时显示菜单图标，可返回时显示返回箭头）
actions List<Widget>? 右侧操作按钮列表
flexibleSpace Widget? AppBar 背后的弹性空间，常用于渐变背景
bottom PreferredSizeWidget? AppBar 底部，常放置 TabBar
elevation double? 阴影高度（Material 3 默认为 0，使用 scrolledUnderElevation）
backgroundColor Color? 背景颜色
centerTitle bool? 标题是否居中（Android 默认靠左，iOS 默认居中）
toolbarHeight double? 工具栏高度（默认 kToolbarHeight = 56.0）

2.2 基础 AppBar 示例

import 'package:flutter/material.dart';

void main() => runApp(const MyApp());

class MyApp extends StatelessWidget {
  const MyApp({super.key});

  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      theme: ThemeData(
        colorSchemeSeed: Colors.indigo,
        useMaterial3: true,
      ),
      home: const AppBarDemo(),
    );
  }
}

class AppBarDemo extends StatelessWidget {
  const AppBarDemo({super.key});

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(
        // 左侧按钮：自定义 leading
        leading: IconButton(
          icon: const Icon(Icons.menu),
          onPressed: () {
            debugPrint('点击了菜单按钮');
          },
        ),
        // 标题
        title: const Text('AppBar 示例'),
        // 标题居中
        centerTitle: true,
        // 右侧操作按钮
        actions: [
          IconButton(
            icon: const Icon(Icons.search),
            onPressed: () {
              debugPrint('点击了搜索按钮');
            },
          ),
          IconButton(
            icon: const Icon(Icons.more_vert),
            onPressed: () {
              debugPrint('点击了更多按钮');
            },
          ),
        ],
        // 阴影高度
        elevation: 2,
        // 自定义工具栏高度
        toolbarHeight: 60,
      ),
      body: const Center(
        child: Text('AppBar 基础示例'),
      ),
    );
  }
}

2.4 使用 bottom 放置 TabBar

bottom 属性接受 PreferredSizeWidget 类型，最常见的用途是放置 TabBar：

import 'package:flutter/material.dart';

void main() => runApp(const MyApp());

class MyApp extends StatelessWidget {
  const MyApp({super.key});

  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      theme: ThemeData(
        colorSchemeSeed: Colors.blue,
        useMaterial3: true,
      ),
      home: const TabBarAppBarDemo(),
    );
  }
}

class TabBarAppBarDemo extends StatelessWidget {
  const TabBarAppBarDemo({super.key});

  @override
  Widget build(BuildContext context) {
    // DefaultTabController 为子树中的 TabBar 和 TabBarView 提供控制器
    return DefaultTabController(
      length: 3,
      child: Scaffold(
        appBar: AppBar(
          title: const Text('带 TabBar 的 AppBar'),
          // bottom 放置 TabBar
          bottom: const TabBar(
            tabs: [
              Tab(icon: Icon(Icons.home), text: '首页'),
              Tab(icon: Icon(Icons.star), text: '收藏'),
              Tab(icon: Icon(Icons.person), text: '我的'),
            ],
          ),
        ),
        body: const TabBarView(
          children: [
            Center(child: Text('首页内容')),
            Center(child: Text('收藏内容')),
            Center(child: Text('我的内容')),
          ],
        ),
      ),
    );
  }
}

最佳实践：

  • automaticallyImplyLeading 默认为 true，如果当前页面可以 Navigator.pop()，AppBar 会自动显示返回箭头；如果 Scaffold 有 Drawer，会自动显示菜单图标。通常不需要手动设置 leading，除非你需要完全自定义。
  • Material 3 中 AppBar 默认没有阴影（elevation = 0），滚动时会通过 scrolledUnderElevation 显示阴影。如果需要始终显示阴影，请手动设置 elevation。

3. Body 页面主体内容

body 是 Scaffold 最核心的区域，占据 AppBar 和 BottomNavigationBar 之间的全部空间。你可以在这里放置任何 Widget。

3.1 基础用法

Scaffold(
  appBar: AppBar(title: const Text('示例')),
  // body 可以是任意 Widget
  body: ListView.builder(
    itemCount: 50,
    itemBuilder: (context, index) {
      return ListTile(
        leading: CircleAvatar(child: Text('${index + 1}')),
        title: Text('列表项 $index'),
      );
    },
  ),
)

3.2 SafeArea 处理刘海屏

现代手机常有刘海、挖孔、圆角等异形屏设计，内容可能被遮挡。SafeArea 会自动计算安全区域并添加内边距：

import 'package:flutter/material.dart';

void main() => runApp(const MyApp());

class MyApp extends StatelessWidget {
  const MyApp({super.key});

  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      theme: ThemeData(
        colorSchemeSeed: Colors.teal,
        useMaterial3: true,
      ),
      home: const SafeAreaDemo(),
    );
  }
}

class SafeAreaDemo extends StatelessWidget {
  const SafeAreaDemo({super.key});

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      // 注意：这里没有 AppBar，所以内容会从屏幕顶部开始
      body: SafeArea(
        // 可以分别控制四个方向是否应用安全区域
        top: true,
        bottom: true,
        left: true,
        right: true,
        // minimum 可以设置最小内边距
        minimum: const EdgeInsets.all(16),
        child: Column(
          crossAxisAlignment: CrossAxisAlignment.start,
          children: [
            Text(
              '使用 SafeArea',
              style: Theme.of(context).textTheme.headlineMedium,
            ),
            const SizedBox(height: 16),
            const Text('这段文字不会被刘海或底部手势条遮挡。'),
            const SizedBox(height: 16),
            // 查看实际的安全区域内边距
            Builder(
              builder: (context) {
                final padding = MediaQuery.paddingOf(context);
                return Text(
                  '安全区域 padding：\n'
                  '  top: ${padding.top}\n'
                  '  bottom: ${padding.bottom}\n'
                  '  left: ${padding.left}\n'
                  '  right: ${padding.right}',
                );
              },
            ),
          ],
        ),
      ),
    );
  }
}

注意事项：

  • 当 Scaffold 有 AppBar 时，body 区域已经在 AppBar 下方，通常不需要对顶部使用 SafeArea。
  • 当使用 bottomNavigationBar 时，body 区域已在其上方，通常不需要对底部使用 SafeArea。
  • 如果你的页面没有 AppBar（例如全屏图片或自定义头部），务必使用 SafeArea 避免内容被遮挡。

4. BottomNavigationBar 底部导航栏

底部导航栏是移动端最常见的导航模式，适用于 3~5 个顶层页面的切换。

4.1 核心属性

属性 类型 说明
items List<BottomNavigationBarItem> 导航项列表，至少 2 个
currentIndex int 当前选中的索引
onTap ValueChanged<int>? 点击回调，返回被点击项的索引
type BottomNavigationBarType? 显示类型：fixed（固定）或 shifting（位移）
selectedItemColor Color? 选中项颜色
unselectedItemColor Color? 未选中项颜色
showUnselectedLabels bool? 是否显示未选中项的文字
elevation double? 阴影高度

4.2 完整的底部导航 + 页面切换方案

这是一个实际项目中常用的底部导航实现模式，使用 IndexedStack 保持页面状态：

import 'package:flutter/material.dart';

void main() => runApp(const MyApp());

class MyApp extends StatelessWidget {
  const MyApp({super.key});

  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      title: '底部导航示例',
      theme: ThemeData(
        colorSchemeSeed: Colors.blue,
        useMaterial3: true,
      ),
      home: const MainScreen(),
    );
  }
}

class MainScreen extends StatefulWidget {
  const MainScreen({super.key});

  @override
  State<MainScreen> createState() => _MainScreenState();
}

class _MainScreenState extends State<MainScreen> {
  // 当前选中的页面索引
  int _currentIndex = 0;

  // 页面列表——使用 IndexedStack 时所有页面都会被创建，
  // 但只有当前索引的页面会被显示（其他页面保持状态但不可见）
  final List<Widget> _pages = const [
    HomePage(),
    CategoryPage(),
    MessagePage(),
    ProfilePage(),
  ];

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      // 使用 IndexedStack 实现页面切换，切换时保持各页面状态
      body: IndexedStack(
        index: _currentIndex,
        children: _pages,
      ),
      bottomNavigationBar: BottomNavigationBar(
        // 当前选中索引
        currentIndex: _currentIndex,
        // 点击回调
        onTap: (index) {
          setState(() {
            _currentIndex = index;
          });
        },
        // fixed 类型：所有项始终显示图标和文字
        // shifting 类型：选中项会放大并显示文字，未选中项只显示图标
        type: BottomNavigationBarType.fixed,
        // 选中/未选中颜色
        selectedItemColor: Theme.of(context).colorScheme.primary,
        unselectedItemColor: Colors.grey,
        // 导航项
        items: const [
          BottomNavigationBarItem(
            icon: Icon(Icons.home_outlined),
            activeIcon: Icon(Icons.home),
            label: '首页',
          ),
          BottomNavigationBarItem(
            icon: Icon(Icons.category_outlined),
            activeIcon: Icon(Icons.category),
            label: '分类',
          ),
          BottomNavigationBarItem(
            icon: Icon(Icons.message_outlined),
            activeIcon: Icon(Icons.message),
            label: '消息',
          ),
          BottomNavigationBarItem(
            icon: Icon(Icons.person_outlined),
            activeIcon: Icon(Icons.person),
            label: '我的',
          ),
        ],
      ),
    );
  }
}

4.3 使用 PageView 替代 IndexedStack

PageView 支持滑动切换页面，适用于需要滑动手势的场景：

import 'package:flutter/material.dart';

void main() => runApp(const MyApp());

class MyApp extends StatelessWidget {
  const MyApp({super.key});

  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      theme: ThemeData(
        colorSchemeSeed: Colors.purple,
        useMaterial3: true,
      ),
      home: const PageViewNavDemo(),
    );
  }
}

class PageViewNavDemo extends StatefulWidget {
  const PageViewNavDemo({super.key});

  @override
  State<PageViewNavDemo> createState() => _PageViewNavDemoState();
}

class _PageViewNavDemoState extends State<PageViewNavDemo> {
  int _currentIndex = 0;
  // PageController 用于编程式控制 PageView 的页面切换
  final PageController _pageController = PageController();

  @override
  void dispose() {
    _pageController.dispose();
    super.dispose();
  }

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(title: const Text('PageView + BottomNav')),
      body: PageView(
        controller: _pageController,
        // 当用户滑动切换页面时同步更新底部导航栏
        onPageChanged: (index) {
          setState(() {
            _currentIndex = index;
          });
        },
        children: const [
          Center(child: Text('页面 1', style: TextStyle(fontSize: 24))),
          Center(child: Text('页面 2', style: TextStyle(fontSize: 24))),
          Center(child: Text('页面 3', style: TextStyle(fontSize: 24))),
        ],
      ),
      bottomNavigationBar: BottomNavigationBar(
        currentIndex: _currentIndex,
        onTap: (index) {
          // 点击底部导航栏时，使用动画跳转到对应页面
          _pageController.animateToPage(
            index,
            duration: const Duration(milliseconds: 300),
            curve: Curves.easeInOut,
          );
        },
        items: const [
          BottomNavigationBarItem(icon: Icon(Icons.looks_one), label: '第一页'),
          BottomNavigationBarItem(icon: Icon(Icons.looks_two), label: '第二页'),
          BottomNavigationBarItem(icon: Icon(Icons.looks_3), label: '第三页'),
        ],
      ),
    );
  }
}

IndexedStack vs PageView 对比：

特性 IndexedStack PageView
切换方式 即时切换，无动画 支持滑动手势和动画
页面状态 所有页面始终存在，状态保持 默认会销毁不可见页面（需配合 AutomaticKeepAliveClientMixin 保持状态）
内存消耗 较高（所有页面同时在内存中） 较低（可按需创建和销毁）
适用场景 页面少、需保持状态 页面可滑动切换、页面较多

5. NavigationBar（Material 3）与 NavigationRail

5.1 NavigationBar — Material 3 新版底部导航

NavigationBar 是 Material 3 推荐的底部导航组件，用于替代旧版 BottomNavigationBar。视觉风格更现代，选中项带有药丸形状的指示器。

import 'package:flutter/material.dart';

void main() => runApp(const MyApp());

class MyApp extends StatelessWidget {
  const MyApp({super.key});

  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      theme: ThemeData(
        colorSchemeSeed: Colors.green,
        useMaterial3: true,
      ),
      home: const NavigationBarDemo(),
    );
  }
}

class NavigationBarDemo extends StatefulWidget {
  const NavigationBarDemo({super.key});

  @override
  State<NavigationBarDemo> createState() => _NavigationBarDemoState();
}

class _NavigationBarDemoState extends State<NavigationBarDemo> {
  int _selectedIndex = 0;

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(title: const Text('NavigationBar 示例')),
      body: Center(
        child: Text(
          '当前页面：$_selectedIndex',
          style: Theme.of(context).textTheme.headlineMedium,
        ),
      ),
      // Material 3 推荐的底部导航组件
      bottomNavigationBar: NavigationBar(
        // 当前选中索引
        selectedIndex: _selectedIndex,
        // 选中回调
        onDestinationSelected: (index) {
          setState(() {
            _selectedIndex = index;
          });
        },
        // 标签行为：alwaysShow 始终显示 / onlyShowSelected 仅选中时显示
        labelBehavior: NavigationDestinationLabelBehavior.alwaysShow,
        // 导航目的地
        destinations: const [
          NavigationDestination(
            icon: Icon(Icons.explore_outlined),
            selectedIcon: Icon(Icons.explore),
            label: '发现',
          ),
          NavigationDestination(
            icon: Badge(
              label: Text('3'),
              child: Icon(Icons.notifications_outlined),
            ),
            selectedIcon: Badge(
              label: Text('3'),
              child: Icon(Icons.notifications),
            ),
            label: '通知',
          ),
          NavigationDestination(
            icon: Icon(Icons.bookmark_outline),
            selectedIcon: Icon(Icons.bookmark),
            label: '书签',
          ),
        ],
      ),
    );
  }
}

5.2 NavigationRail — 侧边导航

NavigationRail 适用于平板和桌面端，垂直排列导航项于屏幕侧边。在响应式布局中，通常手机端使用底部导航，平板/桌面端使用 NavigationRail。

import 'package:flutter/material.dart';

void main() => runApp(const MyApp());

class MyApp extends StatelessWidget {
  const MyApp({super.key});

  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      theme: ThemeData(
        colorSchemeSeed: Colors.orange,
        useMaterial3: true,
      ),
      home: const NavigationRailDemo(),
    );
  }
}

class NavigationRailDemo extends StatefulWidget {
  const NavigationRailDemo({super.key});

  @override
  State<NavigationRailDemo> createState() => _NavigationRailDemoState();
}

class _NavigationRailDemoState extends State<NavigationRailDemo> {
  int _selectedIndex = 0;
  bool _extended = false;

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(title: const Text('NavigationRail 示例')),
      body: Row(
        children: [
          // NavigationRail 放在 Row 的左侧
          NavigationRail(
            // 当前选中索引
            selectedIndex: _selectedIndex,
            // 选中回调
            onDestinationSelected: (index) {
              setState(() {
                _selectedIndex = index;
              });
            },
            // 是否展开（展开时显示文字标签）
            extended: _extended,
            // 顶部可放置组件（常放 logo 或菜单按钮）
            leading: IconButton(
              icon: const Icon(Icons.menu),
              onPressed: () {
                setState(() {
                  _extended = !_extended;
                });
              },
            ),
            // 底部可放置组件（常放设置按钮）
            trailing: Expanded(
              child: Align(
                alignment: Alignment.bottomCenter,
                child: Padding(
                  padding: const EdgeInsets.only(bottom: 16),
                  child: IconButton(
                    icon: const Icon(Icons.settings),
                    onPressed: () {},
                  ),
                ),
              ),
            ),
            // 标签显示方式
            labelType: _extended
                ? NavigationRailLabelType.none
                : NavigationRailLabelType.selected,
            // 导航目的地
            destinations: const [
              NavigationRailDestination(
                icon: Icon(Icons.home_outlined),
                selectedIcon: Icon(Icons.home),
                label: Text('首页'),
              ),
              NavigationRailDestination(
                icon: Icon(Icons.favorite_outline),
                selectedIcon: Icon(Icons.favorite),
                label: Text('收藏'),
              ),
              NavigationRailDestination(
                icon: Icon(Icons.search),
                selectedIcon: Icon(Icons.search),
                label: Text('搜索'),
              ),
              NavigationRailDestination(
                icon: Icon(Icons.settings_outlined),
                selectedIcon: Icon(Icons.settings),
                label: Text('设置'),
              ),
            ],
          ),
          // 分割线
          const VerticalDivider(thickness: 1, width: 1),
          // 主体内容区域
          Expanded(
            child: Center(
              child: Text(
                '选中了第 $_selectedIndex 项',
                style: Theme.of(context).textTheme.headlineMedium,
              ),
            ),
          ),
        ],
      ),
    );
  }
}

最佳实践 — 响应式导航选择：

  • 手机（< 600dp）：使用 NavigationBar（底部导航）
  • 平板（600 ~ 840dp）：使用 NavigationRail（侧边导航，折叠态）
  • 桌面（> 840dp）：使用 NavigationRail（侧边导航，展开态）或 NavigationDrawer

可以通过 MediaQuery 或 LayoutBuilder 动态切换。

6. Drawer 与 EndDrawer 侧边抽屉导航

Drawer 是从屏幕侧边滑出的面板，通常用于放置应用的导航菜单。Scaffold 提供了 drawer（左侧）和 endDrawer（右侧）两个属性。

6.1 核心组件

  • Drawer：抽屉容器本身
  • DrawerHeader：抽屉顶部区域（通常展示 logo 或应用名称）
  • UserAccountsDrawerHeader：带用户头像、姓名、邮箱的抽屉头部
  • ListTile：菜单项

7. FloatingActionButton 浮动操作按钮

FloatingActionButton（FAB）是 Material Design 中最醒目的操作按钮，悬浮在页面内容之上，用于承载页面的主要操作。

7.1 基础用法与形状定制

import 'package:flutter/material.dart';

7.2 FAB 位置定制

Scaffold 的 floatingActionButtonLocation 属性可以控制 FAB 的位置：

import 'package:flutter/material.dart';

void main() => runApp(const MyApp());

class MyApp extends StatelessWidget {
  const MyApp({super.key});

  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      theme: ThemeData(
        colorSchemeSeed: Colors.teal,
        useMaterial3: true,
      ),
      home: const FABLocationDemo(),
    );
  }
}

class FABLocationDemo extends StatefulWidget {
  const FABLocationDemo({super.key});

  @override
  State<FABLocationDemo> createState() => _FABLocationDemoState();
}

class _FABLocationDemoState extends State<FABLocationDemo> {
  // 所有可用的 FAB 位置
  final List<FloatingActionButtonLocation> _locations = [
    FloatingActionButtonLocation.endFloat,       // 默认：右下角浮动
    FloatingActionButtonLocation.centerFloat,    // 底部居中浮动
    FloatingActionButtonLocation.startFloat,     // 左下角浮动
    FloatingActionButtonLocation.endDocked,      // 右下角嵌入 BottomAppBar
    FloatingActionButtonLocation.centerDocked,   // 底部居中嵌入 BottomAppBar
    FloatingActionButtonLocation.startDocked,    // 左下角嵌入 BottomAppBar
    FloatingActionButtonLocation.endTop,         // 右上角（AppBar 下方）
    FloatingActionButtonLocation.startTop,       // 左上角（AppBar 下方）
    FloatingActionButtonLocation.centerTop,      // 顶部居中
    FloatingActionButtonLocation.miniStartFloat,
    FloatingActionButtonLocation.miniEndFloat,
    FloatingActionButtonLocation.miniCenterFloat,
  ];

  final List<String> _locationNames = [
    'endFloat（默认）',
    'centerFloat',
    'startFloat',
    'endDocked',
    'centerDocked',
    'startDocked',
    'endTop',
    'startTop',
    'centerTop',
    'miniStartFloat',
    'miniEndFloat',
    'miniCenterFloat',
  ];

  int _currentLocationIndex = 0;

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(title: const Text('FAB 位置演示')),
      body: ListView.builder(
        itemCount: _locations.length,
        itemBuilder: (context, index) {
          return ListTile(
            title: Text(_locationNames[index]),
            leading: Radio<int>(
              value: index,
              groupValue: _currentLocationIndex,
              onChanged: (value) {
                setState(() {
                  _currentLocationIndex = value!;
                });
              },
            ),
            onTap: () {
              setState(() {
                _currentLocationIndex = index;
              });
            },
          );
        },
      ),
      floatingActionButton: FloatingActionButton(
        onPressed: () {},
        child: const Icon(Icons.add),
      ),
      // 动态切换 FAB 位置
      floatingActionButtonLocation: _locations[_currentLocationIndex],
      bottomNavigationBar: BottomAppBar(
        child: Row(
          children: [
            IconButton(icon: const Icon(Icons.menu), onPressed: () {}),
            const Spacer(),
            IconButton(icon: const Icon(Icons.search), onPressed: () {}),
            IconButton(icon: const Icon(Icons.more_vert), onPressed: () {}),
          ],
        ),
      ),
    );
  }
}

最佳实践：

  • 每个页面最多使用一个 FAB，承载该页面最重要的操作。
  • 如果同一个 Scaffold 树中有多个 FAB（例如上面的展示示例），需要为每个 FAB 设置不同的 heroTag，否则会报错。
  • Material 3 中 FAB 默认是圆角矩形（不再是正圆），颜色取自 primaryContainer。

8. BottomAppBar 底部应用栏

BottomAppBar 与 BottomNavigationBar 不同——它是一个通用的底部工具栏，通常与 FloatingActionButton 配合使用，并可以通过 shape 属性在 FAB 位置创建缺口效果。

8.1 BottomAppBar + FAB 缺口配合

8.2 BottomAppBar 充当底部导航

你也可以用 BottomAppBar 来实现自定义的底部导航效果，获得比 BottomNavigationBar 更灵活的布局：

import 'package:flutter/material.dart';

void main() => runApp(const MyApp());

class MyApp extends StatelessWidget {
  const MyApp({super.key});

  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      theme: ThemeData(
        colorSchemeSeed: Colors.indigo,
        useMaterial3: true,
      ),
      home: const CustomBottomNavDemo(),
    );
  }
}

class CustomBottomNavDemo extends StatefulWidget {
  const CustomBottomNavDemo({super.key});

  @override
  State<CustomBottomNavDemo> createState() => _CustomBottomNavDemoState();
}

class _CustomBottomNavDemoState extends State<CustomBottomNavDemo> {
  int _selectedIndex = 0;

  final List<_NavItem> _navItems = [
    _NavItem(Icons.home_outlined, Icons.home, '首页'),
    _NavItem(Icons.explore_outlined, Icons.explore, '发现'),
    // 中间留给 FAB，这里不放导航项
    _NavItem(Icons.favorite_outline, Icons.favorite, '收藏'),
    _NavItem(Icons.person_outlined, Icons.person, '我的'),
  ];

  @override
  Widget build(BuildContext context) {
    final colorScheme = Theme.of(context).colorScheme;

    return Scaffold(
      appBar: AppBar(title: Text(_navItems[_selectedIndex].label)),
      body: Center(
        child: Text(
          '${_navItems[_selectedIndex].label}页面',
          style: Theme.of(context).textTheme.headlineMedium,
        ),
      ),
      floatingActionButton: FloatingActionButton(
        onPressed: () {
          debugPrint('点击了中间按钮');
        },
        child: const Icon(Icons.add),
      ),
      floatingActionButtonLocation: FloatingActionButtonLocation.centerDocked,
      bottomNavigationBar: BottomAppBar(
        shape: const CircularNotchedRectangle(),
        notchMargin: 8,
        child: Row(
          // 使 4 个导航按钮均匀分布，中间留缺口
          children: [
            // 前两个按钮
            for (int i = 0; i < 2; i++)
              Expanded(
                child: _buildNavButton(i, colorScheme),
              ),
            // 中间给 FAB 让出空间
            const Expanded(child: SizedBox()),
            // 后两个按钮
            for (int i = 2; i < 4; i++)
              Expanded(
                child: _buildNavButton(i, colorScheme),
              ),
          ],
        ),
      ),
    );
  }

  Widget _buildNavButton(int index, ColorScheme colorScheme) {
    final item = _navItems[index];
    final isSelected = _selectedIndex == index;

    return InkWell(
      onTap: () {
        setState(() {
          _selectedIndex = index;
        });
      },
      child: Column(
        mainAxisSize: MainAxisSize.min,
        mainAxisAlignment: MainAxisAlignment.center,
        children: [
          Icon(
            isSelected ? item.activeIcon : item.icon,
            color: isSelected ? colorScheme.primary : colorScheme.onSurfaceVariant,
          ),
          const SizedBox(height: 2),
          Text(
            item.label,
            style: TextStyle(
              fontSize: 12,
              color: isSelected ? colorScheme.primary : colorScheme.onSurfaceVariant,
              fontWeight: isSelected ? FontWeight.w600 : FontWeight.normal,
            ),
          ),
        ],
      ),
    );
  }
}

class _NavItem {
  final IconData icon;
  final IconData activeIcon;
  final String label;

  _NavItem(this.icon, this.activeIcon, this.label);
}

注意事项：

  • BottomAppBar 的 shape 属性在 Material 3 中已不太常用（M3 的 FAB 不再强调缺口效果），但在 Material 2 风格的应用中非常经典。
  • notchMargin 控制缺口边缘与 FAB 之间的间距，默认为 4.0。
  • 当 FAB 的 floatingActionButtonLocation 为 centerDocked 或 endDocked 时，FAB 才会嵌入 BottomAppBar 并触发缺口效果。

9. Scaffold 完整属性一览

下面是 Scaffold 构造函数的所有主要属性汇总，方便查阅：

属性 类型 说明
appBar PreferredSizeWidget? 顶部应用栏
body Widget? 页面主体内容
floatingActionButton Widget? 浮动操作按钮
floatingActionButtonLocation FloatingActionButtonLocation? FAB 位置
floatingActionButtonAnimator FloatingActionButtonAnimator? FAB 切换动画
persistentFooterButtons List<Widget>? 底部持久按钮（显示在 bottomNavigationBar 上方）
drawer Widget? 左侧抽屉
endDrawer Widget? 右侧抽屉
onDrawerChanged DrawerCallback? 左侧抽屉开关回调
onEndDrawerChanged DrawerCallback? 右侧抽屉开关回调
bottomNavigationBar Widget? 底部导航栏
bottomSheet Widget? 持久底部面板
backgroundColor Color? Scaffold 背景色
resizeToAvoidBottomInset bool? 弹出键盘时是否调整 body 大小（默认 true）
primary bool 是否将 AppBar 显示在状态栏下方（默认 true）
extendBody bool body 是否延伸到 bottomNavigationBar 下方（默认 false）
extendBodyBehindAppBar bool body 是否延伸到 AppBar 下方（默认 false，常用于透明 AppBar）
drawerScrimColor Color? 抽屉打开时遮罩颜色
drawerEdgeDragWidth double? 打开抽屉的边缘拖动区域宽度
drawerEnableOpenDragGesture bool 是否允许手势打开 drawer（默认 true）
endDrawerEnableOpenDragGesture bool 是否允许手势打开 endDrawer（默认 true）

10. 综合实战：完整的页面骨架

将本课所学的组件组合起来，构建一个包含 AppBar、Drawer、BottomAppBar、FAB 的完整页面：

import 'package:flutter/material.dart';

void main() => runApp(const MyApp());

class MyApp extends StatelessWidget {
  const MyApp({super.key});

  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      title: 'Scaffold 综合实战',
      theme: ThemeData(
        colorSchemeSeed: Colors.blue,
        useMaterial3: true,
      ),
      home: const ComprehensiveScaffoldDemo(),
    );
  }
}

class ComprehensiveScaffoldDemo extends StatefulWidget {
  const ComprehensiveScaffoldDemo({super.key});

  @override
  State<ComprehensiveScaffoldDemo> createState() =>
      _ComprehensiveScaffoldDemoState();
}

class _ComprehensiveScaffoldDemoState
    extends State<ComprehensiveScaffoldDemo> {
  int _currentTabIndex = 0;
  final List<String> _notes = ['学习 Flutter', '完成第 10 课作业', '阅读官方文档'];

  final _tabLabels = ['笔记', '日历', '统计'];
  final _tabIcons = [Icons.note, Icons.calendar_today, Icons.bar_chart];

  @override
  Widget build(BuildContext context) {
    final colorScheme = Theme.of(context).colorScheme;

    return Scaffold(
      // ---- AppBar ----
      appBar: AppBar(
        title: Text(_tabLabels[_currentTabIndex]),
        centerTitle: true,
        actions: [
          IconButton(
            icon: const Icon(Icons.search),
            onPressed: () {},
          ),
        ],
      ),

      // ---- Drawer ----
      drawer: NavigationDrawer(
        selectedIndex: _currentTabIndex,
        onDestinationSelected: (index) {
          setState(() {
            _currentTabIndex = index;
          });
          Navigator.pop(context);
        },
        children: [
          Padding(
            padding: const EdgeInsets.fromLTRB(28, 16, 16, 10),
            child: Text(
              '我的笔记本',
              style: Theme.of(context).textTheme.titleSmall,
            ),
          ),
          ...List.generate(_tabLabels.length, (index) {
            return NavigationDrawerDestination(
              icon: Icon(_tabIcons[index]),
              label: Text(_tabLabels[index]),
            );
          }),
          const Divider(indent: 28, endIndent: 28),
          Padding(
            padding: const EdgeInsets.symmetric(horizontal: 28),
            child: Text(
              '标签',
              style: Theme.of(context).textTheme.titleSmall,
            ),
          ),
          const NavigationDrawerDestination(
            icon: Icon(Icons.label_outline),
            label: Text('工作'),
          ),
          const NavigationDrawerDestination(
            icon: Icon(Icons.label_outline),
            label: Text('生活'),
          ),
        ],
      ),

      // ---- Body ----
      body: IndexedStack(
        index: _currentTabIndex,
        children: [
          // 笔记页
          _notes.isEmpty
              ? const Center(child: Text('暂无笔记，点击 + 添加'))
              : ListView.builder(
                  padding: const EdgeInsets.only(top: 8, bottom: 80),
                  itemCount: _notes.length,
                  itemBuilder: (context, index) {
                    return Card(
                      margin: const EdgeInsets.symmetric(
                        horizontal: 16,
                        vertical: 4,
                      ),
                      child: ListTile(
                        leading: CircleAvatar(
                          backgroundColor: colorScheme.primaryContainer,
                          child: Text(
                            '${index + 1}',
                            style: TextStyle(color: colorScheme.onPrimaryContainer),
                          ),
                        ),
                        title: Text(_notes[index]),
                        subtitle: const Text('刚刚'),
                        trailing: IconButton(
                          icon: const Icon(Icons.delete_outline),
                          onPressed: () {
                            setState(() {
                              _notes.removeAt(index);
                            });
                          },
                        ),
                      ),
                    );
                  },
                ),
          // 日历页
          const Center(child: Text('日历功能开发中...')),
          // 统计页
          const Center(child: Text('统计功能开发中...')),
        ],
      ),

      // ---- FAB ----
      floatingActionButton: FloatingActionButton(
        onPressed: () {
          setState(() {
            _notes.add('新笔记 ${_notes.length + 1}');
          });
          ScaffoldMessenger.of(context).showSnackBar(
            const SnackBar(
              content: Text('已添加新笔记'),
              duration: Duration(seconds: 1),
            ),
          );
        },
        child: const Icon(Icons.add),
      ),
      floatingActionButtonLocation: FloatingActionButtonLocation.centerDocked,

      // ---- BottomAppBar ----
      bottomNavigationBar: BottomAppBar(
        shape: const CircularNotchedRectangle(),
        notchMargin: 6,
        child: Row(
          children: [
            for (int i = 0; i < _tabLabels.length; i++) ...[
              if (i == (_tabLabels.length ~/ 2))
                const Spacer(),
              Expanded(
                child: InkWell(
                  onTap: () {
                    setState(() {
                      _currentTabIndex = i;
                    });
                  },
                  child: Column(
                    mainAxisSize: MainAxisSize.min,
                    mainAxisAlignment: MainAxisAlignment.center,
                    children: [
                      Icon(
                        _tabIcons[i],
                        color: _currentTabIndex == i
                            ? colorScheme.primary
                            : colorScheme.onSurfaceVariant,
                      ),
                      Text(
                        _tabLabels[i],
                        style: TextStyle(
                          fontSize: 12,
                          color: _currentTabIndex == i
                              ? colorScheme.primary
                              : colorScheme.onSurfaceVariant,
                        ),
                      ),
                    ],
                  ),
                ),
              ),
            ],
          ],
        ),
      ),
    );
  }
}

本课小结

本课作为导航系列的第一课，我们系统学习了 Scaffold 页面脚手架的各个组成部分：

  1. MaterialApp 与 Scaffold 的关系：MaterialApp 负责全局配置（主题、路由），Scaffold 负责单个页面的结构布局。
  2. AppBar：页面顶部工具栏，支持标题、前导按钮、操作按钮、底部 TabBar、弹性空间等丰富定制。
  3. Body 与 SafeArea：页面主体内容区域，使用 SafeArea 处理异形屏幕的安全区域。
  4. BottomNavigationBar：经典底部导航栏，配合 IndexedStack（保持状态）或 PageView（支持滑动）实现页面切换。
  5. NavigationBar 与 NavigationRail：Material 3 推荐的底部导航和侧边导航组件，适用于响应式布局。
  6. Drawer 与 EndDrawer：侧边抽屉导航，支持用户信息头部和菜单列表。
  7. FloatingActionButton：浮动操作按钮，支持多种尺寸、形状和位置定制。
  8. BottomAppBar：底部应用栏，可与 FAB 配合实现缺口效果，布局更灵活。

核心要点：Scaffold 就像一个"页面模板"，它预定义了 Material Design 页面的所有标准槽位。理解 Scaffold 的各个属性，是构建规范 Flutter 页面的基础。

练习题

练习 1：基础练习 — 自定义 AppBar

创建一个页面，要求：

  • AppBar 使用渐变背景（通过 flexibleSpace）
  • 左侧显示返回按钮（leading）
  • 标题居中显示
  • 右侧有搜索按钮和设置按钮
  • 自定义 toolbarHeight 为 70

练习 2：进阶练习 — 完整底部导航应用

创建一个包含 4 个页面的底部导航应用，要求：

  • 使用 NavigationBar（Material 3）作为底部导航
  • 使用 IndexedStack 实现页面切换并保持各页面状态
  • 每个页面包含一个计数器（StatefulWidget），验证切换页面后状态是否保留
  • 第一个页面包含一个 ListView，滚动位置在切换后也应保持

练习 3：综合练习 — 社交应用主框架

构建一个仿社交应用的页面框架，要求：

  • 包含左侧 Drawer（带 UserAccountsDrawerHeader 和多个菜单项）
  • 使用 BottomAppBar + FloatingActionButton（centerDocked，带缺口效果）
  • FAB 点击后弹出 SnackBar 提示"发布新动态"
  • BottomAppBar 中有 4 个导航按钮（首页、发现、消息、我的），中间为 FAB 留出空间
  • 支持通过 Drawer 菜单或底部导航切换内容

练习 4：挑战练习 — 响应式导航

实现一个响应式页面，要求：

  • 使用 LayoutBuilder 或 MediaQuery 判断屏幕宽度
  • 窄屏（< 600）：使用 NavigationBar 底部导航
  • 中等屏幕（600 ~ 1200）：使用 NavigationRail 侧边导航（折叠态）
  • 宽屏（> 1200）：使用 NavigationRail 侧边导航（展开态）
  • 三种模式共享同一套导航项和页面内容
