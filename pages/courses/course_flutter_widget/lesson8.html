第08课 弹性布局与流式布局

在前面的课程中，我们学习了 Row/Column（线性布局）和 Stack（层叠布局）。但在实际开发中，你一定遇到过这样的问题：当子组件的总宽度超出 Row 的可用空间时，Flutter 会抛出溢出错误（黄黑条纹警告）。本课将学习 Wrap、Flow、LayoutBuilder 和 MediaQuery，彻底解决这类布局难题，并掌握响应式布局的核心技巧。

1. Wrap —— 自动换行布局

1.1 为什么需要 Wrap？

在 Row 中放置过多子组件时，当总宽度超出父容器限制，Flutter 不会自动换行，而是报溢出错误：

// ❌ 这段代码会导致溢出
Row(
  children: List.generate(10, (i) => Chip(label: Text('标签 $i'))),
)

Wrap 的出现正是为了解决这个问题——它会在主轴空间不足时自动将子组件换到下一行（或下一列）。

1.2 基础用法

import 'package:flutter/material.dart';

void main() => runApp(const MyApp());

class MyApp extends StatelessWidget {
  const MyApp({super.key});

  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      home: Scaffold(
        appBar: AppBar(title: const Text('Wrap 基础用法')),
        body: Padding(
          padding: const EdgeInsets.all(16.0),
          child: Wrap(
            spacing: 8.0,       // 主轴方向子组件间距
            runSpacing: 12.0,   // 交叉轴方向（行与行之间）间距
            children: List.generate(
              12,
              (index) => Chip(
                avatar: CircleAvatar(child: Text('${index + 1}')),
                label: Text('标签 ${index + 1}'),
                backgroundColor: Colors.blue.shade100,
              ),
            ),
          ),
        ),
      ),
    );
  }
}

运行效果：12 个 Chip 会根据屏幕宽度自动排列，放不下时自动换到下一行。

1.3 核心属性详解

属性 类型 默认值 说明
direction Axis Axis.horizontal 主轴方向，水平或垂直
spacing double 0.0 主轴方向子组件之间的间距
runSpacing double 0.0 交叉轴方向（行/列之间）的间距
alignment WrapAlignment WrapAlignment.start 主轴方向的对齐方式
runAlignment WrapAlignment WrapAlignment.start 交叉轴方向各行的对齐方式
crossAxisAlignment WrapCrossAlignment WrapCrossAlignment.start 每一行内部子组件在交叉轴方向的对齐

direction —— 主轴方向

import 'package:flutter/material.dart';

void main() => runApp(const MyApp());

class MyApp extends StatelessWidget {
  const MyApp({super.key});

  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      home: Scaffold(
        appBar: AppBar(title: const Text('Wrap direction 示例')),
        body: Row(
          crossAxisAlignment: CrossAxisAlignment.start,
          children: [
            // 水平方向的 Wrap（默认）
            Expanded(
              child: Column(
                crossAxisAlignment: CrossAxisAlignment.start,
                children: [
                  const Padding(
                    padding: EdgeInsets.all(8.0),
                    child: Text('水平方向 (默认)',
                        style: TextStyle(fontWeight: FontWeight.bold)),
                  ),
                  Wrap(
                    spacing: 8,
                    runSpacing: 8,
                    direction: Axis.horizontal, // 默认值
                    children: _buildChips(),
                  ),
                ],
              ),
            ),
            const VerticalDivider(),
            // 垂直方向的 Wrap
            Expanded(
              child: Column(
                crossAxisAlignment: CrossAxisAlignment.start,
                children: [
                  const Padding(
                    padding: EdgeInsets.all(8.0),
                    child: Text('垂直方向',
                        style: TextStyle(fontWeight: FontWeight.bold)),
                  ),
                  SizedBox(
                    height: 200, // 垂直方向需要限定高度
                    child: Wrap(
                      spacing: 8,
                      runSpacing: 8,
                      direction: Axis.vertical, // 垂直排列
                      children: _buildChips(),
                    ),
                  ),
                ],
              ),
            ),
          ],
        ),
      ),
    );
  }

  static List<Widget> _buildChips() {
    return List.generate(
      8,
      (i) => Chip(label: Text('Tag $i')),
    );
  }
}
  • Axis.horizontal（默认）：子组件从左到右排列，满行后换到下一行。
  • Axis.vertical：子组件从上到下排列，满列后换到下一列。注意垂直方向时需要给 Wrap 一个有限的高度约束。

alignment —— 主轴对齐

WrapAlignment 提供六种对齐方式，与 MainAxisAlignment 类似：

import 'package:flutter/material.dart';

void main() => runApp(const MyApp());

class MyApp extends StatelessWidget {
  const MyApp({super.key});

  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      home: Scaffold(
        appBar: AppBar(title: const Text('Wrap alignment 对比')),
        body: SingleChildScrollView(
          padding: const EdgeInsets.all(16),
          child: Column(
            crossAxisAlignment: CrossAxisAlignment.stretch,
            children: [
              for (final align in WrapAlignment.values)
                _buildSection(align),
            ],
          ),
        ),
      ),
    );
  }

  Widget _buildSection(WrapAlignment alignment) {
    return Column(
      crossAxisAlignment: CrossAxisAlignment.start,
      children: [
        const SizedBox(height: 16),
        Text(
          'alignment: ${alignment.name}',
          style: const TextStyle(fontWeight: FontWeight.bold, fontSize: 14),
        ),
        const SizedBox(height: 8),
        Container(
          color: Colors.grey.shade200,
          width: double.infinity,
          child: Wrap(
            alignment: alignment,
            spacing: 8,
            runSpacing: 8,
            children: const [
              Chip(label: Text('Flutter')),
              Chip(label: Text('Dart')),
              Chip(label: Text('Widget')),
              Chip(label: Text('布局')),
              Chip(label: Text('响应式')),
            ],
          ),
        ),
      ],
    );
  }
}
  • start：靠主轴起始端对齐
  • end：靠主轴末端对齐
  • center：居中
  • spaceBetween：首尾贴边，中间均匀分布
  • spaceAround：每个子组件两侧有相等间距
  • spaceEvenly：所有间距完全相等

runAlignment 控制的是多行之间在交叉轴方向上的分布方式，只有当 Wrap 的交叉轴有剩余空间时才能看到效果：

2. Flow —— 高性能自定义流式布局

2.1 什么是 Flow？

Flow 是 Flutter 提供的一个高性能布局组件，它通过 FlowDelegate 将布局和绘制的控制权完全交给开发者。与 Wrap 的"自动换行"不同，Flow 需要你手动计算每个子组件的位置。

Flow 的核心优势：

  1. 极高的性能：Flow 使用变换矩阵（transformation matrix）来定位子组件，而不是重新布局。当需要重新定位子组件时，只需调用 paintChildren，不会触发子组件的重新布局（relayout），因此性能远优于其他布局组件。
  2. 完全的布局控制：你可以实现任何自定义的布局逻辑，比如扇形菜单、瀑布流等。

Flow 的使用场景：

  • 需要频繁改变子组件位置的动画场景（如展开/收起菜单）
  • 自定义的复杂布局算法
  • 对性能要求极高的大量子组件定位

2.2 FlowDelegate 详解

使用 Flow 的关键在于实现 FlowDelegate，它有三个核心方法：

方法 说明
paintChildren(FlowPaintingContext context) 必须实现。控制每个子组件的绘制位置和变换
shouldRelayout(covariant FlowDelegate oldDelegate) 必须实现。决定是否需要重新布局
getSize(BoxConstraints constraints) 可选。返回 Flow 组件本身的尺寸，默认尽可能大
shouldRepaint(covariant FlowDelegate oldDelegate) 可选。决定是否需要重绘
getConstraintsForChild(int i, BoxConstraints constraints) 可选。为每个子组件设置约束

2.3 基础示例：自定义流式排列

import 'package:flutter/material.dart';

void main() => runApp(const MyApp());

class MyApp extends StatelessWidget {
  const MyApp({super.key});

  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      home: Scaffold(
        appBar: AppBar(title: const Text('Flow 基础示例')),
        body: Padding(
          padding: const EdgeInsets.all(16),
          child: Flow(
            delegate: SimpleFlowDelegate(spacing: 8),
            children: List.generate(
              15,
              (index) => Container(
                width: 60 + (index % 3) * 20.0, // 不同宽度
                height: 40,
                alignment: Alignment.center,
                decoration: BoxDecoration(
                  color: Colors.primaries[index % Colors.primaries.length],
                  borderRadius: BorderRadius.circular(8),
                ),
                child: Text(
                  '项目 $index',
                  style: const TextStyle(color: Colors.white, fontSize: 12),
                ),
              ),
            ),
          ),
        ),
      ),
    );
  }
}

/// 自定义 FlowDelegate：实现自动换行排列
class SimpleFlowDelegate extends FlowDelegate {
  final double spacing;

  SimpleFlowDelegate({required this.spacing});

  @override
  void paintChildren(FlowPaintingContext context) {
    double x = 0; // 当前绘制位置 x
    double y = 0; // 当前绘制位置 y
    double rowHeight = 0; // 当前行的最大高度

    for (int i = 0; i < context.childCount; i++) {
      final childSize = context.getChildSize(i)!;

      // 如果当前行放不下，换到下一行
      if (x + childSize.width > context.size.width && x > 0) {
        x = 0;
        y += rowHeight + spacing;
        rowHeight = 0;
      }

      // 通过变换矩阵定位子组件（这正是 Flow 高性能的关键）
      context.paintChild(i, transform: Matrix4.translationValues(x, y, 0));

      x += childSize.width + spacing;
      rowHeight = rowHeight > childSize.height ? rowHeight : childSize.height;
    }
  }

  @override
  Size getSize(BoxConstraints constraints) {
    // 返回 Flow 的尺寸，宽度尽量大，高度也尽量大
    return Size(constraints.maxWidth, constraints.maxHeight);
  }

  @override
  bool shouldRelayout(SimpleFlowDelegate oldDelegate) {
    return oldDelegate.spacing != spacing;
  }

  @override
  bool shouldRepaint(SimpleFlowDelegate oldDelegate) {
    return oldDelegate.spacing != spacing;
  }
}

2.4 进阶示例：展开/收起的动画菜单

这是 Flow 最经典的应用场景——利用其高性能变换实现流畅的动画效果：

import 'package:flutter/material.dart';
import 'dart:math';

void main() => runApp(const MyApp());

class MyApp extends StatelessWidget {
  const MyApp({super.key});

  @override
  Widget build(BuildContext context) {
    return const MaterialApp(home: FlowMenuDemo());
  }
}

class FlowMenuDemo extends StatefulWidget {
  const FlowMenuDemo({super.key});

  @override
  State<FlowMenuDemo> createState() => _FlowMenuDemoState();
}

class _FlowMenuDemoState extends State<FlowMenuDemo>
    with SingleTickerProviderStateMixin {
  late AnimationController _controller;

  @override
  void initState() {
    super.initState();
    _controller = AnimationController(
      duration: const Duration(milliseconds: 300),
      vsync: this,
    );
  }

  @override
  void dispose() {
    _controller.dispose();
    super.dispose();
  }

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(title: const Text('Flow 动画菜单')),
      body: Stack(
        children: [
          const Center(child: Text('点击右下角按钮展开菜单')),
          Positioned(
            right: 20,
            bottom: 20,
            child: Flow(
              delegate: _AnimatedMenuDelegate(animation: _controller),
              children: [
                // 最后一个是主按钮（触发展开/收起）
                _buildMenuItem(Icons.menu, Colors.blue, onTap: _toggleMenu),
                _buildMenuItem(Icons.camera_alt, Colors.green, onTap: () {}),
                _buildMenuItem(Icons.photo, Colors.orange, onTap: () {}),
                _buildMenuItem(Icons.videocam, Colors.red, onTap: () {}),
                _buildMenuItem(Icons.mic, Colors.purple, onTap: () {}),
              ],
            ),
          ),
        ],
      ),
    );
  }

  Widget _buildMenuItem(IconData icon, Color color, {VoidCallback? onTap}) {
    return GestureDetector(
      onTap: onTap,
      child: Container(
        width: 56,
        height: 56,
        decoration: BoxDecoration(
          color: color,
          shape: BoxShape.circle,
          boxShadow: const [
            BoxShadow(color: Colors.black26, blurRadius: 4, offset: Offset(0, 2)),
          ],
        ),
        child: Icon(icon, color: Colors.white),
      ),
    );
  }

  void _toggleMenu() {
    if (_controller.isCompleted) {
      _controller.reverse();
    } else {
      _controller.forward();
    }
  }
}

/// 扇形展开动画的 FlowDelegate
class _AnimatedMenuDelegate extends FlowDelegate {
  final Animation<double> animation;

  _AnimatedMenuDelegate({required this.animation})
      : super(repaint: animation); // 关键：监听动画变化来触发重绘

  @override
  void paintChildren(FlowPaintingContext context) {
    final childCount = context.childCount;
    if (childCount == 0) return;

    final size = 56.0; // 按钮尺寸
    final radius = 120.0; // 扇形展开的半径

    // 先绘制展开的子按钮（索引 1 到 childCount-1）
    for (int i = 1; i < childCount; i++) {
      // 计算每个子按钮的角度（从 180° 到 270°，即左上方扇形）
      final angle = pi + (pi / 2) * (i - 1) / (childCount - 2);
      final x = radius * cos(angle) * animation.value;
      final y = radius * sin(angle) * animation.value;

      context.paintChild(
        i,
        transform: Matrix4.translationValues(x, y, 0)
          ..scale(animation.value), // 缩放效果
        opacity: animation.value,   // 透明度变化
      );
    }

    // 最后绘制主按钮（始终在原位）
    context.paintChild(0, transform: Matrix4.identity());
  }

  @override
  Size getSize(BoxConstraints constraints) {
    return const Size(56, 56); // Flow 只占主按钮的大小
  }

  @override
  bool shouldRelayout(_AnimatedMenuDelegate oldDelegate) => false;

  @override
  bool shouldRepaint(_AnimatedMenuDelegate oldDelegate) {
    return animation != oldDelegate.animation;
  }
}

注意事项：

  1. Flow 的子组件不能设置为 const，因为 Flow 需要获取其尺寸。
  2. Flow 子组件会被绘制到 Flow 的边界之外（overflow），这是设计如此——类似于 Stack 的 clipBehavior: Clip.none。
  3. FlowDelegate 的构造函数中传入 repaint: animation 可以在动画值变化时自动触发重绘，无需 setState。

2.5 Wrap vs Flow 对比

对比维度 Wrap Flow
使用复杂度 简单，声明式 复杂，需要实现 FlowDelegate
布局方式 自动计算换行 完全手动控制
性能 一般（改变时重新布局） 极高（只重绘，不重新布局子组件）
适用场景 标签、筛选条件等静态或低频变化布局 动画菜单、需要频繁变化位置的高性能场景
子组件溢出 自动处理 可以绘制到边界之外

最佳实践： 大多数场景下优先使用 Wrap，它简单且够用。只有当你需要实现自定义布局算法或高性能动画定位时，才考虑使用 Flow。

3. LayoutBuilder —— 根据父约束动态构建 UI

3.1 什么是 LayoutBuilder？

LayoutBuilder 可以在 build 阶段获取父组件传递的布局约束（BoxConstraints），从而根据可用空间动态决定构建什么 UI。这是实现响应式布局的基础工具之一。

与 MediaQuery 不同的是：

  • MediaQuery 获取的是整个屏幕的信息
  • LayoutBuilder 获取的是父组件给予的约束，更适合组件级别的自适应

3.2 BoxConstraints 参数解读

BoxConstraints 包含四个关键属性：

属性 说明
minWidth 最小宽度约束
maxWidth 最大宽度约束（double.infinity 表示无限制）
minHeight 最小高度约束
maxHeight 最大高度约束（double.infinity 表示无限制）

常用判断方法：

  • constraints.hasBoundedWidth：宽度是否有限
  • constraints.hasBoundedHeight：高度是否有限
  • constraints.isTight：是否是严格约束（min == max）
  • constraints.biggest：约束范围内最大的 Size
  • constraints.smallest：约束范围内最小的 Size

3.3 基础示例：根据宽度切换布局

import 'package:flutter/material.dart';

void main() => runApp(const MyApp());

class MyApp extends StatelessWidget {
  const MyApp({super.key});

  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      home: Scaffold(
        appBar: AppBar(title: const Text('LayoutBuilder 示例')),
        body: Padding(
          padding: const EdgeInsets.all(16),
          child: LayoutBuilder(
            builder: (BuildContext context, BoxConstraints constraints) {
              // 根据父组件给予的最大宽度动态选择布局
              if (constraints.maxWidth > 600) {
                // 宽屏：使用两列布局
                return _buildWideLayout();
              } else {
                // 窄屏：使用单列布局
                return _buildNarrowLayout();
              }
            },
          ),
        ),
      ),
    );
  }

  Widget _buildWideLayout() {
    return Row(
      crossAxisAlignment: CrossAxisAlignment.start,
      children: [
        Expanded(
          flex: 1,

4. MediaQuery —— 获取屏幕尺寸与设备信息

4.1 什么是 MediaQuery？

MediaQuery 是一个 InheritedWidget，它在 Widget 树中向下传递设备和屏幕的媒体查询信息。在 MaterialApp 或 WidgetsApp 中会自动创建 MediaQuery，你可以在任何后代组件中通过 MediaQuery.of(context) 获取。

4.4 性能优化：使用细粒度的 MediaQuery 方法

Flutter 3.10+ 引入了细粒度的 MediaQuery 访问方法，强烈推荐使用：

// ❌ 不推荐：监听了整个 MediaQueryData 的变化
// 当任何一个属性（如键盘弹出导致 viewInsets 变化）改变时，都会重建
final data = MediaQuery.of(context);
final width = data.size.width;

// ✅ 推荐：只监听 size 的变化
// 键盘弹出等其他变化不会触发重建
final size = MediaQuery.sizeOf(context);

// ✅ 推荐：只监听 padding 的变化
final padding = MediaQuery.paddingOf(context);

// ✅ 推荐：只监听 viewInsets 的变化
final viewInsets = MediaQuery.viewInsetsOf(context);

// ✅ 推荐：只监听亮度模式的变化
final brightness = MediaQuery.platformBrightnessOf(context);

// ✅ 推荐：只监听方向的变化
final orientation = MediaQuery.orientationOf(context);

最佳实践： 始终使用 MediaQuery.xxxOf(context) 而非 MediaQuery.of(context).xxx，这样可以避免不必要的组件重建，显著提升性能。

5.3 封装响应式工具类

在实际项目中，建议将断点判断封装成工具类，方便复用：

import 'package:flutter/material.dart';

/// 响应式断点工具类
class Responsive {
  /// 标准断点值
  static const double mobileBreakpoint = 600;
  static const double tabletBreakpoint = 1024;

  /// 判断当前是否为手机布局
  static bool isMobile(BuildContext context) =>
      MediaQuery.sizeOf(context).width < mobileBreakpoint;

  /// 判断当前是否为平板布局
  static bool isTablet(BuildContext context) {
    final width = MediaQuery.sizeOf(context).width;
    return width >= mobileBreakpoint && width < tabletBreakpoint;
  }

  /// 判断当前是否为桌面布局
  static bool isDesktop(BuildContext context) =>
      MediaQuery.sizeOf(context).width >= tabletBreakpoint;

  /// 根据设备类型返回不同的值
  static T value<T>(
    BuildContext context, {
    required T mobile,
    T? tablet,
    T? desktop,
  }) {
    if (isDesktop(context)) return desktop ?? tablet ?? mobile;
    if (isTablet(context)) return tablet ?? mobile;
    return mobile;
  }
}

/// 响应式布局构建器 Widget
class ResponsiveBuilder extends StatelessWidget {
  final Widget Function(BuildContext context) mobile;
  final Widget Function(BuildContext context)? tablet;
  final Widget Function(BuildContext context)? desktop;

  const ResponsiveBuilder({
    super.key,
    required this.mobile,
    this.tablet,
    this.desktop,
  });

  @override
  Widget build(BuildContext context) {
    return LayoutBuilder(
      builder: (context, constraints) {
        if (constraints.maxWidth >= Responsive.tabletBreakpoint) {
          return (desktop ?? tablet ?? mobile)(context);
        } else if (constraints.maxWidth >= Responsive.mobileBreakpoint) {
          return (tablet ?? mobile)(context);
        }
        return mobile(context);
      },
    );
  }
}

  const MyApp({super.key});

  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      home: Scaffold(
        appBar: AppBar(title: const Text('响应式工具类示例')),
        body: ResponsiveBuilder(
          mobile: (context) => const Center(child: Text('手机布局：单列')),
          tablet: (context) => const Center(child: Text('平板布局：双列')),
          desktop: (context) => const Center(child: Text('桌面布局：三列')),
        ),
      ),
    );
  }
}

5.4 OrientationBuilder —— 监听屏幕方向

除了 MediaQuery.orientationOf，Flutter 还提供了专门的 OrientationBuilder：

import 'package:flutter/material.dart';

void main() => runApp(const MyApp());

class MyApp extends StatelessWidget {
  const MyApp({super.key});

  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      home: Scaffold(
        appBar: AppBar(title: const Text('OrientationBuilder 示例')),
        body: OrientationBuilder(
          builder: (context, orientation) {
            // 根据屏幕方向切换网格列数
            final crossAxisCount =
                orientation == Orientation.portrait ? 2 : 4;

            return GridView.builder(
              padding: const EdgeInsets.all(12),
              gridDelegate: SliverGridDelegateWithFixedCrossAxisCount(
                crossAxisCount: crossAxisCount,
                mainAxisSpacing: 12,
                crossAxisSpacing: 12,
              ),
              itemCount: 20,
              itemBuilder: (context, index) {
                return Container(
                  decoration: BoxDecoration(
                    color: Colors.primaries[index % Colors.primaries.length]
                        .shade200,
                    borderRadius: BorderRadius.circular(12),
                  ),
                  alignment: Alignment.center,
                  child: Text(
                    '项目 ${index + 1}',
                    style: const TextStyle(
                        fontSize: 16, fontWeight: FontWeight.bold),
                  ),
                );
              },
            );
          },
        ),
      ),
    );
  }
}

最佳实践：

  1. 优先使用 LayoutBuilder 实现组件级响应式布局，它比 MediaQuery 更灵活——因为它基于父组件约束而非屏幕尺寸。
  2. MediaQuery 更适合获取设备级别的信息（如安全区域、键盘高度、暗色模式等）。
  3. 将断点常量和判断逻辑统一封装，避免硬编码分散在各处。
  4. 在响应式设计中，使用 Wrap 让内容自然流动，而非为每种屏幕尺寸硬编码布局。

6. 本课小结

本课学习了四个核心组件和一套响应式布局策略：

组件 核心用途 关键要点
Wrap 自动换行布局 解决 Row/Column 溢出问题；spacing、runSpacing 控制间距
Flow 高性能自定义布局 通过 FlowDelegate 手动控制子组件位置；使用变换矩阵，性能极高
LayoutBuilder 基于父约束构建 UI 获取 BoxConstraints，实现组件级响应式布局
MediaQuery 获取设备和屏幕信息 屏幕尺寸、安全区域、键盘高度、暗色模式等；优先使用 xxxOf 方法

核心思路：

  1. 需要自动换行 → 用 Wrap
  2. 需要极致性能的自定义定位 → 用 Flow
  3. 需要根据父组件的可用空间适配 → 用 LayoutBuilder
  4. 需要获取设备级信息 → 用 MediaQuery
  5. 综合运用以上工具，就能构建出适配手机、平板、桌面的响应式界面
