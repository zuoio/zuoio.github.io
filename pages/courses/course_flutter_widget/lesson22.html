1. FutureBuilder：基于 Future 构建 UI

1.1 基本概念

FutureBuilder 是 Flutter 中最常用的异步 UI 组件之一。它接收一个 Future 对象，并根据 Future 的执行状态（等待中、完成、出错）自动调用 builder 回调来构建不同的 UI。

核心参数：

参数 类型 说明
future Future<T>? 要监听的异步操作
initialData T? Future 完成之前使用的初始数据
builder Widget Function(BuildContext, AsyncSnapshot<T>) 根据异步快照构建 UI 的回调

1.2 AsyncSnapshot 详解

builder 回调接收一个 AsyncSnapshot<T> 对象，它包含了异步操作的完整状态信息：

属性/方法 类型 说明
connectionState ConnectionState 当前连接状态
data T? 异步操作返回的数据（成功时）
error Object? 异步操作抛出的错误（失败时）
stackTrace StackTrace? 错误的堆栈信息
hasData bool 是否有数据（data != null）
hasError bool 是否有错误（error != null）
requireData T 获取数据，无数据时抛出异常

1.3 ConnectionState 四种状态

ConnectionState.none    → Future 为 null，尚未开始
ConnectionState.waiting → Future 正在执行，等待结果
ConnectionState.active  → （FutureBuilder 中不常见，主要用于 StreamBuilder）
ConnectionState.done    → Future 已完成（成功或失败）

典型的状态流转：none → waiting → done

1.4 基础示例

import 'package:flutter/material.dart';

void main() {
  runApp(const FutureBuilderDemoApp());
}

class FutureBuilderDemoApp extends StatelessWidget {
  const FutureBuilderDemoApp({super.key});

  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      theme: ThemeData(colorSchemeSeed: Colors.blue, useMaterial3: true),
      home: const FutureBuilderDemoPage(),
    );
  }
}

class FutureBuilderDemoPage extends StatefulWidget {
  const FutureBuilderDemoPage({super.key});

  @override
  State<FutureBuilderDemoPage> createState() => _FutureBuilderDemoPageState();
}

class _FutureBuilderDemoPageState extends State<FutureBuilderDemoPage> {
  // ✅ 将 Future 缓存到 State 变量中，在 initState 中创建
  late Future<String> _userFuture;

  @override
  void initState() {
    super.initState();
    _userFuture = _fetchUserName();
  }

  /// 模拟网络请求，延迟 2 秒返回用户名
  Future<String> _fetchUserName() async {
    await Future.delayed(const Duration(seconds: 2));
    // 模拟 30% 概率失败
    if (DateTime.now().millisecond % 3 == 0) {
      throw Exception('网络连接超时，请检查网络设置');
    }
    return '张三';
  }

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(title: const Text('FutureBuilder 基础示例')),
      body: Center(
        child: FutureBuilder<String>(
          future: _userFuture,
          builder: (context, snapshot) {
            // 根据连接状态构建不同 UI
            switch (snapshot.connectionState) {
              case ConnectionState.none:
                return const Text('尚未开始');

              case ConnectionState.waiting:
                // 加载中状态：显示进度指示器
                return const Column(
                  mainAxisAlignment: MainAxisAlignment.center,
                  children: [
                    CircularProgressIndicator(),
                    SizedBox(height: 16),
                    Text('正在加载用户信息...'),
                  ],
                );

              case ConnectionState.active:
              case ConnectionState.done:
                // 完成状态：区分成功与失败
                if (snapshot.hasError) {
                  // 失败状态：显示错误信息和重试按钮
                  return Column(
                    mainAxisAlignment: MainAxisAlignment.center,
                    children: [
                      Icon(Icons.error_outline,
                          size: 48, color: Colors.red[300]),
                      const SizedBox(height: 16),
                      Text(
                        '加载失败: ${snapshot.error}',
                        textAlign: TextAlign.center,
                        style: TextStyle(color: Colors.red[700]),
                      ),
                      const SizedBox(height: 16),
                      ElevatedButton.icon(
                        onPressed: () {
                          // 重试：创建新的 Future
                          setState(() {
                            _userFuture = _fetchUserName();
                          });
                        },
                        icon: const Icon(Icons.refresh),
                        label: const Text('重试'),
                      ),
                    ],
                  );
                }

                // 成功状态：显示数据
                return Column(
                  mainAxisAlignment: MainAxisAlignment.center,
                  children: [
                    const Icon(Icons.check_circle,
                        size: 48, color: Colors.green),
                    const SizedBox(height: 16),
                    Text(
                      '欢迎, ${snapshot.data}!',
                      style: Theme.of(context).textTheme.headlineMedium,
                    ),
                  ],
                );
            }
          },
        ),
      ),
      floatingActionButton: FloatingActionButton(
        onPressed: () {
          setState(() {
            _userFuture = _fetchUserName();
          });
        },
        child: const Icon(Icons.refresh),
      ),
    );
  }
}

1.5 使用 hasData/hasError 的简洁写法

实际开发中，更常见的写法是直接用 hasData 和 hasError 判断，而不是 switch 所有 ConnectionState：

FutureBuilder<List<String>>(
  future: _dataFuture,
  builder: (context, snapshot) {
    // 优先判断错误
    if (snapshot.hasError) {
      return Center(
        child: Text('出错了: ${snapshot.error}'),
      );
    }

    // 再判断是否有数据
    if (snapshot.hasData) {
      final items = snapshot.data!;
      if (items.isEmpty) {
        return const Center(child: Text('暂无数据'));
      }
      return ListView.builder(
        itemCount: items.length,
        itemBuilder: (context, index) => ListTile(title: Text(items[index])),
      );
    }

    // 默认：加载中
    return const Center(child: CircularProgressIndicator());
  },
)

最佳实践：FutureBuilder 的 builder 回调应按照 错误 → 有数据 → 加载中 的优先级顺序判断。这样能确保错误状态不会被遗漏，同时加载中作为兜底状态出现在最后。

2. StreamBuilder：基于 Stream 实时构建 UI

2.1 基本概念

StreamBuilder 与 FutureBuilder 类似，但它监听的是一个 Stream——一个可以持续发送多次数据的异步序列。每当 Stream 发送新数据，builder 都会被重新调用以更新 UI。

核心参数：

参数 类型 说明
stream Stream<T>? 要监听的数据流
initialData T? Stream 发送第一条数据之前显示的初始值
builder Widget Function(BuildContext, AsyncSnapshot<T>) 根据快照构建 UI

2.2 Stream 中的 ConnectionState

与 FutureBuilder 不同的是，StreamBuilder 会频繁出现 ConnectionState.active 状态：

Stream 为 null            → ConnectionState.none
已订阅，等待第一条数据    → ConnectionState.waiting
收到数据                  → ConnectionState.active（snapshot.data 为最新数据）
Stream 关闭（done 事件）  → ConnectionState.done

2.3 基础示例：实时计时器

import 'package:flutter/material.dart';

void main() {
  runApp(const StreamBuilderDemoApp());
}

class StreamBuilderDemoApp extends StatelessWidget {
  const StreamBuilderDemoApp({super.key});

  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      theme: ThemeData(colorSchemeSeed: Colors.teal, useMaterial3: true),
      home: const StreamBuilderDemoPage(),
    );
  }
}

class StreamBuilderDemoPage extends StatefulWidget {
  const StreamBuilderDemoPage({super.key});

  @override
  State<StreamBuilderDemoPage> createState() => _StreamBuilderDemoPageState();
}

class _StreamBuilderDemoPageState extends State<StreamBuilderDemoPage> {
  /// 创建一个每秒发射递增整数的 Stream
  Stream<int> _createTimerStream() {
    return Stream.periodic(
      const Duration(seconds: 1),
      (count) => count + 1, // 从 1 开始递增
    );
  }

  late Stream<int> _timerStream;

  @override
  void initState() {
    super.initState();
    _timerStream = _createTimerStream();
  }

  String _formatDuration(int totalSeconds) {
    final hours = (totalSeconds ~/ 3600).toString().padLeft(2, '0');
    final minutes = ((totalSeconds % 3600) ~/ 60).toString().padLeft(2, '0');
    final seconds = (totalSeconds % 60).toString().padLeft(2, '0');
    return '$hours:$minutes:$seconds';
  }

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(title: const Text('StreamBuilder 实时计时器')),
      body: Center(
        child: StreamBuilder<int>(
          stream: _timerStream,
          initialData: 0, // 初始显示 0
          builder: (context, snapshot) {
            // 展示当前连接状态（便于学习理解）
            final stateText = switch (snapshot.connectionState) {
              ConnectionState.none => '未连接',
              ConnectionState.waiting => '等待数据...',
              ConnectionState.active => '实时接收中',
              ConnectionState.done => 'Stream 已关闭',
            };

            return Column(
              mainAxisAlignment: MainAxisAlignment.center,
              children: [
                // 连接状态标签
                Chip(
                  avatar: Icon(
                    snapshot.connectionState == ConnectionState.active
                        ? Icons.sensors
                        : Icons.sensors_off,
                    size: 18,
                  ),
                  label: Text(stateText),
                  backgroundColor:
                      snapshot.connectionState == ConnectionState.active
                          ? Colors.green[50]
                          : Colors.grey[200],
                ),
                const SizedBox(height: 32),
                // 计时器显示
                Text(
                  _formatDuration(snapshot.data ?? 0),
                  style: const TextStyle(
                    fontSize: 56,
                    fontWeight: FontWeight.w200,
                    fontFamily: 'monospace',
                    letterSpacing: 4,
                  ),
                ),
                const SizedBox(height: 16),
                Text(
                  '已运行 ${snapshot.data ?? 0} 秒',
                  style: Theme.of(context).textTheme.bodyLarge?.copyWith(
                        color: Colors.grey[600],
                      ),
                ),
              ],
            );
          },
        ),
      ),
    );
  }
}

2.4 initialData 的作用

initialData 可以避免 Stream 尚未发送第一条数据时出现空白或加载状态：

StreamBuilder<int>(
  stream: _counterStream,
  initialData: 0, // Stream 未发数据前，snapshot.data 为 0，hasData 为 true
  builder: (context, snapshot) {
    // 因为有 initialData，首帧就能渲染有意义的 UI
    return Text('计数: ${snapshot.data}');
  },
)

注意事项：设置 initialData 后，snapshot.hasData 在首帧就为 true，connectionState 也会直接从 waiting 开始（而非 none）。如果你的 UI 逻辑依赖"是否收到过真实数据"来做判断，需要额外考虑这一点。

3. ValueListenableBuilder：监听 ValueNotifier 变化

3.1 基本概念

ValueListenableBuilder 用于监听 ValueNotifier（或任何实现了 ValueListenable 接口的对象）的值变化，并在值改变时自动重建 UI。与 FutureBuilder/StreamBuilder 不同，它面向的是同步的值变更通知。

核心参数：

参数 类型 说明
valueListenable ValueListenable<T> 要监听的 ValueNotifier 对象
builder Widget Function(BuildContext, T, Widget?) 值变化时的构建回调
child Widget? 不依赖值变化的子组件，用于性能优化

3.2 builder 回调的三个参数

ValueListenableBuilder<int>(
  valueListenable: _counter,
  builder: (BuildContext context, int value, Widget? child) {
    // context: 当前构建上下文
    // value:   ValueNotifier 的当前值
    // child:   下方传入的 child 参数（不会随值变化而重建）
    return Row(
      children: [
        Text('计数: $value'),  // 依赖 value，每次值变化都重建
        child!,                // 不依赖 value，始终复用同一个实例
      ],
    );
  },
  // child 只构建一次，作为 builder 回调的第三个参数传入
  child: const Icon(Icons.star, size: 32),
)

3.3 child 参数的优化原理

child 参数是 ValueListenableBuilder 提供的重要性能优化手段。传入 child 的 Widget 只会构建一次，后续值变化时不会重新构建，直接复用引用。这对于复杂的、不依赖 notifier 值的子组件树非常有价值。

3.4 完整示例

import 'package:flutter/material.dart';

void main() {
  runApp(const ValueListenableDemoApp());
}

class ValueListenableDemoApp extends StatelessWidget {
  const ValueListenableDemoApp({super.key});

  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      theme: ThemeData(colorSchemeSeed: Colors.purple, useMaterial3: true),
      home: const ValueListenableDemoPage(),
    );
  }
}

class ValueListenableDemoPage extends StatefulWidget {
  const ValueListenableDemoPage({super.key});

  @override
  State<ValueListenableDemoPage> createState() =>
      _ValueListenableDemoPageState();
}

class _ValueListenableDemoPageState extends State<ValueListenableDemoPage> {
  // 创建 ValueNotifier，初始值为 0
  final ValueNotifier<int> _counter = ValueNotifier<int>(0);
  final ValueNotifier<Color> _color = ValueNotifier<Color>(Colors.blue);

  final List<Color> _colorOptions = [
    Colors.blue,
    Colors.red,
    Colors.green,
    Colors.orange,
    Colors.purple,
    Colors.teal,
  ];

  @override
  void dispose() {
    // ValueNotifier 也需要在 dispose 中释放
    _counter.dispose();
    _color.dispose();
    super.dispose();
  }

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(title: const Text('ValueListenableBuilder 示例')),
      body: Center(
        child: Column(
          mainAxisAlignment: MainAxisAlignment.center,
          children: [
            // 监听计数器变化
            ValueListenableBuilder<int>(
              valueListenable: _counter,
              // child 参数优化：Icon 不依赖 _counter 值，只构建一次
              child: const Icon(Icons.favorite, size: 40, color: Colors.pink),
              builder: (context, count, child) {
                return Column(
                  children: [
                    child!, // 复用不变的 Icon
                    const SizedBox(height: 8),
                    Text(
                      '$count',
                      style: const TextStyle(
                        fontSize: 64,
                        fontWeight: FontWeight.bold,
                      ),
                    ),
                  ],
                );
              },
            ),
            const SizedBox(height: 24),

            // 监听颜色变化
            ValueListenableBuilder<Color>(
              valueListenable: _color,
              builder: (context, color, child) {
                return AnimatedContainer(
                  duration: const Duration(milliseconds: 300),
                  width: 120,
                  height: 120,
                  decoration: BoxDecoration(
                    color: color,
                    borderRadius: BorderRadius.circular(16),
                    boxShadow: [
                      BoxShadow(
                        color: color.withOpacity(0.4),
                        blurRadius: 12,
                        offset: const Offset(0, 6),
                      ),
                    ],
                  ),
                  child: child, // 复用不变的文本
                );
              },
              child: const Center(
                child: Text(
                  '点我\n换色',
                  textAlign: TextAlign.center,
                  style: TextStyle(
                    color: Colors.white,
                    fontWeight: FontWeight.bold,
                  ),
                ),
              ),
            ),
            const SizedBox(height: 32),

            // 操作按钮
            Row(
              mainAxisAlignment: MainAxisAlignment.center,
              children: [
                FilledButton.icon(
                  onPressed: () => _counter.value++,
                  icon: const Icon(Icons.add),
                  label: const Text('计数 +1'),
                ),
                const SizedBox(width: 12),
                OutlinedButton.icon(
                  onPressed: () {
                    final currentIndex = _colorOptions.indexOf(_color.value);
                    final nextIndex =
                        (currentIndex + 1) % _colorOptions.length;
                    _color.value = _colorOptions[nextIndex];
                  },
                  icon: const Icon(Icons.palette),
                  label: const Text('换颜色'),
                ),
                const SizedBox(width: 12),
                TextButton.icon(
                  onPressed: () {
                    _counter.value = 0;
                    _color.value = Colors.blue;
                  },
                  icon: const Icon(Icons.restart_alt),
                  label: const Text('重置'),
                ),
              ],
            ),
          ],
        ),
      ),
    );
  }
}

最佳实践：

  • ValueNotifier 非常适合管理简单的、局部的状态（计数器、开关状态、选中项等），避免不必要的全局 setState 重建
  • 永远记得在 dispose 中调用 ValueNotifier.dispose() 释放资源
  • 善用 child 参数，将不依赖值变化的 Widget 提取出来传入，减少不必要的重建

4. StreamController 结合 StreamBuilder 实战

4.1 StreamController 基础

StreamController 是创建和管理自定义 Stream 的核心工具。你可以通过它的 sink 向 Stream 中添加数据，通过 stream 属性获取数据流供消费端监听。

// 创建单订阅 StreamController
final controller = StreamController<String>();

// 发送数据
controller.sink.add('Hello');
controller.sink.add('World');

// 发送错误
controller.sink.addError(Exception('出错了'));

// 监听数据
controller.stream.listen(
  (data) => print('收到: $data'),
  onError: (error) => print('错误: $error'),
  onDone: () => print('Stream 已关闭'),
);

// 使用完毕后关闭
controller.close();

4.2 单订阅流 vs 广播流

特性 单订阅流 StreamController() 广播流 StreamController.broadcast()
监听者数量 只允许一个监听者 允许多个监听者
数据缓存 在被监听之前，数据会被缓存 不缓存，监听前发送的数据会丢失
适用场景 文件读取、HTTP 响应等一次性场景 事件总线、用户输入事件等多处消费的场景
StreamBuilder 使用 ✅ 常用 ✅ 适合多个 Widget 监听同一 Stream

4.3 完整示例：聊天消息流

import 'dart:async';
import 'package:flutter/material.dart';

void main() {
  runApp(const StreamControllerDemoApp());
}

class StreamControllerDemoApp extends StatelessWidget {
  const StreamControllerDemoApp({super.key});

  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      theme: ThemeData(colorSchemeSeed: Colors.indigo, useMaterial3: true),
      home: const ChatDemoPage(),
    );
  }
}

class ChatMessage {
  final String sender;
  final String content;
  final DateTime timestamp;

  ChatMessage({
    required this.sender,
    required this.content,
    DateTime? timestamp,
  }) : timestamp = timestamp ?? DateTime.now();
}

class ChatDemoPage extends StatefulWidget {
  const ChatDemoPage({super.key});

  @override
  State<ChatDemoPage> createState() => _ChatDemoPageState();
}

class _ChatDemoPageState extends State<ChatDemoPage> {
  // 使用广播流，允许多个地方同时监听（如消息列表 + 未读计数）
  final StreamController<ChatMessage> _messageController =
      StreamController<ChatMessage>.broadcast();

  final TextEditingController _textController = TextEditingController();
  final List<ChatMessage> _messages = [];
  final ScrollController _scrollController = ScrollController();

  // 模拟对方自动回复
  Timer? _autoReplyTimer;

  @override
  void initState() {
    super.initState();
    // 监听消息流，将消息追加到列表
    _messageController.stream.listen((message) {
      setState(() {
        _messages.add(message);
      });
      // 滚动到底部
      WidgetsBinding.instance.addPostFrameCallback((_) {
        if (_scrollController.hasClients) {
          _scrollController.animateTo(
            _scrollController.position.maxScrollExtent,
            duration: const Duration(milliseconds: 300),
            curve: Curves.easeOut,
          );
        }
      });
    });
  }

  void _sendMessage(String text) {
    if (text.trim().isEmpty) return;

    // 通过 sink.add 发送消息到 Stream
    _messageController.sink.add(
      ChatMessage(sender: '我', content: text.trim()),
    );
    _textController.clear();

    // 模拟对方 1.5 秒后自动回复
    _autoReplyTimer?.cancel();
    _autoReplyTimer = Timer(const Duration(milliseconds: 1500), () {
      _messageController.sink.add(
        ChatMessage(sender: '助手', content: '收到你的消息: "$text"'),
      );
    });
  }

  @override
  void dispose() {
    _autoReplyTimer?.cancel();
    // 关闭 StreamController 释放资源
    _messageController.close();
    _textController.dispose();
    _scrollController.dispose();
    super.dispose();
  }

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(
        title: const Text('StreamController 聊天演示'),
        // 使用 StreamBuilder 监听广播流，显示消息计数
        actions: [
          Padding(
            padding: const EdgeInsets.only(right: 16),
            child: Center(
              child: Text('消息数: ${_messages.length}'),
            ),
          ),
        ],
      ),
      body: Column(
        children: [
          // 消息列表
          Expanded(
            child: _messages.isEmpty
                ? const Center(
                    child: Text('还没有消息，发送第一条吧！',
                        style: TextStyle(color: Colors.grey)),
                  )
                : ListView.builder(
                    controller: _scrollController,
                    padding: const EdgeInsets.all(16),
                    itemCount: _messages.length,
                    itemBuilder: (context, index) {
                      final msg = _messages[index];
                      final isMe = msg.sender == '我';
                      return Align(
                        alignment: isMe
                            ? Alignment.centerRight
                            : Alignment.centerLeft,
                        child: Container(
                          margin: const EdgeInsets.only(bottom: 8),
                          padding: const EdgeInsets.symmetric(
                            horizontal: 16,
                            vertical: 10,
                          ),
                          constraints: BoxConstraints(
                            maxWidth:
                                MediaQuery.of(context).size.width * 0.7,
                          ),
                          decoration: BoxDecoration(
                            color: isMe
                                ? Theme.of(context).colorScheme.primary
                                : Theme.of(context)
                                    .colorScheme
                                    .surfaceContainerHighest,
                            borderRadius: BorderRadius.only(
                              topLeft: const Radius.circular(16),
                              topRight: const Radius.circular(16),
                              bottomLeft:
                                  Radius.circular(isMe ? 16 : 4),
                              bottomRight:
                                  Radius.circular(isMe ? 4 : 16),
                            ),
                          ),
                          child: Column(
                            crossAxisAlignment: CrossAxisAlignment.start,
                            children: [
                              Text(
                                msg.content,
                                style: TextStyle(
                                  color: isMe ? Colors.white : null,
                                ),
                              ),
                              const SizedBox(height: 4),
                              Text(

// 创建广播流
final _eventBus = StreamController<String>.broadcast();

// Widget A 监听
StreamBuilder<String>(
  stream: _eventBus.stream,
  builder: (context, snapshot) {
    return Text('Widget A 收到: ${snapshot.data ?? "无"}');
  },
)

// Widget B 也可以同时监听同一个 stream
StreamBuilder<String>(
  stream: _eventBus.stream,
  builder: (context, snapshot) {
    return Text('Widget B 收到: ${snapshot.data ?? "无"}');
  },
)

// 任意位置发送数据，两个 Widget 都会收到
_eventBus.sink.add('新事件!');

注意事项：

  • 单订阅的 StreamController 只能被一个 StreamBuilder（或一个 listen）监听。如果尝试多次监听，会抛出 Bad state: Stream has already been listened to 异常
  • 使用完 StreamController 后必须调用 close() 释放资源，否则会导致内存泄漏
  • 在 State.dispose() 中关闭 StreamController 是标准做法

5. FutureBuilder 常见陷阱

5.1 陷阱：在 build 中创建 Future

这是使用 FutureBuilder 时最常见也最严重的错误。如果在 build 方法中直接创建 Future，每次 Widget 重建（setState、父组件更新、键盘弹出等）都会创建一个新的 Future 实例，导致：

  1. 重复的网络请求
  2. UI 闪烁（从完成状态回到加载状态）
  3. 资源浪费和潜在的竞态条件

5.2 错误示范

class BadFutureBuilderPage extends StatefulWidget {
  const BadFutureBuilderPage({super.key});

  @override
  State<BadFutureBuilderPage> createState() => _BadFutureBuilderPageState();
}

class _BadFutureBuilderPageState extends State<BadFutureBuilderPage> {
  int _otherState = 0;

  /// 模拟网络请求
  Future<List<String>> _fetchItems() async {
    debugPrint('⚠️ _fetchItems 被调用了！每次 build 都会触发！');
    await Future.delayed(const Duration(seconds: 2));
    return ['项目 A', '项目 B', '项目 C'];
  }

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(title: const Text('❌ 错误示范')),
      body: Column(
        children: [
          // 这个按钮会触发 setState → build → 创建新 Future → 重新加载
          Padding(
            padding: const EdgeInsets.all(16),
            child: ElevatedButton(
              onPressed: () => setState(() => _otherState++),
              child: Text('修改无关状态 (当前值: $_otherState)'),
            ),
          ),
          Expanded(
            child: FutureBuilder<List<String>>(
              // ❌ 错误！在 build 中直接调用方法创建 Future
              // 每次 build 都会产生新的 Future 实例
              future: _fetchItems(),
              builder: (context, snapshot) {
                if (snapshot.connectionState == ConnectionState.waiting) {
                  return const Center(child: CircularProgressIndicator());
                }
                if (snapshot.hasData) {
                  return ListView(
                    children: snapshot.data!
                        .map((item) => ListTile(title: Text(item)))
                        .toList(),
                  );
                }
                return const SizedBox.shrink();
              },
            ),
          ),
        ],
      ),
    );
  }
}

点击"修改无关状态"按钮后，你会看到列表瞬间回到加载状态，然后 2 秒后重新出现。控制台也会反复打印 _fetchItems 被调用了。在真实项目中，这意味着每次键盘弹出、主题切换、甚至父组件的微小更新都会触发重复的网络请求。

5.3 正确示范

class GoodFutureBuilderPage extends StatefulWidget {
  const GoodFutureBuilderPage({super.key});

  @override
  State<GoodFutureBuilderPage> createState() => _GoodFutureBuilderPageState();
}

class _GoodFutureBuilderPageState extends State<GoodFutureBuilderPage> {
  int _otherState = 0;

  // ✅ 将 Future 缓存为 State 的成员变量
  late Future<List<String>> _itemsFuture;

  @override
  void initState() {
    super.initState();
    // ✅ 在 initState 中创建 Future，只执行一次
    _itemsFuture = _fetchItems();
  }

  Future<List<String>> _fetchItems() async {
    debugPrint('✅ _fetchItems 只在 initState 时调用一次');
    await Future.delayed(const Duration(seconds: 2));
    return ['项目 A', '项目 B', '项目 C'];
  }

  /// 需要主动刷新时，显式地创建新 Future
  void _refresh() {
    setState(() {
      _itemsFuture = _fetchItems();
    });
  }

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(title: const Text('✅ 正确示范')),
      body: Column(
        children: [
          Padding(
            padding: const EdgeInsets.all(16),
            child: Row(
              children: [
                ElevatedButton(
                  // 修改无关状态不会触发重新请求
                  onPressed: () => setState(() => _otherState++),
                  child: Text('修改无关状态 (当前值: $_otherState)'),
                ),
                const SizedBox(width: 12),
                OutlinedButton.icon(
                  // 只有用户主动点击刷新时才重新请求
                  onPressed: _refresh,
                  icon: const Icon(Icons.refresh),
                  label: const Text('刷新数据'),
                ),
              ],
            ),
          ),
          Expanded(
            child: FutureBuilder<List<String>>(
              // ✅ 传入缓存的 Future 变量，build 重建时不会创建新 Future
              future: _itemsFuture,
              builder: (context, snapshot) {
                if (snapshot.connectionState == ConnectionState.waiting) {
                  return const Center(child: CircularProgressIndicator());
                }
                if (snapshot.hasData) {
                  return ListView(
                    children: snapshot.data!
                        .map((item) => ListTile(title: Text(item)))
                        .toList(),
                  );
                }
                return const SizedBox.shrink();
              },
            ),
          ),
        ],
      ),
    );
  }
}

5.4 原理分析

FutureBuilder 内部会在 didUpdateWidget 中比较新旧 future 对象的引用。如果引用不同（widget.future != oldWidget.future），它会认为是一个新的异步操作，从而重新订阅并将状态重置为 ConnectionState.waiting。

在 build 中直接调用 _fetchItems() 每次都返回一个新的 Future 实例（新引用），所以每次重建都会触发重新订阅。而将 Future 缓存到变量后，多次 build 传入的是同一个引用，FutureBuilder 知道它没变，不会重复执行。

最佳实践总结：

  1. 永远不要在 build 方法中创建 Future，应在 initState 中创建并缓存
  2. 需要刷新数据时，在 setState 中重新赋值 Future 变量
  3. 如果 Future 依赖 widget 的参数（从父组件传入），在 didUpdateWidget 中判断参数是否变化，变化时才重新创建 Future

  Widget build(BuildContext context) {

      debugShowCheckedModeBanner: false,

class ArticleListPage extends StatefulWidget {
  const ArticleListPage({super.key});

  @override
  State<ArticleListPage> createState() => _ArticleListPageState();
}

class _ArticleListPageState extends State<ArticleListPage> {
  // ✅ 在 State 中缓存 Future
  late Future<List<Article>> _articlesFuture;

  @override
  void initState() {
    super.initState();
    _articlesFuture = _loadArticles();
  }

  Future<List<Article>> _loadArticles() {
    // 设置 30% 失败率以便观察错误状态
    return ArticleApi.fetchArticles(failRate: 0.3);
  }

  /// 刷新数据
  void _refresh() {
    setState(() {
      _articlesFuture = _loadArticles();
    });
  }

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(
        title: const Text('技术文章'),
        actions: [
          IconButton(
            onPressed: _refresh,
            icon: const Icon(Icons.refresh),
            tooltip: '刷新',
          ),
        ],
      ),
      body: FutureBuilder<List<Article>>(
        future: _articlesFuture,
        builder: (context, snapshot) {
          // 状态1：加载中 → 显示骨架屏
          if (snapshot.connectionState == ConnectionState.waiting) {
            return const ArticleSkeletonList();
          }

          // 状态2：加载失败 → 显示错误页面 + 重试按钮
          if (snapshot.hasError) {
            return ErrorStatePage(
              message: snapshot.error.toString(),
              onRetry: _refresh,
            );
          }

          // 状态3：加载成功但数据为空 → 显示空状态
          final articles = snapshot.data ?? [];
          if (articles.isEmpty) {
            return EmptyStatePage(
              message: '还没有文章哦\n去关注一些感兴趣的话题吧',
              actionText: '刷新试试',
              onAction: _refresh,
            );
          }

          // 状态4：加载成功有数据 → 显示列表（支持下拉刷新）
          return RefreshIndicator(
            onRefresh: () async {
              // 下拉刷新：创建新 Future 并等待完成
              final future = _loadArticles();
              setState(() {
                _articlesFuture = future;
              });
              // 等待请求完成，让 RefreshIndicator 知道何时停止转圈
              await future;
            },
            child: ListView.separated(
              physics: const AlwaysScrollableScrollPhysics(),
              itemCount: articles.length,
              separatorBuilder: (_, __) => const Divider(
                height: 1,
                indent: 76,
              ),
              itemBuilder: (context, index) {
                return ArticleCard(article: articles[index]);
              },
            ),
          );
        },
      ),
    );
  }
}

7. 三大 Builder 组件对比

特性 FutureBuilder StreamBuilder ValueListenableBuilder
数据源类型 Future<T> Stream<T> ValueListenable<T>
数据特征 一次性异步结果 持续的异步数据序列 同步值变更通知
典型场景 网络请求、数据库查询 WebSocket、定时器、事件流 简单状态监听（计数器、开关）
状态信息 AsyncSnapshot AsyncSnapshot 直接获得值 T
child 优化 不支持 不支持 ✅ 支持
常见陷阱 build 中创建 Future Stream 多次订阅 忘记 dispose

本课小结

本课系统讲解了 Flutter 中三大异步 UI 构建组件：

  1. FutureBuilder：适用于一次性异步操作（如网络请求）。核心是理解 AsyncSnapshot 和 ConnectionState，根据不同状态返回不同 UI。最重要的原则是不要在 build 中创建 Future，应缓存到 State 变量中。

  2. StreamBuilder：适用于持续数据流（如 WebSocket、定时器、事件总线）。每当 Stream 发出新数据，builder 就会被调用。ConnectionState.active 表示正在持续接收数据。

  3. ValueListenableBuilder：适用于同步的 ValueNotifier 值变更监听。child 参数是其独有的性能优化手段，可以避免不依赖值变化的子组件被重建。

  4. StreamController：创建自定义 Stream 的核心工具。区分单订阅流和广播流（broadcast），使用完毕后必须 close() 释放资源。

  5. 三态处理模式：在实际项目中，异步页面必须完整处理加载中（骨架屏/进度指示器）、成功（正常内容/空状态）、失败（错误信息 + 重试按钮）三种状态，这是生产级应用的基本要求。

课后练习

练习1：天气查询器（基础）

使用 FutureBuilder 构建一个天气查询页面：

  • 用 TextField 输入城市名称
  • 点击查询按钮后，模拟一个 2 秒的异步请求返回天气数据（温度、天气状况、风力）
  • 完整处理加载中、成功、失败三种状态
  • 确保 Future 不会在每次 build 时重复创建

练习2：实时搜索（中级）

结合 StreamController 和 StreamBuilder 实现一个实时搜索功能：

  • 用户在 TextField 输入文字时，每次变化通过 StreamController 发送搜索关键词
  • 对输入进行 300ms 防抖（debounce）：用户停止输入 300ms 后才执行搜索
  • 用 StreamBuilder 监听搜索结果 Stream，实时展示匹配的列表项
  • 处理空结果和错误状态

练习3：多 ValueNotifier 组合（中级）

创建一个购物车页面，使用多个 ValueNotifier 管理状态：

  • ValueNotifier<List<CartItem>> 管理购物车商品列表
  • ValueNotifier<double> 管理总价（根据商品列表计算）
  • 使用 ValueListenableBuilder 分别构建商品列表和底部总价栏
  • 充分利用 child 参数优化不需要重建的 UI 部分

练习4：生产级异步页面改造（进阶）

将本课的实战示例扩展为更完整的生产级页面：

  • 添加分页加载（上拉加载更多）
  • 骨架屏在首次加载时显示，分页加载时底部显示 CircularProgressIndicator
  • 添加搜索过滤功能
  • 将三态处理逻辑封装为通用的 AsyncContentBuilder<T> 组件，接受 future、loadingBuilder、errorBuilder、emptyBuilder、contentBuilder 参数，实现跨页面复用
