上一课我们学习了 ListView 的各种用法，掌握了列表的基本构建方式。本课是列表系列的第二课，也是进阶内容。我们将从 GridView（网格列表）出发，逐步深入到 Flutter 滚动体系的核心——Sliver。Sliver 是 Flutter 实现高性能滚动视图的基石，理解它的运作机制，是构建复杂滚动页面（如电商首页、社交应用个人主页）的关键。本课内容较多，建议分模块消化。

1. GridView 网格布局

GridView 是 Flutter 中最常用的网格组件，可以将子组件按行列排列展示。它和 ListView 同属于 ScrollView 的子类，核心区别在于 GridView 使用二维网格排列子组件，而 ListView 是一维线性排列。

1.1 GridView.count：固定列数网格

GridView.count 是最简单直观的网格构造方式。你只需指定 crossAxisCount（交叉轴方向的子项数量，即列数），Flutter 就会自动将可用空间等分给每一列。

核心参数：

参数 类型 说明
crossAxisCount int 交叉轴方向的子项数量（垂直滚动时为列数）
mainAxisSpacing double 主轴方向的间距（垂直滚动时为行间距）
crossAxisSpacing double 交叉轴方向的间距（垂直滚动时为列间距）
childAspectRatio double 子项的宽高比，默认 1.0（正方形）
children List<Widget> 子组件列表 
import 'package:flutter/material.dart';

void main() => runApp(const MyApp());

class MyApp extends StatelessWidget {
  const MyApp({super.key});

  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      home: Scaffold(
        appBar: AppBar(title: const Text('GridView.count 示例')),
        body: GridView.count(
          crossAxisCount: 3, // 每行 3 列
          mainAxisSpacing: 8.0, // 行间距 8
          crossAxisSpacing: 8.0, // 列间距 8
          padding: const EdgeInsets.all(8.0),
          childAspectRatio: 1.0, // 宽高比 1:1（正方形）
          children: List.generate(12, (index) {
            return Container(
              decoration: BoxDecoration(
                color: Colors.primaries[index % Colors.primaries.length],
                borderRadius: BorderRadius.circular(8),
              ),
              alignment: Alignment.center,
              child: Text(
                '项目 $index',
                style: const TextStyle(
                  color: Colors.white,
                  fontWeight: FontWeight.bold,
                ),
              ),
            );
          }),
        ),
      ),
    );
  }
}

childAspectRatio 是一个容易被忽略但非常重要的参数。它定义了每个子项的宽高比（width / height）。当 childAspectRatio 为 1.0 时，子项为正方形；大于 1.0 时，子项变得扁平（宽 > 高）；小于 1.0 时，子项变得细高（高 > 宽）。

注意事项：

  • GridView.count 会一次性创建所有子组件，适合子项数量较少的场景（几十个以内）。如果子项数量很大，应使用 GridView.builder 进行懒加载。
  • childAspectRatio 只能统一设定所有子项的宽高比。如果需要子项具有不同的高度，需要使用第三方库如 flutter_staggered_grid_view。

1.2 GridView.extent：固定子项最大宽度

GridView.extent 不指定列数，而是指定每个子项的最大交叉轴范围（maxCrossAxisExtent）。Flutter 会根据可用空间自动计算需要多少列。这种方式特别适合响应式布局——在不同屏幕宽度下，列数会自动调整。

核心参数：

参数 类型 说明
maxCrossAxisExtent double 每个子项在交叉轴方向的最大范围
mainAxisSpacing double 主轴方向的间距
crossAxisSpacing double 交叉轴方向的间距
childAspectRatio double 子项宽高比 
import 'package:flutter/material.dart';

void main() => runApp(const MyApp());

class MyApp extends StatelessWidget {
  const MyApp({super.key});

  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      home: Scaffold(
        appBar: AppBar(title: const Text('GridView.extent 示例')),
        body: GridView.extent(
          maxCrossAxisExtent: 150, // 每个子项最大宽度 150
          mainAxisSpacing: 8.0,
          crossAxisSpacing: 8.0,
          padding: const EdgeInsets.all(8.0),
          childAspectRatio: 3 / 4, // 宽高比 3:4，子项略高
          children: List.generate(20, (index) {
            return Card(
              elevation: 2,
              child: Column(
                mainAxisAlignment: MainAxisAlignment.center,
                children: [
                  Icon(
                    Icons.image,
                    size: 40,
                    color: Colors.primaries[index % Colors.primaries.length],
                  ),
                  const SizedBox(height: 8),
                  Text('商品 $index'),
                ],
              ),
            );
          }),
        ),
      ),
    );
  }
}

count 与 extent 的选择策略：

场景 推荐方式
列数固定不变（如照片九宫格） GridView.count
需要适配不同屏幕宽度 GridView.extent
子项有固定的最佳显示宽度 GridView.extent

1.3 GridView.builder：懒加载网格

当网格中的子项数量很大（或无限）时，GridView.builder 是必须使用的构造方式。它和 ListView.builder 类似，只在子项即将出现在视口中时才创建对应的 Widget，极大节省内存和 CPU 资源。

GridView.builder 需要一个 gridDelegate 参数来指定网格的排列规则。这是我们第一次正式接触 SliverGridDelegate。

import 'package:flutter/material.dart';

void main() => runApp(const MyApp());

class MyApp extends StatelessWidget {
  const MyApp({super.key});

  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      home: Scaffold(
        appBar: AppBar(title: const Text('GridView.builder 示例')),
        body: GridView.builder(
          padding: const EdgeInsets.all(8.0),
          gridDelegate: const SliverGridDelegateWithFixedCrossAxisCount(
            crossAxisCount: 2, // 每行 2 列
            mainAxisSpacing: 10,
            crossAxisSpacing: 10,
            childAspectRatio: 1.0,
          ),
          itemCount: 100, // 总共 100 个子项
          itemBuilder: (context, index) {
            // 只有当子项即将显示时才会调用此回调
            return Container(
              decoration: BoxDecoration(
                gradient: LinearGradient(
                  colors: [
                    Colors.primaries[index % Colors.primaries.length],
                    Colors.primaries[(index + 3) % Colors.primaries.length],
                  ],
                ),
                borderRadius: BorderRadius.circular(12),
              ),
              alignment: Alignment.center,
              child: Text(
                '# $index',
                style: const TextStyle(
                  color: Colors.white,
                  fontSize: 18,
                  fontWeight: FontWeight.bold,
                ),
              ),
            );
          },
        ),
      ),
    );
  }
}

最佳实践：

  • 只要子项数量超过屏幕可见范围，就应该优先使用 GridView.builder。
  • itemCount 可以不传（默认无限），用于分页加载场景。但如果你能确定总数，传入 itemCount 可以避免越界错误。
  • 在 itemBuilder 中避免执行耗时操作（网络请求、复杂计算），应该在外部准备好数据后仅负责构建 UI。

2. SliverGridDelegate 详解

SliverGridDelegate 是控制网格布局的核心类。无论是 GridView.builder、GridView.custom，还是后面将学到的 SliverGrid，都依赖它来决定子项的排列方式。Flutter 提供了两个内置实现：

2.1 SliverGridDelegateWithFixedCrossAxisCount

按固定交叉轴数量排列。对应 GridView.count 的底层实现。

完整参数：

参数 类型 默认值 说明
crossAxisCount int 必填 交叉轴方向的子项数量
mainAxisSpacing double 0.0 主轴方向子项之间的间距
crossAxisSpacing double 0.0 交叉轴方向子项之间的间距
childAspectRatio double 1.0 子项的宽高比（宽 / 高）
mainAxisExtent double? null 子项在主轴方向的固定尺寸（设置后 childAspectRatio 失效）

mainAxisExtent 是一个非常实用的参数。当你需要子项具有固定的高度而非依赖宽高比计算时，直接设置 mainAxisExtent 比反复调试 childAspectRatio 更直观。

import 'package:flutter/material.dart';

void main() => runApp(const MyApp());

class MyApp extends StatelessWidget {
  const MyApp({super.key});

  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      home: Scaffold(
        appBar: AppBar(title: const Text('FixedCrossAxisCount 参数演示')),
        body: GridView.builder(
          padding: const EdgeInsets.all(12),
          gridDelegate: const SliverGridDelegateWithFixedCrossAxisCount(
            crossAxisCount: 3,
            mainAxisSpacing: 12,
            crossAxisSpacing: 12,
            // 使用 mainAxisExtent 直接指定子项高度为 120
            // 设置此值后 childAspectRatio 将被忽略
            mainAxisExtent: 120,
          ),
          itemCount: 15,
          itemBuilder: (context, index) {
            return Container(
              decoration: BoxDecoration(
                color: Colors.teal.withValues(alpha: 0.2 + (index % 5) * 0.15),
                borderRadius: BorderRadius.circular(8),
                border: Border.all(color: Colors.teal),
              ),
              alignment: Alignment.center,
              child: Text('高度固定\n120px', textAlign: TextAlign.center),
            );
          },
        ),
      ),
    );
  }
}

2.2 SliverGridDelegateWithMaxCrossAxisExtent

按最大交叉轴范围排列。对应 GridView.extent 的底层实现。Flutter 会根据可用宽度和 maxCrossAxisExtent 自动计算最佳列数。

完整参数：

参数 类型 默认值 说明
maxCrossAxisExtent double 必填 子项在交叉轴方向的最大范围
mainAxisSpacing double 0.0 主轴方向间距
crossAxisSpacing double 0.0 交叉轴方向间距
childAspectRatio double 1.0 子项宽高比
mainAxisExtent double? null 子项在主轴方向的固定尺寸

列数计算公式：列数 = (可用宽度 / maxCrossAxisExtent).ceil()

例如，屏幕可用宽度为 400px，maxCrossAxisExtent 设为 150：

  • 计算：400 / 150 = 2.67，向上取整得 3 列
  • 每列实际宽度：(400 - 间距总和) / 3 ≈ 125px（小于 150，满足约束）
import 'package:flutter/material.dart';

void main() => runApp(const MyApp());

class MyApp extends StatelessWidget {
  const MyApp({super.key});

  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      home: Scaffold(
        appBar: AppBar(title: const Text('MaxCrossAxisExtent 参数演示')),
        body: LayoutBuilder(
          builder: (context, constraints) {
            return GridView.builder(
              padding: const EdgeInsets.all(12),
              gridDelegate:
                  const SliverGridDelegateWithMaxCrossAxisExtent(
                maxCrossAxisExtent: 200, // 每个子项最宽 200
                mainAxisSpacing: 12,
                crossAxisSpacing: 12,
                childAspectRatio: 4 / 3, // 宽高比 4:3
              ),
              itemCount: 12,
              itemBuilder: (context, index) {
                return Card(
                  elevation: 3,
                  child: Padding(
                    padding: const EdgeInsets.all(8.0),
                    child: Column(
                      mainAxisAlignment: MainAxisAlignment.center,
                      children: [
                        Icon(Icons.photo_album, size: 32),
                        const SizedBox(height: 8),
                        Text(
                          '相册 $index',
                          style: const TextStyle(fontWeight: FontWeight.bold),
                        ),
                        Text(
                          '可用宽度: ${constraints.maxWidth.toStringAsFixed(0)}',
                          style: const TextStyle(
                            fontSize: 11,
                            color: Colors.grey,
                          ),
                        ),
                      ],
                    ),
                  ),
                );
              },
            );
          },
        ),
      ),
    );
  }
}

注意事项：

  • mainAxisExtent 的优先级高于 childAspectRatio。当两者同时设置时，childAspectRatio 会被忽略。
  • childAspectRatio 的计算基于子项的实际交叉轴尺寸（扣除间距后），而不是 maxCrossAxisExtent。
  • 在 SliverGridDelegateWithMaxCrossAxisExtent 中，子项的实际宽度可能小于 maxCrossAxisExtent，因为 Flutter 会把空间均匀分配给整数个列。

3. CustomScrollView 与 Sliver 体系

从这里开始，我们进入本课的重头戏——Sliver 体系。这是 Flutter 滚动系统的核心，也是构建复杂滚动页面不可绕过的知识点。

3.1 什么是 Sliver？

Sliver（中文可理解为"条片"）是 Flutter 滚动框架中的一种特殊 RenderObject。与普通的盒子布局（Box layout）不同，Sliver 使用一套专门的滚动协议来进行布局：

  • 普通 Widget（Box）：接收盒约束（BoxConstraints），返回固定尺寸（Size）
  • Sliver Widget：接收 SliverConstraints（包含视口信息、滚动偏移等），返回 SliverGeometry（描述自己在滚动方向上占用多少空间、绘制多少像素等）

简单来说，Sliver 是一种知道自己正处于滚动环境中的组件，它能根据当前的滚动位置智能地决定"我应该画多少、我占了多大地方"。

为什么需要 Sliver？

想象一个电商首页：顶部是一个可折叠的大图 Banner（SliverAppBar），下面是一段推荐文字（普通 Widget），再下面是热门商品网格（SliverGrid），最后是商品列表（SliverList）。这些不同类型的内容需要在同一个滚动视图中协调滚动。

如果用普通的 ListView 或 GridView，你会面临以下问题：

  • ListView 的 children 必须是同一种排列方式（线性），无法混合网格
  • 在 ListView 中嵌套 GridView 会产生滚动冲突（两个 ScrollView 各自维护自己的滚动位置）
  • 给嵌套的 GridView 设置 shrinkWrap: true 虽然能解决，但会失去懒加载优势，所有子项一次性构建

Sliver 体系正是为了解决这些问题而设计的。通过 CustomScrollView，你可以将多个 Sliver 组件组合在同一个滚动视图中，它们共享同一个滚动位置，各自只渲染视口内可见的部分，既灵活又高效。

3.2 CustomScrollView 基础

CustomScrollView 是所有 Sliver 组件的容器。它本身不包含任何子组件布局逻辑，而是通过 slivers 参数接收一个 Sliver 列表，由这些 Sliver 自行决定如何渲染。

CustomScrollView(
  slivers: [
    // 这里只能放 Sliver 类型的 Widget
    SliverAppBar(...),
    SliverList(...),
    SliverGrid(...),
    SliverToBoxAdapter(child: ...),
  ],
)

与普通 ScrollView 的对比：

对比项 ListView / GridView CustomScrollView
子组件类型 普通 Widget（Box） Sliver Widget
混合布局 不支持（单一排列方式） 支持（自由组合多种 Sliver）
可折叠头部 不支持 SliverAppBar
吸顶效果 不支持 SliverPersistentHeader
性能 单一布局高效 混合布局也能保持懒加载

下面是一个完整的 CustomScrollView 基础示例，展示了如何将不同类型的 Sliver 组合在一起：

import 'package:flutter/material.dart';

void main() => runApp(const MyApp());

class MyApp extends StatelessWidget {
  const MyApp({super.key});

  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      home: Scaffold(
        body: CustomScrollView(
          slivers: [
            // 1. 应用栏（Sliver 版本）
            const SliverAppBar(
              title: Text('CustomScrollView 基础'),
              floating: true,
            ),

            // 2. 一段普通文字（需要用 SliverToBoxAdapter 包装）
            const SliverToBoxAdapter(
              child: Padding(
                padding: EdgeInsets.all(16),
                child: Text(
                  '下面是一个网格区域，再下面是一个列表区域。'
                  '它们在同一个滚动视图中，共享滚动位置。',
                  style: TextStyle(fontSize: 16),
                ),
              ),
            ),

            // 3. 网格区域
            SliverGrid(
              gridDelegate: const SliverGridDelegateWithFixedCrossAxisCount(
                crossAxisCount: 3,
                mainAxisSpacing: 8,
                crossAxisSpacing: 8,
                childAspectRatio: 1,
              ),
              delegate: SliverChildBuilderDelegate(
                (context, index) {
                  return Container(
                    margin: const EdgeInsets.symmetric(horizontal: 4),
                    color: Colors.amber[(index % 9 + 1) * 100],
                    alignment: Alignment.center,
                    child: Text('网格 $index'),
                  );
                },
                childCount: 9,
              ),
            ),

            // 4. 间隔
            const SliverToBoxAdapter(
              child: Padding(
                padding: EdgeInsets.all(16),
                child: Text(
                  '下面是列表区域：',
                  style: TextStyle(fontSize: 16, fontWeight: FontWeight.bold),
                ),
              ),
            ),

            // 5. 列表区域
            SliverList(
              delegate: SliverChildBuilderDelegate(
                (context, index) {
                  return ListTile(
                    leading: CircleAvatar(child: Text('$index')),
                    title: Text('列表项 $index'),
                    subtitle: const Text('这是列表区域的子项'),
                  );
                },
                childCount: 20,
              ),
            ),
          ],
        ),
      ),
    );
  }
}

注意事项：

  • CustomScrollView 的 slivers 列表中只能放 Sliver 类型的 Widget。如果你尝试放入普通 Widget（如 Container、Text），会得到运行时错误。
  • 如果需要在 Sliver 环境中嵌入普通 Widget，必须使用 SliverToBoxAdapter 包装（后面详述）。

4. SliverList 与 SliverGrid

SliverList 和 SliverGrid 是 CustomScrollView 中最常用的两个内容 Sliver，分别对应普通的 ListView 和 GridView。

4.1 SliverList

SliverList 在 CustomScrollView 中创建线性列表。它通过一个 delegate（SliverChildDelegate）来生成子组件。

常用的 delegate 有两种：

  • SliverChildListDelegate：接收一个完整的子组件列表，类似 ListView(children: [...])
  • SliverChildBuilderDelegate：按需构建子组件，类似 ListView.builder
import 'package:flutter/material.dart';

void main() => runApp(const MyApp());

class MyApp extends StatelessWidget {
  const MyApp({super.key});

  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      home: Scaffold(
        body: CustomScrollView(
          slivers: [
            const SliverAppBar(title: Text('SliverList 示例')),

            // 方式一：SliverChildListDelegate（适合少量固定子项）
            SliverList(
              delegate: SliverChildListDelegate([
                const ListTile(
                  leading: Icon(Icons.star, color: Colors.orange),
                  title: Text('固定项 1：VIP 推荐'),
                ),
                const ListTile(
                  leading: Icon(Icons.star, color: Colors.orange),
                  title: Text('固定项 2：热门活动'),
                ),
                const Divider(thickness: 2),
              ]),
            ),

            // 方式二：SliverChildBuilderDelegate（适合大量动态子项）
            SliverList(
              delegate: SliverChildBuilderDelegate(
                (context, index) {
                  return ListTile(
                    leading: CircleAvatar(
                      backgroundColor: Colors.blue,
                      child: Text('${index + 1}'),
                    ),
                    title: Text('动态列表项 ${index + 1}'),
                    subtitle: Text('这是第 ${index + 1} 条内容'),
                    trailing: const Icon(Icons.chevron_right),
                  );
                },
                childCount: 30,
              ),
            ),
          ],
        ),
      ),
    );
  }
}

Flutter 3.x 新增：从 Flutter 3.7 开始，你还可以使用更简洁的 SliverList.builder、SliverList.list、SliverList.separated 构造函数，与 ListView 的 API 风格保持一致：

// 等价于使用 SliverChildBuilderDelegate
SliverList.builder(
  itemCount: 20,
  itemBuilder: (context, index) => ListTile(title: Text('项目 $index')),
)

// 等价于使用 SliverChildListDelegate
SliverList.list(
  children: [Text('A'), Text('B'), Text('C')],
)

// 带分割线的列表（类似 ListView.separated）
SliverList.separated(
  itemCount: 20,
  itemBuilder: (context, index) => ListTile(title: Text('项目 $index')),
  separatorBuilder: (context, index) => const Divider(),
)

4.2 SliverGrid

SliverGrid 在 CustomScrollView 中创建网格布局，需要同时提供 gridDelegate（排列规则）和 delegate（子组件生成方式）。

import 'package:flutter/material.dart';

void main() => runApp(const MyApp());

class MyApp extends StatelessWidget {
  const MyApp({super.key});

  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      home: Scaffold(
        body: CustomScrollView(
          slivers: [
            const SliverAppBar(
              title: Text('SliverGrid 示例'),
              backgroundColor: Colors.deepPurple,
              foregroundColor: Colors.white,
            ),

            // 标题区域
            const SliverToBoxAdapter(
              child: Padding(
                padding: EdgeInsets.all(16),
                child: Text(
                  '热门推荐',
                  style: TextStyle(fontSize: 20, fontWeight: FontWeight.bold),
                ),
              ),
            ),

            // 网格区域
            SliverGrid(
              gridDelegate: const SliverGridDelegateWithMaxCrossAxisExtent(
                maxCrossAxisExtent: 200,
                mainAxisSpacing: 10,
                crossAxisSpacing: 10,
                childAspectRatio: 3 / 4,
              ),
              delegate: SliverChildBuilderDelegate(
                (context, index) {
                  return Card(
                    margin: EdgeInsets.only(
                      left: index % 2 == 0 ? 12 : 4,
                      right: index % 2 == 1 ? 12 : 4,
                    ),
                    clipBehavior: Clip.antiAlias,
                    child: Column(
                      crossAxisAlignment: CrossAxisAlignment.stretch,
                      children: [
                        Expanded(
                          flex: 3,
                          child: Container(
                            color: Colors.primaries[index % Colors.primaries.length]
                                .withValues(alpha: 0.3),
                            child: const Icon(Icons.image, size: 48),
                          ),
                        ),
                        Expanded(
                          flex: 1,
                          child: Padding(
                            padding: const EdgeInsets.all(8),
                            child: Column(

5. SliverAppBar：可折叠应用栏

SliverAppBar 是 Sliver 体系中最具视觉冲击力的组件。它能实现随滚动折叠/展开的应用栏效果，是社交应用个人主页、新闻详情页等场景的标配。

5.1 核心参数

参数 类型 默认值 说明
expandedHeight double? null 展开时的高度（包含 toolbar 高度）
flexibleSpace Widget? null 展开时显示的弹性空间内容，通常用 FlexibleSpaceBar
pinned bool false 折叠后是否固定在顶部（保留 toolbar）
floating bool false 向下滚动时是否立即显示
snap bool false 配合 floating，展开/收起时是否自动吸附到完整状态
stretch bool false 过度下拉时是否允许拉伸效果
collapsedHeight double? null 折叠后的高度
toolbarHeight double kToolbarHeight 工具栏部分的高度

几个关键布尔参数的交互效果：

pinned floating snap 效果描述
true false false 折叠后固定在顶部，只有滚动到顶部才会重新展开
false true false 向下滚动时立即开始展开，向上滚动时立即收起
false true true 向下滚动时自动完整展开（吸附效果），向上滚动时自动完整收起
true true false 折叠后固定，向下滚动时 toolbar 立即出现，继续下拉才完全展开

FlexibleSpaceBar 是 SliverAppBar.flexibleSpace 的标准搭配组件，提供以下效果：

  • title：折叠过程中标题会平滑缩小并移动到 toolbar 位置
  • background：展开时显示的背景内容（通常是图片）
  • collapseMode：折叠模式（parallax 视差效果、pin 固定、none 无效果）
  • stretchModes：过度拉伸时的效果（缩放背景图、模糊背景、渐变标题）

6. SliverToBoxAdapter：在 Sliver 中嵌入普通 Widget

SliverToBoxAdapter 是连接 Sliver 世界和普通 Box Widget 世界的桥梁。它将一个普通的 Widget 包装成 Sliver，使其能放入 CustomScrollView 的 slivers 列表中。

6.1 为什么需要它？

7. SliverPersistentHeader：自定义吸顶效果

SliverPersistentHeader 能在滚动时将一个 Widget 固定在视口顶部（"吸顶"效果），常用于分组标题、筛选栏等场景。它比 SliverAppBar 更灵活，因为内容完全由你自定义。

7.1 SliverPersistentHeaderDelegate

使用 SliverPersistentHeader 需要提供一个 SliverPersistentHeaderDelegate 子类。这个 delegate 定义了 header 的最大高度、最小高度以及如何构建 header 的 UI。

必须重写的四个方法/属性：

成员 说明
minExtent header 在完全折叠时的最小高度
maxExtent header 在完全展开时的最大高度
build(context, shrinkOffset, overlapsContent) 构建 header 的 UI，shrinkOffset 是当前的折叠偏移量
shouldRebuild(oldDelegate) 当 delegate 更新时是否需要重建 
import 'package:flutter/material.dart';

void main() => runApp(const MyApp());

class MyApp extends StatelessWidget {
  const MyApp({super.key});

  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      home: const PersistentHeaderDemo(),
    );
  }
}

/// 自定义 SliverPersistentHeaderDelegate
class _SectionHeaderDelegate extends SliverPersistentHeaderDelegate {
  final String title;
  final Color color;
  final double expandedHeight;
  final double collapsedHeight;

  _SectionHeaderDelegate({
    required this.title,
    required this.color,
    this.expandedHeight = 120.0,
    this.collapsedHeight = 50.0,
  });

  @override
  double get maxExtent => expandedHeight;

  @override
  double get minExtent => collapsedHeight;

  @override
  Widget build(
    BuildContext context,
    double shrinkOffset,
    bool overlapsContent,
  ) {
    // shrinkOffset: 从 0（完全展开）到 maxExtent - minExtent（完全折叠）
    // 计算折叠进度（0.0 = 展开，1.0 = 折叠）
    final progress = shrinkOffset / (maxExtent - minExtent);
    final clampedProgress = progress.clamp(0.0, 1.0);

    return Container(
      color: Color.lerp(color.withValues(alpha: 0.3), color, clampedProgress),
      alignment: Alignment.centerLeft,
      padding: const EdgeInsets.symmetric(horizontal: 16),
      child: Row(
        children: [
          Icon(
            Icons.label,
            // 图标大小随折叠进度从 32 缩小到 20
            size: 32 - 12 * clampedProgress,
            color: Colors.white,
          ),
          const SizedBox(width: 12),
          Text(
            title,
            style: TextStyle(
              color: Colors.white,
              // 字号随折叠进度从 24 缩小到 16
              fontSize: 24 - 8 * clampedProgress,
              fontWeight: FontWeight.bold,
            ),
          ),
        ],
      ),
    );
  }

  @override
  bool shouldRebuild(covariant _SectionHeaderDelegate oldDelegate) {
    return title != oldDelegate.title ||
        color != oldDelegate.color ||
        expandedHeight != oldDelegate.expandedHeight ||
        collapsedHeight != oldDelegate.collapsedHeight;
  }
}

7.2 pinned 与 floating 的区别

SliverPersistentHeader 也有 pinned 和 floating 参数：

  • pinned: true：header 折叠后固定在视口顶部，直到对应的分组内容全部滚出视口才被下一个 pinned header 顶走。这就是经典的分组吸顶效果。
  • floating: true：header 在向下滚动时立即出现（类似 SliverAppBar 的 floating 效果）。
  • 两者都为 false：header 跟随内容正常滚动，没有特殊效果。

最佳实践：

  • shouldRebuild 方法要认真实现。如果返回 true 过于频繁，会导致不必要的重建；如果返回 false 但数据实际变了，UI 不会更新。
  • shrinkOffset 的范围是 0 到 maxExtent - minExtent。利用它可以实现丰富的折叠动画效果（渐变颜色、缩放图标、位移文字等）。
  • 当多个 pinned: true 的 header 同时存在时，后面的 header 滚上来会自动"顶走"前面的 header，形成分组吸顶的连贯效果。

8. NestedScrollView：嵌套滚动协调

8.1 为什么需要 NestedScrollView？

在实际开发中，有一种非常常见的页面布局：顶部是一个可折叠的 AppBar 和一排 TabBar，下方是一个 TabBarView，每个 Tab 页面内各有一个可滚动的列表。

如果直接用 CustomScrollView + SliverAppBar + SliverToBoxAdapter(child: TabBarView(...))，你会遇到一个棘手的问题：TabBarView 内的列表和外层的 CustomScrollView 是两个独立的滚动视图，它们的滚动行为无法协调。具体表现为：

  • 手指在列表上滑动时，内层列表滚动了但外层 AppBar 不会折叠
  • 或者 AppBar 折叠了但内层列表不跟着滚动

NestedScrollView 正是为解决这个问题而设计的。它在内外两个滚动视图之间建立了协调机制，使它们能像一个整体一样流畅滚动。

8.2 核心参数

参数 类型 说明
headerSliverBuilder NestedScrollViewHeaderSliversBuilder 构建头部 Sliver 列表（AppBar、TabBar 等）
body Widget 主体内容（通常是 TabBarView 或可滚动组件）
scrollDirection Axis 滚动方向，默认垂直
controller ScrollController? 外层滚动控制器
floatHeaderSlivers bool 头部 Sliver 是否 float，默认 false

headerSliverBuilder 返回一个 Sliver 列表，这些 Sliver 构成了页面的头部区域（在 body 上方）。典型的组合是 SliverAppBar（包含 TabBar 在 bottom 中）。

body 是主体内容区域。body 内的可滚动组件（如 ListView、GridView）会与头部 Sliver 协调滚动。

8.3 经典场景：TabBar + 列表嵌套滚动

import 'package:flutter/material.dart';

void main() => runApp(const MyApp());

class MyApp extends StatelessWidget {
  const MyApp({super.key});

  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      theme: ThemeData(useMaterial3: true, colorSchemeSeed: Colors.indigo),
      home: const NestedScrollDemo(),
    );
  }
}

class NestedScrollDemo extends StatefulWidget {
  const NestedScrollDemo({super.key});

  @override
  State<NestedScrollDemo> createState() => _NestedScrollDemoState();
}

class _NestedScrollDemoState extends State<NestedScrollDemo>
    with TickerProviderStateMixin {
  late final TabController _tabController;

  final List<String> _tabs = ['推荐', '热门', '最新', '关注'];

  @override
  void initState() {
    super.initState();
    _tabController = TabController(length: _tabs.length, vsync: this);
  }

  @override
  void dispose() {
    _tabController.dispose();
    super.dispose();
  }

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      body: NestedScrollView(
        headerSliverBuilder: (BuildContext context, bool innerBoxIsScrolled) {
          // innerBoxIsScrolled: body 内的滚动视图是否已经开始滚动
          return [
            SliverAppBar(
              expandedHeight: 200.0,
              pinned: true,
              // forceElevated 确保在内层列表滚动时 AppBar 显示阴影
              forceElevated: innerBoxIsScrolled,
              flexibleSpace: FlexibleSpaceBar(
                title: const Text('我的主页'),
                background: Container(
                  decoration: const BoxDecoration(
                    gradient: LinearGradient(
                      begin: Alignment.topCenter,
                      end: Alignment.bottomCenter,
                      colors: [Color(0xFF3949AB), Color(0xFF1A237E)],
                    ),
                  ),
                  child: const Center(
                    child: Column(
                      mainAxisAlignment: MainAxisAlignment.center,
                      children: [
                        CircleAvatar(
                          radius: 36,
                          backgroundColor: Colors.white24,
                          child: Icon(Icons.person, size: 40, color: Colors.white),
                        ),
                        SizedBox(height: 8),
                        Text(
                          '用户昵称',
                          style: TextStyle(color: Colors.white70, fontSize: 14),
                        ),
                      ],
                    ),
                  ),
                ),
              ),
              // 将 TabBar 放在 AppBar 的 bottom 位置
              bottom: TabBar(
                controller: _tabController,
                tabs: _tabs.map((name) => Tab(text: name)).toList(),
                indicatorColor: Colors.white,
                labelColor: Colors.white,
                unselectedLabelColor: Colors.white60,
              ),
            ),
          ];
        },
        // body 中放 TabBarView
        body: TabBarView(
          controller: _tabController,
          children: _tabs.map((tabName) {
            return _buildTabContent(tabName);
          }).toList(),
        ),
      ),
    );
  }

  Widget _buildTabContent(String tabName) {
    // 每个 Tab 页的内容是一个可滚动的列表
    // NestedScrollView 会自动协调内外层的滚动
    return Builder(
      builder: (context) {
        return CustomScrollView(
          // 关键：使用 NestedScrollView 提供的 controller
          // 这是让内外层滚动协调的核心
          slivers: [
            SliverOverlapAbsorber(
              handle: NestedScrollView.sliverOverlapAbsorberHandleFor(context),
              sliver: const SliverToBoxAdapter(child: SizedBox.shrink()),
            ),
            SliverPadding(
              padding: const EdgeInsets.all(8.0),
              sliver: SliverList.builder(
                itemCount: 30,
                itemBuilder: (context, index) {
                  return Card(
                    margin: const EdgeInsets.symmetric(
                      horizontal: 8,
                      vertical: 4,
                    ),
                    child: ListTile(
                      leading: CircleAvatar(
                        backgroundColor:
                            Colors.primaries[index % Colors.primaries.length],
                        child: Text('${index + 1}'),
                      ),
                      title: Text('$tabName - 内容 ${index + 1}'),
                      subtitle: Text('这是「$tabName」标签下的第 ${index + 1} 条数据'),
                      trailing: const Icon(Icons.arrow_forward_ios, size: 14),
                    ),
                  );
                },
              ),
            ),
          ],
        );
      },
    );
  }
}

8.4 更简洁的写法

上面的写法使用了 SliverOverlapAbsorber 来处理重叠区域。在大多数简单场景中，如果 body 内直接使用 ListView 而不是 CustomScrollView，可以省去这些复杂处理：

import 'package:flutter/material.dart';

void main() => runApp(const MyApp());

class MyApp extends StatelessWidget {
  const MyApp({super.key});

  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      theme: ThemeData(useMaterial3: true, colorSchemeSeed: Colors.teal),
      home: const SimpleNestedScrollDemo(),
    );
  }
}

class SimpleNestedScrollDemo extends StatelessWidget {
  const SimpleNestedScrollDemo({super.key});

  @override
  Widget build(BuildContext context) {
    final tabs = ['全部', '图片', '视频'];

    return DefaultTabController(
      length: tabs.length,
      child: Scaffold(
        body: NestedScrollView(
          headerSliverBuilder: (context, innerBoxIsScrolled) {
            return [
              SliverAppBar(
                expandedHeight: 160,
                pinned: true,
                forceElevated: innerBoxIsScrolled,
                title: const Text('动态'),
                flexibleSpace: FlexibleSpaceBar(
                  background: Container(
                    color: Colors.teal.withValues(alpha: 0.2),
                    alignment: Alignment.center,
                    child: const Padding(
                      padding: EdgeInsets.only(top: 40),
                      child: Icon(Icons.dynamic_feed, size: 60, color: Colors.teal),
                    ),
                  ),
                ),
                bottom: TabBar(
                  tabs: tabs.map((t) => Tab(text: t)).toList(),
                ),
              ),
            ];
          },
          // 直接使用 TabBarView + ListView.builder
          body: TabBarView(
            children: tabs.map((tabName) {
              return ListView.builder(
                // 这里不需要手动指定 controller
                // NestedScrollView 会自动注入 PrimaryScrollController
                padding: const EdgeInsets.only(top: 8),
                itemCount: 25,
                itemBuilder: (context, index) {
                  return ListTile(
                    leading: Icon(
                      tabName == '图片'
                          ? Icons.photo
                          : tabName == '视频'
                              ? Icons.videocam
                              : Icons.article,
                    ),
                    title: Text('$tabName · 项目 ${index + 1}'),
                  );
                },
              );
            }).toList(),
          ),
        ),
      ),
    );
  }
}

注意事项：

  • NestedScrollView 的 body 内的可滚动组件不应该自己设置 primary: false 或手动指定 ScrollController，否则会破坏内外层的滚动协调。
  • headerSliverBuilder 的第二个参数 innerBoxIsScrolled 表示 body 内的滚动视图是否已经滚动过（偏移 > 0）。可以利用它来控制 SliverAppBar 的 forceElevated，在内容滚动时显示阴影分隔线。
  • 如果 TabBarView 中的每个页面需要保持状态（切换 Tab 后不丢失滚动位置），在对应的页面 Widget 中使用 AutomaticKeepAliveClientMixin。
  • NestedScrollView 目前只能处理一层嵌套。如果你需要多层嵌套滚动（极少见），可能需要自定义 ScrollPhysics 或使用第三方方案。

最佳实践：

  • 当页面结构为"可折叠头部 + TabBar + 多个列表"时，NestedScrollView 是最标准的解决方案。
  • 将 TabBar 放在 SliverAppBar.bottom 中，这样 TabBar 会在折叠后自然地固定在顶部。
  • 如果不需要 Tab 切换，只是想让一个列表和头部协调滚动，直接用 CustomScrollView + SliverAppBar + SliverList 更简单，不需要 NestedScrollView。

9. Sliver 体系整体梳理

学习了以上各个 Sliver 组件后，让我们用一张表格做一个整体梳理：

Sliver 组件 用途 常见搭配
SliverAppBar 可折叠应用栏 FlexibleSpaceBar、TabBar
SliverList 线性列表 SliverChildBuilderDelegate
SliverGrid 网格列表 SliverGridDelegate + SliverChildBuilderDelegate
SliverToBoxAdapter 嵌入单个普通 Widget 标题、Banner、分割线
SliverPersistentHeader 吸顶/浮动 header 自定义 SliverPersistentHeaderDelegate
SliverPadding 为 Sliver 添加内边距 包裹任何 Sliver
SliverFillRemaining 填充视口剩余空间 空状态页、底部区域
SliverFixedExtentList 固定高度的列表（比 SliverList 更高效） 所有子项等高的场景

选择指南：

  • 只有一种内容类型的简单滚动 → ListView 或 GridView
  • 需要混合多种布局（列表 + 网格 + 头部） → CustomScrollView + Sliver 组合
  • 需要可折叠头部 + TabBar + 内部列表 → NestedScrollView

本课小结

本课是列表系列的进阶篇，内容量较大，我们来回顾核心知识点：

  1. GridView 三种构造方式：

    • GridView.count：固定列数，简单直观，适合列数不变的场景
    • GridView.extent：固定最大宽度，自动计算列数，适合响应式布局
    • GridView.builder：懒加载，配合 gridDelegate，适合大量子项

  2. SliverGridDelegate：

    • SliverGridDelegateWithFixedCrossAxisCount：按固定列数排列
    • SliverGridDelegateWithMaxCrossAxisExtent：按最大宽度自动排列
    • mainAxisExtent 优先级高于 childAspectRatio

  3. Sliver 体系的核心思想：Sliver 是一种感知滚动环境的特殊组件，通过 SliverConstraints 和 SliverGeometry 进行布局，能在混合布局中保持高效的懒加载

  4. CustomScrollView：Sliver 组件的容器，通过 slivers 列表组合多种 Sliver

  5. 核心 Sliver 组件：

    • SliverList / SliverGrid：列表和网格的 Sliver 版本
    • SliverAppBar：可折叠应用栏，pinned、floating、snap 控制行为
    • SliverToBoxAdapter：普通 Widget 到 Sliver 的桥梁
    • SliverPersistentHeader：自定义吸顶效果

  6. NestedScrollView：解决"可折叠头部 + TabBarView + 内部列表"的滚动协调问题

课后练习

练习1：照片九宫格（基础）

使用 GridView.count 创建一个 3×3 的照片展示页面。要求：

  • 照片之间有 4px 的间距
  • 每张照片使用不同的颜色和数字占位
  • 点击某张照片时，在底部弹出 SnackBar 显示"你点击了照片 X"

练习2：响应式商品网格（中级）

使用 GridView.builder 和 SliverGridDelegateWithMaxCrossAxisExtent 创建一个商品展示页面。要求：

  • 每个商品卡片包含图片区域、名称和价格
  • 在手机竖屏（窄屏）下显示 2 列，横屏或平板（宽屏）下自动变为 3 列或更多
  • 使用 LayoutBuilder 获取屏幕宽度并展示在页面顶部

练习3：电商首页布局（中级）

使用 CustomScrollView 构建一个模拟电商首页，包含以下区域（从上到下）：

  • SliverAppBar：可折叠的顶部 Banner（展开高度 200）
  • SliverToBoxAdapter：搜索栏
  • SliverToBoxAdapter：横向滚动的分类图标
  • SliverPersistentHeader：一个吸顶的"热门商品"标题栏
  • SliverGrid：2 列的商品网格（至少 20 个商品）

练习4：分组吸顶列表（中级）

使用 CustomScrollView + 多组 SliverPersistentHeader + SliverList 实现一个通讯录风格的分组列表。要求：

  • 按字母 A-Z 分组（至少实现 5 个分组）
  • 每个分组标题在滚动时吸顶
  • 上方分组标题被下方新的分组标题"顶走"
  • 分组标题的样式随折叠进度产生颜色或大小变化

练习5：个人主页（进阶）

使用 NestedScrollView 实现一个完整的社交应用个人主页。要求：

  • 顶部 SliverAppBar：展开时显示用户头像、昵称、个人简介，折叠后只显示用户名
  • SliverAppBar.bottom 放置 3 个 Tab（动态、相册、收藏）
  • "动态" Tab：ListView 展示动态列表
  • "相册" Tab：GridView 展示照片网格
  • "收藏" Tab：ListView 展示收藏内容
  • 三个 Tab 的滚动与顶部折叠栏协调一致
  • 切换 Tab 后列表位置不重置（使用 AutomaticKeepAliveClientMixin）
