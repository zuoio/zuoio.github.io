1. SingleChildScrollView：单子组件滚动

1.1 基本概念

SingleChildScrollView 是 Flutter 中最简单的可滚动组件。顾名思义，它只能包含一个子组件，当子组件的尺寸超过可视区域时，提供滚动能力。

它的定位类似于 Android 中的 ScrollView 或 Web 中设置了 overflow: auto 的容器。

核心特征：

  • 只接受一个 child
  • 子组件会被一次性全部渲染，没有懒加载机制
  • 适合内容量较少但可能超出屏幕的场景（如表单页面、设置页面）

1.2 常用属性

属性 类型 说明
scrollDirection Axis 滚动方向，默认 Axis.vertical（垂直滚动），可设为 Axis.horizontal（水平滚动）
reverse bool 是否反转滚动方向，默认 false。设为 true 时，内容从底部开始排列
padding EdgeInsetsGeometry? 内容区域的内边距
physics ScrollPhysics? 滚动物理效果（如弹性效果、夹紧效果等），详见第6节
controller ScrollController? 滚动控制器，用于监听滚动事件和编程式滚动，详见第5节
primary bool? 是否使用 PrimaryScrollController。当为 true 时，不需要显式传 controller
clipBehavior Clip 内容裁剪行为，默认 Clip.hardEdge

2. ListView：四种构造方式总览

ListView 是 Flutter 中最常用的可滚动列表组件，几乎每个 Flutter 应用都会用到它。它提供了四种不同的构造方式，每种适用于不同的场景。

2.1 四种构造方式概览

构造方式 子组件创建时机 适用场景 性能
ListView(children: [...]) 一次性全部创建 子组件数量少且固定（< 20 个） ⭐⭐
ListView.builder() 按需懒加载创建 长列表、动态列表、数据源数量大 ⭐⭐⭐⭐⭐
ListView.separated() 按需懒加载创建 + 分割线 需要分割线的长列表 ⭐⭐⭐⭐⭐
ListView.custom() 完全自定义 需要自定义子组件管理策略 ⭐⭐⭐⭐⭐

2.2 性能差异的本质原因

理解四种构造方式性能差异的关键在于 Sliver 懒加载协议：

  • 默认构造 ListView(children: [])：内部使用 SliverChildListDelegate，在构建时会创建所有子 Widget。即使某个列表项在屏幕之外不可见，它的 Widget 也已经被创建了（虽然不一定会布局和绘制）。
  • ListView.builder() 和 ListView.separated()：内部使用 SliverChildBuilderDelegate，只有当列表项即将进入可视区域时，才会调用 itemBuilder 创建对应的 Widget。滚出可视区域的列表项会被销毁回收。

这意味着，对于一个 10,000 条数据的列表：

  • 默认构造会在初始化时创建 10,000 个 Widget 对象 → 内存暴涨、构建缓慢
  • ListView.builder 只会创建可视区域内的约 10~20 个 Widget → 内存平稳、构建迅速

3.2 适用场景

默认构造适合以下场景：

  • 列表项数量少且固定（通常不超过 20 个）
  • 列表项类型各不相同（如设置页面中混合了开关、按钮、文本等不同类型的组件）
  • 不需要动态生成列表项

最佳实践：如果你发现自己在默认构造的 children 中使用了 List.generate() 或 .map().toList() 来生成大量列表项，那说明你应该用 ListView.builder 替代。

4. ListView.builder：懒加载长列表

4.1 核心概念

ListView.builder 是开发中最常用的 ListView 构造方式。它实现了按需创建（懒加载）的机制：只有即将进入屏幕可视区域的列表项才会被创建，滚出屏幕的列表项会被回收销毁。

4.2 关键参数

参数 类型 说明
itemBuilder Widget Function(BuildContext, int) 必填。根据索引构建列表项的回调函数
itemCount int? 列表项总数。不传则为无限列表
itemExtent double? 每个列表项的固定高度（垂直列表）或宽度（水平列表）
prototypeItem Widget? 原型列表项，框架会测量它的尺寸，作为所有列表项的固定尺寸
padding EdgeInsetsGeometry? 列表内边距
scrollDirection Axis 滚动方向
reverse bool 是否反转
shrinkWrap bool 是否根据内容收缩尺寸，默认 false
physics ScrollPhysics? 滚动物理效果
controller ScrollController? 滚动控制器
cacheExtent double? 预渲染区域的大小（像素），默认 250.0

4.4 itemExtent 与 prototypeItem 性能优化

当所有列表项的高度一致时，通过 itemExtent 或 prototypeItem 指定固定高度，可以显著提升性能。

原因：如果不指定固定高度，ListView 需要对每个子组件进行布局计算以确定其高度，才能知道滚动范围。指定固定高度后，ListView 可以直接通过数学计算得到每个列表项的位置和总滚动范围，跳过布局计算。

import 'package:flutter/material.dart';

void main() {
  runApp(const ItemExtentDemoApp());
}

class ItemExtentDemoApp extends StatelessWidget {
  const ItemExtentDemoApp({super.key});

  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      theme: ThemeData(colorSchemeSeed: Colors.teal, useMaterial3: true),
      home: const ItemExtentDemoPage(),
    );
  }
}

class ItemExtentDemoPage extends StatelessWidget {
  const ItemExtentDemoPage({super.key});

  @override
  Widget build(BuildContext context) {
    return DefaultTabController(
      length: 3,
      child: Scaffold(
        appBar: AppBar(
          title: const Text('itemExtent 性能优化'),
          bottom: const TabBar(
            tabs: [
              Tab(text: '无固定高度'),
              Tab(text: 'itemExtent'),
              Tab(text: 'prototypeItem'),
            ],
          ),
        ),
        body: TabBarView(
          children: [
            // 方式1：不指定固定高度 —— 每个列表项需要单独布局计算
            ListView.builder(
              itemCount: 10000,
              itemBuilder: (context, index) => _buildItem(index),
            ),
            // 方式2：使用 itemExtent 指定固定高度（像素值）
            // 适合确切知道列表项高度的场景
            ListView.builder(
              itemCount: 10000,
              itemExtent: 72.0, // 每个列表项固定高度 72 像素
              itemBuilder: (context, index) => _buildItem(index),
            ),
            // 方式3：使用 prototypeItem 提供原型
            // 框架会测量原型的高度，作为所有列表项的高度
            // 适合不确定确切像素值但能提供一个"样本"的场景
            ListView.builder(
              itemCount: 10000,
              prototypeItem: _buildItem(0), // 提供一个原型列表项
              itemBuilder: (context, index) => _buildItem(index),
            ),
          ],
        ),
      ),
    );
  }

  static Widget _buildItem(int index) {
    return ListTile(
      leading: CircleAvatar(child: Text('${index + 1}')),
      title: Text('列表项 ${index + 1}'),
      subtitle: Text('这是第 ${index + 1} 个列表项的描述文字'),
    );
  }
}

最佳实践：

  1. 当列表项高度一致时，优先使用 itemExtent（已知像素值）或 prototypeItem（提供样本），尤其是长列表（超过 100 项）。
  2. itemExtent 和 prototypeItem 不能同时使用，二者互斥。
  3. 使用 itemExtent 后，jumpTo 和 animateTo 等滚动定位操作也会更精确、更快速。

4.5 无限列表（不传 itemCount）

当 itemCount 为 null 时，ListView.builder 会创建一个无限列表，itemBuilder 会被无限调用。常用于"加载更多"场景。

import 'package:flutter/material.dart';

void main() {
  runApp(const InfiniteListDemoApp());
}

class InfiniteListDemoApp extends StatelessWidget {
  const InfiniteListDemoApp({super.key});

  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      theme: ThemeData(colorSchemeSeed: Colors.orange, useMaterial3: true),
      home: const InfiniteListDemoPage(),
    );
  }
}

class InfiniteListDemoPage extends StatefulWidget {
  const InfiniteListDemoPage({super.key});

  @override
  State<InfiniteListDemoPage> createState() => _InfiniteListDemoPageState();
}

class _InfiniteListDemoPageState extends State<InfiniteListDemoPage> {
  final List<String> _items = List.generate(20, (i) => '初始项 ${i + 1}');
  bool _isLoading = false;

  Future<void> _loadMore() async {
    if (_isLoading) return;
    setState(() => _isLoading = true);

    // 模拟网络请求延迟
    await Future.delayed(const Duration(seconds: 1));

    setState(() {
      final currentLength = _items.length;
      _items.addAll(
        List.generate(20, (i) => '加载项 ${currentLength + i + 1}'),
      );
      _isLoading = false;
    });
  }

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(title: Text('无限列表（已加载 ${_items.length} 项）')),
      body: ListView.builder(
        // itemCount 多加 1，用于显示底部的"加载中"指示器
        itemCount: _items.length + 1,
        itemBuilder: (context, index) {
          // 最后一个位置显示加载指示器
          if (index == _items.length) {
            // 触发加载更多
            _loadMore();
            return const Padding(
              padding: EdgeInsets.all(16),
              child: Center(child: CircularProgressIndicator()),
            );
          }
          return ListTile(
            leading: CircleAvatar(child: Text('${index + 1}')),
            title: Text(_items[index]),
          );
        },
      ),
    );
  }
}

5. ListView.separated：带分割线的列表

5.1 核心概念

ListView.separated 在 ListView.builder 的基础上，额外提供了一个 separatorBuilder 参数，用于在相邻列表项之间插入分割组件。它同样具备懒加载能力。

与手动在 itemBuilder 中添加 Divider 相比的优势：

  • 分割线逻辑与列表项逻辑分离，代码更清晰
  • 可以轻松实现不同位置使用不同样式的分割线
  • 分割线不占用 itemCount 的索引

5.2 关键参数

参数 类型 说明
itemBuilder Widget Function(BuildContext, int) 构建列表项的回调
separatorBuilder Widget Function(BuildContext, int) 构建分割线的回调，index 是分割线上方列表项的索引
itemCount int 必填。列表项总数（不含分割线）

7. ScrollController：滚动监听与编程式滚动

7.1 基本概念

ScrollController 是 Flutter 滚动体系的核心控制器，它提供两大能力：

  1. 滚动监听：通过 addListener 监听滚动位置变化
  2. 编程式滚动：通过 animateTo、jumpTo 等方法控制滚动位置

7.2 常用属性与方法

属性/方法 说明
offset 当前滚动偏移量（像素）
position ScrollPosition 对象，包含更详细的滚动信息
position.maxScrollExtent 最大可滚动距离
position.minScrollExtent 最小可滚动距离（通常为 0）
position.pixels 等同于 offset
position.viewportDimension 可视区域的尺寸
addListener(callback) 添加滚动监听回调
removeListener(callback) 移除滚动监听回调
animateTo(offset, ...) 动画滚动到指定位置
jumpTo(offset) 瞬间跳转到指定位置（无动画）
dispose() 释放资源，必须在 State.dispose 中调用

7.3 实战：回到顶部按钮 + 滚动监听

这是一个非常实用的示例，展示了 ScrollController 的典型用法：当用户向下滚动超过一定距离时，显示"回到顶部"按钮。

import 'package:flutter/material.dart';

void main() {
  runApp(const ScrollControllerDemoApp());
}

class ScrollControllerDemoApp extends StatelessWidget {
  const ScrollControllerDemoApp({super.key});

  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      theme: ThemeData(colorSchemeSeed: Colors.blue, useMaterial3: true),
      home: const ScrollControllerDemoPage(),
    );
  }
}

class ScrollControllerDemoPage extends StatefulWidget {
  const ScrollControllerDemoPage({super.key});

  @override
  State<ScrollControllerDemoPage> createState() =>
      _ScrollControllerDemoPageState();
}

class _ScrollControllerDemoPageState extends State<ScrollControllerDemoPage> {
  // 创建 ScrollController
  final ScrollController _scrollController = ScrollController();

  // 是否显示"回到顶部"按钮
  bool _showBackToTop = false;
  // 当前滚动位置信息（用于界面展示）
  double _scrollOffset = 0;
  double _maxScrollExtent = 0;
  double _scrollPercent = 0;

  @override
  void initState() {
    super.initState();
    // 添加滚动监听
    _scrollController.addListener(_onScroll);
  }

  void _onScroll() {
    final offset = _scrollController.offset;
    final maxExtent = _scrollController.position.maxScrollExtent;

    setState(() {
      _scrollOffset = offset;
      _maxScrollExtent = maxExtent;
      _scrollPercent = maxExtent > 0 ? (offset / maxExtent * 100) : 0;
      // 滚动超过 300 像素时显示回到顶部按钮
      _showBackToTop = offset > 300;
    });
  }

  // 动画滚动到顶部
  void _scrollToTop() {
    _scrollController.animateTo(
      0, // 目标位置：顶部
      duration: const Duration(milliseconds: 500), // 动画时长
      curve: Curves.easeInOut, // 动画曲线
    );
  }

  // 动画滚动到底部
  void _scrollToBottom() {
    _scrollController.animateTo(
      _scrollController.position.maxScrollExtent, // 目标位置：底部
      duration: const Duration(milliseconds: 500),
      curve: Curves.easeInOut,
    );
  }

  // 跳转到指定位置（无动画）
  void _jumpToMiddle() {
    _scrollController.jumpTo(
      _scrollController.position.maxScrollExtent / 2,
    );
  }

  @override
  void dispose() {
    // 必须释放 ScrollController，防止内存泄漏
    _scrollController.dispose();
    super.dispose();
  }

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(
        title: const Text('ScrollController 演示'),
        actions: [
          // 跳到中间按钮
          IconButton(
            onPressed: _jumpToMiddle,
            icon: const Icon(Icons.vertical_align_center),
            tooltip: '跳到中间',
          ),
          // 滚动到底部按钮
          IconButton(
            onPressed: _scrollToBottom,
            icon: const Icon(Icons.vertical_align_bottom),
            tooltip: '滚到底部',
          ),
        ],
      ),
      body: Column(
        children: [
          // 滚动状态信息面板
          Container(
            padding: const EdgeInsets.all(12),
            color: Theme.of(context).colorScheme.surfaceContainerHighest,
            child: Row(
              mainAxisAlignment: MainAxisAlignment.spaceAround,
              children: [
                _buildInfoChip('偏移', '${_scrollOffset.toStringAsFixed(0)}px'),
                _buildInfoChip('最大', '${_maxScrollExtent.toStringAsFixed(0)}px'),
                _buildInfoChip('进度', '${_scrollPercent.toStringAsFixed(1)}%'),
              ],
            ),
          ),
          // 列表区域
          Expanded(
            child: ListView.builder(
              controller: _scrollController, // 绑定控制器
              itemCount: 100,
              itemBuilder: (context, index) {
                return ListTile(
                  leading: CircleAvatar(
                    backgroundColor:
                        Colors.primaries[index % Colors.primaries.length],
                    child: Text('${index + 1}',
                        style: const TextStyle(color: Colors.white, fontSize: 12)),
                  ),
                  title: Text('列表项 ${index + 1}'),
                  subtitle: Text('向下滚动查看更多内容 (index: $index)'),
                );
              },
            ),
          ),
        ],
      ),
      // 回到顶部的浮动按钮
      floatingActionButton: _showBackToTop
          ? FloatingActionButton(
              onPressed: _scrollToTop,
              tooltip: '回到顶部',
              child: const Icon(Icons.arrow_upward),
            )
          : null,
    );
  }

  Widget _buildInfoChip(String label, String value) {
    return Column(
      children: [
        Text(label, style: const TextStyle(fontSize: 11, color: Colors.grey)),
        const SizedBox(height: 2),
        Text(value,
            style: const TextStyle(fontSize: 13, fontWeight: FontWeight.bold)),
      ],
    );
  }
}

7.4 使用 NotificationListener 监听滚动

除了 ScrollController.addListener，还可以使用 NotificationListener<ScrollNotification> 来监听滚动事件。这种方式不需要创建 ScrollController，且能获取更丰富的滚动状态信息。

import 'package:flutter/material.dart';

void main() {
  runApp(const NotificationListenerDemoApp());
}

class NotificationListenerDemoApp extends StatelessWidget {
  const NotificationListenerDemoApp({super.key});

  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      theme: ThemeData(colorSchemeSeed: Colors.purple, useMaterial3: true),
      home: const NotificationListenerDemoPage(),
    );
  }
}

class NotificationListenerDemoPage extends StatefulWidget {
  const NotificationListenerDemoPage({super.key});

  @override
  State<NotificationListenerDemoPage> createState() =>
      _NotificationListenerDemoPageState();
}

class _NotificationListenerDemoPageState
    extends State<NotificationListenerDemoPage> {
  String _scrollStatus = '未开始滚动';
  double _progress = 0;

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(title: const Text('NotificationListener 监听滚动')),
      body: Column(
        children: [
          // 滚动状态显示
          Container(
            padding: const EdgeInsets.all(16),
            color: Theme.of(context).colorScheme.surfaceContainerHighest,
            child: Column(
              children: [
                Text('状态: $_scrollStatus',
                    style: const TextStyle(fontSize: 16)),
                const SizedBox(height: 8),
                LinearProgressIndicator(value: _progress),
              ],
            ),
          ),
          // 使用 NotificationListener 包裹可滚动组件
          Expanded(
            child: NotificationListener<ScrollNotification>(
              onNotification: (ScrollNotification notification) {
                // 滚动开始
                if (notification is ScrollStartNotification) {
                  setState(() => _scrollStatus = '开始滚动 ▶');
                }
                // 滚动中（持续触发）
                else if (notification is ScrollUpdateNotification) {
                  final metrics = notification.metrics;
                  setState(() {
                    _scrollStatus =
                        '滚动中 ↕ offset: ${metrics.pixels.toStringAsFixed(0)}';
                    _progress = metrics.maxScrollExtent > 0
                        ? metrics.pixels / metrics.maxScrollExtent
                        : 0;
                  });
                }
                // 滚动结束
                else if (notification is ScrollEndNotification) {
                  setState(() => _scrollStatus = '滚动结束 ■');
                }
                // 滚动到边界（过度滚动）
                else if (notification is OverscrollNotification) {
                  setState(
                      () => _scrollStatus = '到达边界！ overscroll: ${notification.overscroll.toStringAsFixed(1)}');
                }
                // 返回 false 表示不拦截通知，继续向上冒泡
                return false;
              },
              child: ListView.builder(
                itemCount: 50,
                itemBuilder: (context, index) {
                  return ListTile(
                    leading: CircleAvatar(child: Text('${index + 1}')),
                    title: Text('列表项 ${index + 1}'),
                  );
                },
              ),
            ),
          ),
        ],
      ),
    );
  }
}

注意事项：

  1. ScrollController 在使用完毕后必须调用 dispose()，否则会导致内存泄漏。
  2. 一个 ScrollController 只能同时绑定一个可滚动组件。如果需要控制多个列表，请创建多个 ScrollController。
  3. 在 addListener 的回调中调用 setState 时要注意频率，每次滚动都会触发，可能导致过度重建。考虑使用 NotificationListener 代替，或者在 addListener 回调中只更新必要的状态。

8. ScrollPhysics：滚动物理效果

8.1 基本概念

ScrollPhysics 定义了可滚动组件的滚动行为，包括：到达边界时如何表现、滚动惯性如何衰减、是否允许滚动等。Flutter 提供了多种内置的 ScrollPhysics 实现。

8.2 内置 ScrollPhysics 类型

Physics 类型 效果 典型平台
BouncingScrollPhysics 到达边界时产生弹性回弹效果 iOS 默认行为
ClampingScrollPhysics 到达边界时产生**发光（微光）**效果，不回弹 Android 默认行为
NeverScrollableScrollPhysics 完全禁止滚动，用户无法通过手势滚动 嵌套滚动场景
AlwaysScrollableScrollPhysics 即使内容不足以填满视口也允许滚动 下拉刷新场景
PageScrollPhysics 页面吸附效果，滚动结束后自动对齐到最近的页面 PageView
FixedExtentScrollPhysics 固定尺寸项的吸附效果 ListWheelScrollView

              // 让 ListView 根据内容高度自适应，而非占满父容器
              shrinkWrap: true,
              itemCount: 15,
              itemBuilder: (context, index) {
                return Card(
                  child: ListTile(
                    leading: CircleAvatar(child: Text('${index + 1}')),
                    title: Text('嵌套列表项 ${index + 1}'),
                  ),
                );
              },
            ),
            const SizedBox(height: 16),
            const Text('底部区域', style: TextStyle(fontSize: 20, fontWeight: FontWeight.bold)),
            Container(
              height: 100,
              decoration: BoxDecoration(
                color: Colors.green.withOpacity(0.1),
                borderRadius: BorderRadius.circular(12),
              ),
              child: const Center(child: Text('页脚内容')),
            ),
          ],
        ),
      ),
    );
  }
}

注意事项：shrinkWrap: true 会让 ListView 一次性计算所有子组件的高度来确定自身高度，丧失懒加载优势。如果嵌套列表项很多（超过几十个），应该考虑使用 CustomScrollView + SliverList 代替，这将在下一课介绍。

9. RefreshIndicator：下拉刷新

9.1 基本概念

RefreshIndicator 是 Material Design 风格的下拉刷新组件。它包裹一个可滚动组件，当用户从列表顶部下拉时，显示一个旋转的进度指示器，触发数据刷新。

关键要求：

  • 子组件必须是一个可滚动组件（如 ListView、GridView、CustomScrollView）
  • onRefresh 回调必须返回一个 Future，RefreshIndicator 会在 Future 完成后自动隐藏指示器
  • 可滚动组件必须能够过度滚动（即不能使用 NeverScrollableScrollPhysics）

9.2 代码示例

import 'package:flutter/material.dart';
import 'dart:math';

void main() {
  runApp(const RefreshIndicatorDemoApp());
}

class RefreshIndicatorDemoApp extends StatelessWidget {
  const RefreshIndicatorDemoApp({super.key});

  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      theme: ThemeData(colorSchemeSeed: Colors.green, useMaterial3: true),
      home: const RefreshIndicatorDemoPage(),
    );
  }
}

class RefreshIndicatorDemoPage extends StatefulWidget {
  const RefreshIndicatorDemoPage({super.key});

  @override
  State<RefreshIndicatorDemoPage> createState() =>
      _RefreshIndicatorDemoPageState();
}

class _RefreshIndicatorDemoPageState extends State<RefreshIndicatorDemoPage> {
  final Random _random = Random();
  List<Map<String, dynamic>> _items = [];
  int _refreshCount = 0;

  @override
  void initState() {
    super.initState();
    _generateItems();
  }

  void _generateItems() {
    _items = List.generate(20, (index) {
      return {
        'title': '新闻 #${_random.nextInt(10000)}',
        'views': _random.nextInt(9999),
        'color': Colors.primaries[_random.nextInt(Colors.primaries.length)],
      };
    });
  }

  // onRefresh 必须返回 Future
  Future<void> _onRefresh() async {
    // 模拟网络请求延迟
    await Future.delayed(const Duration(seconds: 2));

    setState(() {
      _refreshCount++;
      _generateItems();
    });
  }

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(
        title: const Text('下拉刷新'),
        actions: [
          Center(
            child: Padding(
              padding: const EdgeInsets.only(right: 16),
              child: Text('已刷新 $_refreshCount 次'),
            ),
          ),
        ],
      ),
      body: RefreshIndicator(
        // 核心回调：必须返回 Future
        onRefresh: _onRefresh,
        // 指示器颜色
        color: Colors.green,
        // 指示器背景色
        backgroundColor: Colors.white,
        // 触发刷新所需的下拉距离
        displacement: 40,
        // 指示器到达顶部的边距
        edgeOffset: 0,
        // 描边宽度
        strokeWidth: 3,
        child: ListView.builder(
          // AlwaysScrollableScrollPhysics 确保即使内容不足也能下拉
          physics: const AlwaysScrollableScrollPhysics(),
          itemCount: _items.length,
          itemBuilder: (context, index) {
            final item = _items[index];
            return Card(
              margin: const EdgeInsets.symmetric(horizontal: 12, vertical: 4),
              child: ListTile(
                leading: CircleAvatar(
                  backgroundColor: item['color'] as Color,
                  child: const Icon(Icons.article, color: Colors.white),
                ),
                title: Text(item['title'] as String),
                subtitle: Text('浏览量: ${item['views']}'),
                trailing: const Icon(Icons.chevron_right),
              ),
            );
          },
        ),
      ),
    );
  }
}

最佳实践：

  1. onRefresh 回调中应包含实际的数据刷新逻辑（如网络请求），并在数据到达后再 setState。
  2. 配合 AlwaysScrollableScrollPhysics 使用，确保列表内容不足一屏时也能触发下拉刷新。
  3. onRefresh 返回的 Future 完成后指示器才会自动收起，所以确保 Future 最终会完成（加上超时机制）。
  4. 如果不希望使用 Material 风格的刷新指示器，可以使用第三方包如 pull_to_refresh 来自定义样式。

10. Scrollbar：滚动条显示

10.1 基本概念

Scrollbar 组件为可滚动子组件添加一个平台自适应的滚动条。在桌面端和 Web 上，滚动条默认始终可见且可交互（拖拽）；在移动端，滚动条通常只在滚动时短暂出现。

10.2 常用属性

属性 类型 说明
thumbVisibility bool? 滚动条是否始终可见。true 为始终可见，false（默认）为自动隐藏
thickness double? 滚动条粗细（像素）
radius Radius? 滚动条圆角
trackVisibility bool? 滚动条轨道是否可见
controller ScrollController? 关联的滚动控制器（当子组件有多个可滚动区域时需要显式指定）
interactive bool? 是否可交互（拖拽滚动条）
scrollbarOrientation ScrollbarOrientation? 滚动条位置（左/右/上/下）

10.4 通过主题全局配置 Scrollbar

可以通过 ScrollbarThemeData 在 ThemeData 中全局配置 Scrollbar 样式，避免每处单独设置。

import 'package:flutter/material.dart';

void main() {
  runApp(const ScrollbarThemeDemoApp());
}

class ScrollbarThemeDemoApp extends StatelessWidget {
  const ScrollbarThemeDemoApp({super.key});

  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      theme: ThemeData(
        colorSchemeSeed: Colors.indigo,
        useMaterial3: true,
        // 全局配置 Scrollbar 主题
        scrollbarTheme: ScrollbarThemeData(
          thumbVisibility: WidgetStateProperty.all(true),
          thickness: WidgetStateProperty.all(6),
          radius: const Radius.circular(3),
          thumbColor: WidgetStateProperty.resolveWith((states) {
            if (states.contains(WidgetState.dragged)) {
              return Colors.indigo; // 拖拽时的颜色
            }
            if (states.contains(WidgetState.hovered)) {
              return Colors.indigo.withOpacity(0.8); // 悬停时的颜色
            }
            return Colors.indigo.withOpacity(0.5); // 默认颜色
          }),
        ),
      ),
      home: Scaffold(
        appBar: AppBar(title: const Text('Scrollbar 全局主题')),
        // Scrollbar 会自动应用全局主题
        body: Scrollbar(
          child: ListView.builder(
            itemCount: 100,
            itemBuilder: (context, index) {
              return ListTile(
                leading: CircleAvatar(child: Text('${index + 1}')),
                title: Text('全局主题列表项 ${index + 1}'),
              );
            },
          ),
        ),
      ),
    );
  }
}

注意事项：当 Scrollbar 的 thumbVisibility 为 true 且子组件有自己的 ScrollController 时，Scrollbar 必须也绑定相同的 ScrollController，否则会报错。

11. 综合实战：联系人列表应用

下面的示例综合运用本课所学知识，实现一个功能完整的联系人列表：包含下拉刷新、滚动条、回到顶部按钮和滚动监听。

import 'package:flutter/material.dart';
import 'dart:math';

void main() {
  runApp(const ContactListApp());
}

class ContactListApp extends StatelessWidget {
  const ContactListApp({super.key});

  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      debugShowCheckedModeBanner: false,
      theme: ThemeData(
        colorSchemeSeed: Colors.blue,
        useMaterial3: true,
      ),
      home: const ContactListPage(),
    );
  }
}

class ContactListPage extends StatefulWidget {
  const ContactListPage({super.key});

  @override
  State<ContactListPage> createState() => _ContactListPageState();
}

class _ContactListPageState extends State<ContactListPage> {
  final ScrollController _scrollController = ScrollController();
  final Random _random = Random();

  List<Map<String, dynamic>> _contacts = [];
  bool _showBackToTop = false;
  bool _isLoadingMore = false;

  @override
  void initState() {
    super.initState();
    _generateContacts(30);
    _scrollController.addListener(_onScroll);
  }

  void _generateContacts(int count) {
    final names = ['张伟', '王芳', '李明', '赵静', '刘洋', '陈磊', '杨帆',
      '黄丽', '周涛', '吴强', '孙倩', '马超', '朱婷', '胡杰', '林琳'];
    _contacts = List.generate(count, (index) {
      final name = names[_random.nextInt(names.length)];
      return {
        'id': index,
        'name': '$name${_random.nextInt(100)}',
        'phone': '1${_random.nextInt(9) + 3}${List.generate(9, (_) => _random.nextInt(10)).join()}',
        'isOnline': _random.nextBool(),
      };
    });
  }

  void _onScroll() {
    final offset = _scrollController.offset;
    final maxExtent = _scrollController.position.maxScrollExtent;

    // 控制回到顶部按钮的显示
    if (offset > 500 && !_showBackToTop) {
      setState(() => _showBackToTop = true);
    } else if (offset <= 500 && _showBackToTop) {
      setState(() => _showBackToTop = false);
    }

    // 滚动到接近底部时触发加载更多
    if (offset >= maxExtent - 200 && !_isLoadingMore) {
      _loadMore();
    }
  }

  Future<void> _onRefresh() async {
    await Future.delayed(const Duration(seconds: 1));
    setState(() {
      _generateContacts(30);
    });
  }

  Future<void> _loadMore() async {
    setState(() => _isLoadingMore = true);
    await Future.delayed(const Duration(seconds: 1));
    if (!mounted) return;
    setState(() {
      final currentLength = _contacts.length;
      final names = ['张伟', '王芳', '李明', '赵静', '刘洋'];
      _contacts.addAll(List.generate(15, (index) {
        final name = names[_random.nextInt(names.length)];
        return {
          'id': currentLength + index,
          'name': '$name${_random.nextInt(100)}',
          'phone': '1${_random.nextInt(9) + 3}${List.generate(9, (_) => _random.nextInt(10)).join()}',
          'isOnline': _random.nextBool(),
        };
      }));
      _isLoadingMore = false;
    });
  }

  void _scrollToTop() {
    _scrollController.animateTo(
      0,
      duration: const Duration(milliseconds: 400),
      curve: Curves.easeOut,
    );
  }

  @override
  void dispose() {
    _scrollController.dispose();
    super.dispose();
  }

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(
        title: const Text('联系人'),
        actions: [
          Center(
            child: Padding(
              padding: const EdgeInsets.only(right: 16),
              child: Text('${_contacts.length} 人'),
            ),
          ),
        ],
      ),
      body: RefreshIndicator(
        onRefresh: _onRefresh,
        child: Scrollbar(
          controller: _scrollController,
          thumbVisibility: true,
          radius: const Radius.circular(4),
          child: ListView.separated(
            controller: _scrollController,
            physics: const AlwaysScrollableScrollPhysics(
              parent: BouncingScrollPhysics(),
            ),
            itemCount: _contacts.length + (_isLoadingMore ? 1 : 0),
            itemBuilder: (context, index) {
              // 底部加载指示器
              if (index == _contacts.length) {
                return const Padding(
                  padding: EdgeInsets.all(16),
                  child: Center(child: CircularProgressIndicator()),
                );
              }

              final contact = _contacts[index];
              final bool isOnline = contact['isOnline'] as bool;

              return ListTile(
                leading: Stack(
                  children: [
                    CircleAvatar(
                      backgroundColor: Colors
                          .primaries[index % Colors.primaries.length]
                          .withOpacity(0.7),
                      child: Text(
                        (contact['name'] as String).substring(0, 1),
                        style: const TextStyle(color: Colors.white),
                      ),
                    ),
                    // 在线状态指示点
                    Positioned(
                      right: 0,
                      bottom: 0,
                      child: Container(
                        width: 12,
                        height: 12,
                        decoration: BoxDecoration(
                          color: isOnline ? Colors.green : Colors.grey,
                          shape: BoxShape.circle,
                          border: Border.all(color: Colors.white, width: 2),
                        ),
                      ),
                    ),
                  ],
                ),
                title: Text(contact['name'] as String),
                subtitle: Text(contact['phone'] as String),
                trailing: Row(
                  mainAxisSize: MainAxisSize.min,
                  children: [
                    if (isOnline)
                      const Text('在线',
                          style: TextStyle(color: Colors.green, fontSize: 12)),
                    const SizedBox(width: 8),
                    const Icon(Icons.chevron_right, color: Colors.grey),
                  ],
                ),
              );
            },
            separatorBuilder: (context, index) {
              return const Divider(height: 1, indent: 72);
            },
          ),
        ),
      ),
      floatingActionButton: AnimatedOpacity(
        opacity: _showBackToTop ? 1.0 : 0.0,
        duration: const Duration(milliseconds: 200),
        child: _showBackToTop
            ? FloatingActionButton.small(
                onPressed: _scrollToTop,
                child: const Icon(Icons.arrow_upward),
              )
            : null,
      ),
    );
  }
}

本课小结

本课系统讲解了 Flutter 列表与滚动组件的基础知识，以下是核心要点：

  1. SingleChildScrollView：最简单的滚动容器，只包含一个子组件，适合内容量可控的页面（如表单、设置页）。不具备懒加载能力，不适用于长列表。

  2. ListView 四种构造方式：

    • 默认构造：一次性创建所有子组件，适合少量固定内容
    • builder：懒加载按需创建，长列表首选，95% 场景使用它
    • separated：在 builder 基础上增加分割线支持
    • custom：完全自定义子组件管理策略，高级场景使用

  3. 性能优化：使用 itemExtent 或 prototypeItem 指定固定列表项尺寸，跳过布局计算，显著提升长列表性能。

  4. ScrollController：提供滚动监听（addListener）和编程式滚动（animateTo、jumpTo）能力。使用后必须 dispose 释放资源。

  5. ScrollPhysics：控制滚动行为，常用的有 BouncingScrollPhysics（iOS 弹性）、ClampingScrollPhysics（Android 微光）、NeverScrollableScrollPhysics（禁止滚动）和 AlwaysScrollableScrollPhysics（始终可滚动）。

  6. RefreshIndicator：Material 风格下拉刷新，onRefresh 必须返回 Future，常配合 AlwaysScrollableScrollPhysics 使用。

  7. Scrollbar：为可滚动组件添加滚动条，通过 thumbVisibility、thickness、radius 等属性定制样式，可通过 ScrollbarThemeData 全局配置。
