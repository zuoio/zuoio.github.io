第05课 基础容器组件

本课是 Flutter 常用组件系统学习课程的第五课。我们将深入学习 Flutter 中最常用的基础容器组件，它们是构建 UI 布局的基石。掌握这些组件的用法和区别，将帮助你在实际开发中做出更合理的选择。

1. Container —— 最常用的"万能"容器

1.1 Container 的本质

Container 是 Flutter 中最常用的容器组件，但它并不是一个单一功能的 Widget。Container 本质上是一个"便捷组合组件"（convenience widget），它在内部根据你设置的属性，自动组合了多个单一功能组件。

当你使用 Container 时，Flutter 实际上会根据你设置的属性，在内部构建一棵类似下面的 Widget 树：

Container
  → Align（如果设置了 alignment）
    → Padding（如果设置了 padding）
      → DecoratedBox（如果设置了 decoration）
        → ConstrainedBox（如果设置了 constraints 或 width/height）
          → Transform（如果设置了 transform）
            → child

理解这一点非常重要：如果你只需要其中一个功能，直接使用对应的单一组件（如 Padding、Align、DecoratedBox 等）会更高效，因为可以减少 Widget 树的嵌套层级。

1.2 基本用法：width、height 与 color

最简单的 Container 用法是设置固定宽高和背景色。

import 'package:flutter/material.dart';

void main() => runApp(const MyApp());

class MyApp extends StatelessWidget {
  const MyApp({super.key});

  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      home: Scaffold(
        appBar: AppBar(title: const Text('Container 基本用法')),
        body: Center(
          child: Container(
            width: 200,    // 宽度 200 逻辑像素
            height: 150,   // 高度 150 逻辑像素
            color: Colors.blue, // 背景色
            child: const Center(
              child: Text(
                'Hello Container',
                style: TextStyle(color: Colors.white, fontSize: 18),
              ),
            ),
          ),
        ),
      ),
    );
  }
}

注意：color 属性和 decoration 属性不能同时使用。如果你需要在 decoration 中设置颜色，应使用 BoxDecoration(color: Colors.blue) 而不是同时设置 Container.color 和 Container.decoration。

1.3 alignment —— 子组件对齐

alignment 属性控制子组件在 Container 内部的对齐方式，类型为 AlignmentGeometry，最常用的是 Alignment 类。

Alignment 使用坐标系统：(-1, -1) 为左上角，(0, 0) 为正中心，(1, 1) 为右下角。

import 'package:flutter/material.dart';

void main() => runApp(const MyApp());

class MyApp extends StatelessWidget {
  const MyApp({super.key});

  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      home: Scaffold(
        appBar: AppBar(title: const Text('Container alignment')),
        body: Column(
          children: [
            // 居中对齐（默认无 alignment 时，子组件会占满 Container）
            Container(
              width: 200,
              height: 100,
              color: Colors.blue.shade100,
              alignment: Alignment.center,
              child: const Text('居中'),
            ),
            const SizedBox(height: 10),
            // 右下角对齐
            Container(
              width: 200,
              height: 100,
              color: Colors.green.shade100,
              alignment: Alignment.bottomRight,
              child: const Text('右下角'),
            ),
            const SizedBox(height: 10),
            // 自定义位置：水平偏左 30%，垂直偏上 50%
            Container(
              width: 200,
              height: 100,
              color: Colors.orange.shade100,
              alignment: const Alignment(-0.3, -0.5),
              child: const Text('自定义位置'),
            ),
          ],
        ),
      ),
    );
  }
}

常用的预定义对齐常量：

常量 说明
Alignment.topLeft 左上角 (-1, -1)
Alignment.topCenter 顶部居中 (0, -1)
Alignment.topRight 右上角 (1, -1)
Alignment.centerLeft 左侧居中 (-1, 0)
Alignment.center 正中心 (0, 0)
Alignment.centerRight 右侧居中 (1, 0)
Alignment.bottomLeft 左下角 (-1, 1)
Alignment.bottomCenter 底部居中 (0, 1)
Alignment.bottomRight 右下角 (1, 1)

1.4 padding 与 margin —— 内外间距

  • padding：Container 内部与子组件之间的间距（内边距）
  • margin：Container 外部与其他组件之间的间距（外边距）

两者都接收 EdgeInsetsGeometry 类型，最常用的是 EdgeInsets。

import 'package:flutter/material.dart';

void main() => runApp(const MyApp());

class MyApp extends StatelessWidget {
  const MyApp({super.key});

  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      home: Scaffold(
        appBar: AppBar(title: const Text('padding 与 margin')),
        body: Center(
          child: Container(
            // 外边距：上下左右各 20
            margin: const EdgeInsets.all(20),
            // 内边距：水平方向 16，垂直方向 8
            padding: const EdgeInsets.symmetric(
              horizontal: 16,
              vertical: 8,
            ),
            decoration: BoxDecoration(
              color: Colors.blue.shade100,
              border: Border.all(color: Colors.blue), // 添加边框便于观察
            ),
            child: const Text(
              '观察 padding 和 margin 的效果',
              style: TextStyle(fontSize: 16),
            ),
          ),
        ),
      ),
    );
  }
}

最佳实践：如果只需要设置内边距，直接使用 Padding 组件比用 Container 更轻量；如果只需要外边距，很多时候通过布局组件的 spacing 或手动添加 SizedBox 来控制间距更清晰。Container 的 margin 本质上就是在外部套了一层 Padding。

1.5 decoration —— BoxDecoration 装饰

decoration 是 Container 最强大的属性之一，通常传入 BoxDecoration 对象，可以实现边框、圆角、渐变、阴影、背景图等丰富的视觉效果。

import 'package:flutter/material.dart';

void main() => runApp(const MyApp());

class MyApp extends StatelessWidget {
  const MyApp({super.key});

  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      home: Scaffold(
        appBar: AppBar(title: const Text('Container decoration')),
        body: Center(
          child: Column(
            mainAxisAlignment: MainAxisAlignment.center,
            children: [
              // 圆角 + 边框 + 背景色
              Container(
                width: 200,
                height: 80,
                decoration: BoxDecoration(
                  color: Colors.white,
                  borderRadius: BorderRadius.circular(12), // 圆角半径 12
                  border: Border.all(
                    color: Colors.blue,
                    width: 2,
                  ),
                ),
                alignment: Alignment.center,
                child: const Text('圆角边框'),
              ),
              const SizedBox(height: 20),
              // 渐变背景 + 阴影
              Container(
                width: 200,
                height: 80,
                decoration: BoxDecoration(
                  gradient: const LinearGradient(
                    colors: [Colors.purple, Colors.blue],
                    begin: Alignment.topLeft,
                    end: Alignment.bottomRight,
                  ),
                  borderRadius: BorderRadius.circular(16),
                  boxShadow: [
                    BoxShadow(
                      color: Colors.blue.withValues(alpha: 0.4),
                      blurRadius: 12,    // 模糊半径
                      offset: const Offset(0, 6), // 阴影偏移
                    ),
                  ],
                ),
                alignment: Alignment.center,
                child: const Text(
                  '渐变 + 阴影',
                  style: TextStyle(color: Colors.white, fontWeight: FontWeight.bold),
                ),
              ),
              const SizedBox(height: 20),
              // 圆形容器
              Container(
                width: 100,
                height: 100,
                decoration: BoxDecoration(
                  color: Colors.orange,
                  shape: BoxShape.circle, // 圆形
                  boxShadow: [
                    BoxShadow(
                      color: Colors.orange.withValues(alpha: 0.5),
                      blurRadius: 10,
                      spreadRadius: 2, // 扩散半径
                    ),
                  ],
                ),
                alignment: Alignment.center,
                child: const Icon(Icons.star, color: Colors.white, size: 40),
              ),
            ],
          ),
        ),
      ),
    );
  }
}

注意：BoxDecoration 的 shape 属性设为 BoxShape.circle 时，不能同时设置 borderRadius，否则会抛出异常。圆形的 Container 的 width 和 height 应相同。

1.6 constraints —— 约束控制

constraints 属性接收一个 BoxConstraints 对象，用于限制 Container 的最小/最大宽高。

import 'package:flutter/material.dart';

void main() => runApp(const MyApp());

class MyApp extends StatelessWidget {
  const MyApp({super.key});

  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      home: Scaffold(
        appBar: AppBar(title: const Text('Container constraints')),
        body: Center(
          child: Container(
            // 最小宽度 100，最大宽度 300
            // 最小高度 50，最大高度 200
            constraints: const BoxConstraints(
              minWidth: 100,
              maxWidth: 300,
              minHeight: 50,
              maxHeight: 200,
            ),
            color: Colors.teal.shade100,
            alignment: Alignment.center,
            // 子组件很小，但 Container 至少有 100x50 的大小
            child: const Text('约束控制'),
          ),
        ),
      ),
    );
  }
}

BoxConstraints 常用构造方法：

  • BoxConstraints.tight(Size size)：宽高固定为指定值
  • BoxConstraints.loose(Size size)：宽高最大为指定值，最小为 0
  • BoxConstraints.expand()：尽可能占满父容器
  • BoxConstraints.tightFor(width: 100)：仅固定宽或高

注意：当同时设置 width/height 和 constraints 时，width/height 实际上会被合并到 constraints 中。例如 width: 100 等价于在 constraints 中设置 minWidth: 100, maxWidth: 100。

1.7 transform —— 变换

transform 属性接收一个 Matrix4 对象，可以对 Container 进行旋转、缩放、平移等变换。变换是在绘制阶段执行的，不会影响布局。

import 'package:flutter/material.dart';
import 'dart:math' as math;

void main() => runApp(const MyApp());

class MyApp extends StatelessWidget {
  const MyApp({super.key});

  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      home: Scaffold(
        appBar: AppBar(title: const Text('Container transform')),
        body: Center(
          child: Column(
            mainAxisAlignment: MainAxisAlignment.center,
            children: [
              // 旋转 15 度
              Container(
                width: 150,
                height: 80,
                transform: Matrix4.rotationZ(15 * math.pi / 180),
                transformAlignment: Alignment.center, // 变换原点设为中心
                decoration: BoxDecoration(
                  color: Colors.red.shade200,
                  borderRadius: BorderRadius.circular(8),
                ),
                alignment: Alignment.center,
                child: const Text('旋转 15°'),
              ),
              const SizedBox(height: 40),
              // 缩放 1.2 倍
              Container(
                width: 150,
                height: 80,
                transform: Matrix4.diagonal3Values(1.2, 1.2, 1),
                transformAlignment: Alignment.center,
                decoration: BoxDecoration(
                  color: Colors.green.shade200,
                  borderRadius: BorderRadius.circular(8),
                ),
                alignment: Alignment.center,
                child: const Text('缩放 1.2x'),
              ),
              const SizedBox(height: 40),
              // 平移
              Container(
                width: 150,
                height: 80,
                transform: Matrix4.translationValues(30, 0, 0), // 向右平移 30
                decoration: BoxDecoration(
                  color: Colors.blue.shade200,
                  borderRadius: BorderRadius.circular(8),
                ),
                alignment: Alignment.center,
                child: const Text('右移 30px'),
              ),
            ],
          ),
        ),
      ),
    );
  }
}

最佳实践：如果只需要旋转、缩放、平移变换，建议直接使用 Transform 组件（如 Transform.rotate、Transform.scale），语义更清晰。Container 的 transform 适合在已有 Container 的前提下顺便添加简单变换。

1.9 Container 注意事项与最佳实践

  1. 不要滥用 Container：很多初学者习惯用 Container 来设置任何样式，但如果你只需要一个功能（比如只需要 padding），应该直接使用对应的单一组件。这样可以减少 Widget 树的复杂度。
  2. color 和 decoration 互斥：不能同时设置 color 和 decoration，需要在 BoxDecoration 中设置 color。
  3. 没有 child 时的行为：当 Container 没有 child 且没有 width/height/constraints 时，它会尽可能地填满父容器；当有 child 时，它会尽可能地缩小以包裹 child。
  4. 调试技巧：给 Container 加一个 decoration: BoxDecoration(border: Border.all()) 可以快速可视化它的边界，便于调试布局问题。

2. SizedBox —— 固定尺寸盒子

2.1 基本概念

SizedBox 是一个非常简单但极其常用的组件，它的唯一功能就是给子组件一个固定的宽高约束。相比 Container，SizedBox 没有装饰、变换等额外功能，因此更轻量。

import 'package:flutter/material.dart';

void main() => runApp(const MyApp());

class MyApp extends StatelessWidget {
  const MyApp({super.key});

  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      home: Scaffold(
        appBar: AppBar(title: const Text('SizedBox 基本用法')),
        body: Center(
          child: SizedBox(
            width: 200,
            height: 100,
            child: Container(
              color: Colors.blue.shade200,
              alignment: Alignment.center,
              child: const Text('200 x 100 的盒子'),
            ),
          ),
        ),
      ),
    );
  }
}

2.2 作为间距使用

SizedBox 最高频的使用场景之一是在组件之间创建固定间距，这比使用 Container 或 Padding 更简洁、语义更清晰。

import 'package:flutter/material.dart';

void main() => runApp(const MyApp());

class MyApp extends StatelessWidget {
  const MyApp({super.key});

  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      home: Scaffold(
        appBar: AppBar(title: const Text('SizedBox 作为间距')),
        body: Center(
          child: Column(
            mainAxisAlignment: MainAxisAlignment.center,
            children: [
              Container(width: 100, height: 50, color: Colors.red),
              const SizedBox(height: 20), // 垂直间距 20
              Container(width: 100, height: 50, color: Colors.green),
              const SizedBox(height: 20), // 垂直间距 20
              Container(width: 100, height: 50, color: Colors.blue),
            ],
          ),
        ),
      ),
    );
  }
}

最佳实践：在 Column 中使用 SizedBox(height: n) 做垂直间距，在 Row 中使用 SizedBox(width: n) 做水平间距。这是 Flutter 社区广泛认可的间距用法。

2.3 SizedBox.expand —— 填满父容器

SizedBox.expand 等价于 SizedBox(width: double.infinity, height: double.infinity)，会让子组件尽可能地填满父容器。

import 'package:flutter/material.dart';

void main() => runApp(const MyApp());

class MyApp extends StatelessWidget {
  const MyApp({super.key});

  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      home: Scaffold(
        appBar: AppBar(title: const Text('SizedBox.expand')),
        body: Padding(
          padding: const EdgeInsets.all(32),
          child: SizedBox.expand(
            // 填满 Padding 内的所有可用空间
            child: Container(
              color: Colors.teal.shade200,
              alignment: Alignment.center,
              child: const Text(
                '我填满了父容器',
                style: TextStyle(fontSize: 18),
              ),
            ),
          ),
        ),
      ),
    );
  }
}

2.4 SizedBox.shrink —— 零尺寸盒子

SizedBox.shrink 等价于 SizedBox(width: 0, height: 0)，常用于条件渲染中，当不想显示任何内容时作为占位符。

import 'package:flutter/material.dart';

void main() => runApp(const MyApp());

class MyApp extends StatelessWidget {
  const MyApp({super.key});

  @override
  Widget build(BuildContext context) {
    const bool showBanner = false;

    return MaterialApp(
      home: Scaffold(
        appBar: AppBar(title: const Text('SizedBox.shrink')),
        body: Column(
          children: [
            // 根据条件决定是否显示 Banner
            showBanner
                ? Container(
                    width: double.infinity,
                    padding: const EdgeInsets.all(12),
                    color: Colors.amber,
                    child: const Text('这是一个 Banner'),
                  )
                : const SizedBox.shrink(), // 不显示时用零尺寸盒子占位
            const Padding(
              padding: EdgeInsets.all(16),
              child: Text('页面内容区域'),
            ),
          ],
        ),
      ),
    );
  }
}

2.5 SizedBox.fromSize

SizedBox.fromSize 接收一个 Size 对象来设置尺寸，在需要从其他地方获取 Size 值时比较方便。

import 'package:flutter/material.dart';

void main() => runApp(const MyApp());

class MyApp extends StatelessWidget {
  const MyApp({super.key});

  @override
  Widget build(BuildContext context) {
    const cardSize = Size(220, 120);

    return MaterialApp(
      home: Scaffold(
        appBar: AppBar(title: const Text('SizedBox.fromSize')),
        body: Center(
          child: SizedBox.fromSize(
            size: cardSize,
            child: Container(
              decoration: BoxDecoration(
                color: Colors.deepPurple.shade100,
                borderRadius: BorderRadius.circular(12),
              ),
              alignment: Alignment.center,
              child: Text('尺寸: ${cardSize.width} x ${cardSize.height}'),
            ),
          ),
        ),
      ),
    );
  }
}

3. DecoratedBox 与 BoxDecoration —— 装饰盒子

3.1 DecoratedBox 基本概念

DecoratedBox 是一个只负责绘制装饰（背景、边框等）的组件。Container 内部的装饰功能实际上就是通过 DecoratedBox 来实现的。

当你只需要给子组件添加装饰效果而不需要 Container 的其他功能（padding、margin、alignment 等）时，直接使用 DecoratedBox 会更高效。

import 'package:flutter/material.dart';

void main() => runApp(const MyApp());

class MyApp extends StatelessWidget {
  const MyApp({super.key});

  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      home: Scaffold(
        appBar: AppBar(title: const Text('DecoratedBox 基本用法')),
        body: Center(
          child: DecoratedBox(
            decoration: BoxDecoration(
              color: Colors.blue.shade50,
              borderRadius: BorderRadius.circular(12),
              border: Border.all(color: Colors.blue, width: 2),
            ),
            child: const Padding(
              padding: EdgeInsets.symmetric(horizontal: 24, vertical: 12),
              child: Text('DecoratedBox', style: TextStyle(fontSize: 16)),
            ),
          ),
        ),
      ),
    );
  }
}

3.2 边框 —— Border

Border 类用于定义 BoxDecoration 的边框。可以为四条边设置不同的样式。

import 'package:flutter/material.dart';

void main() => runApp(const MyApp());

class MyApp extends StatelessWidget {
  const MyApp({super.key});

  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      home: Scaffold(
        appBar: AppBar(title: const Text('Border 边框')),
        body: Center(
          child: Column(
            mainAxisAlignment: MainAxisAlignment.center,
            children: [
              // 统一边框
              DecoratedBox(
                decoration: BoxDecoration(
                  border: Border.all(color: Colors.red, width: 3),
                ),
                child: const SizedBox(
                  width: 150, height: 60,
                  child: Center(child: Text('统一边框')),
                ),
              ),
              const SizedBox(height: 20),
              // 不同方向不同样式的边框
              DecoratedBox(
                decoration: const BoxDecoration(
                  border: Border(
                    top: BorderSide(color: Colors.red, width: 3),
                    bottom: BorderSide(color: Colors.blue, width: 3),
                    left: BorderSide(color: Colors.green, width: 3),
                    right: BorderSide(color: Colors.orange, width: 3),
                  ),
                ),
                child: const SizedBox(
                  width: 150, height: 60,
                  child: Center(child: Text('四色边框')),
                ),
              ),
              const SizedBox(height: 20),
              // 只有底部边框（常用于分割线效果）
              DecoratedBox(
                decoration: const BoxDecoration(
                  border: Border(
                    bottom: BorderSide(color: Colors.grey, width: 1),
                  ),
                ),
                child: const SizedBox(
                  width: 200, height: 50,
                  child: Center(child: Text('底部边框')),
                ),
              ),
            ],
          ),
        ),
      ),
    );
  }
}

注意：当 BoxDecoration 同时设置了 borderRadius 和 border 时，Border 的四条边必须样式一致（颜色和宽度相同），否则 Flutter 无法正确渲染圆角边框。

3.3 圆角 —— BorderRadius

BorderRadius 用于定义矩形的圆角半径，支持统一圆角和分别设置每个角的圆角。

import 'package:flutter/material.dart';

4. Padding —— 间距控制

何时使用 Padding 而非 Container？ 当你只需要添加内边距而不需要背景色、边框、阴影等其他装饰时，Padding 是更好的选择。

4.2 EdgeInsets 的各种构造方式

EdgeInsets 提供了多种构造方式来满足不同场景的需求：

import 'package:flutter/material.dart';

void main() => runApp(const MyApp());

class MyApp extends StatelessWidget {
  const MyApp({super.key});

  Widget _buildPaddingDemo(String label, EdgeInsets padding) {
    return Container(
      margin: const EdgeInsets.only(bottom: 12),
      color: Colors.grey.shade200,
      child: Padding(
        padding: padding,
        child: Container(
          color: Colors.blue.shade100,
          padding: const EdgeInsets.all(8),
          child: Text(label, style: const TextStyle(fontSize: 14)),
        ),
      ),
    );
  }

  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      home: Scaffold(
        appBar: AppBar(title: const Text('EdgeInsets 构造方式')),
        body: SingleChildScrollView(
          child: Column(
            crossAxisAlignment: CrossAxisAlignment.stretch,
            children: [
              // 1. all：四个方向相同的间距
              _buildPaddingDemo(
                'EdgeInsets.all(16)',
                const EdgeInsets.all(16),
              ),

              // 2. symmetric：水平和垂直方向分别设置
              _buildPaddingDemo(
                'EdgeInsets.symmetric(h:32, v:8)',
                const EdgeInsets.symmetric(horizontal: 32, vertical: 8),
              ),

              // 3. only：只设置某些方向
              _buildPaddingDemo(
                'EdgeInsets.only(left:40, top:20)',
                const EdgeInsets.only(left: 40, top: 20),
              ),

              // 4. fromLTRB：分别设置左、上、右、下
              _buildPaddingDemo(
                'EdgeInsets.fromLTRB(10, 20, 30, 40)',
                const EdgeInsets.fromLTRB(10, 20, 30, 40),
              ),

              // 5. zero：无间距（与 EdgeInsets.all(0) 等价）
              _buildPaddingDemo(
                'EdgeInsets.zero',
                EdgeInsets.zero,
              ),
            ],
          ),
        ),
      ),
    );
  }
}

各构造方式使用场景对比：

构造方式 适用场景
EdgeInsets.all(n) 四个方向相同间距，最常用
EdgeInsets.symmetric(horizontal: h, vertical: v) 水平/垂直方向对称间距，常用于列表项
EdgeInsets.only(left: l, top: t, ...) 只设置个别方向的间距
EdgeInsets.fromLTRB(l, t, r, b) 四个方向各不相同
EdgeInsets.zero 显式表示无间距

注意：对于需要支持 RTL（从右到左）语言的应用，应使用 EdgeInsetsDirectional 代替 EdgeInsets。EdgeInsetsDirectional 使用 start/end 代替 left/right，Flutter 会根据当前文本方向自动调整。

5. Center —— 居中组件

5.1 基本概念

Center 组件将子组件放置在自身的正中心。它实际上是 Align(alignment: Alignment.center) 的简写形式。

import 'package:flutter/material.dart';

void main() => runApp(const MyApp());

class MyApp extends StatelessWidget {
  const MyApp({super.key});

  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      home: Scaffold(
        appBar: AppBar(title: const Text('Center 基本用法')),

            height: 150,

          ),
        ),
      ),
    );
  }
}

5.2 widthFactor 与 heightFactor

默认情况下，Center 会尽可能占满父容器的所有可用空间。通过 widthFactor 和 heightFactor 可以让 Center 的尺寸基于子组件的尺寸乘以一个比例因子。

import 'package:flutter/material.dart';

void main() => runApp(const MyApp());

class MyApp extends StatelessWidget {
  const MyApp({super.key});

  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      home: Scaffold(
        appBar: AppBar(title: const Text('Center widthFactor & heightFactor')),
        body: Column(
          children: [
            const SizedBox(height: 20),
            // 没有 factor：Center 占满父容器宽度
            Container(
              color: Colors.grey.shade300,
              child: Center(
                child: Container(
                  width: 100,
                  height: 50,
                  color: Colors.blue,
                  alignment: Alignment.center,
                  child: const Text('无 factor',
                      style: TextStyle(color: Colors.white, fontSize: 12)),
                ),
              ),
            ),
            const SizedBox(height: 20),
            // 有 factor：Center 的宽度 = 子组件宽度 × widthFactor
            Container(
              color: Colors.grey.shade300,
              child: Center(
                widthFactor: 2.0,   // Center 宽度 = 100 × 2 = 200
                heightFactor: 1.5,  // Center 高度 = 50 × 1.5 = 75
                child: Container(
                  width: 100,
                  height: 50,
                  color: Colors.green,
                  alignment: Alignment.center,
                  child: const Text('有 factor',
                      style: TextStyle(color: Colors.white, fontSize: 12)),
                ),
              ),
            ),
          ],
        ),
      ),
    );
  }
}

上面第二个例子中，灰色区域（Center 的实际大小）变小了，因为 Center 不再占满父容器，而是根据子组件尺寸乘以因子来决定自身大小。

提示：widthFactor 和 heightFactor 在 Align 组件中也同样存在，功能一致。这个特性在需要精确控制 Align/Center 自身尺寸时非常有用。

ColoredBox 是一个极其简单的组件，它唯一的功能就是给子组件绘制一个纯色背景。在组件树中，它是最轻量的背景色方案。

6.2 ColoredBox 与 Container(color:) 的区别

当你写 Container(color: Colors.blue, child: ...) 时，Container 内部实际创建了一个 ColoredBox。但 Container 本身还会引入额外的 Widget 层级（如 LimitedBox、ConstrainedBox 等），即使你只设置了 color。

性能对比：

写法 Widget 树复杂度 推荐场景
ColoredBox(color: ...) 最简单，只有 1 层 只需要纯色背景
Container(color: ...) 较复杂，包含多层 需要组合多种功能 
import 'package:flutter/material.dart';

void main() => runApp(const MyApp());

class MyApp extends StatelessWidget {
  const MyApp({super.key});

  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      home: Scaffold(
        appBar: AppBar(title: const Text('ColoredBox vs Container')),
        body: Column(
          mainAxisAlignment: MainAxisAlignment.center,
          children: [
            // 方式一：ColoredBox（推荐，更轻量）
            const ColoredBox(
              color: Colors.lightBlue,
              child: SizedBox(
                width: 200,
                height: 60,
                child: Center(
                  child: Text(
                    'ColoredBox（更轻量）',
                    style: TextStyle(color: Colors.white),
                  ),
                ),
              ),
            ),
            const SizedBox(height: 20),
            // 方式二：Container（功能更多，但更重）
            Container(
              width: 200,
              height: 60,
              color: Colors.indigo,
              alignment: Alignment.center,
              child: const Text(
                'Container（更重）',
                style: TextStyle(color: Colors.white),
              ),
            ),
          ],
        ),
      ),
    );
  }
}

最佳实践：如果你只需要添加一个纯色背景，优先使用 ColoredBox。当你需要同时控制尺寸、间距、对齐等多个属性时，再考虑使用 Container。在大型列表等性能敏感场景中，这种优化尤其有意义。

7.2 PhysicalShape

PhysicalShape 功能与 PhysicalModel 类似，但它接受一个 CustomClipper<Path> 来定义任意的裁剪形状，提供更大的灵活性。

/// 自定义菱形裁剪器
class _DiamondClipper extends CustomClipper<Path> {
  @override
  Path getClip(Size size) {
    final path = Path()
      ..moveTo(size.width / 2, 0)           // 顶部中点
      ..lineTo(size.width, size.height / 2)  // 右侧中点
      ..lineTo(size.width / 2, size.height)  // 底部中点
      ..lineTo(0, size.height / 2)           // 左侧中点
      ..close();
    return path;
  }

  @override
  bool shouldReclip(covariant CustomClipper<Path> oldClipper) => false;
}

7.3 PhysicalModel vs BoxDecoration 阴影对比

特性 PhysicalModel / PhysicalShape BoxDecoration + BoxShadow
阴影真实感 更贴近物理投影 简单的偏移模糊阴影
裁剪 自动裁剪子组件到形状内 不裁剪，需配合 ClipRRect 等
自定义形状 PhysicalShape 支持任意 Path 仅矩形/圆形
性能 使用 Skia 引擎原生阴影渲染 手动绘制阴影
灵活性 阴影参数较少 boxShadow 支持多层、颜色、扩散等

最佳实践：当你需要 Material Design 风格的层级阴影效果（特别是带裁剪的）时，优先使用 PhysicalModel。当你需要更细致地控制阴影外观（多层阴影、发光效果等）时，使用 BoxDecoration 的 boxShadow。在大多数日常开发中，使用 Card 或 Material 组件即可获得恰当的阴影效果。

8. 容器组件选择指南

在实际开发中，面对这么多容器组件，应该如何选择？下面是一份快速决策指南：

需求 推荐组件 原因
只需要固定间距 SizedBox 最轻量
只需要内边距 Padding 单一职责，语义清晰
只需要居中 Center 语义清晰
只需要纯色背景 ColoredBox 最轻量的背景色方案
只需要装饰（边框/圆角/渐变） DecoratedBox 单一职责
需要物理阴影 + 裁剪 PhysicalModel 物理级阴影渲染
需要组合多种功能 Container 便捷组合组件
只需要固定尺寸 SizedBox 比 Container 轻量

核心原则：能用单一功能组件解决的，就不要用 Container。Container 适合需要同时使用多种功能的场景，避免手动嵌套过多单一组件导致代码可读性下降。

本课小结

  1. Container 是一个便捷组合组件，内部组合了 Align、Padding、DecoratedBox、ConstrainedBox、Transform 等多个单一功能组件。它功能强大但不是万能的，应该根据实际需要选择合适的组件。

  2. SizedBox 是最轻量的尺寸控制组件，也是 Flutter 社区推荐的间距创建方式。SizedBox.expand 用于填满父容器，SizedBox.shrink 用于零尺寸占位。

  3. DecoratedBox 配合 BoxDecoration 可以实现丰富的视觉效果：边框（Border）、圆角（BorderRadius）、渐变（LinearGradient / RadialGradient / SweepGradient）、阴影（BoxShadow）、背景图（DecorationImage）。

  4. Padding 搭配 EdgeInsets 的多种构造方式（all、symmetric、only、fromLTRB）可以灵活控制间距。支持 RTL 语言时应使用 EdgeInsetsDirectional。

  5. Center 是 Align(alignment: Alignment.center) 的简写，通过 widthFactor 和 heightFactor 可以控制自身尺寸。

  6. ColoredBox 是最轻量的纯色背景方案，Container(color:) 内部就是用它实现的。

  7. PhysicalModel / PhysicalShape 提供物理级阴影渲染和自动裁剪，适合 Material Design 的层级效果。

  8. 组件选择的核心原则：优先使用单一功能组件，只有在需要组合多种功能时才使用 Container。
