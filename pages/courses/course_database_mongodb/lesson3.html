第03章 文档的 CRUD 操作（Shell 篇）

本章在 mongosh 中操作一个博客数据库 blog，包含 users（用户）和 articles（文章）两个集合。 所有示例均可直接在 mongosh 中执行。

// 切换到 blog 数据库（不存在时自动创建）
use blog

3.1 插入文档：insertOne / insertMany

3.1.1 insertOne —— 插入单个文档

语法：

db.collection.insertOne(document, options)

示例：插入一个用户文档

db.users.insertOne({
  username: "zhangsan",
  email: "zhangsan@example.com",
  password: "hashed_password_123",
  nickname: "张三",
  age: 28,
  role: "author",
  tags: ["Java", "MongoDB", "Spring Boot"],
  profile: {
    bio: "全栈开发工程师",
    avatar: "/avatars/zhangsan.jpg",
    location: "北京"
  },
  createdAt: new Date(),
  updatedAt: new Date()
})

返回结果：

{
  acknowledged: true,
  insertedId: ObjectId('665a1b2c3d4e5f6a7b8c9d0e')
}
  • acknowledged: true 表示写入操作被确认。
  • insertedId 是 MongoDB 自动生成的 _id。

3.1.3 _id 的自动生成机制

MongoDB 中每个文档都必须有一个 _id 字段作为主键。如果插入时不指定 _id，MongoDB 会自动生成一个 ObjectId。

ObjectId 的结构（12 字节）：

字节 含义
1-4 时间戳（秒级，Unix 时间）
5-9 随机值（每个进程生成一次）
10-12 自增计数器（初始值随机） 
// 手动指定 _id
db.users.insertOne({
  _id: "user_lisi",
  username: "lisi",
  email: "lisi@example.com",
  nickname: "李四",
  age: 25,
  role: "reader",
  tags: ["前端", "Vue"],
  profile: {
    bio: "前端开发工程师",
    avatar: "/avatars/lisi.jpg",
    location: "上海"
  },
  createdAt: new Date(),
  updatedAt: new Date()
})

// 从 ObjectId 中提取时间戳
let id = ObjectId('665a1b2c3d4e5f6a7b8c9d0e')
id.getTimestamp()  // 返回 ISODate("2024-05-31T...")

注意： 如果插入的文档 _id 与已有文档重复，MongoDB 会抛出 DuplicateKeyError。

3.1.4 ordered 参数

insertMany 的 ordered 参数控制批量插入时的行为：

  • ordered: true（默认）：按顺序逐条插入，遇到错误立即停止，已插入的不会回滚。
  • ordered: false：并行插入所有文档，遇到错误跳过该条，继续插入其他文档。
// 假设第2条的 _id 冲突
db.users.insertMany(
  [
    { _id: "u1", username: "user1" },
    { _id: "u1", username: "user1_dup" },  // _id 重复！
    { _id: "u3", username: "user3" }
  ],
  { ordered: true }
)
// 结果：只插入了 u1，u1_dup 报错后 u3 不再执行

db.users.insertMany(
  [
    { _id: "u4", username: "user4" },
    { _id: "u4", username: "user4_dup" },  // _id 重复！
    { _id: "u6", username: "user6" }
  ],
  { ordered: false }
)
// 结果：u4 和 u6 都成功插入，u4_dup 报错但不影响 u6

选择建议：

  • 数据有序依赖时，使用默认的 ordered: true。
  • 大批量导入、容忍部分失败时，使用 ordered: false（性能更好）。

3.2 查询文档：find / findOne 与查询操作符

3.2.1 find() 和 findOne() 基本用法

// find() —— 返回匹配的所有文档（游标）
db.articles.find()            // 查询所有文章
db.articles.find({})          // 等价写法

// findOne() —— 返回匹配的第一个文档
db.articles.findOne()         // 返回第一篇文章

// 带条件查询
db.articles.find({ author: "zhangsan" })          // 查询张三的所有文章
db.articles.findOne({ status: "published" })       // 查询第一篇已发布文章

// 美化输出
db.articles.find({ author: "zhangsan" }).pretty()

3.2.2 比较操作符

操作符 含义 等价 SQL
$eq 等于 = value
$ne 不等于 != value
$gt 大于 > value
$gte 大于等于 >= value
$lt 小于 < value
$lte 小于等于 <= value 
// $eq —— 等于（直接写值就是隐式 $eq）
db.articles.find({ status: { $eq: "published" } })
db.articles.find({ status: "published" })  // 等价的简写形式

// $ne —— 不等于：查询非草稿状态的文章
db.articles.find({ status: { $ne: "draft" } })

// $gt —— 大于：查询阅读量大于 100 的文章
db.articles.find({ viewCount: { $gt: 100 } })

// $gte —— 大于等于：查询阅读量 >= 200 的文章
db.articles.find({ viewCount: { $gte: 200 } })

// $lt —— 小于：查询阅读量小于 200 的文章
db.articles.find({ viewCount: { $lt: 200 } })

// $lte —— 小于等于：查询 2024-02-01 之前（含）创建的文章
db.articles.find({ createdAt: { $lte: new Date("2024-02-01") } })

// 组合使用：查询阅读量在 100 到 300 之间的文章
db.articles.find({ viewCount: { $gt: 100, $lte: 300 } })

3.2.3 逻辑操作符

操作符 含义 说明
$and 逻辑与 所有条件都满足
$or 逻辑或 至少一个条件满足
$not 逻辑非 对条件取反
$nor 逻辑或非 所有条件都不满足 
// $and —— 查询张三发布的已发布文章
// 隐式 $and（同一层级多个字段时自动 AND）
db.articles.find({ author: "zhangsan", status: "published" })

// 显式 $and（同一字段多个条件时必须使用）
db.articles.find({
  $and: [
    { viewCount: { $gte: 100 } },
    { viewCount: { $lte: 300 } }
  ]
})

// $or —— 查询分类为"Java"或"数据库"的文章
db.articles.find({
  $or: [
    { category: "Java" },
    { category: "数据库" }
  ]
})

// $and + $or 组合：查询已发布且（分类为 Java 或阅读量 > 200）的文章
db.articles.find({
  status: "published",
  $or: [
    { category: "Java" },
    { viewCount: { $gt: 200 } }
  ]
})

// $not —— 查询阅读量不大于 100 的文章（即 <= 100）
db.articles.find({
  viewCount: { $not: { $gt: 100 } }
})

// $nor —— 查询既不是草稿、分类也不是 DevOps 的文章
db.articles.find({
  $nor: [
    { status: "draft" },
    { category: "DevOps" }
  ]
})

3.2.4 元素操作符

// $exists —— 查询包含 profile 字段的用户
db.users.find({ profile: { $exists: true } })

// 查询没有 avatar 字段的用户
db.users.find({ "profile.avatar": { $exists: false } })

// $type —— 查询 age 字段类型为数值的文档
db.users.find({ age: { $type: "number" } })

// 也可以使用 BSON 类型编号：16 = int32, 18 = int64, 1 = double
db.users.find({ age: { $type: 16 } })

// $type 可以接收数组，匹配多种类型
db.users.find({ age: { $type: ["int", "double"] } })

3.2.5 数组操作符

// $in —— 查询分类为 "Java" 或 "数据库" 的文章（等价于 $or 对同一字段的写法）
db.articles.find({ category: { $in: ["Java", "数据库"] } })

// $nin —— 查询分类不是 "Java" 也不是 "DevOps" 的文章
db.articles.find({ category: { $nin: ["Java", "DevOps"] } })

// $in 用于数组字段：查询 tags 中包含 "MongoDB" 或 "Redis" 的文章
db.articles.find({ tags: { $in: ["MongoDB", "Redis"] } })

// $all —— 查询 tags 同时包含 "Java" 和 "并发" 的文章
db.articles.find({ tags: { $all: ["Java", "并发"] } })

// $size —— 查询恰好有 2 条评论的文章
db.articles.find({ comments: { $size: 2 } })

// $elemMatch —— 查询评论中存在 user 为 "zhangsan" 的文章
db.articles.find({
  comments: {
    $elemMatch: {
      user: "zhangsan",
      createdAt: { $gte: new Date("2024-02-01") }
    }
  }
})

$in vs $all 的区别：

  • $in：数组中至少包含一个指定值（OR 语义）。
  • $all：数组中必须包含所有指定值（AND 语义）。

3.2.6 投影（Projection）：指定返回字段

投影用来控制查询结果中包含或排除哪些字段，减少网络传输量。

// 只返回 title 和 author 字段（_id 默认返回）
db.articles.find({}, { title: 1, author: 1 })

// 返回 title 和 author，排除 _id
db.articles.find({}, { title: 1, author: 1, _id: 0 })

// 排除 content 字段（其他字段全部返回）
db.articles.find({}, { content: 0 })

// 对嵌套字段投影
db.users.find({}, { username: 1, "profile.bio": 1 })

注意： 投影中不能混合使用包含（1）和排除（0），_id 除外。 即 { title: 1, content: 0 } 是非法的，但 { title: 1, _id: 0 } 是合法的。

3.2.7 正则表达式查询

// 查询标题包含 "MongoDB" 的文章
db.articles.find({ title: /MongoDB/ })

// 等价写法
db.articles.find({ title: { $regex: "MongoDB" } })

// 不区分大小写
db.articles.find({ title: { $regex: "mongodb", $options: "i" } })

// 查询标题以 "Spring" 开头的文章
db.articles.find({ title: { $regex: "^Spring" } })

// 查询邮箱以 @example.com 结尾的用户
db.users.find({ email: { $regex: "example\\.com$" } })

// 查询用户名是 3-10 个字母的用户
db.users.find({ username: { $regex: "^[a-zA-Z]{3,10}$" } })

3.2.8 分页与排序：skip(), limit(), sort()

// sort() —— 1 升序，-1 降序
// 按创建时间倒序
db.articles.find().sort({ createdAt: -1 })

// 按阅读量降序，阅读量相同则按创建时间升序
db.articles.find().sort({ viewCount: -1, createdAt: 1 })

// limit() —— 限制返回数量
db.articles.find().limit(3)  // 只返回前 3 篇

// skip() —— 跳过指定数量
db.articles.find().skip(2)   // 跳过前 2 篇

// 分页查询：每页 2 条，查询第 2 页
let pageSize = 2
let pageNum = 2
db.articles.find({ status: "published" })
  .sort({ createdAt: -1 })
  .skip((pageNum - 1) * pageSize)
  .limit(pageSize)

// 等价于 SQL: SELECT * FROM articles WHERE status='published'
//              ORDER BY createdAt DESC LIMIT 2 OFFSET 2

性能提示： 当数据量大时，skip() 的性能较差（需要扫描并跳过前 N 条）。 大数据量分页推荐使用基于游标的分页（记录上一页最后一条的排序字段值）：

// 基于游标的分页（第一页之后的分页请求）
db.articles.find({
  status: "published",
  createdAt: { $lt: lastSeenCreatedAt }  // 上一页最后一条的时间
}).sort({ createdAt: -1 }).limit(pageSize)

3.2.9 综合查询示例

以下是更多查询场景示例，展示各种操作符的灵活组合：

// 示例1：查询张三在2024年1月发布的文章标题
db.articles.find(
  {
    author: "zhangsan",
    status: "published",
    createdAt: {
      $gte: new Date("2024-01-01"),
      $lt: new Date("2024-02-01")
    }
  },
  { title: 1, createdAt: 1, _id: 0 }
)

// 示例2：查询有评论且阅读量大于 100 的文章
db.articles.find({
  comments: { $not: { $size: 0 } },
  viewCount: { $gt: 100 }
})

// 示例3：查询标签数量大于等于 3 的文章
// 注意：$size 不支持范围查询，需要用 $expr 配合 $size 聚合表达式
db.articles.find({
  $expr: { $gte: [{ $size: "$tags" }, 3] }
})

// 示例4：查询嵌套字段——位于北京的用户
db.users.find({ "profile.location": "北京" })

// 示例5：查询角色为 author 或 admin 的用户，按年龄升序
db.users.find({ role: { $in: ["author", "admin"] } }).sort({ age: 1 })

// 示例6：查询没有评论的草稿文章
db.articles.find({
  status: "draft",
  comments: { $size: 0 }
})

// 示例7：统计查询结果数量
db.articles.find({ status: "published" }).count()
// 或使用 countDocuments（推荐）
db.articles.countDocuments({ status: "published" })

// 示例8：查询标签中包含 "Java" 但不包含 "入门" 的文章
db.articles.find({
  tags: "Java",
  tags: { $ne: "入门" }
})
// 上面的写法有问题（同键覆盖），正确写法：
db.articles.find({
  $and: [
    { tags: "Java" },
    { tags: { $ne: "入门" } }
  ]
})

// 示例9：查询 email 字段存在且不为 null 的用户
db.users.find({
  email: { $exists: true, $ne: null }
})

// 示例10：查询标题包含 "实践" 或 "详解" 的已发布文章，只返回标题和作者
db.articles.find(
  {
    status: "published",
    title: { $regex: "实践|详解" }
  },
  { title: 1, author: 1, _id: 0 }
)

// 示例11：使用 distinct 查询所有不重复的分类
db.articles.distinct("category")
// 返回: ["数据库", "Java", "DevOps"]

// 示例12：查询结果转为数组
db.articles.find({ status: "published" }).toArray()

3.3 更新文档：updateOne / updateMany 与更新操作符

3.3.1 updateOne / updateMany 语法

// 语法
db.collection.updateOne(filter, update, options)
db.collection.updateMany(filter, update, options)
  • filter：筛选条件，语法与 find() 相同。
  • update：更新操作，必须使用更新操作符（$set、$inc 等）。
  • options：可选参数，如 upsert。

3.3.2 $set —— 设置字段值

$set 用于设置指定字段的值。如果字段不存在，则创建该字段。

// 更新张三的昵称和邮箱
db.users.updateOne(
  { username: "zhangsan" },
  {
    $set: {
      nickname: "张三丰",
      email: "zhangsan_new@example.com",
      updatedAt: new Date()
    }
  }
)

// 更新嵌套字段
db.users.updateOne(
  { username: "zhangsan" },
  { $set: { "profile.location": "深圳", "profile.bio": "资深全栈工程师" } }
)

// 批量更新：将所有草稿文章的状态改为 "review"
db.articles.updateMany(
  { status: "draft" },
  { $set: { status: "review", updatedAt: new Date() } }
)

返回结果：

{
  acknowledged: true,
  insertedId: null,
  matchedCount: 1,    // 匹配到的文档数
  modifiedCount: 1,   // 实际修改的文档数
  upsertedCount: 0
}

3.3.3 $unset —— 删除字段

// 删除用户的 age 字段
db.users.updateOne(
  { username: "zhangsan" },
  { $unset: { age: "" } }  // 值可以是任意内容，通常写 "" 或 1
)

// 批量删除所有用户的临时字段
db.users.updateMany(
  {},
  { $unset: { tempField: "", legacyField: "" } }
)

3.3.4 $inc —— 递增/递减

// 文章阅读量 +1
db.articles.updateOne(
  { title: "MongoDB 入门指南" },
  { $inc: { viewCount: 1 } }
)

// 文章阅读量 -10（传负数实现递减）
db.articles.updateOne(
  { title: "MongoDB 入门指南" },
  { $inc: { viewCount: -10 } }
)

// 同时递增多个字段
db.articles.updateOne(
  { title: "MongoDB 入门指南" },
  { $inc: { viewCount: 1, commentCount: 1 } }
)

3.3.5 $rename —— 重命名字段

// 将 nickname 字段重命名为 displayName
db.users.updateOne(
  { username: "zhangsan" },
  { $rename: { "nickname": "displayName" } }
)

// 重命名嵌套字段
db.users.updateOne(
  { username: "zhangsan" },
  { $rename: { "profile.bio": "profile.description" } }
)

// 批量重命名
db.users.updateMany(
  {},
  { $rename: { "tags": "skills" } }
)

3.3.6 $push / $addToSet —— 向数组添加元素

// $push —— 向数组末尾添加一个元素（允许重复）
db.articles.updateOne(
  { title: "MongoDB 入门指南" },
  {
    $push: {
      comments: {
        user: "wangwu",
        content: "非常实用的入门教程！",
        createdAt: new Date()
      }
    }
  }
)

// $push 添加多个元素：使用 $each
db.articles.updateOne(
  { title: "MongoDB 入门指南" },
  {
    $push: {
      tags: { $each: ["教程", "新手"] }
    }
  }
)

// $addToSet —— 向数组添加元素（去重，已存在则不添加）
db.users.updateOne(
  { username: "zhangsan" },
  { $addToSet: { tags: "MongoDB" } }  // 如果 tags 中已有 "MongoDB"，不会重复添加
)

// $addToSet + $each —— 批量去重添加
db.users.updateOne(
  { username: "zhangsan" },
  {
    $addToSet: {
      tags: { $each: ["Docker", "Kubernetes", "MongoDB"] }
    }
  }
)

3.3.7 $pull / $pop —— 从数组移除元素

// $pull —— 移除数组中匹配条件的所有元素
// 从标签中移除 "入门"
db.articles.updateOne(
  { title: "MongoDB 入门指南" },
  { $pull: { tags: "入门" } }
)

// $pull 配合条件：移除某用户的所有评论
db.articles.updateOne(
  { title: "Spring Boot 3.x 新特性" },
  {
    $pull: {
      comments: { user: "lisi" }
    }
  }
)

// $pull 更复杂的条件：移除 2024-02-01 之前的评论
db.articles.updateMany(
  {},
  {
    $pull: {
      comments: { createdAt: { $lt: new Date("2024-02-01") } }
    }
  }
)

// $pop —— 移除数组的第一个或最后一个元素
// 移除 tags 数组的最后一个元素
db.articles.updateOne(
  { title: "MongoDB 入门指南" },
  { $pop: { tags: 1 } }   // 1 = 最后一个，-1 = 第一个
)

// 移除 tags 数组的第一个元素
db.articles.updateOne(
  { title: "MongoDB 入门指南" },
  { $pop: { tags: -1 } }
)

3.3.8 $each, $position, $slice 修饰符

这些修饰符与 $push 配合使用，提供更精细的数组操作控制。

// $each + $position：在数组指定位置插入元素
// 在 tags 数组的索引 1 位置插入两个新标签
db.articles.updateOne(
  { title: "MongoDB 入门指南" },
  {
    $push: {
      tags: {
        $each: ["实战", "进阶"],
        $position: 1  // 从索引 1 开始插入
      }
    }
  }
)
// 原来: ["MongoDB", "NoSQL", "入门"]
// 之后: ["MongoDB", "实战", "进阶", "NoSQL", "入门"]

// $each + $slice：限制数组最大长度
// 添加评论，但只保留最新的 5 条
db.articles.updateOne(
  { title: "MongoDB 入门指南" },
  {
    $push: {
      comments: {
        $each: [{ user: "newuser", content: "新评论", createdAt: new Date() }],
        $slice: -5  // 负数表示保留最后 N 个，正数表示保留前 N 个
      }
    }
  }
)

// $each + $sort + $slice：添加后排序再截断
// 保留阅读量最高的 10 篇推荐文章
db.users.updateOne(
  { username: "zhangsan" },
  {
    $push: {
      recommendedArticles: {
        $each: [{ title: "新文章", viewCount: 500 }],
        $sort: { viewCount: -1 },  // 按阅读量降序排列
        $slice: 10                  // 只保留前 10 个
      }
    }
  }
)

3.3.9 upsert 选项

upsert = true 时：如果 filter 没有匹配到文档，则插入一个新文档（由 filter 和 update 合并而成）。

// 更新用户的登录信息，如果用户不存在则创建
db.users.updateOne(
  { username: "newuser" },
  {
    $set: {
      lastLoginAt: new Date(),
      loginCount: 1
    },
    $setOnInsert: {
      // 仅在插入时设置的字段
      email: "newuser@example.com",
      role: "reader",
      createdAt: new Date()
    }
  },
  { upsert: true }
)

返回结果（插入场景）：

{
  acknowledged: true,
  insertedId: ObjectId('665b...'),  // 新插入的文档 ID
  matchedCount: 0,
  modifiedCount: 0,
  upsertedCount: 1
}

$setOnInsert 是 upsert 的好搭档：它指定的字段仅在执行插入操作时才会写入，更新已有文档时不会生效。

3.3.10 replaceOne —— 整文档替换

replaceOne 用一个全新的文档完全替换匹配的文档（_id 保持不变）。

// 注意：替换文档中不能包含更新操作符
db.users.replaceOne(
  { username: "lisi" },
  {
    username: "lisi",
    email: "lisi_new@example.com",
    nickname: "李四2.0",
    age: 26,
    role: "author",
    tags: ["Vue", "React", "TypeScript"],
    profile: {
      bio: "资深前端工程师",
      avatar: "/avatars/lisi_v2.jpg",
      location: "杭州"
    },
    createdAt: new Date("2024-01-01"),
    updatedAt: new Date()
  }
)

updateOne vs replaceOne：

  • updateOne：使用更新操作符（$set 等），只修改指定字段。
  • replaceOne：用新文档完全替换，未指定的字段会丢失。

3.4 删除文档：deleteOne / deleteMany

3.4.1 基本删除语法和示例

// deleteOne —— 删除匹配的第一个文档
db.articles.deleteOne({ title: "Docker 容器化部署实践" })

// 返回结果
// { acknowledged: true, deletedCount: 1 }

// deleteMany —— 删除匹配的所有文档
// 删除所有草稿文章
db.articles.deleteMany({ status: "draft" })

// 删除阅读量为 0 的文章
db.articles.deleteMany({ viewCount: 0 })

// 删除某用户的所有文章
db.articles.deleteMany({ author: "lisi" })

// 配合日期条件：删除一年前的文章
db.articles.deleteMany({
  createdAt: { $lt: new Date("2023-01-01") }
})

3.4.2 删除所有文档

// 方式一：传入空条件（保留集合和索引）
db.articles.deleteMany({})

// 方式二：使用 drop()（删除整个集合，包括索引）
db.articles.drop()

两者的区别：

特性 deleteMany({}) drop()
删除文档 是 是
删除索引 否（保留） 是
删除集合 否（保留空集合） 是
性能 逐条删除，较慢 直接删除文件，极快
适用场景 清空数据但保留结构 彻底删除重建

3.4.3 drop() 删除整个集合

// 删除 articles 集合
db.articles.drop()
// 返回 true 表示成功

// 删除整个数据库
db.dropDatabase()
// { ok: 1, dropped: 'blog' }

注意： drop() 和 dropDatabase() 是不可逆操作，生产环境务必谨慎使用！

3.5 批量操作：bulkWrite

3.5.1 bulkWrite 语法

bulkWrite 可以在一次请求中执行多种不同类型的写操作（插入、更新、删除），减少网络往返次数，提升性能。

db.collection.bulkWrite(
  [operation1, operation2, ...],
  { ordered: true/false }
)

支持的操作类型：

操作类型 说明
insertOne 插入文档
updateOne 更新单个文档
updateMany 更新多个文档
replaceOne 替换单个文档
deleteOne 删除单个文档
deleteMany 删除多个文档

3.5.2 混合操作示例

db.articles.bulkWrite([
  // 操作1：插入一篇新文章
  {
    insertOne: {
      document: {
        title: "Kubernetes 入门教程",
        content: "Kubernetes 是容器编排领域的事实标准...",
        author: "zhangsan",
        category: "DevOps",
        tags: ["Kubernetes", "容器", "云原生"],
        status: "published",
        viewCount: 0,
        comments: [],
        createdAt: new Date(),
        updatedAt: new Date()
      }
    }
  },

  // 操作2：更新某篇文章的阅读量
  {
    updateOne: {
      filter: { title: "MongoDB 入门指南" },
      update: { $inc: { viewCount: 50 } }
    }
  },

  // 操作3：批量更新所有已发布文章，添加一个 featured 字段
  {
    updateMany: {
      filter: { status: "published" },
      update: { $set: { featured: false } }
    }
  },

  // 操作4：替换一篇文章
  {
    replaceOne: {
      filter: { title: "Redis 缓存最佳实践" },
      replacement: {
        title: "Redis 缓存最佳实践（第二版）",
        content: "全新改版，增加了更多实战案例...",
        author: "zhangsan",
        category: "数据库",
        tags: ["Redis", "缓存", "性能优化", "实战"],
        status: "published",
        viewCount: 210,
        comments: [],
        createdAt: new Date("2024-03-01"),
        updatedAt: new Date()
      }
    }
  },

  // 操作5：删除阅读量为 0 的草稿
  {
    deleteMany: {
      filter: { status: "draft", viewCount: 0 }
    }
  }
])

返回结果：

{
  acknowledged: true,
  insertedCount: 1,
  insertedIds: { '0': ObjectId('665c...') },
  matchedCount: 6,
  modifiedCount: 6,
  deletedCount: 1,
  upsertedCount: 0,
  upsertedIds: {}
}

3.5.3 ordered vs unordered 模式

// ordered: true（默认）—— 按顺序执行，遇错即停
db.articles.bulkWrite(
  [
    { insertOne: { document: { _id: "a1", title: "文章A" } } },
    { insertOne: { document: { _id: "a1", title: "文章A重复" } } },  // 会报错
    { insertOne: { document: { _id: "a3", title: "文章C" } } }       // 不会执行
  ],
  { ordered: true }
)

// ordered: false —— 并行执行，遇错跳过继续
db.articles.bulkWrite(
  [
    { insertOne: { document: { _id: "b1", title: "文章B1" } } },
    { insertOne: { document: { _id: "b1", title: "文章B1重复" } } },  // 报错但跳过
    { insertOne: { document: { _id: "b3", title: "文章B3" } } }       // 仍然执行
  ],
  { ordered: false }
)

选择建议：

场景 推荐模式 原因
操作之间有依赖关系 ordered: true 保证执行顺序
大批量独立操作 ordered: false 性能更高，不会因单条失败而全部停止
数据迁移/同步 ordered: false 容忍部分失败，提高吞吐量
业务逻辑要求原子性 考虑事务 bulkWrite 不是事务，无法保证原子性
