7.1 聚合管道概述与执行流程

7.1.1 什么是聚合管道

聚合管道（Aggregation Pipeline）是 MongoDB 提供的一套数据处理框架，它允许你对集合中的文档进行一系列有序的转换操作，最终输出处理后的结果。

你可以把它想象成一条工厂流水线：

原始文档 → [阶段1: 过滤] → [阶段2: 分组] → [阶段3: 计算] → [阶段4: 排序] → 最终结果

每个阶段（Stage）接收上一个阶段的输出作为输入，进行特定的处理后，将结果传递给下一个阶段。

与 SQL 的对应关系：

SQL 概念 聚合管道阶段
WHERE $match
GROUP BY $group
SELECT (列选择) $project
ORDER BY $sort
LIMIT $limit
OFFSET / SKIP $skip
COUNT(*) $count
LEFT JOIN $lookup
HAVING $match（放在 $group 之后）
AS (别名) $project（字段重命名）

7.1.2 基本语法

db.collection.aggregate([
  { <stage1>: { <stage1-specification> } },
  { <stage2>: { <stage2-specification> } },
  { <stage3>: { <stage3-specification> } },
  // ... 可以有任意多个阶段
])

最简单的聚合查询示例：

// 查询已完成的订单，按金额降序排列，只返回前3条
db.orders.aggregate([
  { $match: { status: "completed" } },
  { $sort: { totalAmount: -1 } },
  { $limit: 3 }
])

7.1.3 管道执行流程详解

以下面这个查询为例，用文字描述每个阶段的数据流转过程：

// 需求：统计每个客户的已完成订单总金额，只看金额大于500的客户，按金额降序
db.orders.aggregate([
  { $match: { status: "completed" } },
  { $group: { _id: "$customerId", totalSpent: { $sum: "$totalAmount" } } },
  { $match: { totalSpent: { $gt: 500 } } },
  { $sort: { totalSpent: -1 } }
])

7.1.4 聚合管道的优化原则

  1. 尽早过滤：将 $match 放在管道的最前面，减少后续阶段处理的文档数量。
  2. 利用索引：管道中第一个 $match 阶段可以利用索引加速查询。
  3. 减少字段：尽早使用 $project 去掉不需要的字段，减少内存占用。
  4. 内存限制：单个聚合管道阶段默认最多使用 100MB 内存。超出时可设置 allowDiskUse: true。
// 大数据量时允许使用磁盘
db.orders.aggregate([
  { $group: { _id: "$customerId", total: { $sum: "$totalAmount" } } },
  { $sort: { total: -1 } }
], { allowDiskUse: true })

7.2 常用阶段详解

7.2.1 $match —— 过滤文档

$match 的语法与 find() 的查询条件完全相同，它会过滤掉不满足条件的文档。

示例1：简单条件过滤

// 查询已完成的订单
db.orders.aggregate([
  { $match: { status: "completed" } }
])

示例2：多条件组合过滤

// 查询2024年4月之后、金额大于500的已完成订单
db.orders.aggregate([
  {
    $match: {
      status: "completed",
      orderDate: { $gte: new Date("2024-04-01") },
      totalAmount: { $gt: 500 }
    }
  }
])

示例3：使用 $or 条件

// 查询状态为 completed 或 shipped 的订单
db.orders.aggregate([
  {
    $match: {
      $or: [
        { status: "completed" },
        { status: "shipped" }
      ]
    }
  }
])

最佳实践：$match 应尽量放在管道的最前面。放在第一位的 $match 可以使用集合上的索引，大幅提升性能。

7.2.2 $group —— 分组聚合

$group 是聚合管道中最核心的阶段，它按照指定字段将文档分组，并对每组文档进行各种聚合运算。

基本语法：

{
  $group: {
    _id: <分组表达式>,         // 按什么字段分组，null 表示所有文档为一组
    <字段名>: { <累加器>: <表达式> },
    <字段名>: { <累加器>: <表达式> }
  }
}

常用累加器（Accumulator）：

累加器 说明 示例
$sum 求和 { $sum: "$totalAmount" }
$avg 平均值 { $avg: "$totalAmount" }
$min 最小值 { $min: "$totalAmount" }
$max 最大值 { $max: "$totalAmount" }
$first 每组第一个值 { $first: "$orderNo" }
$last 每组最后一个值 { $last: "$orderNo" }
$push 收集为数组 { $push: "$orderNo" }
$addToSet 收集为数组（去重） { $addToSet: "$status" }
$count 计数 { $count: {} }

示例1：统计每个客户的订单数和总金额

db.orders.aggregate([
  {
    $group: {
      _id: "$customerId",
      customerName: { $first: "$customerName" },
      orderCount: { $sum: 1 },
      totalAmount: { $sum: "$totalAmount" }
    }
  }
])

输出：

{ _id: "C001", customerName: "张三", orderCount: 3, totalAmount: 1203 }
{ _id: "C002", customerName: "李四", orderCount: 2, totalAmount: 2626 }
{ _id: "C003", customerName: "王五", orderCount: 1, totalAmount: 1298 }
{ _id: "C004", customerName: "赵六", orderCount: 1, totalAmount: 89 }
{ _id: "C005", customerName: "孙七", orderCount: 1, totalAmount: 4697 }

示例2：统计每种支付方式的使用次数和平均金额

db.orders.aggregate([
  {
    $group: {
      _id: "$paymentMethod",
      count: { $sum: 1 },
      avgAmount: { $avg: "$totalAmount" },
      minAmount: { $min: "$totalAmount" },
      maxAmount: { $max: "$totalAmount" }
    }
  }
])

示例3：统计所有订单的汇总信息（不分组）

当 _id 设置为 null 时，所有文档归为一组：

db.orders.aggregate([
  {
    $group: {
      _id: null,
      totalOrders: { $sum: 1 },
      totalRevenue: { $sum: "$totalAmount" },
      avgOrderAmount: { $avg: "$totalAmount" },
      maxOrderAmount: { $max: "$totalAmount" },
      minOrderAmount: { $min: "$totalAmount" }
    }
  }
])

示例4：按多个字段分组

// 按客户和支付方式分组统计
db.orders.aggregate([
  {
    $group: {
      _id: {
        customer: "$customerId",
        payment: "$paymentMethod"
      },
      orderCount: { $sum: 1 },
      totalAmount: { $sum: "$totalAmount" }
    }
  }
])

示例5：使用 $push 收集每个客户的订单号列表

db.orders.aggregate([
  {
    $group: {
      _id: "$customerId",
      customerName: { $first: "$customerName" },
      orderNos: { $push: "$orderNo" },
      statuses: { $addToSet: "$status" }
    }
  }
])

输出：

{
  _id: "C001",
  customerName: "张三",
  orderNos: ["ORD20240001", "ORD20240003", "ORD20240008"],
  statuses: ["completed", "cancelled"]   // addToSet 自动去重
}

7.2.3 $project —— 投影 / 重塑文档

$project 用于控制输出文档的结构，可以包含、排除、重命名字段，以及创建计算字段。

示例1：选择和排除字段

db.orders.aggregate([
  {
    $project: {
      _id: 0,                    // 排除 _id
      orderNo: 1,                // 包含 orderNo
      customerName: 1,           // 包含 customerName
      totalAmount: 1,            // 包含 totalAmount
      status: 1                  // 包含 status
    }
  }
])

示例2：字段重命名

db.orders.aggregate([
  {
    $project: {
      _id: 0,
      订单号: "$orderNo",
      客户名: "$customerName",
      金额: "$totalAmount",
      状态: "$status"
    }
  }
])

示例3：计算字段

// 计算每笔订单的商品数量和折扣后金额
db.orders.aggregate([
  {
    $project: {
      orderNo: 1,
      customerName: 1,
      totalAmount: 1,
      itemCount: { $size: "$items" },
      discountedAmount: { $multiply: ["$totalAmount", 0.9] }
    }
  }
])

示例4：嵌套字段提取

// 提取订单中第一个商品的名称
db.orders.aggregate([
  {
    $project: {
      orderNo: 1,
      customerName: 1,
      firstProduct: { $arrayElemAt: ["$items.productName", 0] },
      totalAmount: 1
    }
  }
])

7.2.4 $sort —— 排序

示例1：单字段排序

// 按订单金额降序
db.orders.aggregate([
  { $sort: { totalAmount: -1 } }   // 1 升序，-1 降序
])

示例2：多字段排序

// 先按状态升序，再按金额降序
db.orders.aggregate([
  { $sort: { status: 1, totalAmount: -1 } }
])

注意：当 $sort 紧跟在 $match 之后且排序字段有索引时，MongoDB 可以利用索引进行排序，性能更好。

7.2.5 $limit / $skip —— 限制与跳过

$limit：限制输出文档数量

// 查询金额最高的 3 笔订单
db.orders.aggregate([
  { $sort: { totalAmount: -1 } },
  { $limit: 3 }
])

$skip：跳过指定数量的文档

// 分页查询：第2页，每页3条
db.orders.aggregate([
  { $sort: { orderDate: -1 } },
  { $skip: 3 },    // 跳过前3条（第1页）
  { $limit: 3 }    // 取3条（第2页）
])

提示：$skip 和 $limit 通常配合 $sort 使用，确保结果顺序一致。

7.2.6 $count —— 计数

$count 阶段统计输入到该阶段的文档数量，并输出一个包含计数的文档。

// 统计已完成订单的数量
db.orders.aggregate([
  { $match: { status: "completed" } },
  { $count: "completedOrderCount" }
])
// 输出：{ completedOrderCount: 5 }

// 统计金额超过1000的订单数
db.orders.aggregate([
  { $match: { totalAmount: { $gt: 1000 } } },
  { $count: "highValueOrders" }
])
// 输出：{ highValueOrders: 3 }

7.2.7 综合示例：串联多个阶段

需求：查询每个客户的已完成订单统计信息，只显示消费总额大于 500 的客户，按总额降序，返回客户名、订单数、总额和平均订单金额。

db.orders.aggregate([
  // 阶段1：过滤已完成订单
  { $match: { status: "completed" } },

  // 阶段2：按客户分组统计
  {
    $group: {
      _id: "$customerId",
      customerName: { $first: "$customerName" },
      orderCount: { $sum: 1 },
      totalSpent: { $sum: "$totalAmount" },
      avgOrderAmount: { $avg: "$totalAmount" }
    }
  },

  // 阶段3：过滤总消费大于500的客户（相当于 HAVING）
  { $match: { totalSpent: { $gt: 500 } } },

  // 阶段4：重塑输出文档
  {
    $project: {
      _id: 0,
      customerId: "$_id",
      customerName: 1,
      orderCount: 1,
      totalSpent: 1,
      avgOrderAmount: { $round: ["$avgOrderAmount", 2] }
    }
  },

  // 阶段5：按总消费降序
  { $sort: { totalSpent: -1 } }
])

输出：

{ customerId: "C005", customerName: "孙七", orderCount: 1, totalSpent: 4697, avgOrderAmount: 4697 }
{ customerId: "C002", customerName: "李四", orderCount: 2, totalSpent: 2626, avgOrderAmount: 1313 }
{ customerId: "C001", customerName: "张三", orderCount: 2, totalSpent: 904, avgOrderAmount: 452 }

7.3 进阶阶段

7.3.1 $lookup —— 关联查询

$lookup 实现了类似 SQL 中 LEFT JOIN 的功能，可以将当前集合与另一个集合的数据关联起来。

基本语法：

{
  $lookup: {
    from: "<关联的集合名>",
    localField: "<当前集合的关联字段>",
    foreignField: "<关联集合的匹配字段>",
    as: "<输出数组字段名>"
  }
}

示例1：订单关联商品详细信息

假设我们需要展开订单中的每个商品条目，并关联到商品集合获取完整信息：

db.orders.aggregate([
  // 先展开订单中的商品数组
  { $unwind: "$items" },

  // 关联商品集合
  {
    $lookup: {
      from: "products",
      localField: "items.productId",
      foreignField: "_id",
      as: "productDetail"
    }
  },

  // productDetail 是数组，取第一个元素
  { $unwind: "$productDetail" },

  // 投影输出
  {
    $project: {
      orderNo: 1,
      customerName: 1,
      "items.productName": 1,
      "items.quantity": 1,
      "items.price": 1,
      "productDetail.category": 1,
      "productDetail.stock": 1,
      "productDetail.tags": 1
    }
  }
])

示例2：查询每个商品被购买的次数

db.products.aggregate([
  // 将 orders 中 items.productId 与 products._id 关联
  {
    $lookup: {
      from: "orders",
      let: { pid: "$_id" },
      pipeline: [
        { $unwind: "$items" },
        { $match: { $expr: { $eq: ["$items.productId", "$$pid"] } } },
        {
          $group: {
            _id: null,
            totalQuantity: { $sum: "$items.quantity" },
            orderCount: { $sum: 1 }
          }
        }
      ],
      as: "salesInfo"
    }
  },

  // 展开关联结果（可能为空数组，用 preserveNullAndEmptyArrays 保留没有销售记录的商品）
  {
    $unwind: {
      path: "$salesInfo",
      preserveNullAndEmptyArrays: true
    }
  },

  // 重塑输出
  {
    $project: {
      name: 1,
      category: 1,
      price: 1,
      totalSold: { $ifNull: ["$salesInfo.totalQuantity", 0] },
      orderCount: { $ifNull: ["$salesInfo.orderCount", 0] }
    }
  },

  { $sort: { totalSold: -1 } }
])

$lookup 的管道语法（Pipeline 形式）：

上面的示例2使用了 $lookup 的高级语法，它比基本语法更强大，支持在关联时使用额外的过滤条件：

{
  $lookup: {
    from: "<关联集合>",
    let: { <变量名>: <当前集合的字段表达式> },   // 声明变量
    pipeline: [                                  // 在关联集合上执行的子管道
      { $match: { $expr: { ... } } },
      { $project: { ... } }
    ],
    as: "<输出字段>"
  }
}

示例3：关联时只取特定状态的订单

// 查询每个商品的已完成订单销售情况
db.products.aggregate([
  {
    $lookup: {
      from: "orders",
      let: { productId: "$_id" },
      pipeline: [
        { $match: { status: "completed" } },
        { $unwind: "$items" },
        { $match: { $expr: { $eq: ["$items.productId", "$$productId"] } } },
        {
          $group: {
            _id: null,
            revenue: { $sum: { $multiply: ["$items.quantity", "$items.price"] } }
          }
        }
      ],
      as: "completedSales"
    }
  },
  {
    $project: {
      name: 1,
      price: 1,
      revenue: {
        $ifNull: [{ $arrayElemAt: ["$completedSales.revenue", 0] }, 0]
      }
    }
  },
  { $sort: { revenue: -1 } }
])

7.3.2 $unwind —— 展开数组字段

$unwind 将文档中的数组字段拆分，为数组中的每个元素生成一个独立的文档。

基本语法：

// 简写形式
{ $unwind: "$arrayField" }

// 完整形式
{
  $unwind: {
    path: "$arrayField",
    includeArrayIndex: "indexFieldName",          // 可选：输出数组元素的索引
    preserveNullAndEmptyArrays: true              // 可选：保留空数组或不存在的字段
  }
}

示例1：展开订单的商品列表

db.orders.aggregate([
  { $match: { orderNo: "ORD20240001" } },
  { $unwind: "$items" }
])

输出：一个文档变成了两个文档（items 数组有 2 个元素）

{
  orderNo: "ORD20240001", customerName: "张三",
  items: { productId: "P001", productName: "机械键盘", quantity: 1, price: 399 },
  totalAmount: 657, status: "completed"
}
{
  orderNo: "ORD20240001", customerName: "张三",
  items: { productId: "P002", productName: "无线鼠标", quantity: 2, price: 129 },
  totalAmount: 657, status: "completed"
}

示例2：展开并保留数组索引

db.orders.aggregate([
  { $match: { orderNo: "ORD20240001" } },
  {
    $unwind: {
      path: "$items",
      includeArrayIndex: "itemIndex"
    }
  },
  { $project: { orderNo: 1, itemIndex: 1, "items.productName": 1, "items.price": 1 } }
])

输出：

{ orderNo: "ORD20240001", itemIndex: 0, items: { productName: "机械键盘", price: 399 } }
{ orderNo: "ORD20240001", itemIndex: 1, items: { productName: "无线鼠标", price: 129 } }

示例3：preserveNullAndEmptyArrays 的作用

// 默认行为：如果 items 为空数组或不存在，该文档会被丢弃
db.orders.aggregate([
  { $unwind: "$items" }
])

// 设置 preserveNullAndEmptyArrays: true 后，空数组的文档也会保留
db.orders.aggregate([
  {
    $unwind: {
      path: "$items",
      preserveNullAndEmptyArrays: true
    }
  }
])

示例4：利用 $unwind 实现"最畅销商品"统计

db.orders.aggregate([
  { $match: { status: "completed" } },
  { $unwind: "$items" },
  {
    $group: {
      _id: "$items.productId",
      productName: { $first: "$items.productName" },
      totalQuantity: { $sum: "$items.quantity" },
      totalRevenue: { $sum: { $multiply: ["$items.quantity", "$items.price"] } }
    }
  },
  { $sort: { totalQuantity: -1 } },
  { $limit: 5 }
])

7.3.3 $addFields / $set —— 添加计算字段

$addFields 和 $set（MongoDB 4.2+ 的别名）用于向文档中添加新字段或覆盖已有字段，而不影响其他字段。与 $project 不同的是，它不会移除未提及的字段。

示例1：添加折扣后金额字段

db.orders.aggregate([
  {
    $addFields: {
      discountedAmount: { $multiply: ["$totalAmount", 0.85] },
      itemCount: { $size: "$items" }
    }
  }
])

示例2：使用 $set（$addFields 的别名）

// $set 和 $addFields 功能完全相同，只是名字更简洁
db.orders.aggregate([
  {
    $set: {
      isHighValue: { $gte: ["$totalAmount", 1000] },
      year: { $year: "$orderDate" },
      month: { $month: "$orderDate" }
    }
  }
])

示例3：基于条件添加字段

db.orders.aggregate([
  {
    $addFields: {
      orderLevel: {
        $switch: {
          branches: [
            { case: { $gte: ["$totalAmount", 3000] }, then: "大额订单" },
            { case: { $gte: ["$totalAmount", 1000] }, then: "中额订单" },
            { case: { $gte: ["$totalAmount", 500] }, then: "普通订单" }
          ],
          default: "小额订单"
        }
      }
    }
  },
  { $project: { orderNo: 1, totalAmount: 1, orderLevel: 1 } }
])

7.3.4 $bucket / $bucketAuto —— 分桶统计

$bucket 将文档按指定字段的值分成多个范围区间（桶），类似于直方图统计。

$bucket 语法：

{
  $bucket: {
    groupBy: <表达式>,           // 分桶依据的字段
    boundaries: [b1, b2, b3],    // 桶的边界值（左闭右开）
    default: "<其他桶名>",        // 超出边界的文档归入此桶
    output: {                    // 每个桶的输出字段
      count: { $sum: 1 },
      ...
    }
  }
}

示例1：按订单金额分桶统计

db.orders.aggregate([
  {
    $bucket: {
      groupBy: "$totalAmount",
      boundaries: [0, 100, 500, 1000, 5000],
      default: "5000以上",
      output: {
        count: { $sum: 1 },
        orders: { $push: "$orderNo" },
        avgAmount: { $avg: "$totalAmount" }
      }
    }
  }
])

输出：

{ _id: 0,    count: 1, orders: ["ORD20240005"], avgAmount: 89 }          // [0, 100)
{ _id: 100,  count: 2, orders: ["ORD20240003", "ORD20240008"], avgAmount: 273 }  // [100, 500)
{ _id: 500,  count: 2, orders: ["ORD20240001", "ORD20240006"], avgAmount: 692 }  // [500, 1000)
{ _id: 1000, count: 3, orders: ["ORD20240002", "ORD20240004", "ORD20240007"], avgAmount: 2631.33 } // [1000, 5000)

示例2：按商品价格分桶

db.products.aggregate([
  {
    $bucket: {
      groupBy: "$price",
      boundaries: [0, 100, 300, 500, 1000, 2000],
      default: "2000以上",
      output: {
        count: { $sum: 1 },
        products: { $push: "$name" },
        avgPrice: { $avg: "$price" }
      }
    }
  }
])

$bucketAuto —— 自动分桶：

$bucketAuto 让 MongoDB 自动决定桶的边界值，你只需要指定桶的数量。

// 将订单自动分成4个桶
db.orders.aggregate([
  {
    $bucketAuto: {
      groupBy: "$totalAmount",
      buckets: 4,
      output: {
        count: { $sum: 1 },
        avgAmount: { $avg: "$totalAmount" },
        minAmount: { $min: "$totalAmount" },
        maxAmount: { $max: "$totalAmount" }
      }
    }
  }
])

7.3.5 $facet —— 多面聚合

$facet 允许在一次聚合操作中并行执行多条管道，每条管道独立计算，最终合并为一个文档输出。非常适合需要从同一数据集中获取多维度统计结果的场景。

示例1：订单多维度统计看板

db.orders.aggregate([
  {
    $facet: {
      // 维度1：按状态统计
      byStatus: [
        { $group: { _id: "$status", count: { $sum: 1 }, total: { $sum: "$totalAmount" } } },
        { $sort: { count: -1 } }
      ],

      // 维度2：按支付方式统计
      byPayment: [
        { $group: { _id: "$paymentMethod", count: { $sum: 1 }, total: { $sum: "$totalAmount" } } },
        { $sort: { total: -1 } }
      ],

      // 维度3：金额最高的3笔订单
      topOrders: [
        { $sort: { totalAmount: -1 } },
        { $limit: 3 },
        { $project: { orderNo: 1, customerName: 1, totalAmount: 1 } }
      ],

      // 维度4：总体汇总
      summary: [
        {
          $group: {
            _id: null,
            totalOrders: { $sum: 1 },
            totalRevenue: { $sum: "$totalAmount" },
            avgOrderAmount: { $avg: "$totalAmount" }
          }
        }
      ]
    }
  }
])

输出一个包含四个数组字段的文档：

{
  byStatus: [
    { _id: "completed", count: 5, total: 8270 },
    { _id: "shipped", count: 1, total: 1298 },
    { _id: "pending", count: 1, total: 89 },
    { _id: "cancelled", count: 1, total: 299 }
  ],
  byPayment: [
    { _id: "alipay", count: 4, total: 1932 },
    { _id: "credit_card", count: 2, total: 5995 },
    { _id: "wechat", count: 2, total: 1988 }
  ],
  topOrders: [
    { orderNo: "ORD20240007", customerName: "孙七", totalAmount: 4697 },
    { orderNo: "ORD20240002", customerName: "李四", totalAmount: 1899 },
    { orderNo: "ORD20240004", customerName: "王五", totalAmount: 1298 }
  ],
  summary: [
    { _id: null, totalOrders: 8, totalRevenue: 9956, avgOrderAmount: 1244.5 }
  ]
}

示例2：分页查询 + 总数（一次请求实现）

db.orders.aggregate([
  { $match: { status: "completed" } },
  {
    $facet: {
      // 分页数据
      data: [
        { $sort: { orderDate: -1 } },
        { $skip: 0 },
        { $limit: 3 },
        { $project: { orderNo: 1, customerName: 1, totalAmount: 1, orderDate: 1 } }

7.4 聚合表达式与运算符

聚合管道中的表达式和运算符用于在 $project、$addFields、$group 等阶段中执行计算和数据转换。

7.4.1 算术运算符

运算符 说明 语法
$add 加法（也可用于日期加毫秒） { $add: [expr1, expr2] }
$subtract 减法（也可用于日期差） { $subtract: [expr1, expr2] }
$multiply 乘法 { $multiply: [expr1, expr2] }
$divide 除法 { $divide: [expr1, expr2] }
$mod 取模 { $mod: [expr1, expr2] }
$round 四舍五入 { $round: [expr, 小数位数] }
$ceil 向上取整 { $ceil: expr }
$floor 向下取整 { $floor: expr }
$abs 绝对值 { $abs: expr }

示例：计算订单的税费和最终价格

db.orders.aggregate([
  {
    $project: {
      orderNo: 1,
      totalAmount: 1,
      tax: {
        $round: [{ $multiply: ["$totalAmount", 0.13] }, 2]
      },
      finalPrice: {
        $round: [{ $multiply: ["$totalAmount", 1.13] }, 2]
      },
      discount: {
        $subtract: ["$totalAmount", { $multiply: ["$totalAmount", 0.85] }]
      }
    }
  }
])

示例：计算日期差（订单处理天数）

db.orders.aggregate([
  { $match: { completedDate: { $exists: true } } },
  {
    $project: {
      orderNo: 1,
      orderDate: 1,
      completedDate: 1,
      processingDays: {
        $divide: [
          { $subtract: ["$completedDate", "$orderDate"] },
          1000 * 60 * 60 * 24   // 毫秒转天数
        ]
      }
    }
  }
])

7.4.2 字符串运算符

运算符 说明 语法
$concat 字符串拼接 { $concat: [str1, str2, ...] }
$substr / $substrBytes 截取子串 { $substr: [str, start, length] }
$toUpper 转大写 { $toUpper: expr }
$toLower 转小写 { $toLower: expr }
$trim 去除首尾空格 { $trim: { input: str } }
$split 按分隔符拆分 { $split: [str, delimiter] }
$strLenCP 字符串长度 { $strLenCP: expr }
$regexMatch 正则匹配 { $regexMatch: { input: str, regex: pattern } }

示例：格式化订单信息

db.orders.aggregate([
  {
    $project: {
      orderInfo: {
        $concat: ["订单号: ", "$orderNo", " | 客户: ", "$customerName"]
      },
      orderNoUpper: { $toUpper: "$orderNo" },
      orderPrefix: { $substr: ["$orderNo", 0, 3] },   // 截取前3个字符
      customerNameLength: { $strLenCP: "$customerName" }
    }
  }
])

示例：判断客户名中是否包含特定字符

db.orders.aggregate([
  {
    $project: {
      orderNo: 1,
      customerName: 1,
      isZhang: {
        $regexMatch: { input: "$customerName", regex: "^张" }
      }
    }
  }
])

运算符 说明 语法
$year 提取年份 { $year: dateExpr }
$month 提取月份 (1-12) { $month: dateExpr }
$dayOfMonth 提取日 (1-31) { $dayOfMonth: dateExpr }
$dayOfWeek 星期几 (1=周日, 7=周六) { $dayOfWeek: dateExpr }
$hour 小时 (0-23) { $hour: dateExpr }
$dateToString 日期格式化为字符串 { $dateToString: { format: ..., date: ... } }
$dateFromString 字符串解析为日期 { $dateFromString: { dateString: ..., format: ... } }

示例1：按月份统计订单

db.orders.aggregate([
  {
    $group: {
      _id: {
        year: { $year: "$orderDate" },
        month: { $month: "$orderDate" }
      },
      orderCount: { $sum: 1 },
      totalAmount: { $sum: "$totalAmount" }
    }
  },
  { $sort: { "_id.year": 1, "_id.month": 1 } }
])

示例2：日期格式化输出

db.orders.aggregate([
  {
    $project: {
      orderNo: 1,
      formattedDate: {
        $dateToString: {
          format: "%Y年%m月%d日",
          date: "$orderDate",
          timezone: "Asia/Shanghai"
        }
      },
      year: { $year: "$orderDate" },
      month: { $month: "$orderDate" },

    }
  }
])

示例3：按季度统计

db.orders.aggregate([
  {
    $addFields: {
      quarter: {
        $ceil: { $divide: [{ $month: "$orderDate" }, 3] }
      }
    }
  },
  {
    $group: {
      _id: {
        year: { $year: "$orderDate" },

])

$cond —— 三元条件（if-then-else）

// 根据金额标记订单级别
db.orders.aggregate([
  {
    $project: {
      orderNo: 1,
      totalAmount: 1,
      level: {
        $cond: {
          if: { $gte: ["$totalAmount", 1000] },
          then: "高价值",
          else: "普通"
        }
      }
    }
  }
])

// 简写形式：$cond: [ <条件>, <true值>, <false值> ]
db.orders.aggregate([
  {
    $project: {
      orderNo: 1,
      totalAmount: 1,
      level: {
        $cond: [{ $gte: ["$totalAmount", 1000] }, "高价值", "普通"]
      }
    }
  }
])

$ifNull —— 空值替换

// 如果 completedDate 为 null，显示 "未完成"
db.orders.aggregate([
  {
    $project: {
      orderNo: 1,
      status: 1,
      completedDate: {
        $ifNull: ["$completedDate", "未完成"]
      }
    }
  }
])

$switch —— 多分支条件（类似 switch-case）

db.orders.aggregate([
  {
    $project: {
      orderNo: 1,
      totalAmount: 1,
      statusCn: {
        $switch: {
          branches: [
            { case: { $eq: ["$status", "completed"] }, then: "已完成" },
            { case: { $eq: ["$status", "shipped"] }, then: "已发货" },
            { case: { $eq: ["$status", "pending"] }, then: "待处理" },
            { case: { $eq: ["$status", "cancelled"] }, then: "已取消" }

        }
      }

  }
])

7.5.2 示例1：简单的 $match + $group（按客户统计订单）

package com.example.mongo.service;

import org.springframework.data.mongodb.core.MongoTemplate;
import org.springframework.data.mongodb.core.aggregation.Aggregation;
import org.springframework.data.mongodb.core.aggregation.AggregationResults;
import org.springframework.data.mongodb.core.query.Criteria;
import org.springframework.stereotype.Service;
import java.util.List;

@Service
public class OrderAggregationService {

    private final MongoTemplate mongoTemplate;

    public OrderAggregationService(MongoTemplate mongoTemplate) {
        this.mongoTemplate = mongoTemplate;
    }

    /**
     * 统计每个客户的已完成订单数和总金额
     */
    public List<CustomerOrderStats> getCustomerOrderStats() {
        Aggregation aggregation = Aggregation.newAggregation(
            // $match: 过滤已完成订单
            Aggregation.match(Criteria.where("status").is("completed")),

            // $group: 按客户分组
            Aggregation.group("customerId")
                .first("customerName").as("customerName")
                .count().as("orderCount")
                .sum("totalAmount").as("totalAmount")
                .avg("totalAmount").as("avgAmount"),

            // $sort: 按总金额降序
            Aggregation.sort(org.springframework.data.domain.Sort.by(
                org.springframework.data.domain.Sort.Direction.DESC, "totalAmount"
            ))
        );

        AggregationResults<CustomerOrderStats> results =
            mongoTemplate.aggregate(aggregation, "orders", CustomerOrderStats.class);

        return results.getMappedResults();
    }

    // 结果映射类
    public static class CustomerOrderStats {
        private String id;           // _id（customerId）
        private String customerName;
        private int orderCount;
        private double totalAmount;
        private double avgAmount;

        // getter / setter
        public String getId() { return id; }
        public void setId(String id) { this.id = id; }
        public String getCustomerName() { return customerName; }
        public void setCustomerName(String customerName) { this.customerName = customerName; }
        public int getOrderCount() { return orderCount; }
        public void setOrderCount(int orderCount) { this.orderCount = orderCount; }
        public double getTotalAmount() { return totalAmount; }
        public void setTotalAmount(double totalAmount) { this.totalAmount = totalAmount; }

    }

    /**
     * 获取订单看板数据：按状态统计 + 按支付方式统计 + Top3 订单
     */
    public Map<String, Object> getDashboardData() {
        // 按状态统计子管道
        MatchOperation matchCompleted = Aggregation.match(Criteria.where("status").is("completed"));
        GroupOperation groupByStatus = Aggregation.group("status")
            .count().as("count")
            .sum("totalAmount").as("total");

        // 按支付方式统计子管道
        GroupOperation groupByPayment = Aggregation.group("paymentMethod")
            .count().as("count")
            .sum("totalAmount").as("total");

        // Top3 订单子管道
        SortOperation sortByAmount = Aggregation.sort(Sort.by(Sort.Direction.DESC, "totalAmount"));
        LimitOperation limitTop3 = Aggregation.limit(3);
        ProjectionOperation projectTop3 = Aggregation.project("orderNo", "customerName", "totalAmount");

        // 构建 $facet
        FacetOperation facetOperation = Aggregation.facet(groupByStatus, Aggregation.sort(Sort.by(Sort.Direction.DESC, "count")))
            .as("byStatus")
            .and(groupByPayment, Aggregation.sort(Sort.by(Sort.Direction.DESC, "total")))
            .as("byPayment")
            .and(sortByAmount, limitTop3, projectTop3)
            .as("topOrders");

        Aggregation aggregation = Aggregation.newAggregation(facetOperation);

        AggregationResults<Map> results =
            mongoTemplate.aggregate(aggregation, "orders", Map.class);

        return results.getUniqueMappedResult();
    }

本章小结

知识点 核心内容
聚合管道概念 数据流经多个阶段依次处理，类似工厂流水线
$match 过滤文档，语法同 find()，尽量放在管道最前面以利用索引
$group 分组聚合，支持 $sum, $avg, $min, $max, $first, $last, $push 等累加器
$project 投影 / 重塑文档，支持字段选择、重命名、计算字段
$sort / $limit / $skip 排序、限制、跳过，常用于分页场景
$count 计算文档数量
$lookup 关联查询（LEFT JOIN），支持基本语法和 Pipeline 高级语法
$unwind 展开数组字段，每个元素生成独立文档
$addFields / $set 添加或覆盖字段，不影响其他字段
$bucket / $bucketAuto 按范围分桶统计，适合直方图 / 区间分析
$facet 多面聚合，一次查询返回多维度结果
聚合表达式 算术、字符串、日期、条件、数组等丰富的运算符
Spring Boot Aggregation 使用 Aggregation.newAggregation() 构建管道，MongoTemplate.aggregate() 执行
TypedAggregation 类型安全的聚合方式，自动从 @Document 获取集合名
