第12章 GridFS 文件存储

MongoDB 的文档大小上限为 16MB，但实际项目中经常需要存储图片、视频、PDF 等大文件。 GridFS 是 MongoDB 提供的一种文件存储规范，它将大文件拆分为多个小块（chunk）分别存储， 突破了 16MB 的限制。本章将从原理到实践，系统讲解 GridFS 的使用方法。

12.1 GridFS 的原理与适用场景

12.1.1 GridFS 的工作原理

GridFS 并不是一个独立的存储引擎，而是 MongoDB 驱动层面的一种文件存储规范。它将一个文件拆分为多个固定大小的 chunk（默认 255KB），分别存储在两个集合中：

                        ┌─────────────────────┐
   大文件（如 10MB）     │     GridFS 存储      │
  ──────────────────>   │                     │
                        │  fs.files 集合       │  ← 文件元数据（文件名、大小、类型等）
                        │  ┌─────────────┐    │
                        │  │ fileId       │    │
                        │  │ filename     │    │
                        │  │ length       │    │
                        │  │ chunkSize    │    │
                        │  │ uploadDate   │    │
                        │  │ metadata     │    │
                        │  └─────────────┘    │
                        │                     │
                        │  fs.chunks 集合      │  ← 文件内容（二进制数据块）
                        │  ┌─────────────┐    │
                        │  │ chunk 0      │    │  255KB
                        │  │ chunk 1      │    │  255KB
                        │  │ chunk 2      │    │  255KB
                        │  │ ...          │    │
                        │  │ chunk N      │    │  剩余部分
                        │  └─────────────┘    │
                        └─────────────────────┘

12.1.2 fs.files 集合结构

fs.files 存储文件的元数据信息：

{
    _id: ObjectId("6571abcd1234567890abcdef"),  // 文件ID
    length: 10485760,                            // 文件大小（字节）
    chunkSize: 261120,                           // 每个 chunk 的大小（默认 255KB = 261120 字节）
    uploadDate: ISODate("2024-01-15T08:30:00Z"), // 上传时间
    filename: "report-2024.pdf",                 // 文件名
    metadata: {                                  // 自定义元数据（可选）
        contentType: "application/pdf",
        uploadedBy: "user001",
        department: "财务部",
        tags: ["年报", "2024"]
    }
}

12.1.3 fs.chunks 集合结构

fs.chunks 存储文件的实际二进制内容：

{
    _id: ObjectId("6571abce..."),
    files_id: ObjectId("6571abcd..."),  // 关联 fs.files 的 _id
    n: 0,                                // chunk 序号（从0开始）
    data: BinData(0, "base64...")        // 二进制数据
}

{
    _id: ObjectId("6571abcf..."),
    files_id: ObjectId("6571abcd..."),
    n: 1,
    data: BinData(0, "base64...")
}

// ... 更多 chunks

自动创建的索引：

// fs.files 上的索引
{ filename: 1, uploadDate: 1 }

// fs.chunks 上的索引（保证唯一性和查询效率）
{ files_id: 1, n: 1 }  // 唯一复合索引

12.1.4 默认 chunk 大小

参数 默认值 说明
chunkSize 255KB（261120 字节） 每个 chunk 的大小
可配置范围 最小不建议低于 4KB，最大不超过 16MB 需要在上传时指定

chunk 大小选择建议：

  • 默认 255KB 适合大多数场景
  • 较大的 chunk（如 1MB）：适合大文件顺序读取，减少 chunk 数量
  • 较小的 chunk（如 64KB）：适合需要随机访问的场景，减少内存占用

12.1.5 适用场景

场景 是否适合使用 GridFS 说明
文件大于 16MB ✅ 非常适合 GridFS 的核心用途
需要部分读取大文件 ✅ 适合 可以只读取特定的 chunks
文件需要与业务数据在同一数据库 ✅ 适合 简化架构，统一备份
小于 16MB 的文件 ⚠️ 可以但非必须 小文件可以直接用 BinData 存在文档中
大量小文件（如缩略图） ❌ 不推荐 每个文件至少占两个文档，开销大
需要 CDN 加速 ❌ 不推荐 对象存储（如 OSS）更适合
海量文件存储（TB级别） ❌ 不推荐 专业对象存储更合适

12.2 文件的上传、下载与删除（Shell / mongofiles）

12.2.1 使用 mongofiles 命令行工具

mongofiles 是 MongoDB 官方提供的 GridFS 命令行工具，包含在 MongoDB Database Tools 中。

安装确认：

mongofiles --version

如果未安装，可以从 MongoDB Database Tools 下载。

12.2.2 常用命令

命令 说明 示例
put 上传文件 mongofiles put myfile.pdf
get 下载文件 mongofiles get myfile.pdf
delete 删除文件 mongofiles delete myfile.pdf
list 列出所有文件 mongofiles list
search 搜索文件 mongofiles search report
get_id 按ID下载 mongofiles get_id "ObjectId(...)"
delete_id 按ID删除 mongofiles delete_id "ObjectId(...)"

12.2.3 完整示例

上传文件：

# 上传到默认数据库 test
mongofiles --db=mydb put report.pdf

# 指定远程连接
mongofiles --uri="mongodb://localhost:27017" --db=mydb put report.pdf

# 上传时指定存储文件名（与本地文件名不同）
mongofiles --db=mydb put --local=./downloads/report-2024-q1.pdf "reports/2024-Q1-财务报告.pdf"

# 使用认证连接
mongofiles --uri="mongodb://admin:password@localhost:27017/mydb?authSource=admin" put avatar.jpg

列出文件：

# 列出所有文件
mongofiles --db=mydb list

# 输出示例：
# report.pdf     2485760
# avatar.jpg     153600
# backup.zip     52428800

# 按前缀过滤
mongofiles --db=mydb list "report"

搜索文件：

# 搜索文件名包含 "report" 的文件
mongofiles --db=mydb search report

下载文件：

# 下载到当前目录
mongofiles --db=mydb get report.pdf

# 下载并保存为不同的文件名
mongofiles --db=mydb get --local=./output/downloaded-report.pdf "report.pdf"

删除文件：

# 按文件名删除
mongofiles --db=mydb delete report.pdf

12.3 在 Spring Boot 中集成 GridFS 实现文件服务

12.3.1 GridFsTemplate 配置

Spring Data MongoDB 内置了对 GridFS 的支持，核心类是 GridFsTemplate。

依赖（pom.xml）：

<dependencies>
    <dependency>
        <groupId>org.springframework.boot</groupId>
        <artifactId>spring-boot-starter-data-mongodb</artifactId>
    </dependency>
    <dependency>
        <groupId>org.springframework.boot</groupId>
        <artifactId>spring-boot-starter-web</artifactId>
    </dependency>
</dependencies>

application.yml：

spring:
  data:
    mongodb:
      uri: mongodb://localhost:27017/mydb
  servlet:
    multipart:
      max-file-size: 100MB       # 最大文件大小
      max-request-size: 100MB    # 最大请求大小

注意：GridFsTemplate 由 Spring Boot 自动配置，无需手动声明 Bean。 只要引入了 spring-boot-starter-data-mongodb，就可以直接注入使用。

12.3.3 完整的文件服务类（FileService）

import com.mongodb.client.gridfs.model.GridFSFile;
import org.bson.Document;
import org.bson.types.ObjectId;
import org.springframework.data.mongodb.core.query.Criteria;
import org.springframework.data.mongodb.core.query.Query;
import org.springframework.data.mongodb.gridfs.GridFsOperations;
import org.springframework.data.mongodb.gridfs.GridFsResource;
import org.springframework.data.mongodb.gridfs.GridFsTemplate;
import org.springframework.stereotype.Service;
import org.springframework.web.multipart.MultipartFile;
import org.slf4j.Logger;
import org.slf4j.LoggerFactory;

import java.io.IOException;
import java.io.InputStream;
import java.util.ArrayList;
import java.util.List;
import java.util.Map;

@Service
public class FileService {

    private static final Logger log = LoggerFactory.getLogger(FileService.class);

    private final GridFsTemplate gridFsTemplate;
    private final GridFsOperations gridFsOperations;

    public FileService(GridFsTemplate gridFsTemplate, GridFsOperations gridFsOperations) {
        this.gridFsTemplate = gridFsTemplate;
        this.gridFsOperations = gridFsOperations;
    }

    /**
     * 上传文件
     *
     * @param file     上传的文件
     * @param metadata 自定义元数据（可选）
     * @return 文件ID
     */
    public String uploadFile(MultipartFile file, Map<String, Object> metadata) {
        if (file.isEmpty()) {
            throw new IllegalArgumentException("上传文件不能为空");
        }

        try (InputStream inputStream = file.getInputStream()) {
            // 构建元数据文档
            Document metadataDoc = new Document();
            metadataDoc.put("contentType", file.getContentType());
            metadataDoc.put("originalFilename", file.getOriginalFilename());
            metadataDoc.put("size", file.getSize());

            if (metadata != null) {
                metadataDoc.putAll(metadata);
            }

            // 存储文件到 GridFS
            ObjectId fileId = gridFsTemplate.store(
                    inputStream,                    // 文件输入流
                    file.getOriginalFilename(),     // 文件名
                    file.getContentType(),          // MIME 类型
                    metadataDoc                     // 元数据
            );

            log.info("文件上传成功: id={}, filename={}, size={}bytes",
                    fileId, file.getOriginalFilename(), file.getSize());

            return fileId.toHexString();

        } catch (IOException e) {
            log.error("文件上传失败: {}", e.getMessage(), e);
            throw new RuntimeException("文件上传失败: " + e.getMessage(), e);
        }
    }

    /**
     * 通过 InputStream 上传文件（适用于非 Web 场景）
     */
    public String uploadFile(InputStream inputStream, String filename,
                             String contentType, Map<String, Object> metadata) {
        Document metadataDoc = new Document();
        metadataDoc.put("contentType", contentType);
        if (metadata != null) {
            metadataDoc.putAll(metadata);
        }

        ObjectId fileId = gridFsTemplate.store(inputStream, filename, contentType, metadataDoc);
        log.info("文件上传成功: id={}, filename={}", fileId, filename);
        return fileId.toHexString();
    }

    /**
     * 根据文件ID查询文件元数据
     */
    public FileInfo getFileInfo(String fileId) {
        GridFSFile gridFSFile = gridFsTemplate.findOne(
                new Query(Criteria.where("_id").is(new ObjectId(fileId)))
        );

        if (gridFSFile == null) {
            throw new FileNotFoundException("文件不存在: " + fileId);
        }

        return convertToFileInfo(gridFSFile);
    }

    /**
     * 根据文件名查询文件
     */
    public FileInfo getFileInfoByName(String filename) {
        GridFSFile gridFSFile = gridFsTemplate.findOne(
                new Query(Criteria.where("filename").is(filename))
        );

        if (gridFSFile == null) {
            throw new FileNotFoundException("文件不存在: " + filename);
        }

        return convertToFileInfo(gridFSFile);
    }

    /**
     * 查询所有文件
     */
    public List<FileInfo> listFiles() {
        List<FileInfo> files = new ArrayList<>();
        gridFsTemplate.find(new Query())
                .into(new ArrayList<>())
                .forEach(file -> files.add(convertToFileInfo(file)));
        return files;
    }

    /**
     * 根据元数据条件查询文件
     */
    public List<FileInfo> findFilesByMetadata(String key, Object value) {
        Query query = new Query(Criteria.where("metadata." + key).is(value));
        List<FileInfo> files = new ArrayList<>();
        gridFsTemplate.find(query)
                .into(new ArrayList<>())
                .forEach(file -> files.add(convertToFileInfo(file)));
        return files;
    }

    /**
     * 下载文件，返回 GridFsResource
     */
    public GridFsResource downloadFile(String fileId) {
        GridFSFile gridFSFile = gridFsTemplate.findOne(
                new Query(Criteria.where("_id").is(new ObjectId(fileId)))
        );

        if (gridFSFile == null) {
            throw new FileNotFoundException("文件不存在: " + fileId);
        }

        return gridFsOperations.getResource(gridFSFile);
    }

    /**
     * 下载文件（按文件名）
     */
    public GridFsResource downloadFileByName(String filename) {
        GridFSFile gridFSFile = gridFsTemplate.findOne(
                new Query(Criteria.where("filename").is(filename))
        );

        if (gridFSFile == null) {
            throw new FileNotFoundException("文件不存在: " + filename);
        }

        return gridFsOperations.getResource(gridFSFile);
    }

    /**
     * 删除文件
     */
    public void deleteFile(String fileId) {
        gridFsTemplate.delete(
                new Query(Criteria.where("_id").is(new ObjectId(fileId)))
        );
        log.info("文件已删除: id={}", fileId);
    }

    /**
     * 按文件名删除
     */
    public void deleteFileByName(String filename) {
        gridFsTemplate.delete(
                new Query(Criteria.where("filename").is(filename))
        );
        log.info("文件已删除: filename={}", filename);
    }

    /**
     * 批量删除（按元数据条件）
     */
    public void deleteFilesByMetadata(String key, Object value) {
        gridFsTemplate.delete(
                new Query(Criteria.where("metadata." + key).is(value))
        );
        log.info("批量删除文件: metadata.{}={}", key, value);
    }

    /**
     * 判断文件是否存在
     */
    public boolean fileExists(String fileId) {
        GridFSFile file = gridFsTemplate.findOne(
                new Query(Criteria.where("_id").is(new ObjectId(fileId)))
        );
        return file != null;
    }

    /**
     * GridFSFile 转 FileInfo
     */
    private FileInfo convertToFileInfo(GridFSFile gridFSFile) {
        FileInfo info = new FileInfo();
        info.setId(gridFSFile.getObjectId().toHexString());
        info.setFilename(gridFSFile.getFilename());
        info.setSize(gridFSFile.getLength());
        info.setUploadDate(gridFSFile.getUploadDate().toInstant());

        Document metadata = gridFSFile.getMetadata();
        if (metadata != null) {
            info.setContentType(metadata.getString("contentType"));
            info.setMetadata(metadata);
        }

        return info;
    }

    /**
     * 自定义异常
     */
    public static class FileNotFoundException extends RuntimeException {
        public FileNotFoundException(String message) {
            super(message);
        }
    }
}

12.3.4 完整的文件上传下载 Controller

import org.springframework.core.io.InputStreamResource;
import org.springframework.data.mongodb.gridfs.GridFsResource;
import org.springframework.http.HttpHeaders;
import org.springframework.http.HttpStatus;
import org.springframework.http.MediaType;
import org.springframework.http.ResponseEntity;
import org.springframework.web.bind.annotation.*;
import org.springframework.web.multipart.MultipartFile;
import org.slf4j.Logger;
import org.slf4j.LoggerFactory;

import java.io.IOException;
import java.net.URLEncoder;
import java.nio.charset.StandardCharsets;
import java.util.HashMap;
import java.util.List;
import java.util.Map;

@RestController
@RequestMapping("/api/files")
public class FileController {

    private static final Logger log = LoggerFactory.getLogger(FileController.class);

    private final FileService fileService;

    public FileController(FileService fileService) {
        this.fileService = fileService;
    }

    /**
     * 上传单个文件
     * POST /api/files/upload
     */
    @PostMapping("/upload")
    public ResponseEntity<Map<String, Object>> uploadFile(
            @RequestParam("file") MultipartFile file,
            @RequestParam(value = "uploadedBy", required = false) String uploadedBy,
            @RequestParam(value = "category", required = false) String category) {

        // 构建自定义元数据
        Map<String, Object> metadata = new HashMap<>();
        if (uploadedBy != null) metadata.put("uploadedBy", uploadedBy);
        if (category != null) metadata.put("category", category);

        String fileId = fileService.uploadFile(file, metadata);

        Map<String, Object> response = new HashMap<>();
        response.put("success", true);
        response.put("fileId", fileId);
        response.put("filename", file.getOriginalFilename());
        response.put("size", file.getSize());
        response.put("contentType", file.getContentType());

        return ResponseEntity.ok(response);
    }

    /**
     * 批量上传文件
     * POST /api/files/upload/batch
     */
    @PostMapping("/upload/batch")
    public ResponseEntity<Map<String, Object>> uploadFiles(
            @RequestParam("files") MultipartFile[] files,
            @RequestParam(value = "category", required = false) String category) {

        List<Map<String, String>> results = new java.util.ArrayList<>();

        for (MultipartFile file : files) {
            Map<String, Object> metadata = new HashMap<>();
            if (category != null) metadata.put("category", category);

            String fileId = fileService.uploadFile(file, metadata);

            Map<String, String> fileResult = new HashMap<>();
            fileResult.put("fileId", fileId);
            fileResult.put("filename", file.getOriginalFilename());
            results.add(fileResult);
        }

        Map<String, Object> response = new HashMap<>();
        response.put("success", true);
        response.put("count", results.size());
        response.put("files", results);

        return ResponseEntity.ok(response);
    }

    /**
     * 下载文件（以附件形式）
     * GET /api/files/download/{fileId}
     */
    @GetMapping("/download/{fileId}")
    public ResponseEntity<InputStreamResource> downloadFile(@PathVariable String fileId) {
        try {
            GridFsResource resource = fileService.downloadFile(fileId);
            FileInfo fileInfo = fileService.getFileInfo(fileId);

            String encodedFilename = URLEncoder.encode(
                    fileInfo.getFilename(), StandardCharsets.UTF_8
            ).replace("+", "%20");

            return ResponseEntity.ok()
                    .contentType(MediaType.parseMediaType(
                            fileInfo.getContentType() != null
                                    ? fileInfo.getContentType()
                                    : MediaType.APPLICATION_OCTET_STREAM_VALUE))
                    .contentLength(fileInfo.getSize())
                    .header(HttpHeaders.CONTENT_DISPOSITION,
                            "attachment; filename*=UTF-8''" + encodedFilename)
                    .body(new InputStreamResource(resource.getInputStream()));

        } catch (FileService.FileNotFoundException e) {
            return ResponseEntity.notFound().build();
        } catch (IOException e) {
            log.error("文件下载失败: {}", e.getMessage(), e);
            return ResponseEntity.status(HttpStatus.INTERNAL_SERVER_ERROR).build();
        }
    }

    /**
     * 在线预览文件（图片、PDF等）
     * GET /api/files/preview/{fileId}
     */
    @GetMapping("/preview/{fileId}")
    public ResponseEntity<InputStreamResource> previewFile(@PathVariable String fileId) {
        try {
            GridFsResource resource = fileService.downloadFile(fileId);
            FileInfo fileInfo = fileService.getFileInfo(fileId);

            return ResponseEntity.ok()
                    .contentType(MediaType.parseMediaType(
                            fileInfo.getContentType() != null
                                    ? fileInfo.getContentType()
                                    : MediaType.APPLICATION_OCTET_STREAM_VALUE))
                    .contentLength(fileInfo.getSize())
                    .header(HttpHeaders.CONTENT_DISPOSITION,
                            "inline; filename*=UTF-8''" + URLEncoder.encode(
                                    fileInfo.getFilename(), StandardCharsets.UTF_8))
                    .body(new InputStreamResource(resource.getInputStream()));

        } catch (FileService.FileNotFoundException e) {
            return ResponseEntity.notFound().build();
        } catch (IOException e) {
            log.error("文件预览失败: {}", e.getMessage(), e);
            return ResponseEntity.status(HttpStatus.INTERNAL_SERVER_ERROR).build();
        }
    }

    /**
     * 获取文件信息
     * GET /api/files/info/{fileId}
     */
    @GetMapping("/info/{fileId}")
    public ResponseEntity<FileInfo> getFileInfo(@PathVariable String fileId) {
        try {
            FileInfo fileInfo = fileService.getFileInfo(fileId);
            return ResponseEntity.ok(fileInfo);
        } catch (FileService.FileNotFoundException e) {
            return ResponseEntity.notFound().build();
        }
    }

    /**
     * 列出所有文件
     * GET /api/files
     */
    @GetMapping
    public ResponseEntity<List<FileInfo>> listFiles() {
        return ResponseEntity.ok(fileService.listFiles());
    }

    /**
     * 按分类查询文件
     * GET /api/files/category/{category}
     */
    @GetMapping("/category/{category}")
    public ResponseEntity<List<FileInfo>> listFilesByCategory(@PathVariable String category) {
        return ResponseEntity.ok(fileService.findFilesByMetadata("category", category));
    }

    /**
     * 删除文件
     * DELETE /api/files/{fileId}
     */
    @DeleteMapping("/{fileId}")
    public ResponseEntity<Map<String, Object>> deleteFile(@PathVariable String fileId) {
        try {
            if (!fileService.fileExists(fileId)) {
                return ResponseEntity.notFound().build();
            }
            fileService.deleteFile(fileId);

            Map<String, Object> response = new HashMap<>();
            response.put("success", true);
            response.put("message", "文件已删除");
            return ResponseEntity.ok(response);

        } catch (Exception e) {
            log.error("文件删除失败: {}", e.getMessage(), e);
            Map<String, Object> response = new HashMap<>();
            response.put("success", false);
            response.put("message", "文件删除失败: " + e.getMessage());
            return ResponseEntity.status(HttpStatus.INTERNAL_SERVER_ERROR).body(response);
        }
    }
}

12.3.5 全局异常处理

import org.springframework.http.HttpStatus;
import org.springframework.http.ResponseEntity;
import org.springframework.web.bind.annotation.ExceptionHandler;
import org.springframework.web.bind.annotation.RestControllerAdvice;
import org.springframework.web.multipart.MaxUploadSizeExceededException;

import java.util.Map;

@RestControllerAdvice
public class FileExceptionHandler {

    @ExceptionHandler(FileService.FileNotFoundException.class)
    public ResponseEntity<Map<String, String>> handleFileNotFound(FileService.FileNotFoundException e) {
        return ResponseEntity.status(HttpStatus.NOT_FOUND)
                .body(Map.of("error", e.getMessage()));
    }

    @ExceptionHandler(MaxUploadSizeExceededException.class)
    public ResponseEntity<Map<String, String>> handleMaxSizeExceeded(MaxUploadSizeExceededException e) {
        return ResponseEntity.status(HttpStatus.PAYLOAD_TOO_LARGE)
                .body(Map.of("error", "文件大小超过限制，最大允许 100MB"));
    }

    @ExceptionHandler(IllegalArgumentException.class)
    public ResponseEntity<Map<String, String>> handleIllegalArgument(IllegalArgumentException e) {
        return ResponseEntity.status(HttpStatus.BAD_REQUEST)
                .body(Map.of("error", e.getMessage()));
    }
}

12.4.2 各方案的优缺点

GridFS 优点：

  • 无需引入额外的存储系统，降低架构复杂度
  • 文件与业务数据在同一数据库，备份恢复一致
  • 支持丰富的元数据查询（利用 MongoDB 查询能力）
  • 可以和业务数据放在同一个事务中
  • 自动处理大文件分块和重组

GridFS 缺点：

  • 占用 MongoDB 存储空间和内存，可能影响业务数据库性能
  • 不支持 CDN 加速，不适合高并发的静态资源访问
  • 读写性能不如专业的文件/对象存储系统
  • 大量文件存储时，MongoDB 的运维压力增大

对象存储（OSS/S3/MinIO）优点：

  • 专为文件存储设计，性能优秀
  • 天然支持 CDN 加速
  • 海量存储能力，成本相对低廉
  • 丰富的权限管理和访问控制

对象存储缺点：

  • 引入额外的系统依赖，增加架构复杂度
  • 文件和业务数据分离，一致性需要额外保证
  • 云服务有网络延迟和带宽成本
  • 自建（MinIO）需要额外的运维投入

12.4.3 选型建议

场景 推荐方案 原因
小型项目 / MVP GridFS 简化架构，快速交付，不需要额外组件
文件与业务数据强关联 GridFS 利用 MongoDB 的查询和事务能力
文件数量少（< 1万）、总量小（< 10GB） GridFS 完全够用，没必要引入额外系统
需要公网访问 / CDN 加速 云 OSS CDN 是刚需，GridFS 无法满足
大量静态资源（图片/视频） 云 OSS / MinIO 专业的对象存储更合适
海量文件（TB 级别） 云 OSS / MinIO 成本和性能更优
私有化部署 + 大量文件 MinIO S3 兼容 API，自建可控
临时文件 / 中间结果 GridFS 或本地文件系统 简单够用，便于清理

务实建议：如果你的项目已经使用了 MongoDB，且文件存储需求不大（几百个文件，总量几个 GB）， GridFS 是最简单直接的选择。当文件量增长到一定规模，或需要 CDN、公网访问等能力时， 再迁移到对象存储也不迟——文件服务通常是独立的模块，迁移成本可控。
