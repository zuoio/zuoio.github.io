第02章 环境搭建

本章将手把手指导你搭建 MongoDB 的完整开发环境。我们会介绍两种主流安装方式（Windows 本地安装和 Docker 安装）、MongoDB Shell 的基本使用、可视化管理工具，以及如何在 Spring Boot 项目中集成 MongoDB。

2.1 MongoDB 安装（Windows / Docker 两种方式）

2.1.1 方式一：Windows 本地安装

第一步：下载 MongoDB

  1. 访问 MongoDB 官方下载页面：https://www.mongodb.com/try/download/community
  2. 选择以下选项：
    • Version：选择最新的稳定版本（推荐 7.0.x 或 8.0.x）
    • Platform：Windows
    • Package：MSI（安装包）

第二步：运行安装程序

  1. 双击下载的 .msi 文件，启动安装向导。
  2. 选择 Complete（完整安装）模式。
  3. Service Configuration（服务配置）页面：
☑ Install MongoDB as a Service（推荐勾选）
  ○ Run service as Network Service user
  ● Run service as a local or domain user:
    Account Name: .\Administrator（或使用默认）

Service Name: MongoDB
Data Directory: C:\Program Files\MongoDB\Server\7.0\data\
Log Directory:  C:\Program Files\MongoDB\Server\7.0\log\

建议：勾选"Install MongoDB as a Service"可以让 MongoDB 随系统启动自动运行，省去手动启动的麻烦。

  1. 可选：勾选 Install MongoDB Compass（图形化管理工具，也可以后面单独安装）。
  2. 点击 Install 完成安装。

第三步：配置环境变量

将 MongoDB 的 bin 目录添加到系统 PATH 环境变量中：

默认安装路径：C:\Program Files\MongoDB\Server\7.0\bin

配置步骤：

  1. 右键 此电脑 → 属性 → 高级系统设置 → 环境变量
  2. 在 系统变量 中找到 Path，点击 编辑
  3. 点击 新建，添加：C:\Program Files\MongoDB\Server\7.0\bin
  4. 点击 确定 保存

或者使用 PowerShell 快速设置：

# 以管理员身份运行 PowerShell
[Environment]::SetEnvironmentVariable(
    "Path",
    $env:Path + ";C:\Program Files\MongoDB\Server\7.0\bin",
    [EnvironmentVariableTarget]::Machine
)

第四步：验证安装

打开新的命令行窗口，执行以下命令：

# 查看 MongoDB 服务端版本
mongod --version

# 输出示例：
# db version v7.0.x
# Build Info: ...

第五步：安装 mongosh（MongoDB Shell）

MongoDB 7.0 开始，Shell 工具 mongosh 需要单独安装：

  1. 访问：https://www.mongodb.com/try/download/shell
  2. 下载 Windows 版本的 MSI 安装包
  3. 运行安装程序，安装路径会自动添加到 PATH

验证 mongosh 安装：

mongosh --version

# 输出示例：
# 2.x.x

第六步：Windows 服务管理

# 查看 MongoDB 服务状态
Get-Service MongoDB

# 启动服务
Start-Service MongoDB

# 停止服务
Stop-Service MongoDB

# 重启服务
Restart-Service MongoDB

# 也可以使用 net 命令
net start MongoDB
net stop MongoDB

2.1.2 方式二：Docker 安装（推荐开发环境使用）

使用 Docker 安装 MongoDB 更加简洁，不会"污染"主机环境，而且可以轻松管理多个版本。

前提条件

确保已安装 Docker Desktop：https://www.docker.com/products/docker-desktop

# 验证 Docker 已安装
docker --version
docker compose version

方式 A：使用 docker run 快速启动

# 拉取 MongoDB 官方镜像
docker pull mongodb/mongodb-community-server:7.0-ubi8

# 快速启动（开发测试用）
docker run -d \
  --name mongodb \
  -p 27017:27017 \
  -e MONGO_INITDB_ROOT_USERNAME=admin \
  -e MONGO_INITDB_ROOT_PASSWORD=admin123 \
  -v mongodb_data:/data/db \
  mongodb/mongodb-community-server:7.0-ubi8

方式 B：使用 docker-compose.yml（推荐）

创建一个 docker-compose.yml 文件，方便管理和复用：

version: '3.8'

services:
  mongodb:
    image: mongodb/mongodb-community-server:7.0-ubi8
    container_name: mongodb
    restart: unless-stopped
    ports:
      - "27017:27017"
    environment:
      MONGO_INITDB_ROOT_USERNAME: admin
      MONGO_INITDB_ROOT_PASSWORD: admin123
      MONGO_INITDB_DATABASE: myapp
    volumes:
      # 数据持久化 —— 容器删除后数据不会丢失
      - mongodb_data:/data/db
      # 配置文件（可选）
      - mongodb_config:/data/configdb
      # 初始化脚本目录（可选，容器首次启动时自动执行）
      - ./mongo-init:/docker-entrypoint-initdb.d
    command: ["mongod", "--auth"]
    healthcheck:
      test: ["CMD", "mongosh", "--eval", "db.adminCommand('ping')", "-u", "admin", "-p", "admin123", "--authenticationDatabase", "admin"]
      interval: 10s
      timeout: 5s
      retries: 5
      start_period: 30s

volumes:
  mongodb_data:
    driver: local
  mongodb_config:
    driver: local

docker-compose.yml 配置说明：

配置项 说明
image 使用 MongoDB 官方社区版镜像，版本 7.0
restart: unless-stopped 除非手动停止，否则容器异常退出后自动重启
ports: "27017:27017" 将容器的 27017 端口映射到主机的 27017 端口
MONGO_INITDB_ROOT_USERNAME root 用户名
MONGO_INITDB_ROOT_PASSWORD root 密码
MONGO_INITDB_DATABASE 初始化时创建的默认数据库
mongodb_data:/data/db 数据文件持久化到 Docker 命名卷
command: ["mongod", "--auth"] 启用认证模式
healthcheck 健康检查配置，确保服务正常运行

（可选）创建初始化脚本 mongo-init/init-user.js，在容器首次启动时自动执行：

// mongo-init/init-user.js
// 该脚本在容器首次启动时自动执行，用于创建应用数据库和用户

db = db.getSiblingDB('myapp');

db.createUser({
    user: 'app_user',
    pwd: 'app_password',
    roles: [
        { role: 'readWrite', db: 'myapp' }
    ]
});

db.createCollection('users');

db.users.insertOne({
    username: 'admin',
    email: 'admin@example.com',
    role: 'ADMIN',
    createdAt: new Date()
});

print('✔ 数据库初始化完成：创建了 myapp 数据库、app_user 用户和 users 集合');

启动与管理命令：

# 启动（后台运行）
docker compose up -d

# 查看运行状态
docker compose ps

# 查看日志
docker compose logs -f mongodb

# 停止
docker compose down

# 停止并删除数据卷（谨慎！会清除所有数据）
docker compose down -v

2.1.3 验证安装成功

无论哪种安装方式，都可以用以下方式验证 MongoDB 是否正常运行：

方式1：使用 mongosh 连接

# 无认证连接（本地安装未开启认证时）
mongosh

# 带认证连接（Docker 方式）
mongosh "mongodb://admin:admin123@localhost:27017/admin"

# 或者分步认证
mongosh --host localhost --port 27017 -u admin -p admin123 --authenticationDatabase admin

方式2：连接后执行测试命令

// 连接成功后会看到类似以下提示：
// Current Mongosh Log ID: 65f3...
// Connecting to: mongodb://localhost:27017
// Using MongoDB: 7.0.x
// mongosh 2.x.x

// 查看 MongoDB 版本
db.version()
// 输出：7.0.x

// 查看服务器状态
db.serverStatus().version

// 执行简单的数据操作测试
use test_db
db.test_collection.insertOne({ message: "Hello MongoDB!", timestamp: new Date() })
db.test_collection.find()

// 输出：
// {
//   _id: ObjectId('65f3...'),
//   message: 'Hello MongoDB!',
//   timestamp: ISODate('2024-03-15T...')
// }

// 清理测试数据
db.test_collection.drop()

如果以上命令都执行成功，说明 MongoDB 环境已经搭建完毕。

2.2 MongoDB Shell（mongosh）的基本使用

mongosh（MongoDB Shell）是 MongoDB 官方提供的交互式命令行工具，用于连接和操作 MongoDB 数据库。它基于 Node.js 运行，支持 JavaScript 语法。

2.2.1 连接 MongoDB

# 最简单的连接方式（连接本地默认端口 27017）
mongosh

# 指定主机和端口
mongosh --host 192.168.1.100 --port 27017

# 使用连接字符串
mongosh "mongodb://localhost:27017"

# 带认证的连接字符串
mongosh "mongodb://username:password@localhost:27017/mydb?authSource=admin"

# 连接远程数据库（含副本集）
mongosh "mongodb://user:pass@host1:27017,host2:27017,host3:27017/mydb?replicaSet=rs0"

连接成功后的界面：

Current Mongosh Log ID: 65f3a1b2c3d4e5f6a7b8c9d0
Connecting to:          mongodb://127.0.0.1:27017/?directConnection=true
Using MongoDB:          7.0.12
Using Mongosh:          2.2.1

For mongosh info see: https://docs.mongodb.com/mongodb-shell/

test>

test> 表示当前连接的数据库是 test（默认数据库）。

2.2.2 数据库操作命令

// ========== 数据库级别操作 ==========

// 查看所有数据库（只显示有数据的数据库）
show dbs
// 输出：
// admin   100.00 KiB
// config   72.00 KiB
// local    72.00 KiB
// myapp     1.25 MiB

// 切换到指定数据库（不存在则创建）
use myapp

// 查看当前所在数据库
db
// 输出：myapp

// 查看当前数据库的统计信息
db.stats()
// 输出：
// {
//   db: 'myapp',
//   collections: 3,
//   views: 0,
//   objects: 1520,
//   avgObjSize: 256.5,
//   dataSize: 389880,
//   storageSize: 180224,
//   indexes: 5,
//   indexSize: 143360,
//   totalSize: 323584,
//   ...
// }

// 删除当前数据库（谨慎操作！）
db.dropDatabase()

2.2.3 集合操作命令

// ========== 集合级别操作 ==========

// 查看当前数据库的所有集合
show collections
// 输出：
// users
// orders
// products

// 创建集合（通常不需要显式创建）
db.createCollection("logs")

// 创建带选项的集合（如设置上限集合）
db.createCollection("capped_logs", {
    capped: true,       // 上限集合：固定大小，自动淘汰最老的文档
    size: 10485760,     // 最大 10MB
    max: 10000          // 最多 10000 个文档
})

// 查看集合信息
db.users.stats()

// 查看集合中的文档数量
db.users.countDocuments()

// 删除集合
db.logs.drop()

// 重命名集合
db.users.renameCollection("members")

2.2.4 基本 CRUD 操作演示

// 切换到演示数据库
use demo

// ========== 插入文档 ==========

// 插入单个文档
db.users.insertOne({
    name: "张三",
    age: 28,
    email: "zhangsan@example.com",
    skills: ["Java", "Spring Boot", "MongoDB"],
    address: {
        city: "北京",
        district: "朝阳区"
    },
    createdAt: new Date()
})
// 输出：
// {
//   acknowledged: true,
//   insertedId: ObjectId('65f3a1b2c3d4e5f6a7b8c9d0')
// }

// 插入多个文档
db.users.insertMany([
    {
        name: "李四",
        age: 32,
        email: "lisi@example.com",
        skills: ["Python", "Django"],
        address: { city: "上海", district: "浦东新区" },
        createdAt: new Date()
    },
    {
        name: "王五",
        age: 25,
        email: "wangwu@example.com",
        skills: ["Java", "MySQL", "Redis"],
        address: { city: "广州", district: "天河区" },
        createdAt: new Date()
    },
    {
        name: "赵六",
        age: 30,
        email: "zhaoliu@example.com",
        skills: ["Go", "Docker", "Kubernetes"],
        address: { city: "深圳", district: "南山区" },
        createdAt: new Date()
    }
])

// ========== 查询文档 ==========

// 查询所有文档
db.users.find()

// 格式化输出（更易读）
db.users.find().pretty()

// 查询单个文档
db.users.findOne({ name: "张三" })
// 输出：
// {
//   _id: ObjectId('65f3...'),
//   name: '张三',
//   age: 28,
//   email: 'zhangsan@example.com',
//   skills: ['Java', 'Spring Boot', 'MongoDB'],
//   address: { city: '北京', district: '朝阳区' },
//   createdAt: ISODate('2024-03-15T...')
// }

// 条件查询：年龄大于 28 的用户
db.users.find({ age: { $gt: 28 } })

// 嵌套字段查询：城市为北京的用户
db.users.find({ "address.city": "北京" })

// 数组包含查询：技能包含 Java 的用户
db.users.find({ skills: "Java" })

// 投影：只返回 name 和 email 字段
db.users.find({}, { name: 1, email: 1, _id: 0 })
// 输出：
// { name: '张三', email: 'zhangsan@example.com' }
// { name: '王五', email: 'wangwu@example.com' }
// ...

// 排序 + 限制：按年龄降序，取前 2 条
db.users.find().sort({ age: -1 }).limit(2)

// ========== 更新文档 ==========

// 更新单个文档：给张三添加一个字段
db.users.updateOne(
    { name: "张三" },
    { $set: { phone: "13800138000", updatedAt: new Date() } }
)

// 更新多个文档：所有北京用户添加标签
db.users.updateMany(
    { "address.city": "北京" },
    { $set: { region: "华北" } }
)

// 数组操作：给张三的技能列表追加一项
db.users.updateOne(
    { name: "张三" },
    { $push: { skills: "Docker" } }
)

// 数值递增：年龄加 1
db.users.updateOne(
    { name: "张三" },
    { $inc: { age: 1 } }
)

// ========== 删除文档 ==========

// 删除单个文档
db.users.deleteOne({ name: "赵六" })

// 删除多个文档：删除年龄小于 26 的用户
db.users.deleteMany({ age: { $lt: 26 } })

// 删除集合中的所有文档（保留集合本身）
db.users.deleteMany({})

2.3 可视化工具介绍（MongoDB Compass / Navicat）

虽然 mongosh 很强大，但在日常开发中使用可视化工具会更加高效直观，特别是在浏览数据、构建查询和分析性能时。

2.3.1 MongoDB Compass（官方免费工具）

MongoDB Compass 是 MongoDB 官方提供的免费图形化管理工具，功能全面且易于使用。

下载安装

  1. 下载地址：https://www.mongodb.com/try/download/compass
  2. 选择 Windows 平台，下载 .exe 安装包
  3. 运行安装程序，按默认选项安装即可

连接配置

启动 Compass 后，在连接界面输入连接字符串：

# 无认证连接
mongodb://localhost:27017

# 带认证连接
mongodb://admin:admin123@localhost:27017/?authSource=admin

# 也可以展开"Advanced Connection Options"，分别填写：
# - Host: localhost
# - Port: 27017
# - Authentication: Username / Password
# - Username: admin
# - Password: admin123
# - Authentication Database: admin

主要功能介绍

功能模块 说明
Databases 查看所有数据库及其大小、集合数量
Collections 浏览集合中的文档，支持列表视图、JSON 视图和表格视图
Documents 增删改查文档，支持可视化编辑
Aggregations 可视化构建聚合管道，拖拽添加阶段，实时预览每个阶段的输出
Schema 自动分析集合的 Schema 分布（字段类型和频率统计）
Explain Plan 查看查询的执行计划，分析索引使用情况
Indexes 查看、创建和管理索引
Validation 配置文档验证规则（Schema Validation）
Performance 实时监控数据库性能指标

Compass 使用示例

1. 查询文档： 在查询栏输入过滤条件

{ "age": { "$gt": 25 }, "address.city": "北京" }

2. 构建聚合管道： 通过可视化界面逐步添加阶段

Stage 1: $match    → { "status": "active" }
Stage 2: $group    → { "_id": "$city", "count": { "$sum": 1 } }
Stage 3: $sort     → { "count": -1 }
Stage 4: $limit    → 10

3. 查看执行计划： 点击 Explain Plan 按钮，可以看到：

  • 查询是否使用了索引（IXSCAN vs COLLSCAN）
  • 扫描了多少文档
  • 执行耗时

2.3.2 Navicat for MongoDB

Navicat 是一款付费的跨平台数据库管理工具，支持 MongoDB、MySQL、PostgreSQL 等多种数据库。如果你的团队已经在使用 Navicat 管理 MySQL，那么用它来管理 MongoDB 可以保持工具的一致性。

主要特点：

  • 统一管理多种数据库（MongoDB + MySQL + PostgreSQL 等）
  • 支持数据建模（ER 图）
  • 内置查询构建器
  • 支持数据导入/导出（JSON, CSV 等格式）
  • 支持数据同步和结构同步

连接配置：

  1. 打开 Navicat，点击 连接 → MongoDB
  2. 填写连接信息：
    • 连接名：本地MongoDB
    • 主机：localhost
    • 端口：27017
  3. 如果开启了认证，切换到 认证 标签页：
    • 认证方式：Password
    • 用户名：admin
    • 密码：admin123
    • 认证数据库：admin
  4. 点击 测试连接，确认连接成功后点击 确定

2.4 Spring Boot 项目初始化与依赖配置

作为 Java 开发者，我们最终会在 Spring Boot 中操作 MongoDB。本节将引导你从零搭建一个 Spring Boot + MongoDB 的项目骨架。

2.4.1 使用 Spring Initializr 创建项目

访问 https://start.spring.io，配置以下选项：

配置项 值
Project Maven
Language Java
Spring Boot 3.3.x（选择最新稳定版）
Group com.example
Artifact mongodb-demo
Name mongodb-demo
Package name com.example.mongodbdemo
Packaging Jar
Java 17（或 21）

添加依赖（Dependencies）：

  • Spring Data MongoDB —— MongoDB 数据访问
  • Spring Web —— Web 支持（可选，后续章节会用到）
  • Lombok —— 简化实体类代码（可选但推荐）

点击 Generate 下载项目压缩包，解压后用 IntelliJ IDEA 打开。

2.4.3 application.yml 配置

在 src/main/resources/application.yml 中配置 MongoDB 连接信息：

配置方式一：单机无认证（本地开发）

spring:
  data:
    mongodb:
      host: localhost
      port: 27017
      database: myapp

# 日志配置：显示 MongoDB 操作日志（开发时建议开启）
logging:
  level:
    org.springframework.data.mongodb: DEBUG
    org.mongodb.driver: WARN

配置方式二：带认证连接（推荐）

spring:
  data:
    mongodb:
      host: localhost
      port: 27017
      database: myapp
      username: app_user
      password: app_password
      authentication-database: admin

logging:
  level:
    org.springframework.data.mongodb: DEBUG
    org.mongodb.driver: WARN

配置方式三：使用 URI 连接字符串（最灵活）

spring:
  data:
    mongodb:
      uri: mongodb://app_user:app_password@localhost:27017/myapp?authSource=admin

logging:
  level:
    org.springframework.data.mongodb: DEBUG
    org.mongodb.driver: WARN

配置方式四：多环境配置

实际项目中，不同环境的 MongoDB 配置通常不同。可以使用 Spring Profiles 管理：

application.yml（公共配置）：

spring:
  profiles:
    active: dev

logging:
  level:
    org.springframework.data.mongodb: DEBUG

application-dev.yml（开发环境）：

spring:
  data:
    mongodb:
      uri: mongodb://localhost:27017/myapp_dev

application-prod.yml（生产环境）：

spring:
  data:
    mongodb:
      uri: mongodb://app_user:${MONGO_PASSWORD}@mongo-host:27017/myapp?authSource=admin&replicaSet=rs0

logging:
  level:
    org.springframework.data.mongodb: INFO
    org.mongodb.driver: WARN

安全提示：生产环境的密码不应写死在配置文件中，应通过环境变量（如 ${MONGO_PASSWORD}）或配置中心（如 Nacos、Consul）注入。

2.4.4 编写启动测试类验证连接

创建一个简单的实体类和启动测试，验证 Spring Boot 能否成功连接 MongoDB。

步骤一：创建实体类

package com.example.mongodbdemo.entity;

import lombok.AllArgsConstructor;
import lombok.Data;
import lombok.NoArgsConstructor;
import org.springframework.data.annotation.Id;
import org.springframework.data.mongodb.core.mapping.Document;

import java.time.LocalDateTime;

@Data
@NoArgsConstructor
@AllArgsConstructor
@Document(collection = "users")
public class User {

    @Id
    private String id;

    private String username;

    private String email;

    private Integer age;

    private LocalDateTime createdAt;
}

注解说明：

注解 说明
@Document(collection = "users") 标记该类映射到 MongoDB 的 users 集合
@Id 标记 id 字段映射到 MongoDB 的 _id 字段
@Data Lombok 注解，自动生成 getter/setter/toString/equals/hashCode
@NoArgsConstructor 生成无参构造器
@AllArgsConstructor 生成全参构造器

步骤二：创建 Repository 接口

package com.example.mongodbdemo.repository;

import com.example.mongodbdemo.entity.User;
import org.springframework.data.mongodb.repository.MongoRepository;

public interface UserRepository extends MongoRepository<User, String> {
}

只需要继承 MongoRepository 接口，Spring Data 会自动提供常用的 CRUD 方法，无需编写实现代码。

步骤三：创建启动测试类

package com.example.mongodbdemo;

import com.example.mongodbdemo.entity.User;
import com.example.mongodbdemo.repository.UserRepository;
import org.springframework.boot.CommandLineRunner;
import org.springframework.boot.SpringApplication;
import org.springframework.boot.autoconfigure.SpringBootApplication;
import org.springframework.context.annotation.Bean;
import org.springframework.data.mongodb.core.MongoTemplate;

import java.time.LocalDateTime;

@SpringBootApplication
public class MongodbDemoApplication {

    public static void main(String[] args) {
        SpringApplication.run(MongodbDemoApplication.class, args);
    }

    @Bean
    CommandLineRunner runner(UserRepository userRepository, MongoTemplate mongoTemplate) {
        return args -> {
            System.out.println("========== MongoDB 连接测试 ==========");

            // 1. 测试 MongoTemplate 连接
            String dbName = mongoTemplate.getDb().getName();
            System.out.println("成功连接到数据库：" + dbName);

            // 2. 使用 Repository 插入一条测试数据
            User testUser = new User();
            testUser.setUsername("test_user");
            testUser.setEmail("test@example.com");
            testUser.setAge(25);
            testUser.setCreatedAt(LocalDateTime.now());

            userRepository.save(testUser);
            System.out.println("插入测试用户成功，ID：" + testUser.getId());

            // 3. 查询刚插入的数据
            User found = userRepository.findById(testUser.getId()).orElse(null);
            if (found != null) {
                System.out.println("查询到用户：" + found.getUsername() + ", Email: " + found.getEmail());
            }

            // 4. 统计文档数量
            long count = userRepository.count();
            System.out.println("users 集合当前文档数量：" + count);

            // 5. 删除测试数据
            userRepository.deleteById(testUser.getId());
            System.out.println("已清理测试数据");

            System.out.println("========== 连接测试完成 ==========");
        };
    }
}
