第04章 Spring Boot 集成 MongoDB

本章将系统学习如何在 Spring Boot 项目中集成 MongoDB。从依赖配置、实体映射注解，到 MongoRepository 的开箱即用 CRUD、方法名派生查询，再到 @Query 自定义查询，带你用 Java 代码实现上一章在 Shell 中完成的所有操作。

本章示例基于 Spring Boot 3.x + Java 17+ + Spring Data MongoDB 4.x。

4.1 spring-boot-starter-data-mongodb 配置详解

4.1.1 引入依赖

在 pom.xml 中添加 Spring Data MongoDB 的 Starter 依赖：

<parent>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-parent</artifactId>
    <version>3.3.0</version>
    <relativePath/>
</parent>

<dependencies>
    <!-- Spring Data MongoDB -->
    <dependency>
        <groupId>org.springframework.boot</groupId>
        <artifactId>spring-boot-starter-data-mongodb</artifactId>
    </dependency>

    <!-- Spring Web（提供 REST 接口能力） -->
    <dependency>
        <groupId>org.springframework.boot</groupId>
        <artifactId>spring-boot-starter-web</artifactId>
    </dependency>

    <!-- Lombok（简化实体类代码，可选） -->
    <dependency>
        <groupId>org.projectlombok</groupId>
        <artifactId>lombok</artifactId>
        <optional>true</optional>
    </dependency>

    <!-- 测试 -->
    <dependency>
        <groupId>org.springframework.boot</groupId>
        <artifactId>spring-boot-starter-test</artifactId>
        <scope>test</scope>
    </dependency>
</dependencies>

spring-boot-starter-data-mongodb 会自动引入 spring-data-mongodb、mongodb-driver-sync 等核心依赖，无需手动管理版本。

4.1.2 自动配置原理简述

Spring Boot 通过 MongoAutoConfiguration 和 MongoDataAutoConfiguration 完成 MongoDB 的自动配置，核心流程如下：

application.yml 中的 spring.data.mongodb.* 配置
        ↓
MongoProperties 读取配置属性
        ↓
MongoAutoConfiguration 创建 MongoClient（底层连接客户端）
        ↓
MongoDataAutoConfiguration 创建 MongoTemplate / MongoDatabaseFactory
        ↓
@EnableMongoRepositories 扫描并生成 Repository 代理对象
        ↓
开发者注入 MongoTemplate 或 Repository 即可使用

关键的自动配置类：

配置类 作用
MongoAutoConfiguration 创建 MongoClient 实例
MongoDataAutoConfiguration 创建 MongoDatabaseFactory、MongoTemplate、MongoConverter
MongoRepositoriesAutoConfiguration 启用 @EnableMongoRepositories，扫描 Repository 接口

只要引入 Starter 依赖并配置连接信息，上述 Bean 都会自动创建，开发者无需手动声明。

4.1.3 基础连接配置

方式一：使用连接字符串（推荐）

# application.yml
spring:
  data:
    mongodb:
      uri: mongodb://root:123456@localhost:27017/blog?authSource=admin

连接字符串格式：

mongodb://[username:password@]host[:port]/database[?options]

方式二：分别指定各项配置

spring:
  data:
    mongodb:
      host: localhost
      port: 27017
      database: blog
      username: root
      password: 123456
      authentication-database: admin

注意：uri 和 host/port/database 两种方式不能混用。如果同时配置了 uri，则 host、port、database 等属性会被忽略。

方式三：无认证的本地连接（开发环境）

spring:
  data:
    mongodb:
      uri: mongodb://localhost:27017/blog

4.1.4 连接池配置

MongoDB Java 驱动内部使用连接池管理与服务端的连接。Spring Boot 中可以通过自定义 MongoClientSettings 来调整连接池参数：

package com.example.mongo.config;

import com.mongodb.ConnectionString;
import com.mongodb.MongoClientSettings;
import com.mongodb.connection.ConnectionPoolSettings;
import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;

import java.util.concurrent.TimeUnit;

@Configuration
public class MongoConfig {

    @Bean
    public MongoClientSettings mongoClientSettings() {
        return MongoClientSettings.builder()
            .applyConnectionString(new ConnectionString(
                "mongodb://root:123456@localhost:27017/blog?authSource=admin"
            ))
            .applyToConnectionPoolSettings(builder ->
                builder.maxSize(50)                                     // 最大连接数（默认 100）
                       .minSize(5)                                      // 最小连接数（默认 0）
                       .maxWaitTime(5000, TimeUnit.MILLISECONDS)        // 获取连接的最大等待时间
                       .maxConnectionIdleTime(60000, TimeUnit.MILLISECONDS) // 连接最大空闲时间
                       .maxConnectionLifeTime(300000, TimeUnit.MILLISECONDS) // 连接最大存活时间
            )
            .applyToSocketSettings(builder ->
                builder.connectTimeout(5000, TimeUnit.MILLISECONDS)     // 连接超时
                       .readTimeout(15000, TimeUnit.MILLISECONDS)       // 读取超时
            )
            .build();
    }
}

常用连接池参数说明：

参数 默认值 说明
maxSize 100 连接池最大连接数
minSize 0 连接池最小连接数（空闲时保持的连接）
maxWaitTime 120s 等待可用连接的最大时间
maxConnectionIdleTime 0（无限制） 连接最大空闲时间，超时后关闭
maxConnectionLifeTime 0（无限制） 连接最大生存时间

4.1.5 多数据源配置

在微服务架构或复杂业务场景中，可能需要连接多个 MongoDB 实例或数据库。Spring Boot 默认只支持单数据源，多数据源需要手动配置。

场景：主库 blog 用于业务数据，日志库 logs 用于操作日志。

第一步：配置文件中定义多个数据源

# application.yml
mongodb:
  primary:
    uri: mongodb://root:123456@localhost:27017/blog?authSource=admin
  secondary:
    uri: mongodb://root:123456@localhost:27017/logs?authSource=admin

第二步：创建主数据源配置

package com.example.mongo.config;

import com.mongodb.client.MongoClient;
import com.mongodb.client.MongoClients;
import org.springframework.beans.factory.annotation.Value;
import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;
import org.springframework.context.annotation.Primary;
import org.springframework.data.mongodb.MongoDatabaseFactory;
import org.springframework.data.mongodb.core.MongoTemplate;
import org.springframework.data.mongodb.core.SimpleMongoClientDatabaseFactory;
import org.springframework.data.mongodb.repository.config.EnableMongoRepositories;

@Configuration
@EnableMongoRepositories(
    basePackages = "com.example.mongo.repository.primary",
    mongoTemplateRef = "primaryMongoTemplate"
)
public class PrimaryMongoConfig {

    @Value("${mongodb.primary.uri}")
    private String primaryUri;

    @Primary
    @Bean(name = "primaryMongoClient")
    public MongoClient primaryMongoClient() {
        return MongoClients.create(primaryUri);
    }

    @Primary
    @Bean(name = "primaryMongoDatabaseFactory")
    public MongoDatabaseFactory primaryMongoDatabaseFactory() {
        return new SimpleMongoClientDatabaseFactory(primaryMongoClient(), "blog");
    }

    @Primary
    @Bean(name = "primaryMongoTemplate")
    public MongoTemplate primaryMongoTemplate() {
        return new MongoTemplate(primaryMongoDatabaseFactory());
    }
}

第四步：使用时注入对应的 MongoTemplate

package com.example.mongo.service;

    }
}

4.2 实体映射注解详解

Spring Data MongoDB 提供了一组注解，用于将 Java 实体类映射到 MongoDB 文档。以下逐一讲解每个核心注解。

4.2.1 @Document —— 标记文档实体

@Document 将一个 Java 类标记为 MongoDB 文档实体，并指定其对应的集合名称。

package com.example.mongo.entity;

import org.springframework.data.mongodb.core.mapping.Document;

// 映射到 MongoDB 中名为 "users" 的集合
@Document(collection = "users")
public class User {
    // 如果不指定 collection，默认使用类名的小写形式："user"
}

// 也可以使用 value 属性（和 collection 等效）
@Document("articles")
public class Article {
}
属性 说明
collection / value 指定集合名称，不指定则默认为类名首字母小写
language 文本索引的语言，默认为 ""
collation 排序规则（用于大小写不敏感查询等场景）

4.2.2 @Id —— 主键映射

@Id 将字段映射为文档的 _id 主键。

package com.example.mongo.entity;

import org.springframework.data.annotation.Id;
import org.springframework.data.mongodb.core.mapping.Document;

@Document(collection = "users")
public class User {

    @Id
    private String id;  // 映射为 MongoDB 的 _id 字段

    private String username;

    // getter / setter
    public String getId() { return id; }
    public void setId(String id) { this.id = id; }
    public String getUsername() { return username; }
    public void setUsername(String username) { this.username = username; }
}

_id 的生成规则：

id 字段类型 行为
String 插入时如果为 null，自动生成 ObjectId 的十六进制字符串
ObjectId 插入时如果为 null，自动生成 ObjectId
Long / Integer 不会自动生成，需要手动赋值

建议：id 字段使用 String 类型最为通用，MongoDB 会自动生成 24 位十六进制字符串的 ObjectId。

4.2.3 @Field —— 字段名映射

@Field 用于指定 Java 字段在 MongoDB 文档中对应的字段名。当 Java 属性名与 MongoDB 字段名不一致时使用。

package com.example.mongo.entity;

import org.springframework.data.annotation.Id;
import org.springframework.data.mongodb.core.mapping.Document;
import org.springframework.data.mongodb.core.mapping.Field;
import org.springframework.data.mongodb.core.mapping.FieldType;

import java.math.BigDecimal;
import java.time.LocalDateTime;

@Document(collection = "articles")
public class Article {

    @Id
    private String id;

    private String title;

    // Java 中用驼峰命名，MongoDB 中用下划线命名
    @Field("author_name")
    private String authorName;

    @Field("view_count")
    private Integer viewCount;

    @Field("created_at")
    private LocalDateTime createdAt;

    // 使用 targetType 指定 BSON 存储类型（适合金额等精度敏感场景）
    @Field(targetType = FieldType.DECIMAL128)
    private BigDecimal price;

    // getter / setter
    public String getId() { return id; }
    public void setId(String id) { this.id = id; }
    public String getTitle() { return title; }
    public void setTitle(String title) { this.title = title; }
    public String getAuthorName() { return authorName; }
    public void setAuthorName(String authorName) { this.authorName = authorName; }
    public Integer getViewCount() { return viewCount; }
    public void setViewCount(Integer viewCount) { this.viewCount = viewCount; }
    public LocalDateTime getCreatedAt() { return createdAt; }
    public void setCreatedAt(LocalDateTime createdAt) { this.createdAt = createdAt; }
    public BigDecimal getPrice() { return price; }
    public void setPrice(BigDecimal price) { this.price = price; }
}

如果不使用 @Field 注解，Spring Data MongoDB 默认使用 Java 属性名作为 MongoDB 字段名（区分大小写）。

4.2.4 @Indexed / @CompoundIndex —— 索引注解

@Indexed：单字段索引

package com.example.mongo.entity;

import org.springframework.data.annotation.Id;
import org.springframework.data.mongodb.core.index.Indexed;
import org.springframework.data.mongodb.core.mapping.Document;

@Document(collection = "users")
public class User {

    @Id
    private String id;

    // 创建唯一索引（用户名不可重复）
    @Indexed(unique = true)
    private String username;

    // 创建普通索引（加速按邮箱查询）
    @Indexed
    private String email;

    // TTL 索引：文档在 expireAt 时间点后自动删除
    @Indexed(expireAfterSeconds = 0)
    private java.util.Date expireAt;

    // TTL 索引：文档在创建后 3600 秒（1小时）自动删除
    @Indexed(expireAfterSeconds = 3600)
    private java.util.Date createdAt;

    // getter / setter 省略
}

@Indexed 常用属性：

属性 说明
unique 是否唯一索引，默认 false
direction 索引方向，ASCENDING（默认）或 DESCENDING
expireAfterSeconds TTL 索引的过期时间（秒）
name 索引名称
sparse 稀疏索引，true 时仅对存在该字段的文档建立索引
background 是否后台创建索引

@CompoundIndex：复合索引

package com.example.mongo.entity;

import org.springframework.data.annotation.Id;
import org.springframework.data.mongodb.core.index.CompoundIndex;
import org.springframework.data.mongodb.core.index.CompoundIndexes;
import org.springframework.data.mongodb.core.mapping.Document;
import org.springframework.data.mongodb.core.mapping.Field;

import java.time.LocalDateTime;
import java.util.List;

@Document(collection = "articles")
@CompoundIndexes({
    // 复合索引：按 author 和 status 联合查询
    @CompoundIndex(name = "idx_author_status", def = "{'author': 1, 'status': 1}"),
    // 复合索引：按 category 升序 + createdAt 降序（适合分类列表分页查询）
    @CompoundIndex(name = "idx_category_created", def = "{'category': 1, 'created_at': -1}")
})
public class Article {

    @Id
    private String id;

    private String title;

    private String author;

    private String category;

    private String status;

    private List<String> tags;

    @Field("view_count")
    private Integer viewCount;

    @Field("created_at")
    private LocalDateTime createdAt;

    // getter / setter 省略
}

注意：Spring Boot 3.x 默认不自动创建注解声明的索引。需要在配置中手动开启：

spring:
  data:
    mongodb:
      auto-index-creation: true

或者通过 Java 配置：

@Configuration
public class MongoConfig extends AbstractMongoClientConfiguration {
    @Override
    protected boolean autoIndexCreation() {
        return true;
    }
}

4.2.5 @DBRef —— 文档引用

@DBRef 用于在文档之间建立引用关系，类似于关系型数据库的外键。MongoDB 会存储被引用文档的 $ref（集合名）和 $id，在查询时自动加载引用的文档。

package com.example.mongo.entity;

import org.springframework.data.annotation.Id;
import org.springframework.data.mongodb.core.mapping.DBRef;
import org.springframework.data.mongodb.core.mapping.Document;
import org.springframework.data.mongodb.core.mapping.Field;

import java.time.LocalDateTime;
import java.util.List;

@Document(collection = "articles")
public class Article {

    @Id
    private String id;

    private String title;

    private String content;

    // 引用 User 文档：查询 Article 时会自动加载关联的 User
    @DBRef
    private User author;

    // 引用 Comment 文档列表
    @DBRef(lazy = true)  // lazy = true：延迟加载，访问时才查询
    private List<Comment> comments;

    @Field("created_at")
    private LocalDateTime createdAt;

    // getter / setter
    public String getId() { return id; }
    public void setId(String id) { this.id = id; }
    public String getTitle() { return title; }
    public void setTitle(String title) { this.title = title; }
    public String getContent() { return content; }
    public void setContent(String content) { this.content = content; }
    public User getAuthor() { return author; }
    public void setAuthor(User author) { this.author = author; }
    public List<Comment> getComments() { return comments; }
    public void setComments(List<Comment> comments) { this.comments = comments; }
    public LocalDateTime getCreatedAt() { return createdAt; }
    public void setCreatedAt(LocalDateTime createdAt) { this.createdAt = createdAt; }
}

存储到 MongoDB 中的文档结构：

{
  "_id": "article_001",
  "title": "MongoDB 入门指南",
  "author": {
    "$ref": "users",
    "$id": "user_001"
  },
  "comments": [
    { "$ref": "comments", "$id": "comment_001" },
    { "$ref": "comments", "$id": "comment_002" }
  ]
}

实际开发建议：@DBRef 的本质是额外发起查询来加载引用文档，存在 N+1 查询问题。在大多数场景中，推荐使用内嵌文档或手动存储 ID + 应用层关联来代替 @DBRef，性能更好。

4.2.6 @Transient —— 忽略字段

@Transient 标记的字段不会被持久化到 MongoDB，仅在 Java 内存中存在。

package com.example.mongo.entity;

import org.springframework.data.annotation.Id;
import org.springframework.data.annotation.Transient;
import org.springframework.data.mongodb.core.mapping.Document;

@Document(collection = "users")
public class User {

    @Id
    private String id;

    private String username;

    private String password;

    // 该字段不会存储到 MongoDB 中
    @Transient
    private String confirmPassword;

    // 计算属性，不需要存储
    @Transient
    private String displayName;

    public String getDisplayName() {
        return this.username + " (ID: " + this.id + ")";
    }

    // 也可以用 Java 的 transient 关键字达到同样效果
    // transient String tempData;

    // getter / setter
    public String getId() { return id; }
    public void setId(String id) { this.id = id; }
    public String getUsername() { return username; }
    public void setUsername(String username) { this.username = username; }
    public String getPassword() { return password; }
    public void setPassword(String password) { this.password = password; }
    public String getConfirmPassword() { return confirmPassword; }
    public void setConfirmPassword(String confirmPassword) { this.confirmPassword = confirmPassword; }
}

注意：@Transient 来自 org.springframework.data.annotation 包，不是 jakarta.persistence.Transient（JPA 的注解）。

4.2.7 @CreatedDate / @LastModifiedDate —— 审计功能

Spring Data 提供审计注解，可以自动填充文档的创建时间和最后修改时间。

第一步：启用审计功能

package com.example.mongo.config;

import org.springframework.context.annotation.Configuration;
import org.springframework.data.mongodb.config.EnableMongoAuditing;

@Configuration
@EnableMongoAuditing  // 启用 MongoDB 审计功能
public class MongoAuditConfig {
}

第二步：在实体类中使用审计注解

package com.example.mongo.entity;

import org.springframework.data.annotation.CreatedDate;
import org.springframework.data.annotation.Id;
import org.springframework.data.annotation.LastModifiedDate;
import org.springframework.data.annotation.CreatedBy;
import org.springframework.data.annotation.LastModifiedBy;
import org.springframework.data.annotation.Version;
import org.springframework.data.mongodb.core.mapping.Document;

import java.time.LocalDateTime;

@Document(collection = "articles")
public class Article {

    @Id
    private String id;

    private String title;

    private String content;

    private String author;

    // 自动填充：文档首次保存时设置，后续更新不会改变
    @CreatedDate
    private LocalDateTime createdAt;

    // 自动填充：每次保存（包括更新）时都会刷新
    @LastModifiedDate
    private LocalDateTime updatedAt;

    // 自动填充操作人（需要配合 AuditorAware 接口使用）
    @CreatedBy
    private String createdBy;

    @LastModifiedBy
    private String updatedBy;

    // 乐观锁版本号：每次 save 时自动递增
    @Version
    private Long version;

    // getter / setter
    public String getId() { return id; }
    public void setId(String id) { this.id = id; }
    public String getTitle() { return title; }
    public void setTitle(String title) { this.title = title; }
    public String getContent() { return content; }
    public void setContent(String content) { this.content = content; }
    public String getAuthor() { return author; }
    public void setAuthor(String author) { this.author = author; }
    public LocalDateTime getCreatedAt() { return createdAt; }
    public void setCreatedAt(LocalDateTime createdAt) { this.createdAt = createdAt; }
    public LocalDateTime getUpdatedAt() { return updatedAt; }
    public void setUpdatedAt(LocalDateTime updatedAt) { this.updatedAt = updatedAt; }
    public String getCreatedBy() { return createdBy; }
    public void setCreatedBy(String createdBy) { this.createdBy = createdBy; }
    public String getUpdatedBy() { return updatedBy; }
    public void setUpdatedBy(String updatedBy) { this.updatedBy = updatedBy; }
    public Long getVersion() { return version; }
    public void setVersion(Long version) { this.version = version; }
}

第三步（可选）：实现 AuditorAware 接口，自动填充操作人

package com.example.mongo.config;

import org.springframework.data.domain.AuditorAware;
import org.springframework.stereotype.Component;

import java.util.Optional;

@Component
public class MongoAuditorAware implements AuditorAware<String> {

    @Override
    public Optional<String> getCurrentAuditor() {
        // 实际项目中从 SecurityContext 或 ThreadLocal 中获取当前登录用户
        // 这里简化为返回固定值
        return Optional.of("system");
    }
}

4.2.8 注解速查表

注解 来源 说明
@Document spring-data-mongodb 标记文档实体，指定集合名
@Id spring-data-commons 映射 _id 主键
@Field spring-data-mongodb 自定义字段名映射
@Indexed spring-data-mongodb 单字段索引
@CompoundIndex spring-data-mongodb 复合索引
@TextIndexed spring-data-mongodb 全文索引字段
@DBRef spring-data-mongodb 文档引用（类似外键）
@Transient spring-data-commons 不持久化该字段

package com.example.mongo.service;

import com.example.mongo.entity.User;
import com.example.mongo.repository.UserRepository;
import org.springframework.data.domain.Page;
import org.springframework.data.domain.PageRequest;
import org.springframework.data.domain.Sort;
import org.springframework.stereotype.Service;

import java.util.List;
import java.util.Optional;

@Service
public class UserService {

    private final UserRepository userRepository;

    public UserService(UserRepository userRepository) {
        this.userRepository = userRepository;
    }

    // ========== 创建 ==========

    public User createUser(User user) {
        if (userRepository.existsByUsername(user.getUsername())) {
            throw new RuntimeException("用户名已存在: " + user.getUsername());
        }
        user.setStatus("ACTIVE");
        return userRepository.insert(user);
    }

    public List<User> batchCreateUsers(List<User> users) {
        return userRepository.insert(users);
    }

    // ========== 查询 ==========

    public Optional<User> findById(String id) {
        return userRepository.findById(id);
    }

    public Optional<User> findByUsername(String username) {
        return userRepository.findByUsername(username);
    }

    public List<User> findAllUsers() {
        return userRepository.findAll(Sort.by(Sort.Direction.DESC, "createdAt"));
    }

    public Page<User> findUsersPage(int page, int size) {
        PageRequest pageRequest = PageRequest.of(page, size, Sort.by(Sort.Direction.DESC, "createdAt"));
        return userRepository.findAll(pageRequest);
    }

    public long countUsers() {
        return userRepository.count();
    }

    // ========== 更新 ==========

    public User updateUser(String id, User updatedUser) {
        User existingUser = userRepository.findById(id)
            .orElseThrow(() -> new RuntimeException("用户不存在: " + id));

        existingUser.setNickname(updatedUser.getNickname());
        existingUser.setEmail(updatedUser.getEmail());
        existingUser.setAge(updatedUser.getAge());
        existingUser.setTags(updatedUser.getTags());

        // save 会触发 @LastModifiedDate 自动更新
        return userRepository.save(existingUser);
    }

    // ========== 删除 ==========

    public void deleteUser(String id) {
        if (!userRepository.existsById(id)) {
            throw new RuntimeException("用户不存在: " + id);
        }
        userRepository.deleteById(id);
    }

    public void deleteAllUsers() {
        userRepository.deleteAll();
    }
}

Controller 层：

package com.example.mongo.controller;

@RestController
@RequestMapping("/api/users")
public class UserController {

        this.userService = userService;
    }

    public ResponseEntity<User> createUser(@RequestBody User user) {

        return ResponseEntity.status(HttpStatus.CREATED).body(created);
    }

    @PostMapping("/batch")
    public ResponseEntity<List<User>> batchCreate(@RequestBody List<User> users) {
        return ResponseEntity.status(HttpStatus.CREATED).body(userService.batchCreateUsers(users));
    }

    public ResponseEntity<User> getUser(@PathVariable String id) {
        return userService.findById(id)
            .map(ResponseEntity::ok)
            .orElse(ResponseEntity.notFound().build());
    }

    public ResponseEntity<User> getUserByUsername(@PathVariable String username) {
        return userService.findByUsername(username)
            .map(ResponseEntity::ok)
            .orElse(ResponseEntity.notFound().build());
    }

            @RequestParam(defaultValue = "10") int size) {

    public ResponseEntity<Long> countUsers() {
        return ResponseEntity.ok(userService.countUsers());
    }

    @PutMapping("/{id}")
    public ResponseEntity<User> updateUser(@PathVariable String id, @RequestBody User user) {
        return ResponseEntity.ok(userService.updateUser(id, user));
    }

4.4 方法名派生查询（Derived Query）规则

Spring Data MongoDB 能够根据 Repository 接口中的方法名自动生成查询实现，无需编写任何查询代码。这一机制称为方法名派生查询（Derived Query Method）。

4.4.1 命名规则

方法名由以下部分组成：

<操作关键字>By<条件字段><条件关键字>[And|Or<条件字段><条件关键字>...][OrderBy<排序字段><方向>]

操作关键字前缀：

前缀 返回类型 说明
findBy / getBy / queryBy / readBy List<T> 或 T 或 Optional<T> 查询
countBy long 统计数量
existsBy boolean 是否存在
deleteBy / removeBy long 或 List<T> 删除

4.4.2 条件关键字详细表

关键字 方法名示例 等价 MongoDB 查询
（无 / Is / Equals） findByUsername(String name) { username: name }
Not findByStatusNot(String status) { status: { $ne: status } }
Between findByAgeBetween(int min, int max) { age: { $gte: min, $lte: max } }
LessThan findByAgeLessThan(int age) { age: { $lt: age } }
LessThanEqual findByAgeLessThanEqual(int age) { age: { $lte: age } }
GreaterThan findByAgeGreaterThan(int age) { age: { $gt: age } }
GreaterThanEqual findByAgeGreaterThanEqual(int age) { age: { $gte: age } }
Before findByCreatedAtBefore(LocalDateTime time) { createdAt: { $lt: time } }
After findByCreatedAtAfter(LocalDateTime time) { createdAt: { $gt: time } }
Like / Containing findByNicknameLike(String name) { nickname: /name/ }
StartingWith findByUsernameStartingWith(String prefix) { username: /^prefix/ }
EndingWith findByEmailEndingWith(String suffix) { email: /suffix$/ }
Regex findByUsernameRegex(String regex) { username: { $regex: regex } }
In findByStatusIn(Collection<String> statuses) { status: { $in: statuses } }
NotIn findByStatusNotIn(Collection<String> statuses) { status: { $nin: statuses } }
IsNull / Null findByEmailIsNull() { email: null }
IsNotNull / NotNull findByEmailIsNotNull() { email: { $ne: null } }
Exists findByProfileExists(boolean exists) { profile: { $exists: exists } }
True findByActiveTrue() { active: true }
False findByActiveFalse() { active: false }
Near findByLocationNear(Point p, Distance d) 地理空间近距离查询
OrderBy findByStatusOrderByCreatedAtDesc(String s) { status: s } + sort
And findByUsernameAndEmail(String u, String e) { username: u, email: e }
Or findByUsernameOrEmail(String u, String e) { $or: [{username: u}, {email: e}] }
First / Top findFirstByOrderByCreatedAtDesc() 返回排序后的第一条
TopN findTop5ByStatusOrderByAgeDesc(String s) 返回前 N 条
Distinct findDistinctByRole(String role) 去重查询

4.5 @Query 注解：自定义查询语句

方法名派生查询虽然方便，但面对复杂条件时方法名会变得冗长且可读性差。@Query 注解允许你直接编写 MongoDB JSON 查询语句，既灵活又清晰。

4.5.1 基本用法

@Query 注解写在 Repository 方法上，值为 MongoDB JSON 查询语句：

package com.example.mongo.repository;

import com.example.mongo.entity.Article;
import org.springframework.data.mongodb.repository.MongoRepository;
import org.springframework.data.mongodb.repository.Query;
import org.springframework.stereotype.Repository;

import java.util.List;

@Repository
public interface ArticleRepository extends MongoRepository<Article, String> {

    // 示例1：基本 @Query —— 查询指定作者的所有文章
    @Query("{ 'author': ?0 }")
    List<Article> findByAuthor(String author);

    // 示例2：多条件查询 —— 查询指定作者和状态的文章
    @Query("{ 'author': ?0, 'status': ?1 }")
    List<Article> findByAuthorAndStatus(String author, String status);
}

?0、?1 分别对应方法的第 1 个、第 2 个参数（从 0 开始编号）。

4.5.2 位置参数 ?0 ?1

@Repository
public interface ArticleRepository extends MongoRepository<Article, String> {

    // 示例3：使用比较操作符 —— 查询阅读量大于指定值的文章
    @Query("{ 'view_count': { $gt: ?0 } }")
    List<Article> findByViewCountGreaterThan(int minViewCount);

    // 示例4：多参数 + 范围查询 —— 查询指定分类且阅读量在某范围内的文章
    @Query("{ 'category': ?0, 'view_count': { $gte: ?1, $lte: ?2 } }")
    List<Article> findByCategoryAndViewCountRange(String category, int minCount, int maxCount);

    // 示例5：$or 查询 —— 查询标题或内容包含关键词的文章
    @Query("{ $or: [ { 'title': { $regex: ?0, $options: 'i' } }, { 'content': { $regex: ?0, $options: 'i' } } ] }")
    List<Article> searchByKeyword(String keyword);
}

4.5.3 SpEL 表达式 :#{#paramName}

当方法参数较多或需要更灵活的表达式时，可以使用 SpEL（Spring Expression Language）引用参数名：

import org.springframework.data.repository.query.Param;

@Repository
public interface ArticleRepository extends MongoRepository<Article, String> {

    // 示例6：使用 SpEL 表达式引用命名参数
    @Query("{ 'author': :#{#author}, 'status': :#{#status} }")
    List<Article> findArticles(@Param("author") String author, @Param("status") String status);

    // 示例7：SpEL + 条件表达式 —— 如果参数为 null 则不加该条件
    // 注意：这种写法需要使用 ?#{} 语法
    @Query("{ 'category': ?0, 'view_count': { $gte: ?#{[1] ?: 0} } }")
    List<Article> findByCategoryWithMinViews(String category, Integer minViews);

    // 示例8：SpEL 引用复杂对象的属性
    @Query("{ 'author': :#{#filter.author}, 'category': :#{#filter.category} }")
    List<Article> findByFilter(@Param("filter") ArticleFilter filter);
}

4.5.4 投影查询 —— 指定返回字段

通过 @Query 的 fields 属性指定只返回哪些字段，减少数据传输量：

@Repository
public interface ArticleRepository extends MongoRepository<Article, String> {

    // 示例9：只返回 title 和 author 字段
    @Query(value = "{ 'status': 'published' }", fields = "{ 'title': 1, 'author': 1 }")
    List<Article> findPublishedTitles();

    // 示例10：排除 content 字段（文章内容可能很大）
    @Query(value = "{ 'author': ?0 }", fields = "{ 'content': 0 }")
    List<Article> findByAuthorWithoutContent(String author);

    // 示例11：只返回嵌套文档的某些字段
    @Query(value = "{ 'status': 'published' }", fields = "{ 'title': 1, 'author': 1, 'view_count': 1, '_id': 0 }")
    List<Article> findPublishedArticleSummaries();
}

投影规则（与 Shell 中一致）：

  • { 'title': 1, 'author': 1 } —— 只返回 title、author 和 _id
  • { 'content': 0 } —— 返回除 content 外的所有字段
  • 包含（1）和排除（0）不能混用，_id 除外

4.5.5 分页与排序

@Query 方法可以通过添加 Pageable 和 Sort 参数实现分页和排序：

import org.springframework.data.domain.Page;
import org.springframework.data.domain.Pageable;
import org.springframework.data.domain.Sort;

@Repository
public interface ArticleRepository extends MongoRepository<Article, String> {

    // 示例12：@Query + 分页
    @Query("{ 'status': ?0 }")
    Page<Article> findByStatusWithPage(String status, Pageable pageable);

    // 示例13：@Query + 排序
    @Query("{ 'category': ?0, 'status': 'published' }")
    List<Article> findPublishedByCategory(String category, Sort sort);

    // 示例14：@Query + 投影 + 分页
    @Query(value = "{ 'author': ?0 }", fields = "{ 'title': 1, 'view_count': 1, 'created_at': 1 }")
    Page<Article> findArticleSummaryByAuthor(String author, Pageable pageable);
}

Service 层调用示例：

package com.example.mongo.service;

import com.example.mongo.entity.Article;
import com.example.mongo.repository.ArticleRepository;
import org.springframework.data.domain.Page;
import org.springframework.data.domain.PageRequest;
import org.springframework.data.domain.Sort;
import org.springframework.stereotype.Service;

import java.util.List;

@Service
public class ArticleService {

    private final ArticleRepository articleRepository;

    public ArticleService(ArticleRepository articleRepository) {
        this.articleRepository = articleRepository;
    }

    // 分页查询已发布文章，按创建时间倒序
    public Page<Article> findPublishedArticles(int page, int size) {
        PageRequest pageRequest = PageRequest.of(page, size, Sort.by(Sort.Direction.DESC, "created_at"));
        return articleRepository.findByStatusWithPage("published", pageRequest);
    }

    // 查询指定分类的文章，按阅读量排序
    public List<Article> findArticlesByCategory(String category) {
        return articleRepository.findPublishedByCategory(category, Sort.by(Sort.Direction.DESC, "view_count"));
    }

    // 查询作者的文章摘要（分页）
    public Page<Article> findAuthorArticleSummary(String author, int page, int size) {
        PageRequest pageRequest = PageRequest.of(page, size, Sort.by(Sort.Direction.DESC, "created_at"));
        return articleRepository.findArticleSummaryByAuthor(author, pageRequest);
    }
}

4.5.7 综合示例：完整的 ArticleRepository

将上面所有的 @Query 示例汇总到一个完整的 Repository 接口中：

package com.example.mongo.repository;

import com.example.mongo.dto.ArticleFilter;
import com.example.mongo.entity.Article;
import org.springframework.data.domain.Page;
import org.springframework.data.domain.Pageable;
import org.springframework.data.domain.Sort;
import org.springframework.data.mongodb.repository.DeleteQuery;
import org.springframework.data.mongodb.repository.MongoRepository;
import org.springframework.data.mongodb.repository.Query;
import org.springframework.data.repository.query.Param;
import org.springframework.stereotype.Repository;

import java.util.List;

@Repository
public interface ArticleRepository extends MongoRepository<Article, String> {

    // ========== 基本查询 ==========

    // 1. 按作者查询
    @Query("{ 'author': ?0 }")
    List<Article> findByAuthor(String author);

    // 2. 按作者和状态查询
    @Query("{ 'author': ?0, 'status': ?1 }")
    List<Article> findByAuthorAndStatus(String author, String status);

    // 3. 阅读量大于指定值
    @Query("{ 'view_count': { $gt: ?0 } }")
    List<Article> findByViewCountGreaterThan(int minViewCount);

    // 4. 分类 + 阅读量范围
    @Query("{ 'category': ?0, 'view_count': { $gte: ?1, $lte: ?2 } }")
    List<Article> findByCategoryAndViewCountRange(String category, int minCount, int maxCount);

    // ========== 正则 / 模糊查询 ==========

    // 5. 全文关键词搜索（标题或内容）
    @Query("{ $or: [ { 'title': { $regex: ?0, $options: 'i' } }, { 'content': { $regex: ?0, $options: 'i' } } ] }")
    List<Article> searchByKeyword(String keyword);

    // ========== SpEL 表达式 ==========

    // 6. 使用命名参数
    @Query("{ 'author': :#{#author}, 'status': :#{#status} }")
    List<Article> findArticles(@Param("author") String author, @Param("status") String status);

    // ========== 投影查询 ==========

    // 7. 只返回标题和作者
    @Query(value = "{ 'status': 'published' }", fields = "{ 'title': 1, 'author': 1 }")
    List<Article> findPublishedTitles();

    // 8. 排除大字段
    @Query(value = "{ 'author': ?0 }", fields = "{ 'content': 0 }")
    List<Article> findByAuthorWithoutContent(String author);

    // ========== 分页 + 排序 ==========

    // 9. 分页查询
    @Query("{ 'status': ?0 }")
    Page<Article> findByStatusWithPage(String status, Pageable pageable);

    // 10. 排序查询
    @Query("{ 'category': ?0, 'status': 'published' }")
    List<Article> findPublishedByCategory(String category, Sort sort);

    // 11. 投影 + 分页
    @Query(value = "{ 'author': ?0 }", fields = "{ 'title': 1, 'view_count': 1, 'created_at': 1 }")
    Page<Article> findArticleSummaryByAuthor(String author, Pageable pageable);

    // ========== 统计 / 存在性 ==========

    // 12. 统计某作者的已发布文章数
    @Query(value = "{ 'author': ?0, 'status': 'published' }", count = true)
    long countPublishedByAuthor(String author);

    // 13. 判断标题是否存在
    @Query(value = "{ 'title': ?0 }", exists = true)
    boolean existsByTitle(String title);

    // ========== 数组查询 ==========

    // 14. 查询包含指定标签的已发布文章
    @Query("{ 'tags': { $all: ?0 }, 'status': 'published' }")
    List<Article> findByAllTags(List<String> tags);

    // 15. 查询标签中包含任意一个指定值的文章
    @Query("{ 'tags': { $in: ?0 } }")
    List<Article> findByAnyTag(List<String> tags);

    // ========== 嵌套文档查询 ==========

    // 16. 查询有特定用户评论的文章
    @Query("{ 'comments.user': ?0 }")
    List<Article> findByCommentUser(String username);

    // ========== 删除查询 ==========

    // 17. 删除指定状态的文章
    @DeleteQuery("{ 'status': ?0 }")
    long deleteByArticleStatus(String status);

    // 18. 删除作者的所有草稿
    @DeleteQuery("{ 'author': ?0, 'status': 'draft' }")
    long deleteDraftsByAuthor(String author);
}

4.5.8 @Query vs 方法名派生查询 —— 如何选择

场景 推荐方式 原因
简单的单字段查询 方法名派生 简洁直观
双字段 AND 查询 方法名派生 方法名不长，可读性好
三个以上条件的查询 @Query 方法名过长，可读性差
正则 / 模糊搜索 @Query $regex + $options 更灵活
$or / $nor 等逻辑组合 @Query 方法名很难表达复杂逻辑
数组操作（$all、$elemMatch） @Query 派生查询不支持
投影查询（返回部分字段） @Query fields 属性很方便
动态条件（条件数量不固定） MongoTemplate @Query 和派生查询都不适合
