第05章 MongoTemplate 详解

在上一章中我们学习了 MongoRepository 的便捷用法。但在实际项目中，面对复杂的查询、动态条件拼接、批量更新等场景，MongoRepository 往往力不从心。本章将深入学习 MongoTemplate —— Spring Data MongoDB 提供的底层模板类，掌握它意味着你可以驾驭几乎所有 MongoDB 操作。

5.1 MongoTemplate vs MongoRepository —— 如何选择

5.1.1 对比总览

对比维度 MongoRepository MongoTemplate
使用方式 继承接口，声明方法名即可 注入 Bean，手动编写查询逻辑
学习成本 低，上手快 中等，需要了解 Criteria/Query API
CRUD 操作 自动生成，开箱即用 需要手动编写，但控制力强
复杂查询 方法名派生有局限，@Query 也受限 支持任意复杂的查询条件组合
动态查询 不擅长（条件不固定时难以处理） 非常擅长，可以动态拼接 Criteria
聚合操作 不支持 完整支持 Aggregation Pipeline
批量操作 有限支持 完整支持 bulkOps
部分字段更新 不方便（save 是整体替换） 精确控制 Update 哪些字段
分页排序 内置支持，使用简单 需手动实现，但更灵活
原子操作 不支持 findAndModify 支持 findAndModify、findAndReplace
返回值控制 返回完整实体 可以投影返回部分字段

5.1.2 适用场景分析

推荐使用 MongoRepository 的场景：

  • 简单的增删改查操作
  • 查询条件固定、明确
  • 快速搭建原型，追求开发效率
  • 标准的分页列表查询

推荐使用 MongoTemplate 的场景：

  • 动态查询条件（如管理后台的多条件筛选）
  • 复杂的嵌套文档查询
  • 需要原子更新操作（如计数器自增、数组元素操作）
  • 聚合统计
  • 需要精细控制返回字段
  • 批量操作，对性能有要求

5.1.3 最佳实践：混合使用

在实际项目中，两者完全可以共存。推荐策略：

// 1. 定义 Repository 接口处理简单 CRUD
public interface UserRepository extends MongoRepository<User, String> {
    List<User> findByStatus(String status);
    Optional<User> findByEmail(String email);
}

// 2. 在 Service 中同时注入两者
@Service
public class UserService {

    private final UserRepository userRepository;
    private final MongoTemplate mongoTemplate;

    public UserService(UserRepository userRepository, MongoTemplate mongoTemplate) {
        this.userRepository = userRepository;
        this.mongoTemplate = mongoTemplate;
    }

    // 简单查询 —— 使用 Repository
    public Optional<User> findByEmail(String email) {
        return userRepository.findByEmail(email);
    }

    // 复杂动态查询 —— 使用 MongoTemplate
    public List<User> searchUsers(String name, Integer minAge, Integer maxAge, String status) {
        Query query = new Query();
        if (name != null) {
            query.addCriteria(Criteria.where("name").regex(name, "i"));
        }
        if (minAge != null) {
            query.addCriteria(Criteria.where("age").gte(minAge));
        }
        if (maxAge != null) {
            query.addCriteria(Criteria.where("age").lte(maxAge));
        }
        if (status != null) {
            query.addCriteria(Criteria.where("status").is(status));
        }
        return mongoTemplate.find(query, User.class);
    }
}

5.2 使用 MongoTemplate 进行 CRUD 操作

前置准备：实体类

后续示例统一使用以下实体类：

package com.example.mongo.entity;

import org.springframework.data.annotation.Id;
import org.springframework.data.mongodb.core.mapping.Document;
import org.springframework.data.mongodb.core.mapping.Field;
import java.time.LocalDateTime;
import java.util.List;

@Document(collection = "users")
public class User {

    @Id
    private String id;

    private String name;

    private Integer age;

    private String email;

    private String status;

    private List<String> tags;

    @Field("created_at")
    private LocalDateTime createdAt;

    // 构造方法
    public User() {}

    public User(String name, Integer age, String email) {
        this.name = name;
        this.age = age;
        this.email = email;
        this.createdAt = LocalDateTime.now();
        this.status = "ACTIVE";
    }

    // getter / setter 省略，实际项目中建议使用 Lombok @Data
}

5.2.1 insert / insertAll：插入文档

insert 方法用于向集合中插入一个新文档。如果文档的 _id 已存在，会抛出 DuplicateKeyException。

插入单个文档：

package com.example.mongo.service;

import com.example.mongo.entity.User;
import org.springframework.data.mongodb.core.MongoTemplate;
import org.springframework.stereotype.Service;

@Service
public class UserService {

    private final MongoTemplate mongoTemplate;

    public UserService(MongoTemplate mongoTemplate) {
        this.mongoTemplate = mongoTemplate;
    }

    public User createUser(User user) {
        // insert 会将生成的 _id 回写到 user 对象中
        return mongoTemplate.insert(user);
    }

    // 指定集合名称插入（当实体类没有 @Document 注解时有用）
    public User createUserToCollection(User user, String collectionName) {
        return mongoTemplate.insert(user, collectionName);
    }
}

批量插入文档：

import java.util.Collection;
import java.util.List;

public Collection<User> batchCreateUsers(List<User> users) {
    // insertAll 批量插入，性能优于循环调用 insert
    return mongoTemplate.insertAll(users);
}

// 也可以使用 insert 的重载方法批量插入到指定集合
public Collection<User> batchInsert(List<User> users) {
    return mongoTemplate.insert(users, "users");
}

使用示例：

// 插入单个
User user = new User("张三", 25, "zhangsan@example.com");
user.setTags(List.of("Java", "MongoDB"));
User saved = userService.createUser(user);
System.out.println("生成的ID: " + saved.getId());

// 批量插入
List<User> users = List.of(
    new User("李四", 30, "lisi@example.com"),
    new User("王五", 28, "wangwu@example.com"),
    new User("赵六", 35, "zhaoliu@example.com")
);
Collection<User> result = userService.batchCreateUsers(users);
System.out.println("插入数量: " + result.size());

注意：insert 和 save 的区别：insert 在 _id 冲突时抛异常，save 在 _id 存在时执行更新。

5.2.2 find / findOne / findById：查询文档

查询所有文档：

import org.springframework.data.mongodb.core.query.Query;
import java.util.List;

public List<User> findAllUsers() {
    return mongoTemplate.findAll(User.class);
}

// 指定集合名称
public List<User> findAllUsersFromCollection(String collectionName) {
    return mongoTemplate.findAll(User.class, collectionName);
}

根据条件查询多个文档：

import org.springframework.data.mongodb.core.query.Criteria;
import org.springframework.data.mongodb.core.query.Query;

public List<User> findActiveUsers() {
    Query query = new Query(Criteria.where("status").is("ACTIVE"));
    return mongoTemplate.find(query, User.class);
}

public List<User> findUsersByAgeRange(int minAge, int maxAge) {
    Query query = new Query(
        Criteria.where("age").gte(minAge).lte(maxAge)
    );
    return mongoTemplate.find(query, User.class);
}

查询单个文档：

public User findOneByEmail(String email) {
    Query query = new Query(Criteria.where("email").is(email));
    // findOne：返回匹配的第一个文档，没有匹配则返回 null
    return mongoTemplate.findOne(query, User.class);
}

根据 ID 查询：

public User findUserById(String id) {
    // findById 是根据 _id 查询的快捷方法
    return mongoTemplate.findById(id, User.class);
}

查询文档数量：

public long countActiveUsers() {
    Query query = new Query(Criteria.where("status").is("ACTIVE"));
    return mongoTemplate.count(query, User.class);
}

判断文档是否存在：

public boolean existsByEmail(String email) {
    Query query = new Query(Criteria.where("email").is(email));
    return mongoTemplate.exists(query, User.class);
}

5.2.3 save：保存文档

save 方法的行为取决于文档是否包含 _id：

  • 如果 _id 为 null → 执行插入操作（等同于 insert）
  • 如果 _id 有值且已存在 → 执行整体替换更新
  • 如果 _id 有值但不存在 → 执行插入操作
public User saveUser(User user) {
    return mongoTemplate.save(user);
}

典型使用场景 —— 查出来修改后保存：

public User updateUserName(String id, String newName) {
    User user = mongoTemplate.findById(id, User.class);
    if (user == null) {
        throw new RuntimeException("用户不存在: " + id);
    }
    user.setName(newName);
    // save 会整体替换该文档（注意：其他字段如果为 null 也会被写入）
    return mongoTemplate.save(user);
}

⚠️ 警告：save 是整体替换文档，如果你只想更新某几个字段，应该使用 updateFirst / updateMulti + Update 对象，避免意外覆盖其他字段。

5.2.4 updateFirst / updateMulti：更新文档

与 save 不同，updateFirst 和 updateMulti 可以只更新指定的字段，不影响文档中的其他字段。

更新第一个匹配的文档：

import org.springframework.data.mongodb.core.query.Update;
import com.mongodb.client.result.UpdateResult;

public long updateUserStatus(String id, String newStatus) {
    Query query = new Query(Criteria.where("_id").is(id));
    Update update = new Update().set("status", newStatus);

    UpdateResult result = mongoTemplate.updateFirst(query, update, User.class);
    return result.getModifiedCount(); // 返回实际修改的文档数
}

更新所有匹配的文档：

public long deactivateOldUsers(int maxAge) {
    Query query = new Query(Criteria.where("age").gt(maxAge));
    Update update = new Update()
        .set("status", "INACTIVE")
        .set("updatedAt", LocalDateTime.now());

    UpdateResult result = mongoTemplate.updateMulti(query, update, User.class);

    System.out.println("匹配数量: " + result.getMatchedCount());
    System.out.println("修改数量: " + result.getModifiedCount());

    return result.getModifiedCount();
}

UpdateResult 常用方法：

UpdateResult result = mongoTemplate.updateFirst(query, update, User.class);

result.getMatchedCount();    // 匹配到的文档数
result.getModifiedCount();   // 实际被修改的文档数
result.getUpsertedId();      // 如果是 upsert 操作，返回新插入文档的 _id
result.wasAcknowledged();    // 操作是否被确认

5.2.5 remove：删除文档

import com.mongodb.client.result.DeleteResult;

// 根据条件删除
public long deleteUserByEmail(String email) {
    Query query = new Query(Criteria.where("email").is(email));
    DeleteResult result = mongoTemplate.remove(query, User.class);
    return result.getDeletedCount();
}

// 根据对象删除（需要对象中包含 _id）
public long deleteUser(User user) {
    DeleteResult result = mongoTemplate.remove(user);
    return result.getDeletedCount();
}

// 删除集合中所有文档
public long deleteAllUsers() {
    Query query = new Query();
    DeleteResult result = mongoTemplate.remove(query, User.class);
    return result.getDeletedCount();
}

// 删除并返回被删除的文档
public User findAndRemove(String email) {
    Query query = new Query(Criteria.where("email").is(email));
    return mongoTemplate.findAndRemove(query, User.class);
}

// 删除所有匹配的文档并返回它们
public List<User> findAllAndRemove(String status) {
    Query query = new Query(Criteria.where("status").is(status));
    return mongoTemplate.findAllAndRemove(query, User.class);
}

5.3 Criteria 与 Query 构建复杂查询条件

Criteria 和 Query 是 MongoTemplate 中构建查询条件的核心 API。Criteria 用于定义条件，Query 用于组装条件并附加排序、分页、投影等设置。

5.3.1 Criteria 基本比较操作符

import org.springframework.data.mongodb.core.query.Criteria;
import org.springframework.data.mongodb.core.query.Query;

// is —— 等于
Query q1 = new Query(Criteria.where("name").is("张三"));

// ne —— 不等于
Query q2 = new Query(Criteria.where("status").ne("DELETED"));

// gt / gte —— 大于 / 大于等于
Query q3 = new Query(Criteria.where("age").gt(18));
Query q4 = new Query(Criteria.where("age").gte(18));

// lt / lte —— 小于 / 小于等于
Query q5 = new Query(Criteria.where("age").lt(60));
Query q6 = new Query(Criteria.where("age").lte(60));

// 链式调用实现范围查询：18 <= age <= 60
Query q7 = new Query(Criteria.where("age").gte(18).lte(60));

// in —— 值在集合中
Query q8 = new Query(Criteria.where("status").in("ACTIVE", "PENDING"));

// nin —— 值不在集合中
Query q9 = new Query(Criteria.where("status").nin("DELETED", "BANNED"));

// exists —— 字段是否存在
Query q10 = new Query(Criteria.where("email").exists(true));

// regex —— 正则匹配（第二个参数 "i" 表示不区分大小写）
Query q11 = new Query(Criteria.where("name").regex("^张", "i"));

5.3.2 逻辑组合查询

AND 组合（默认行为）：

在同一个 Query 上添加多个 Criteria 就是 AND 关系：

// 查询 status 为 ACTIVE 且 age > 25 的用户
Query query = new Query();
query.addCriteria(Criteria.where("status").is("ACTIVE"));
query.addCriteria(Criteria.where("age").gt(25));
// 等价于 MongoDB: { status: "ACTIVE", age: { $gt: 25 } }

使用 andOperator：

当需要对同一个字段组合多个条件，或者需要显式的 $and 时：

// 对同一字段使用 $and（避免 key 冲突）
Criteria criteria = new Criteria().andOperator(
    Criteria.where("age").gt(18),
    Criteria.where("age").lt(60),
    Criteria.where("status").is("ACTIVE")
);
Query query = new Query(criteria);
// MongoDB: { $and: [ {age: {$gt:18}}, {age: {$lt:60}}, {status: "ACTIVE"} ] }

OR 组合 —— orOperator：

// 查询 status 为 ACTIVE 或 age < 20 的用户
Criteria criteria = new Criteria().orOperator(
    Criteria.where("status").is("ACTIVE"),
    Criteria.where("age").lt(20)
);
Query query = new Query(criteria);
// MongoDB: { $or: [ {status: "ACTIVE"}, {age: {$lt: 20}} ] }

AND + OR 混合使用：

// 查询 name 以 "张" 开头，且 (status 为 ACTIVE 或 age > 30) 的用户
Criteria criteria = Criteria.where("name").regex("^张")
    .orOperator(
        Criteria.where("status").is("ACTIVE"),
        Criteria.where("age").gt(30)
    );
Query query = new Query(criteria);
// MongoDB: { name: /^张/, $or: [ {status: "ACTIVE"}, {age: {$gt: 30}} ] }

5.3.3 嵌套文档查询

假设 User 文档中有嵌套的地址信息：

{
  "name": "张三",
  "address": {
    "city": "北京",
    "district": "朝阳区",
    "zipCode": "100000"
  }
}

使用点号表示法查询嵌套字段：

// 查询地址在北京的用户
Query query = new Query(Criteria.where("address.city").is("北京"));
List<User> users = mongoTemplate.find(query, User.class);

// 查询地址在北京市朝阳区的用户
Query query2 = new Query(
    Criteria.where("address.city").is("北京")
            .and("address.district").is("朝阳区")
);
List<User> users2 = mongoTemplate.find(query2, User.class);

5.3.4 数组字段查询

假设用户文档中有如下结构：

{
  "name": "张三",
  "tags": ["Java", "MongoDB", "Spring"],
  "scores": [
    { "subject": "数学", "score": 90 },
    { "subject": "英语", "score": 85 }
  ]
}

基本数组查询：

// 查询 tags 中包含 "Java" 的用户
Query q1 = new Query(Criteria.where("tags").is("Java"));

// 查询 tags 中同时包含 "Java" 和 "MongoDB" 的用户
Query q2 = new Query(Criteria.where("tags").all("Java", "MongoDB"));

// 查询 tags 数组大小为 3 的用户
Query q3 = new Query(Criteria.where("tags").size(3));

elemMatch —— 数组内嵌文档查询：

// 查询 scores 数组中存在「数学成绩大于 80」的用户
Query q4 = new Query(
    Criteria.where("scores").elemMatch(
        Criteria.where("subject").is("数学").and("score").gt(80)
    )
);
List<User> users = mongoTemplate.find(q4, User.class);

5.3.5 综合查询示例（由简到繁）

下面给出 10 个不同复杂度的查询示例：

@Service
public class UserQueryService {

    private final MongoTemplate mongoTemplate;

    public UserQueryService(MongoTemplate mongoTemplate) {
        this.mongoTemplate = mongoTemplate;
    }

    // 示例1：精确匹配单个字段
    public User findByEmail(String email) {
        Query query = new Query(Criteria.where("email").is(email));
        return mongoTemplate.findOne(query, User.class);
    }

    // 示例2：范围查询
    public List<User> findByAgeRange(int min, int max) {
        Query query = new Query(Criteria.where("age").gte(min).lte(max));
        return mongoTemplate.find(query, User.class);
    }

    // 示例3：正则模糊查询
    public List<User> searchByName(String keyword) {
        Query query = new Query(
            Criteria.where("name").regex(keyword, "i")
        );
        return mongoTemplate.find(query, User.class);
    }

    // 示例4：IN 查询 + 排序
    public List<User> findByStatusIn(List<String> statuses) {
        Query query = new Query(
            Criteria.where("status").in(statuses)
        );
        query.with(Sort.by(Sort.Direction.DESC, "createdAt"));
        return mongoTemplate.find(query, User.class);
    }

    // 示例5：多条件 AND 查询
    public List<User> findActiveAdults() {
        Query query = new Query();
        query.addCriteria(Criteria.where("status").is("ACTIVE"));
        query.addCriteria(Criteria.where("age").gte(18));
        return mongoTemplate.find(query, User.class);
    }

    // 示例6：OR 条件查询
    public List<User> findVipOrSenior() {
        Criteria criteria = new Criteria().orOperator(
            Criteria.where("tags").is("VIP"),
            Criteria.where("age").gte(60)
        );
        return mongoTemplate.find(new Query(criteria), User.class);
    }

    // 示例7：嵌套文档 + AND + OR 组合
    public List<User> findBeijingActiveOrYoung() {
        Criteria criteria = Criteria.where("address.city").is("北京")
            .orOperator(
                Criteria.where("status").is("ACTIVE"),
                Criteria.where("age").lt(25)
            );
        return mongoTemplate.find(new Query(criteria), User.class);
    }

    // 示例8：数组 elemMatch + 其他条件
    public List<User> findHighScoreStudents(String subject, int minScore) {
        Query query = new Query();
        query.addCriteria(Criteria.where("status").is("ACTIVE"));
        query.addCriteria(
            Criteria.where("scores").elemMatch(
                Criteria.where("subject").is(subject).and("score").gte(minScore)
            )
        );
        return mongoTemplate.find(query, User.class);
    }

    // 示例9：字段投影（只返回部分字段）+ 分页
    public List<User> findUserNamesAndEmails(int page, int size) {
        Query query = new Query(Criteria.where("status").is("ACTIVE"));
        query.fields().include("name").include("email").exclude("_id");
        query.skip((long) page * size).limit(size);
        return mongoTemplate.find(query, User.class);
    }

    // 示例10：复杂动态查询（管理后台常见场景）
    public List<User> advancedSearch(String name, Integer minAge, Integer maxAge,
                                     String status, String city,
                                     List<String> tags, String sortField,
                                     String sortDirection) {
        Query query = new Query();

        if (name != null && !name.isBlank()) {
            query.addCriteria(Criteria.where("name").regex(name, "i"));
        }
        if (minAge != null) {
            query.addCriteria(Criteria.where("age").gte(minAge));
        }
        if (maxAge != null) {
            // 注意：如果 minAge 和 maxAge 都有值，不能对同一字段 addCriteria 两次
            // 需要合并成一个 Criteria：Criteria.where("age").gte(minAge).lte(maxAge)
            // 这里简化处理，假设 minAge 和 maxAge 互斥传入
            query.addCriteria(Criteria.where("age").lte(maxAge));
        }
        if (status != null) {
            query.addCriteria(Criteria.where("status").is(status));
        }
        if (city != null) {
            query.addCriteria(Criteria.where("address.city").is(city));
        }
        if (tags != null && !tags.isEmpty()) {
            query.addCriteria(Criteria.where("tags").all(tags));
        }

        Sort.Direction direction = "desc".equalsIgnoreCase(sortDirection)
            ? Sort.Direction.DESC : Sort.Direction.ASC;
        query.with(Sort.by(direction, sortField != null ? sortField : "createdAt"));

        return mongoTemplate.find(query, User.class);
    }
}

提示：对同一个字段不能在同一个 Query 中 addCriteria 两次（会报错），需要合并为一个 Criteria 链式调用，如 Criteria.where("age").gte(18).lte(60)。

5.4 Update 对象与部分字段更新

Update 类是 MongoTemplate 中用于构建更新操作的核心类。它对应 MongoDB 的各种更新操作符。

5.4.1 常用 Update 方法

import org.springframework.data.mongodb.core.query.Update;

// set —— 设置字段值（字段不存在则创建）
Update u1 = new Update().set("name", "新名字");

// unset —— 删除字段
Update u2 = new Update().unset("tempField");

// inc —— 字段值自增（支持正负数）
Update u3 = new Update().inc("viewCount", 1);      // +1
Update u4 = new Update().inc("balance", -100);      // -100

// push —— 向数组添加一个元素
Update u5 = new Update().push("tags", "Python");

// push 多个元素
Update u6 = new Update().push("tags").each("Python", "Go", "Rust");

// addToSet —— 向数组添加元素（如果不存在才添加，避免重复）
Update u7 = new Update().addToSet("tags", "Java");

// addToSet 多个元素
Update u8 = new Update().addToSet("tags").each("Java", "Spring");

// pull —— 从数组中移除匹配的元素
Update u9 = new Update().pull("tags", "PHP");

// pop —— 移除数组的第一个或最后一个元素
Update u10 = new Update().pop("tags", Update.Position.LAST);   // 移除最后一个
Update u11 = new Update().pop("tags", Update.Position.FIRST);  // 移除第一个

// rename —— 重命名字段
Update u12 = new Update().rename("oldField", "newField");

// 链式组合多个更新操作
Update combined = new Update()
    .set("name", "新名字")
    .set("status", "ACTIVE")
    .inc("loginCount", 1)
    .set("lastLoginTime", LocalDateTime.now());

@Service
public class UserUpdateService {

    private final MongoTemplate mongoTemplate;

    public UserUpdateService(MongoTemplate mongoTemplate) {
        this.mongoTemplate = mongoTemplate;
    }

    // 更新用户的基本信息
    public long updateUserInfo(String id, String name, String email) {

    public void incrementViewCount(String articleId) {

        mongoTemplate.updateFirst(query, update, "articles");
    }

    // 给用户添加标签（避免重复）
    public void addTag(String userId, String tag) {
        Query query = new Query(Criteria.where("_id").is(userId));
        Update update = new Update().addToSet("tags", tag);
        mongoTemplate.updateFirst(query, update, User.class);
    }

    // 批量给用户添加多个标签
    public void addTags(String userId, List<String> newTags) {
        Query query = new Query(Criteria.where("_id").is(userId));

        mongoTemplate.updateFirst(query, update, User.class);
    }

    // 移除用户的某个标签
    public void removeTag(String userId, String tag) {

        Update update = new Update().pull("tags", tag);
        mongoTemplate.updateFirst(query, update, User.class);
    }

    // 批量更新：将所有超过60岁的用户状态设置为 SENIOR
    public long markSeniorUsers() {
        Query query = new Query(Criteria.where("age").gte(60));
        Update update = new Update()
            .set("status", "SENIOR")
            .set("updatedAt", LocalDateTime.now());
        UpdateResult result = mongoTemplate.updateMulti(query, update, User.class);
        return result.getModifiedCount();
    }
}

5.4.3 findAndModify —— 原子操作

findAndModify 可以在一次原子操作中查找并修改文档，常用于实现计数器、抢锁等场景。

import org.springframework.data.mongodb.core.FindAndModifyOptions;

// 自增序列号生成器 —— 经典用法
public long getNextSequence(String sequenceName) {
    Query query = new Query(Criteria.where("_id").is(sequenceName));
    Update update = new Update().inc("value", 1);

    // returnNew(true)：返回修改后的文档（而不是修改前的）
    // upsert(true)：如果不存在则自动创建
    FindAndModifyOptions options = FindAndModifyOptions.options()
        .returnNew(true)
        .upsert(true);

    Sequence sequence = mongoTemplate.findAndModify(
        query, update, options, Sequence.class
    );
    return sequence != null ? sequence.getValue() : 1;
}

// Sequence 实体
@Document(collection = "sequences")
public class Sequence {
    @Id
    private String id;
    private long value;

    // getter / setter
    public long getValue() { return value; }
    public void setValue(long value) { this.value = value; }
}

带条件的原子更新：

// 抢购场景：库存大于 0 时才扣减
public Product decrementStock(String productId) {
    Query query = new Query(
        Criteria.where("_id").is(productId).and("stock").gt(0)
    );
    Update update = new Update().inc("stock", -1);
    FindAndModifyOptions options = FindAndModifyOptions.options().returnNew(true);

    return mongoTemplate.findAndModify(query, update, options, Product.class);
    // 返回 null 表示库存不足（条件不匹配）
}

5.4.4 Upsert 操作

Upsert = Update + Insert：如果匹配到文档则更新，否则插入新文档。

// 用户设备信息管理：存在则更新登录时间，不存在则创建
public void upsertDeviceInfo(String userId, String deviceId, String deviceName) {
    Query query = new Query(
        Criteria.where("userId").is(userId).and("deviceId").is(deviceId)
    );
    Update update = new Update()
        .set("deviceName", deviceName)
        .set("lastLoginTime", LocalDateTime.now())
        .setOnInsert("createdAt", LocalDateTime.now()); // 仅在插入时设置

    UpdateResult result = mongoTemplate.upsert(query, update, "user_devices");
    if (result.getUpsertedId() != null) {
        System.out.println("新设备已记录: " + result.getUpsertedId());
    } else {
        System.out.println("设备信息已更新");
    }
}

setOnInsert 是一个非常有用的方法：只在 upsert 导致插入新文档时才设置该字段，更新已有文档时不会触发。

5.4.5 批量操作 BulkOperations

当需要执行大量更新操作时，使用 BulkOperations 比循环调用 updateFirst 效率高很多：

import org.springframework.data.mongodb.core.BulkOperations;
import org.springframework.data.util.Pair;
import com.mongodb.bulk.BulkWriteResult;

public void bulkUpdateUserStatus(Map<String, String> userStatusMap) {
    BulkOperations bulkOps = mongoTemplate.bulkOps(
        BulkOperations.BulkMode.UNORDERED, User.class
    );

    for (Map.Entry<String, String> entry : userStatusMap.entrySet()) {
        Query query = new Query(Criteria.where("_id").is(entry.getKey()));
        Update update = new Update().set("status", entry.getValue());
        bulkOps.updateOne(query, update);
    }

    BulkWriteResult result = bulkOps.execute();
    System.out.println("匹配: " + result.getMatchedCount());
    System.out.println("修改: " + result.getModifiedCount());
}

// 使用 Pair 列表的方式
public void bulkUpdateWithPairs(List<Pair<Query, Update>> updates) {
    BulkOperations bulkOps = mongoTemplate.bulkOps(
        BulkOperations.BulkMode.ORDERED, User.class
    );
    bulkOps.updateMulti(updates);
    bulkOps.execute();
}

BulkMode.ORDERED：顺序执行，一个失败后续停止。 BulkMode.UNORDERED：并行执行，一个失败不影响其他操作。

5.5.4 封装完整的分页查询（从 Controller 到 Service）

下面实现一个完整的分页查询功能，包含分页信息的返回。

1) 通用分页响应类：

package com.example.mongo.common;

import java.util.List;

public class PageResult<T> {

    private List<T> content;       // 当前页数据
    private long totalElements;    // 总记录数
    private int totalPages;        // 总页数
    private int currentPage;       // 当前页码（从0开始）
    private int pageSize;          // 每页大小
    private boolean hasNext;       // 是否有下一页
    private boolean hasPrevious;   // 是否有上一页

    public PageResult(List<T> content, long totalElements, int currentPage, int pageSize) {
        this.content = content;
        this.totalElements = totalElements;
        this.currentPage = currentPage;
        this.pageSize = pageSize;
        this.totalPages = (int) Math.ceil((double) totalElements / pageSize);
        this.hasNext = currentPage < totalPages - 1;
        this.hasPrevious = currentPage > 0;
    }

    // getter 方法
    public List<T> getContent() { return content; }
    public long getTotalElements() { return totalElements; }
    public int getTotalPages() { return totalPages; }
    public int getCurrentPage() { return currentPage; }
    public int getPageSize() { return pageSize; }
    public boolean isHasNext() { return hasNext; }
    public boolean isHasPrevious() { return hasPrevious; }
}

2) Service 层 —— 分页查询方法：

package com.example.mongo.service;

import com.example.mongo.common.PageResult;
import com.example.mongo.entity.User;
import org.springframework.data.domain.Sort;
import org.springframework.data.mongodb.core.MongoTemplate;
import org.springframework.data.mongodb.core.query.Criteria;
import org.springframework.data.mongodb.core.query.Query;
import org.springframework.stereotype.Service;
import java.util.List;

@Service
public class UserService {

    private final MongoTemplate mongoTemplate;

    public UserService(MongoTemplate mongoTemplate) {
        this.mongoTemplate = mongoTemplate;
    }

    /**
     * 分页查询用户列表（支持多条件筛选）
     *
     * @param name   用户名（模糊匹配，可为 null）
     * @param status 状态（精确匹配，可为 null）
     * @param minAge 最小年龄（可为 null）
     * @param maxAge 最大年龄（可为 null）
     * @param page   页码（从 0 开始）
     * @param size   每页大小
     * @return 分页结果
     */
    public PageResult<User> findUsersPage(String name, String status,
                                          Integer minAge, Integer maxAge,
                                          int page, int size) {
        // 构建查询条件
        Query query = new Query();

        if (name != null && !name.isBlank()) {
            query.addCriteria(Criteria.where("name").regex(name, "i"));
        }
        if (status != null && !status.isBlank()) {
            query.addCriteria(Criteria.where("status").is(status));
        }
        if (minAge != null && maxAge != null) {
            query.addCriteria(Criteria.where("age").gte(minAge).lte(maxAge));
        } else if (minAge != null) {
            query.addCriteria(Criteria.where("age").gte(minAge));
        } else if (maxAge != null) {
            query.addCriteria(Criteria.where("age").lte(maxAge));
        }

        // 先查询总数（使用相同的条件）
        long total = mongoTemplate.count(query, User.class);

        // 添加排序和分页
        query.with(Sort.by(Sort.Direction.DESC, "createdAt"));
        query.skip((long) page * size);
        query.limit(size);

        // 查询当前页数据
        List<User> users = mongoTemplate.find(query, User.class);

        return new PageResult<>(users, total, page, size);
    }
}

5.5.5 分页性能提示

在数据量较大时（数十万以上），skip + limit 的分页方式性能会下降，因为 MongoDB 需要遍历跳过的文档。优化方案：

  1. 确保排序字段有索引：分页查询的排序字段必须建立索引。
  2. 基于游标的分页（适合"加载更多"场景）：
// 使用上一页最后一条记录的 createdAt 作为游标，避免 skip
public List<User> findUsersAfter(LocalDateTime lastCreatedAt, int size) {
    Query query = new Query();
    if (lastCreatedAt != null) {
        query.addCriteria(Criteria.where("createdAt").lt(lastCreatedAt));
    }
    query.with(Sort.by(Sort.Direction.DESC, "createdAt"));
    query.limit(size);
    return mongoTemplate.find(query, User.class);
}
  1. 限制最大页码：对深度分页做限制，如最多允许翻到第 100 页。

本章小结

知识点 核心内容
MongoTemplate vs Repository Repository 适合简单 CRUD，MongoTemplate 适合复杂查询，可混合使用
CRUD 操作 insert / find / save / updateFirst / updateMulti / remove
Criteria 查询 丰富的操作符（is/gt/lt/in/regex/exists/elemMatch...），支持 AND/OR 组合
Update 更新 set / unset / inc / push / pull / addToSet 等精细操作
findAndModify 原子操作，适合计数器、抢购等场景
Upsert 存在则更新，不存在则插入
BulkOperations 高效批量操作
分页排序 skip/limit 分页 + Sort 排序，封装 PageResult 返回完整分页信息
