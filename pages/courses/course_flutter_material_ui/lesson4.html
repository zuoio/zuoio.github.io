第04课 形状系统与 Elevation

课程系列：Flutter Material Design 系统学习
前置条件：已掌握 Container、BoxDecoration 等容器组件的基本用法
课程目标：深入理解 Material 3 形状 Token 体系与 Elevation 层级模型，掌握 ShapeBorder 家族的使用，能够通过全局主题统一管理组件形状与阴影

形状（Shape）和层级（Elevation）是 Material Design 视觉系统中的两大基石。形状赋予组件独特的视觉识别性，Elevation 定义了界面元素之间的空间层次关系。Material 3 对这两者都做了重要更新——形状有了标准化的 Token 层级体系，Elevation 则从纯阴影表达转变为色调叠加（Surface Tint）模型。

本课将系统讲解这两大设计基础，并通过大量代码示例帮助你掌握实际应用。

1. Material 3 Shape 规范：7 个形状 Token 层级

Material 3 将形状标准化为 7 个 Token 层级，每个层级对应一个固定的圆角半径值。这种标准化使得整个应用的视觉语言保持一致。

1.1 形状 Token 一览表

Token 名称 圆角值 视觉效果 典型用途
None 0dp 直角矩形 全宽横幅、背景面板
Extra Small 4dp 极微圆角 TextField、Badge
Small 8dp 小圆角 Chip、Snackbar
Medium 12dp 中等圆角 Card、Dialog
Large 16dp 大圆角 FAB、NavigationDrawer
Extra Large 28dp 超大圆角 BottomSheet、大面积面板
Full 短边 / 2 完全圆弧 小型 FAB、搜索栏、Slider 轨道

1.2 形状 Token 在代码中的表达

Flutter 中并没有内置的 ShapeToken 枚举，但我们可以自己定义一套标准常量来统一管理：

import 'package:flutter/material.dart';

/// Material 3 形状 Token 常量定义
/// 将设计规范映射为代码中可复用的圆角值
class M3ShapeTokens {
  M3ShapeTokens._();

  /// None: 0dp - 直角矩形
  static const double none = 0;

  /// Extra Small: 4dp
  static const double extraSmall = 4;

  /// Small: 8dp
  static const double small = 8;

  /// Medium: 12dp
  static const double medium = 12;

  /// Large: 16dp
  static const double large = 16;

  /// Extra Large: 28dp
  static const double extraLarge = 28;

  /// 将 Token 转换为 RoundedRectangleBorder
  static RoundedRectangleBorder border(double radius) {
    return RoundedRectangleBorder(
      borderRadius: BorderRadius.circular(radius),
    );
  }

  /// Full: 完全圆弧（StadiumBorder）
  static const ShapeBorder full = StadiumBorder();
}

💡 最佳实践：在项目中定义一套形状常量类（如上面的 M3ShapeTokens），避免在各处硬编码圆角数值。这不仅让代码更语义化，还便于后期批量调整。

2. ShapeBorder 家族详解

Flutter 提供了 ShapeBorder 抽象类及其多个子类，用于描述组件的边框形状。不同的 ShapeBorder 子类适用于不同的视觉需求。

2.1 ShapeBorder 家族继承关系

ShapeBorder
├── OutlinedBorder
│   ├── RoundedRectangleBorder    // 圆角矩形
│   ├── StadiumBorder             // 体育场形（两端半圆）
│   ├── BeveledRectangleBorder    // 斜切角矩形
│   ├── ContinuousRectangleBorder // 超椭圆圆角（squircle）
│   ├── CircleBorder              // 圆形
│   └── StarBorder                // 星形（Flutter 3.10+）
├── RoundedRectangleBorder        // （也可直接用，不带 side）
├── InputBorder
│   ├── OutlineInputBorder        // 输入框轮廓边框
│   └── UnderlineInputBorder      // 输入框下划线边框
└── ...

这是使用频率最高的形状边框，用于创建各种圆角的矩形。

2.5 ContinuousRectangleBorder：超椭圆圆角（Squircle）

ContinuousRectangleBorder 使用超椭圆曲线（superellipse）连接圆角与直边，让过渡更加平滑自然。这种形状在 iOS 设计中被广泛使用（也称为 squircle）。

import 'package:flutter/material.dart';

void main() => runApp(const MaterialApp(home: ContinuousRectDemo()));

class ContinuousRectDemo extends StatelessWidget {
  const ContinuousRectDemo({super.key});

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(title: const Text('ContinuousRectangleBorder')),
      body: Padding(
        padding: const EdgeInsets.all(16),
        child: Column(
          children: [
            // 对比：RoundedRectangleBorder vs ContinuousRectangleBorder
            Row(
              mainAxisAlignment: MainAxisAlignment.spaceEvenly,
              children: [
                Column(
                  children: [
                    Container(
                      width: 120,
                      height: 120,
                      decoration: ShapeDecoration(
                        color: Theme.of(context).colorScheme.primaryContainer,
                        shape: RoundedRectangleBorder(
                          borderRadius: BorderRadius.circular(32),
                        ),
                      ),
                      child: const Center(child: Text('Rounded')),
                    ),
                    const SizedBox(height: 8),
                    const Text('普通圆角'),
                  ],
                ),
                Column(
                  children: [
                    Container(
                      width: 120,
                      height: 120,
                      decoration: ShapeDecoration(
                        color: Theme.of(context).colorScheme.tertiaryContainer,
                        shape: ContinuousRectangleBorder(
                          borderRadius: BorderRadius.circular(64),
                        ),
                      ),
                      child: const Center(child: Text('Continuous')),
                    ),
                    const SizedBox(height: 8),
                    const Text('超椭圆圆角'),
                  ],
                ),
              ],
            ),
            const SizedBox(height: 24),
            Text(
              '注意观察两者的区别：\n'
              'Continuous 的圆角与直边之间过渡更平滑，\n'
              '没有 Rounded 那种突然从直线变为弧线的"断裂感"。',
              style: Theme.of(context).textTheme.bodyMedium,
            ),
          ],
        ),
      ),
    );
  }
}

⚠️ 注意事项：ContinuousRectangleBorder 的 borderRadius 需要设置为普通圆角的约 2 倍 才能达到类似的视觉圆角大小，因为超椭圆曲线的形状特性使其实际圆角看起来比等值的 RoundedRectangleBorder 更小。

2.7 StarBorder：星形边框（Flutter 3.10+）

StarBorder 是 Flutter 3.10 引入的新形状，可以创建星形、多边形等丰富的几何形状。

import 'package:flutter/material.dart';

void main() => runApp(const MaterialApp(home: StarBorderDemo()));

class StarBorderDemo extends StatelessWidget {
  const StarBorderDemo({super.key});

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(title: const Text('StarBorder')),
      body: Padding(
        padding: const EdgeInsets.all(16),
        child: Wrap(
          spacing: 16,
          runSpacing: 16,
          alignment: WrapAlignment.center,
          children: [
            // 默认五角星
            _buildStarCard(
              context,
              label: '五角星',
              shape: const StarBorder(),
            ),

            // 自定义角数 — 六角星
            _buildStarCard(
              context,
              label: '六角星',
              shape: const StarBorder(points: 6),
            ),

            // 调整内半径比 — 更尖锐的星形
            _buildStarCard(
              context,
              label: '尖锐星形',
              shape: const StarBorder(
                points: 5,
                innerRadiusRatio: 0.3, // 值越小，星角越尖锐
              ),
            ),

            // 圆角星形
            _buildStarCard(
              context,
              label: '圆角星形',
              shape: const StarBorder(
                points: 5,
                pointRounding: 0.4, // 星角圆润度
                innerRadiusRatio: 0.5,
              ),
            ),

            // 正多边形（将 innerRadiusRatio 设为 1 时）
            _buildStarCard(
              context,
              label: '正六边形',
              shape: const StarBorder.polygon(sides: 6),
            ),

            // 正八边形
            _buildStarCard(
              context,
              label: '正八边形',
              shape: const StarBorder.polygon(sides: 8),
            ),
          ],
        ),
      ),
    );
  }

  Widget _buildStarCard(
    BuildContext context, {
    required String label,
    required ShapeBorder shape,
  }) {
    return Column(
      mainAxisSize: MainAxisSize.min,
      children: [
        Container(
          width: 100,
          height: 100,
          decoration: ShapeDecoration(
            color: Theme.of(context).colorScheme.primaryContainer,
            shape: shape,
          ),
          child: Center(
            child: Icon(
              Icons.star,
              color: Theme.of(context).colorScheme.onPrimaryContainer,
            ),
          ),
        ),
        const SizedBox(height: 4),
        Text(label, style: Theme.of(context).textTheme.labelSmall),
      ],
    );
  }
}

💡 最佳实践：StarBorder.polygon() 构造函数可以创建任意正多边形，非常适合用于游戏、社交应用中的徽章、成就标识等。

2.8 ShapeBorder 家族对照总结

ShapeBorder 类型 视觉效果 常见用途 关键参数
RoundedRectangleBorder 圆角矩形 Card、Button、Dialog borderRadius
StadiumBorder 胶囊 / 体育场 搜索栏、标签、Chip 无（自动适应）
BeveledRectangleBorder 斜切角 特殊装饰元素 borderRadius（控制切角大小）
ContinuousRectangleBorder 超椭圆 (squircle) iOS 风格图标、卡片 borderRadius（需约 2x）
CircleBorder 正圆 头像、FAB、圆形按钮 无（自动适应）
StarBorder 星形 / 多边形 徽章、评分、装饰 points、innerRadiusRatio

3. 组件默认形状规范对照表

Material 3 为每种组件指定了默认的形状 Token，确保整个应用的视觉一致性。下表列出了常见组件与其默认形状的对应关系：

形状 Token 圆角值 对应组件
None (0dp) 0dp NavigationBar、FullScreenDialog
Extra Small (4dp) 4dp TextField（OutlineInputBorder）、Badge
Small (8dp) 8dp Chip、Snackbar、FilterChip、ActionChip
Medium (12dp) 12dp Card、AlertDialog、DatePicker、TimePicker
Large (16dp) 16dp FAB、ExtendedFAB、NavigationDrawer
Extra Large (28dp) 28dp BottomSheet、LargeTopAppBar
Full 短边/2 SmallFAB、Slider Thumb、SearchBar、标签 Badge

4. 全局形状定制

在实际项目中，你通常需要统一管理所有组件的形状，而不是在每个组件上单独设置。Flutter 提供了通过 ThemeData 的各组件主题属性来全局定制形状的能力。

4.1 通过组件主题统一形状

import 'package:flutter/material.dart';

void main() => runApp(const ShapeThemeApp());

class ShapeThemeApp extends StatelessWidget {
  const ShapeThemeApp({super.key});

  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      theme: ThemeData(

4.2 形状优先级规则

组件的最终形状由以下优先级链决定（从高到低）：

  1. Widget 参数：直接在组件上设置的 shape 属性
  2. 组件主题：ThemeData 中对应的组件主题（如 CardTheme.shape）
  3. Material 默认值：Flutter 框架内置的 Material 3 默认形状
// 优先级 1: Widget 参数 — 最高优先级
Card(
  shape: RoundedRectangleBorder(borderRadius: BorderRadius.circular(24)),
  child: ...,
)

// 优先级 2: 组件主题 — 次优先级
ThemeData(
  cardTheme: CardTheme(
    shape: RoundedRectangleBorder(borderRadius: BorderRadius.circular(16)),
  ),
)

// 优先级 3: Material 默认值 — 最低优先级
// Card 默认使用 Medium (12dp) 圆角

💡 最佳实践：

  • 在 ThemeData 中统一配置全局默认形状（优先级 2）
  • 只在特殊需求时通过组件参数覆盖（优先级 1）
  • 避免在每个组件上逐个设置形状，保持一致性

5. Material 3 中的 Elevation 新模型

Elevation（层级）在 Material Design 中表达界面元素之间的空间关系和视觉重要性。Material 3 对 Elevation 的表达方式做了重大改变。

5.1 M2 vs M3 的 Elevation 表达差异

对比维度 Material 2 Material 3
视觉表达 纯阴影（drop shadow） Surface Tint（色调叠加）+ 弱化阴影
颜色变化 层级变化不影响表面颜色 层级越高，表面色调越深
暗色模式 阴影几乎不可见，靠 elevation overlay Surface Tint 在亮/暗模式下都有效
性能 阴影渲染开销较大 色调混合性能更优
设计意图 模拟物理世界的光影 更抽象的层级语义表达

5.2 Surface Tint 的工作机制

在 Material 3 中，surfaceTintColor（通常取自 colorScheme.surfaceTint，即 primary 颜色）会根据 Elevation 值叠加到组件的背景色上。Elevation 越高，叠加的色调越浓。

最终颜色 = surface 底色 + surfaceTint × 不透明度(由 elevation 决定)

各层级对应的 Surface Tint 不透明度近似值：

Elevation 层级 dp 值 Tint 不透明度（近似）
Level 0 0dp 0%（无色调叠加）
Level 1 1dp 5%
Level 2 3dp 8%
Level 3 6dp 11%
Level 4 8dp 12%
Level 5 12dp 14%

6. Elevation 的 6 个层级

Material 3 定义了 6 个标准 Elevation 层级（Level 0 到 Level 5），每个层级有明确的 dp 值和使用场景。

6.1 Elevation 层级详细规范

层级 dp 值 使用场景 对应组件
Level 0 0dp 页面主体、背景表面 Surface、Scaffold body
Level 1 1dp 可滚动内容、搜索栏 Card（默认）、NavigationBar、SearchBar
Level 2 3dp 需要轻微突出的内容 ElevatedButton、NavigationDrawer（非模态）
Level 3 6dp 模态内容、浮动元素 FAB、Snackbar、NavigationDrawer（模态）
Level 4 8dp （较少使用） 特殊需求
Level 5 12dp （较少使用） 特殊需求

7. Material 组件详解

Material 是 Flutter 中实现 Material Design 视觉效果的核心 Widget。它负责渲染背景色、阴影、形状和水波纹效果。

7.1 Material Widget 的关键属性

属性 类型 说明
type MaterialType 材质类型（canvas/card/circle/transparency 等）
elevation double 层级高度，影响阴影和色调叠加
color Color? 背景颜色
shadowColor Color? 阴影颜色（M3 默认透明）
surfaceTintColor Color? 色调叠加颜色（M3 核心特性）
shape ShapeBorder? 形状边框
clipBehavior Clip 裁剪行为
borderRadius BorderRadiusGeometry? 圆角（仅适用于 MaterialType.canvas 和 card）
borderOnForeground bool 边框是否绘制在子组件前面

9. 本课小结

核心概念回顾

  1. M3 Shape Token 体系：Material 3 定义了 7 个标准形状层级（None → Extra Small → Small → Medium → Large → Extra Large → Full），对应从 0dp 到全圆的圆角值。通过标准化形状 Token 统一应用的视觉语言。

  2. ShapeBorder 家族：Flutter 提供了 6 种主要的 ShapeBorder 子类：

    • RoundedRectangleBorder：最常用的圆角矩形
    • StadiumBorder：胶囊形，自动适应短边半径
    • BeveledRectangleBorder：斜切角，几何风格
    • ContinuousRectangleBorder：超椭圆，iOS 风格的平滑圆角
    • CircleBorder：正圆
    • StarBorder：星形和多边形（Flutter 3.10+）

  3. 全局形状管理：通过 ThemeData 中各组件主题的 shape 属性统一管理形状，遵循优先级链：Widget 参数 > 组件主题 > Material 默认值。

  4. Elevation 新模型：M3 用 Surface Tint（色调叠加）替代 M2 的纯阴影来表达层级，6 个层级分别对应 0dp、1dp、3dp、6dp、8dp、12dp。

  5. Material Widget：通过 type、elevation、shadowColor、surfaceTintColor 等属性精确控制材质的视觉表现。

关键对照表

组件 默认形状 Token 默认 Elevation
Scaffold body None Level 0 (0dp)
Card Medium (12dp) Level 1 (1dp)
Chip Small (8dp) Level 0 (0dp)
TextField Extra Small (4dp) Level 0 (0dp)
FAB Large (16dp) Level 3 (6dp)
ElevatedButton Full Level 1 (1dp)
NavigationBar None Level 2 (3dp)
BottomSheet Extra Large (28dp) Level 1 (1dp)
Dialog Extra Large (28dp) Level 3 (6dp)

10. 练习题

练习 1：形状探索（基础）

创建一个页面，用 6 种 ShapeBorder 各包裹一个包含文字和图标的 Container。要求：

  • 每种形状使用不同的背景色
  • 每个容器至少 80×80 像素
  • 使用 Wrap 布局自动换行

练习 2：Elevation 阶梯（基础）

创建一个纵向排列的 6 张卡片，分别设置 Level 0 到 Level 5 的 Elevation 值。要求：

  • 使用 Material Widget（不要用 Card）
  • 同时展示 Surface Tint 效果
  • 在每张卡片上显示当前层级编号和 dp 值
  • 尝试切换到暗色主题观察 Surface Tint 的效果

练习 3：自定义品牌形状主题（中级）

为一个虚构的品牌 App 配置全局形状主题。要求：

  • 所有按钮使用 StadiumBorder（胶囊形）
  • Card 使用 20dp 圆角
  • Chip 使用 ContinuousRectangleBorder（超椭圆）
  • Dialog 使用 24dp 圆角
  • FAB 使用 CircleBorder
  • 在一个展示页面中验证所有组件的形状效果

练习 4：Shape & Elevation 动态调节器（进阶）

构建一个交互式页面，包含：

  • 一个中央展示区域的 Material Widget
  • 一个 Slider 控制圆角半径（0~50dp）
  • 一个 Slider 控制 Elevation（0~12dp）
  • 一个 SegmentedButton 切换形状类型（Rounded / Stadium / Beveled / Continuous）
  • 一个 Switch 切换 Surface Tint 开关
  • 实时预览形状和 Elevation 的变化效果

练习 5：Material 层级系统设计（进阶）

模拟一个内容阅读 App 的页面层级设计：

  • 底层（Level 0）：页面背景
  • 第一层（Level 1）：内容卡片列表
  • 第二层（Level 2）：浮动操作栏
  • 第三层（Level 3）：FAB
  • 点击卡片弹出详情 Dialog（Level 3）
  • 使用 BottomSheet（Level 1）展示评论区
  • 要求所有层级的 Surface Tint 效果清晰可辨
