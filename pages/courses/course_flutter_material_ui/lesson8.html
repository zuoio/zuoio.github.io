1. Material Design 响应式布局指南

1.1 四种窗口尺寸类（Window Size Classes）

Material Design 3 将所有设备屏幕宽度划分为四个尺寸类。这不是随意划分——每个尺寸类都对应着截然不同的用户交互模式：

尺寸类 宽度范围 典型设备 交互特征
Compact < 600dp 手机竖屏 单手操作，屏幕空间有限，以纵向滚动为主
Medium 600–840dp 手机横屏、小平板 双手操作，有一定横向空间，可考虑侧边导航
Expanded 840–1200dp 平板、小桌面窗口 可同时展示多栏内容，鼠标/触控混合
Large > 1200dp 大桌面显示器 大量空间，适合三栏布局、永久侧边栏

注意：这里的 dp 在 Flutter 中对应逻辑像素（logical pixels），即 MediaQuery.sizeOf(context).width 返回的值。

2. 断点判断实现

2.1 WindowSizeClass 工具类

首先，我们需要一个可靠的断点判断工具。以下是按照 Material 3 规范实现的 WindowSizeClass：

import 'package:flutter/widgets.dart';

/// Material 3 窗口尺寸类枚举
enum WindowSizeClass {
  /// 紧凑型：< 600dp，手机竖屏
  compact,

  /// 中等型：600-840dp，手机横屏或小平板
  medium,

  /// 扩展型：840-1200dp，平板或小桌面
  expanded,

  /// 大型：> 1200dp，大桌面显示器
  large,
}

/// 窗口尺寸类判断工具
class WindowSizeClassHelper {
  static const double compactMaxWidth = 600;
  static const double mediumMaxWidth = 840;
  static const double expandedMaxWidth = 1200;

  /// 根据给定宽度返回尺寸类
  static WindowSizeClass fromWidth(double width) {
    if (width < compactMaxWidth) return WindowSizeClass.compact;
    if (width < mediumMaxWidth) return WindowSizeClass.medium;
    if (width < expandedMaxWidth) return WindowSizeClass.expanded;
    return WindowSizeClass.large;
  }

  /// 从 BuildContext 获取当前窗口尺寸类
  static WindowSizeClass of(BuildContext context) {
    final width = MediaQuery.sizeOf(context).width;
    return fromWidth(width);
  }

  /// 判断当前窗口是否至少为指定尺寸类
  static bool isAtLeast(BuildContext context, WindowSizeClass sizeClass) {
    final current = of(context);
    return current.index >= sizeClass.index;
  }
}

2.2 在 Widget 中使用断点

import 'package:flutter/material.dart';

class BreakpointDemoPage extends StatelessWidget {
  const BreakpointDemoPage({super.key});

  @override
  Widget build(BuildContext context) {
    // 推荐使用 MediaQuery.sizeOf 而非 MediaQuery.of，性能更好
    final windowClass = WindowSizeClassHelper.of(context);
    final width = MediaQuery.sizeOf(context).width;

    return Scaffold(
      appBar: AppBar(title: const Text('断点演示')),
      body: Center(
        child: Column(
          mainAxisAlignment: MainAxisAlignment.center,
          children: [
            Icon(
              _iconForClass(windowClass),
              size: 64,
              color: Theme.of(context).colorScheme.primary,
            ),
            const SizedBox(height: 16),
            Text(
              '当前窗口宽度：${width.toStringAsFixed(0)}dp',
              style: Theme.of(context).textTheme.headlineSmall,
            ),
            const SizedBox(height: 8),
            Chip(
              avatar: const Icon(Icons.aspect_ratio),
              label: Text('尺寸类：${windowClass.name}'),
            ),
            const SizedBox(height: 24),
            Text(
              _descriptionForClass(windowClass),
              style: Theme.of(context).textTheme.bodyLarge,
              textAlign: TextAlign.center,
            ),
          ],
        ),
      ),
    );
  }

  IconData _iconForClass(WindowSizeClass sizeClass) {
    return switch (sizeClass) {
      WindowSizeClass.compact => Icons.phone_android,
      WindowSizeClass.medium => Icons.tablet_android,
      WindowSizeClass.expanded => Icons.tablet_mac,
      WindowSizeClass.large => Icons.desktop_windows,
    };
  }

  String _descriptionForClass(WindowSizeClass sizeClass) {
    return switch (sizeClass) {
      WindowSizeClass.compact => '手机竖屏模式\n使用底部导航 + 单列布局',
      WindowSizeClass.medium => '中等屏幕模式\n使用侧边导航栏 + 可选双栏布局',
      WindowSizeClass.expanded => '扩展屏幕模式\n使用导航抽屉 + 双栏/三栏布局',
      WindowSizeClass.large => '大桌面模式\n使用永久导航抽屉 + 三栏布局',
    };
  }
}

最佳实践：使用 MediaQuery.sizeOf(context) 而非 MediaQuery.of(context).size。前者只在窗口尺寸变化时触发重建，后者会在任何 MediaQueryData 属性变化（如键盘弹出、字体缩放等）时都触发重建，造成不必要的性能开销。

3. 导航模式随窗口自适应

这是响应式布局中最核心的部分——根据窗口大小自动切换导航模式。

3.1 三种导航模式

尺寸类 导航组件 特点
Compact NavigationBar 底部导航栏，最多 5 个目的地
Medium NavigationRail 侧边窄轨导航，可配合 leading FAB
Expanded / Large NavigationDrawer 永久展示的侧边栏导航，可显示分组和标签

3.2 完整的自适应导航 Shell 实现

以下是一个完整的、可直接运行的自适应导航框架：

import 'package:flutter/material.dart';

void main() {
  runApp(const AdaptiveNavApp());
}

class AdaptiveNavApp extends StatelessWidget {
  const AdaptiveNavApp({super.key});

  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      title: '自适应导航演示',
      theme: ThemeData(
        colorSchemeSeed: Colors.indigo,
        useMaterial3: true,
      ),
      home: const AdaptiveNavShell(),
    );
  }
}

/// 导航目的地定义
class NavDestination {
  final String label;
  final IconData icon;
  final IconData selectedIcon;
  final Widget body;

  const NavDestination({
    required this.label,
    required this.icon,
    required this.selectedIcon,
    required this.body,
  });
}

/// 自适应导航外壳
class AdaptiveNavShell extends StatefulWidget {
  const AdaptiveNavShell({super.key});

  @override
  State<AdaptiveNavShell> createState() => _AdaptiveNavShellState();
}

class _AdaptiveNavShellState extends State<AdaptiveNavShell> {
  int _selectedIndex = 0;

  /// 定义所有导航目的地
  static final List<NavDestination> _destinations = [
    NavDestination(
      label: '首页',
      icon: Icons.home_outlined,
      selectedIcon: Icons.home,
      body: const _PlaceholderPage(title: '首页', icon: Icons.home),
    ),
    NavDestination(
      label: '发现',
      icon: Icons.explore_outlined,
      selectedIcon: Icons.explore,
      body: const _PlaceholderPage(title: '发现', icon: Icons.explore),
    ),
    NavDestination(
      label: '收藏',
      icon: Icons.bookmark_border,
      selectedIcon: Icons.bookmark,
      body: const _PlaceholderPage(title: '收藏', icon: Icons.bookmark),
    ),
    NavDestination(
      label: '设置',
      icon: Icons.settings_outlined,
      selectedIcon: Icons.settings,
      body: const _PlaceholderPage(title: '设置', icon: Icons.settings),
    ),
  ];

  @override
  Widget build(BuildContext context) {
    final windowClass = WindowSizeClassHelper.of(context);

    return switch (windowClass) {
      WindowSizeClass.compact => _buildCompactLayout(),
      WindowSizeClass.medium => _buildMediumLayout(),
      WindowSizeClass.expanded || WindowSizeClass.large =>
        _buildExpandedLayout(),
    };
  }

  /// Compact：底部导航栏
  Widget _buildCompactLayout() {
    return Scaffold(
      appBar: AppBar(title: const Text('自适应导航')),
      body: _destinations[_selectedIndex].body,
      bottomNavigationBar: NavigationBar(
        selectedIndex: _selectedIndex,
        onDestinationSelected: (index) {
          setState(() => _selectedIndex = index);
        },
        destinations: _destinations
            .map((d) => NavigationDestination(
                  icon: Icon(d.icon),
                  selectedIcon: Icon(d.selectedIcon),
                  label: d.label,
                ))
            .toList(),
      ),
    );
  }

  /// Medium：NavigationRail + 内容区
  Widget _buildMediumLayout() {
    return Scaffold(
      appBar: AppBar(title: const Text('自适应导航')),
      body: Row(
        children: [
          NavigationRail(
            selectedIndex: _selectedIndex,
            onDestinationSelected: (index) {
              setState(() => _selectedIndex = index);
            },
            labelType: NavigationRailLabelType.all,
            // 在 Rail 顶部放置一个 FAB
            leading: FloatingActionButton(
              onPressed: () {},
              child: const Icon(Icons.add),
            ),
            destinations: _destinations
                .map((d) => NavigationRailDestination(
                      icon: Icon(d.icon),
                      selectedIcon: Icon(d.selectedIcon),
                      label: Text(d.label),
                    ))
                .toList(),
          ),
          const VerticalDivider(thickness: 1, width: 1),
          // 内容区占满剩余空间
          Expanded(child: _destinations[_selectedIndex].body),
        ],
      ),
    );
  }

  /// Expanded/Large：永久 NavigationDrawer + 内容区
  Widget _buildExpandedLayout() {
    return Scaffold(
      body: Row(
        children: [
          // 永久展示的导航抽屉
          NavigationDrawer(
            selectedIndex: _selectedIndex,
            onDestinationSelected: (index) {
              setState(() => _selectedIndex = index);
            },
            children: [
              const Padding(
                padding: EdgeInsets.fromLTRB(28, 16, 16, 8),
                child: Text(
                  '自适应导航',
                  style: TextStyle(
                    fontSize: 18,
                    fontWeight: FontWeight.bold,
                  ),
                ),
              ),
              const Divider(indent: 28, endIndent: 28),
              ..._destinations.map(
                (d) => NavigationDrawerDestination(
                  icon: Icon(d.icon),
                  selectedIcon: Icon(d.selectedIcon),
                  label: Text(d.label),
                ),
              ),
            ],
          ),
          const VerticalDivider(thickness: 1, width: 1),
          // 内容区
          Expanded(child: _destinations[_selectedIndex].body),
        ],
      ),
    );
  }
}

3.3 使用 LayoutBuilder 进行局部断点判断

当你需要在某个组件内部（而非页面级别）根据可用空间自适应时，使用 LayoutBuilder 比 MediaQuery 更合适：

class AdaptiveCard extends StatelessWidget {
  const AdaptiveCard({super.key});

  @override
  Widget build(BuildContext context) {
    return LayoutBuilder(
      builder: (context, constraints) {
        // 根据父容器给的最大宽度来决定布局
        if (constraints.maxWidth < 400) {
          // 窄空间：纵向排列
          return const Column(
            children: [
              _CardImage(),
              _CardContent(),
            ],
          );
        } else {
          // 宽空间：横向排列
          return const Row(
            children: [
              Expanded(child: _CardImage()),
              Expanded(child: _CardContent()),
            ],
          );
        }
      },
    );
  }
}

class _CardImage extends StatelessWidget {
  const _CardImage();

  @override
  Widget build(BuildContext context) {
    return Container(
      height: 200,
      color: Theme.of(context).colorScheme.primaryContainer,
      child: const Center(child: Icon(Icons.image, size: 48)),
    );
  }
}

class _CardContent extends StatelessWidget {
  const _CardContent();

  @override
  Widget build(BuildContext context) {
    return Padding(
      padding: const EdgeInsets.all(16),
      child: Column(
        crossAxisAlignment: CrossAxisAlignment.start,
        children: [
          Text('卡片标题', style: Theme.of(context).textTheme.titleLarge),
          const SizedBox(height: 8),
          Text('这是卡片的描述内容，会根据可用空间调整布局方向。',
              style: Theme.of(context).textTheme.bodyMedium),
        ],
      ),
    );
  }
}

最佳实践：MediaQuery 基于整个窗口尺寸，适合页面级布局决策（如导航模式切换）；LayoutBuilder 基于父容器约束，适合组件级自适应（如卡片内部布局）。两者结合使用，才能实现真正灵活的响应式设计。

4. Canonical Layouts（规范化布局模式）

Material Design 3 定义了三种规范化布局，分别适用于不同的内容展示场景。

4.1 列表-详情（List-Detail）

这是最常见的自适应模式：在手机上，列表和详情分别占一整屏；在平板/桌面上，左侧列表右侧详情同时展示。

import 'package:flutter/material.dart';

void main() {
  runApp(const ListDetailApp());
}

class ListDetailApp extends StatelessWidget {
  const ListDetailApp({super.key});

  @override
  Widget build(BuildContext context) {
    return MaterialApp(

      theme: ThemeData(
        colorSchemeSeed: Colors.teal,
        useMaterial3: true,

      home: const ListDetailPage(),

  State<ListDetailPage> createState() => _ListDetailPageState();

class _ListDetailPageState extends State<ListDetailPage> {
  Article? _selectedArticle;

  @override
  Widget build(BuildContext context) {
    final isExpanded =
        WindowSizeClassHelper.isAtLeast(context, WindowSizeClass.medium);

    return Scaffold(
      appBar: AppBar(title: const Text('列表-详情布局')),
      body: isExpanded
          ? _buildTwoPaneLayout()   // 大屏：双栏并排
          : _buildSinglePaneLayout(), // 小屏：列表单栏
    );
  }

  /// 大屏双栏布局
  Widget _buildTwoPaneLayout() {
    return Row(
      children: [
        // 左侧列表，固定宽度或按比例
        SizedBox(
          width: 360,
          child: _ArticleListView(
            articles: _articles,
            selectedArticle: _selectedArticle,
            onArticleSelected: (article) {
              setState(() => _selectedArticle = article);
            },
          ),
        ),
        const VerticalDivider(width: 1),
        // 右侧详情
        Expanded(
          child: _selectedArticle != null
              ? _ArticleDetailView(article: _selectedArticle!)
              : const Center(
                  child: Text('请从左侧列表选择一篇文章'),
                ),
        ),
      ],
    );
  }

  /// 小屏单栏布局
  Widget _buildSinglePaneLayout() {
    return _ArticleListView(
      articles: _articles,
      selectedArticle: _selectedArticle,
      onArticleSelected: (article) {
        // 小屏上通过导航跳转到详情页
        Navigator.push(
          context,
          MaterialPageRoute(
            builder: (_) => Scaffold(
              appBar: AppBar(title: Text(article.title)),
              body: _ArticleDetailView(article: article),
            ),
          ),
        );
      },
    );
  }
}

class _ArticleListView extends StatelessWidget {

  final Article? selectedArticle;

    required this.onArticleSelected,

    return ListView.builder(
      itemCount: articles.length,
      itemBuilder: (context, index) {
        final article = articles[index];

        return ListTile(
          title: Text(article.title),

            article.summary,
            maxLines: 2,
            overflow: TextOverflow.ellipsis,
          ),
          selected: isSelected,
          selectedTileColor:
              Theme.of(context).colorScheme.secondaryContainer,
          leading: CircleAvatar(
            child: Text('${index + 1}'),
          ),
          onTap: () => onArticleSelected(article),
        );
      },
    );
  }
}

4.2 Feed（信息流）

信息流布局在窄屏上以单列展示，宽屏上自动排列为多列瀑布式网格：

import 'package:flutter/material.dart';

class AdaptiveFeedPage extends StatelessWidget {
  const AdaptiveFeedPage({super.key});

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(title: const Text('自适应信息流')),
      body: LayoutBuilder(
        builder: (context, constraints) {
          // 根据可用宽度计算列数
          final columnCount = _calculateColumns(constraints.maxWidth);

          return GridView.builder(
            padding: const EdgeInsets.all(16),
            gridDelegate: SliverGridDelegateWithFixedCrossAxisCount(
              crossAxisCount: columnCount,
              mainAxisSpacing: 16,
              crossAxisSpacing: 16,
              // 根据列数调整子项宽高比
              childAspectRatio: columnCount == 1 ? 2.5 : 1.2,
            ),
            itemCount: 20,
            itemBuilder: (context, index) {
              return _FeedCard(index: index);
            },
          );
        },
      ),
    );
  }

  int _calculateColumns(double width) {
    if (width < 600) return 1;       // Compact：单列
    if (width < 840) return 2;       // Medium：双列
    if (width < 1200) return 3;      // Expanded：三列
    return 4;                        // Large：四列
  }
}

5. 自适应组件行为

除了布局和导航，许多 Material 组件自身也需要在不同屏幕尺寸上有不同表现。

5.1 Dialog 在大屏上的宽度限制

默认情况下，showDialog 在大屏上会把对话框拉得非常宽。应当给对话框设置最大宽度：

import 'package:flutter/material.dart';

class AdaptiveDialogDemo extends StatelessWidget {
  const AdaptiveDialogDemo({super.key});

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(title: const Text('自适应对话框')),
      body: Center(
        child: FilledButton(
          onPressed: () => _showAdaptiveDialog(context),
          child: const Text('显示对话框'),
        ),
      ),
    );
  }

  void _showAdaptiveDialog(BuildContext context) {
    final windowClass = WindowSizeClassHelper.of(context);

    if (windowClass == WindowSizeClass.compact) {
      // 手机上使用全屏对话框，信息更完整
      showDialog(
        context: context,
        builder: (_) => Dialog.fullscreen(
          child: Scaffold(
            appBar: AppBar(
              title: const Text('编辑内容'),
              leading: IconButton(
                icon: const Icon(Icons.close),
                onPressed: () => Navigator.pop(context),
              ),
              actions: [
                TextButton(
                  onPressed: () => Navigator.pop(context),
                  child: const Text('保存'),
                ),
              ],
            ),
            body: const Padding(
              padding: EdgeInsets.all(16),
              child: TextField(
                maxLines: null,
                decoration: InputDecoration(
                  hintText: '请输入内容...',
                  border: OutlineInputBorder(),
                ),
              ),
            ),
          ),
        ),
      );
    } else {
      // 大屏使用标准对话框，并限制最大宽度

        context: context,
        builder: (_) => AlertDialog(
          title: const Text('编辑内容'),
          content: ConstrainedBox(
            constraints: const BoxConstraints(
              maxWidth: 560, // Material 规范推荐的对话框最大宽度
              minHeight: 200,

            child: const TextField(
              maxLines: 8,
              decoration: InputDecoration(
                hintText: '请输入内容...',
                border: OutlineInputBorder(),
              ),
            ),
          ),
          actions: [
            TextButton(
              onPressed: () => Navigator.pop(context),

            ),
            FilledButton(

5.3 Drawer 在大屏上转为永久导航

在第 3 节中我们已经展示了这一模式。核心思想是：在 Compact 尺寸使用可滑出的 Drawer，在 Expanded 及以上尺寸使用永久展示的 NavigationDrawer。

import 'package:flutter/material.dart';

class AdaptiveDrawerPage extends StatefulWidget {
  const AdaptiveDrawerPage({super.key});

  @override
  State<AdaptiveDrawerPage> createState() => _AdaptiveDrawerPageState();
}

class _AdaptiveDrawerPageState extends State<AdaptiveDrawerPage> {
  int _selectedIndex = 0;

  @override
  Widget build(BuildContext context) {
    final isExpanded =
        WindowSizeClassHelper.isAtLeast(context, WindowSizeClass.expanded);

    final drawerContent = NavigationDrawer(
      selectedIndex: _selectedIndex,
      onDestinationSelected: (index) {
        setState(() => _selectedIndex = index);
        // 小屏上选择后关闭 Drawer
        if (!isExpanded && Navigator.canPop(context)) {
          Navigator.pop(context);
        }
      },
      children: const [
        Padding(
          padding: EdgeInsets.fromLTRB(28, 16, 16, 8),
          child: Text('导航菜单', style: TextStyle(fontWeight: FontWeight.bold)),
        ),
        NavigationDrawerDestination(
          icon: Icon(Icons.inbox_outlined),
          selectedIcon: Icon(Icons.inbox),
          label: Text('收件箱'),
        ),
        NavigationDrawerDestination(
          icon: Icon(Icons.send_outlined),
          selectedIcon: Icon(Icons.send),
          label: Text('已发送'),
        ),
        NavigationDrawerDestination(
          icon: Icon(Icons.drafts_outlined),
          selectedIcon: Icon(Icons.drafts),
          label: Text('草稿'),
        ),
      ],
    );

    if (isExpanded) {
      // 大屏：永久侧边栏
      return Scaffold(
        body: Row(
          children: [
            drawerContent,
            const VerticalDivider(width: 1),
            Expanded(
              child: _buildBody(),
            ),
          ],
        ),
      );
    } else {
      // 小屏：可滑出的 Drawer
      return Scaffold(
        appBar: AppBar(title: const Text('邮件')),
        drawer: drawerContent,
        body: _buildBody(),
      );
    }
  }

  Widget _buildBody() {
    final pages = ['收件箱', '已发送', '草稿'];
    return Center(
      child: Text(
        pages[_selectedIndex],
        style: Theme.of(context).textTheme.headlineLarge,
      ),
    );
  }
}

注意事项：在大屏上使用永久 NavigationDrawer 时，不要再通过 Scaffold.drawer 属性挂载，而是直接放在 Row 中作为固定子组件。否则用户在桌面上仍然需要通过汉堡菜单按钮打开它，违反了大屏交互的直觉。

6. Platform Adaptive（平台自适应）

除了屏幕尺寸自适应，Material Design 还需要考虑不同操作系统平台的交互习惯差异。

ThemeData.platform 控制 Flutter 组件的平台行为。你可以通过它在任意平台上模拟其他平台的交互风格：

最佳实践：

  • 不要过度使用平台自适应。Material Design 在所有平台上都能提供一致的体验，只在确实需要遵循特定平台惯例时才切换。
  • 推荐的策略是：核心 UI 使用 Material，仅在特定控件（如开关、滑块、进度条）上使用 .adaptive。
  • 使用 Theme.of(context).platform 而非 dart:io 的 Platform，因为前者在 Web 上也能正常工作，且支持通过主题覆盖。

7.2 代码架构解析

上述代码的核心设计思想：

  1. 状态上提：_selectedEmail 和 _navIndex 状态由顶层 ResponsiveHomePage 管理，各布局组件通过回调通信。这确保了在窗口尺寸变化时（如旋转平板），状态不丢失。

  2. 布局组件独立：_CompactLayout、_MediumLayout、_ExpandedLayout 是三个独立的布局组件，各自负责在特定窗口尺寸下的呈现方式。

  3. 内容组件共享：EmailListView、EmailDetailView 等内容组件被所有布局共享。改一处，处处生效。

  4. 渐进式增强：

    • Compact → 单列 + 路由跳转
    • Medium → Rail + 双栏
    • Expanded → Drawer + 双栏 + 工具栏
    • Large → Drawer + 双栏 + 工具栏 + 第三栏信息面板
┌──────────────────────────────────────────────────────────────────────────┐
│                        架构示意                                          │
│                                                                          │
│  ResponsiveHomePage (状态管理: navIndex, selectedEmail)                   │
│       │                                                                  │
│       ├── Compact  → NavigationBar + EmailListView → 路由 → Detail       │
│       ├── Medium   → NavigationRail + EmailListView + Detail             │
│       ├── Expanded → NavigationDrawer + EmailListView + Detail           │
│       └── Large    → NavigationDrawer + EmailListView + Detail + Panel   │
│                                                                          │
│  共享组件: EmailListView, EmailDetailView, _DetailToolbar                │
└──────────────────────────────────────────────────────────────────────────┘

注意事项：

  1. 在实际项目中，建议使用路由框架（如 go_router）来管理导航，它原生支持 ShellRoute 等自适应导航模式。
  2. 窗口尺寸变化时（如用户调整桌面窗口大小、旋转平板），build 方法会自动重新执行，布局自动切换。你无需监听任何事件。
  3. 在 Large 断点的三栏布局中，要注意各栏的最小宽度约束，避免内容被挤压到不可读。

8. 本课小结

核心知识回顾

知识点 要点
窗口尺寸类 Compact（< 600dp）、Medium（600-840dp）、Expanded（840-1200dp）、Large（> 1200dp）
断点判断 MediaQuery.sizeOf(context) 用于页面级，LayoutBuilder 用于组件级
导航自适应 Compact → NavigationBar，Medium → NavigationRail，Expanded → NavigationDrawer
规范化布局 List-Detail（列表详情）、Feed（信息流）、Supporting Panel（辅助面板）
组件自适应 Dialog 全屏/限宽切换、BottomSheet 宽度约束、Drawer 永久化
平台自适应 .adaptive 构造函数、Theme.of(context).platform 判断、按需 Cupertino

最佳实践总结

  1. 优先使用 MediaQuery.sizeOf 而非 MediaQuery.of，减少不必要的重建
  2. 将断点逻辑封装为工具类，避免魔法数字散落在各处
  3. 内容组件与布局组件分离，同一内容在不同布局下复用
  4. 状态提升到布局切换层之上，确保断点切换时状态不丢失
  5. 使用 LayoutBuilder 处理组件级自适应，不要所有断点都用 MediaQuery
  6. 优雅降级：先设计大屏体验，再考虑如何在小屏上简化；或反过来，先确保手机可用，再逐步增强
  7. 不要过度自适应：只在 Material 规范明确建议的地方做断点切换，避免过多的条件分支
