1. Material Design 演进：从 Material 1 到 Material 3

Material Design 是 Google 于 2014 年推出的设计语言，至今经历了三个大版本。理解这条演进脉络，才能真正理解 Material 3 背后的设计决策。

1.1 Material 1（2014）—— 拟物化数字材料

Material Design 1.0 在 Google I/O 2014 上首次发布，它的核心理念是将物理世界的材料属性带入数字界面。

核心特征：

维度 说明
隐喻 将 UI 元素想象为"纸张"和"墨水"，有真实的厚度和阴影
阴影 通过 Elevation（海拔高度）表达层级关系，阴影是核心视觉语言
颜色 预定义 19 种颜色色板（Red、Pink、Purple...），每种色板包含 14 个色阶（50~900）
动效 强调有意义的动画过渡，模拟物理运动规律
排版 使用 Roboto 作为默认字体，定义了有限的文字层级

Material 1 奠定了整个设计语言的基础，但它有一个明显问题：所有 Google 产品看起来都一样。品牌个性很难在这套严格的规范中得到表达。

1.2 Material 2（2018）—— 引入主题化与品牌表达

Material Design 2.0（也叫 Material Theming）于 Google I/O 2018 推出，核心变化是让品牌能在 Material 框架内表达自我。

核心特征：

维度 相比 M1 的变化
主题系统 引入 Color Theme、Typography Theme、Shape Theme，允许全局定制
颜色 从固定色板改为 Primary/Secondary 颜色角色体系
形状 允许组件使用不同的形状（圆角、切角等），不再只有直角/小圆角
排版 扩展为 13 种文字样式（headline16、subtitle12、body1~2、caption、button、overline）
组件 新增 Backdrop、Extended FAB 等组件
暗色主题 正式定义了 Dark Theme 规范

Material 2 是 Flutter 早期版本的默认设计语言。如果你之前学习的组件课程中没有特别设置 useMaterial3: true，那么看到的默认样式就是 Material 2。

1.3 Material 3（2021）—— Material You，个性化设计

Material Design 3（也称 Material You）于 Google I/O 2021 随 Android 12 一同发布，它代表了设计语言的一次根本性转变：从统一走向个性化。

核心特征：

维度 相比 M2 的变化
动态取色 可从用户壁纸自动提取主题色（Dynamic Color），每个用户看到的界面配色都不同
颜色系统 引入 Tonal Palette（色调调色板）和完整的颜色角色体系（Primary、Secondary、Tertiary、Error、Surface 等）
种子颜色 从一个"种子色"（Seed Color）算法生成完整的调色板，保证色彩和谐
形状 标准化为 7 级形状 Token（None → Extra Small → Small → Medium → Large → Extra Large → Full）
排版 简化为 5 大类 × 3 级别 = 15 种文字样式（Display、Headline、Title、Body、Label × Large/Medium/Small）
阴影 弱化真实阴影，引入 Surface Tint（表面色调叠加）表达层级
组件 大量组件视觉更新，新增 NavigationBar、SegmentedButton、SearchBar 等

设计理念的根本变化：

Material 1: "这是规范，所有人都要遵守"       → 统一但缺乏个性
Material 2: "这是规范，但你可以自定义主题"    → 品牌可以表达自我
Material 3: "这是规范，而且会自动适配用户偏好" → 每个用户都独一无二

2. Material 3 三大核心支柱

Material 3 的设计系统建立在三大核心支柱之上：Color（颜色）、Typography（排版） 和 Shape（形状）。它们共同定义了应用的视觉身份（Visual Identity）。

2.1 Color —— 颜色系统

颜色是 Material 3 中变化最大、也最核心的部分。

M3 颜色系统的核心概念：

  • 种子颜色（Seed Color）：只需提供一个品牌色，算法自动生成完整的调色板
  • 色调调色板（Tonal Palette）：从种子色生成 5 组色调调色板（Primary、Secondary、Tertiary、Neutral、Neutral Variant），每组包含 13 个色阶
  • 颜色角色（Color Roles）：从色调调色板中挑选特定色阶，赋予语义化的角色名（如 primary、onPrimary、primaryContainer、onPrimaryContainer）
  • 动态颜色（Dynamic Color）：在 Android 12+ 上可从用户壁纸自动提取种子色

      ),
    );

  final String name;
  final double radius;

3. Design Tokens：从设计规范到代码的桥梁

3.1 什么是 Design Tokens

Design Tokens 是设计系统中最小的样式单元，它们是将设计决策从抽象概念转化为具体代码值的桥梁。

你可以把 Design Tokens 理解为设计系统的"变量"——它不是某种具体颜色或字号，而是带有语义的命名引用：

设计规范层：  "主操作按钮背景色"
    ↓ 映射
Token 层：    md.sys.color.primary
    ↓ 映射
代码层：      ColorScheme.primary → Color(0xFF6750A4)

3.2 Token 的三个层级

Material 3 的 Token 体系分为三个层级：

层级 名称 说明 示例
Reference Token 参考令牌 原始设计值的集合，与用途无关 md.ref.palette.primary40 = #6750A4
System Token 系统令牌 赋予语义角色，引用参考令牌 md.sys.color.primary → md.ref.palette.primary40
Component Token 组件令牌 具体组件的样式，引用系统令牌 md.comp.filled-button.container.color → md.sys.color.primary

注意事项：Design Tokens 是 Material 3 设计规范的核心概念。Flutter 框架已经将这些 Token 内建到了 ThemeData、ColorScheme、TextTheme 和各组件主题中。开发者不需要手动管理 Token 映射——只需正确配置 ThemeData，Flutter 会自动将 Token 应用到每个组件。

6. MaterialApp 配置全景

MaterialApp 是 Flutter 应用的根组件，它提供了丰富的主题配置选项来控制整个应用的视觉表现。

6.1 核心主题属性一览

属性 类型 说明
theme ThemeData 亮色主题（必配）
darkTheme ThemeData? 暗色主题
themeMode ThemeMode 主题模式：system（跟随系统）/ light / dark
highContrastTheme ThemeData? 系统开启高对比度时使用的亮色主题（无障碍）
highContrastDarkTheme ThemeData? 系统开启高对比度时使用的暗色主题（无障碍）
themeAnimationDuration Duration 主题切换动画时长（默认 200ms）
themeAnimationCurve Curve 主题切换动画曲线（默认 Curves.linear）

        colorSchemeSeed: _seedColor,
        brightness: Brightness.light,
      ),

      // ========== 暗色主题 ==========
      darkTheme: ThemeData(
        useMaterial3: true,
        colorSchemeSeed: _seedColor,
        brightness: Brightness.dark,
      ),

      // ========== 主题模式 ==========
      // ThemeMode.system: 跟随系统设置（推荐默认值）
      // ThemeMode.light:  强制亮色
      // ThemeMode.dark:   强制暗色
      themeMode: _themeMode,

      // ========== 高对比度主题（无障碍支持） ==========

      highContrastTheme: ThemeData(
        useMaterial3: true,
        colorScheme: ColorScheme.fromSeed(
          seedColor: _seedColor,
          brightness: Brightness.light,
          // 高对比度场景可使用 contrastLevel 参数
          // 范围 -1.0 到 1.0，1.0 为最高对比度

          primary: Colors.black,
          onPrimary: Colors.white,
          surface: Colors.white,
          onSurface: Colors.black,

      highContrastDarkTheme: ThemeData(
        useMaterial3: true,

                  subtitle: const Text('观察暗色主题下的颜色适配'),
                  trailing: const Icon(Icons.chevron_right),
                ),
                const Divider(height: 1),
                SwitchListTile(
                  title: const Text('暗色模式'),
                  subtitle: Text(isDark ? '已开启' : '已关闭'),
                  value: isDark,
                  onChanged: (value) {
                    onThemeModeChanged(
                        value ? ThemeMode.dark : ThemeMode.light);
                  },
                ),
              ],
            ),
          ),
        ],
      ),
    );
  }
}

最佳实践：

  1. 始终提供 darkTheme：现代应用应该支持暗色模式，这已经成为用户的基本期望
  2. 默认使用 ThemeMode.system：让应用跟随系统设置，同时提供手动切换选项
  3. 考虑高对比度主题：这是无障碍（Accessibility）的重要组成部分，虽然不是所有应用都需要，但值得关注
  4. 使用相同的 colorSchemeSeed：亮色和暗色主题使用同一个种子颜色，通过 brightness 参数区分，确保品牌一致性
  5. 主题切换动画时长建议 200~400ms：过短会显得突兀，过长会让用户等待

本课小结

本课作为 Material Design 系列课程的开篇，建立了以下核心认知：

  1. Material Design 演进脉络：从 M1（拟物化纸墨隐喻）到 M2（品牌主题化）再到 M3（个性化动态取色），每一代都在统一性与个性化之间寻找更好的平衡点。理解这条演进线索，才能真正理解 M3 的设计决策。

  2. 三大核心支柱：Color（颜色系统）基于种子色算法生成完整的色调调色板和颜色角色；Typography（排版系统）简化为 5×3=15 种语义化的文字样式；Shape（形状系统）标准化为 7 级形状 Token。三大支柱共同构成应用的视觉身份。

  3. Design Tokens：是设计规范与代码实现之间的桥梁，分为参考令牌、系统令牌、组件令牌三个层级。在 Flutter 中通过 ThemeData、ColorScheme、TextTheme 等类直接映射。

  4. Material 3 启用与配置：通过 useMaterial3: true 启用（Flutter 3.16+ 默认），使用 colorSchemeSeed 快速生成主题，MaterialApp 提供了 theme、darkTheme、themeMode、highContrastTheme 等完整的主题管理能力。

  5. M2 与 M3 视觉差异：按钮从小圆角变为胶囊形、卡片从阴影变为 Surface Tint、FAB 从圆形变为圆角矩形、Dialog 圆角从 4dp 变为 28dp、导航栏新增 Pill 指示器——这些视觉变化反映了 M3 "更柔和、更有机"的设计哲学。

  6. MaterialApp 配置全景：掌握了 theme/darkTheme/themeMode 三件套、高对比度主题的无障碍支持、以及主题切换动画的配置方式。
