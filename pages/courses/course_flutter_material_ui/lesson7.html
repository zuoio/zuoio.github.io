课程系列：Flutter Material Design 系统学习

1. 为什么动效规范很重要？

在 Material Design 体系中，动效（Motion）并非锦上添花的装饰，而是设计语言的核心组成部分。一个没有动效规范的 App，就像一本没有段落和标点的书——虽然内容都在，但阅读体验大打折扣。

动效在 UI 中承担三大职责：

  • 引导用户注意力：让用户知道界面发生了什么变化
  • 建立空间关系：帮助用户理解页面之间的层级和导航结构
  • 传达品牌个性：通过缓动曲线和时长传达 App 的气质

2. Material Motion 设计原则

Material Motion 建立在三个核心设计原则之上：

2.1 Informative（信息性）

动效应该帮助用户理解界面的空间关系和层级结构。例如：

  • 一个列表项展开为详情页时，用户能直观看到"详情从列表项中展开"，理解两者的从属关系
  • 页面前进/后退时沿同一轴线滑动，暗示它们处于同一导航层级

2.2 Focused（聚焦性）

动效应该引导用户的注意力到最重要的内容上，避免不必要的视觉干扰。例如：

  • 同一时刻只有一个主要的过渡动画在进行
  • 次要元素以淡入淡出的方式过渡，不与主要动画争夺注意力

2.3 Expressive（表达性）

动效应该传达品牌个性和操作的重要程度。例如：

  • 重要的转场（如打开新页面）使用较长的持续时间和显眼的缓动曲线
  • 微小的状态变化（如按钮按下）使用短暂而细腻的动画

最佳实践：设计动效时，先问自己三个问题：

  1. 这个动效传达了什么信息？（Informative）
  2. 用户的注意力应该被引导到哪里？（Focused）
  3. 这个动效是否与 App 的整体气质一致？（Expressive）

3. animations 包安装与导入

Material Motion 的四大转场模式由 Google 官方提供的 animations 包实现。

3.1 安装

在项目根目录执行：

flutter pub add animations

或者在 pubspec.yaml 中添加依赖：

dependencies:
  flutter:
    sdk: flutter
  animations: ^2.0.11

然后运行 flutter pub get。

import 'package:animations/animations.dart';

4. Material Motion 四大转场模式

4.1 Container Transform（容器变形）

Container Transform 是四大转场中最具表现力的一种。它让一个元素（源容器）平滑变形为另一个元素（目标容器），在视觉上建立了两者之间的强关联。

典型使用场景：

  • 列表项展开为详情页
  • FAB 展开为新页面
  • 卡片展开为全屏内容
  • 搜索框展开为搜索结果页

4.1.1 OpenContainer 核心属性

属性 类型 说明
closedBuilder CloseContainerBuilder 收起状态（源容器）的构建器
openBuilder OpenContainerBuilder 展开状态（目标容器）的构建器
transitionType ContainerTransitionType 过渡类型：fade 或 fadeThroughColor
transitionDuration Duration 过渡持续时间，默认 300ms
closedElevation double 收起状态的阴影高度
openElevation double 展开状态的阴影高度
closedShape ShapeBorder 收起状态的形状
openShape ShapeBorder 展开状态的形状
closedColor Color 收起状态的背景色
openColor Color 展开状态的背景色
onClosed ClosedCallback 从展开状态返回时的回调

        closedElevation: 6,
        closedShape: const RoundedRectangleBorder(
          borderRadius: BorderRadius.all(Radius.circular(16)),
        ),

        closedBuilder: (context, openContainer) {
          return SizedBox(
            width: 56,

    return Padding(
      padding: const EdgeInsets.only(bottom: 8),
      child: OpenContainer<bool>(
        // 过渡类型：fade 表示内容渐变切换
        transitionType: ContainerTransitionType.fade,
        transitionDuration: const Duration(milliseconds: 400),
        // 收起状态的外观
        closedElevation: 1,
        closedShape: RoundedRectangleBorder(
          borderRadius: BorderRadius.circular(12),
        ),
        closedColor: Theme.of(context).colorScheme.surface,
        // 展开状态的外观
        openElevation: 0,
        openColor: Theme.of(context).colorScheme.surface,
        // 收起状态：列表项
        closedBuilder: (context, openContainer) {
          return ListTile(
            leading: CircleAvatar(
              backgroundColor: Colors.primaries[index % Colors.primaries.length],
              child: Text('${index + 1}'),
            ),
            title: Text('项目 ${index + 1}'),
            subtitle: const Text('点击查看详情'),
            trailing: const Icon(Icons.arrow_forward_ios, size: 16),
            onTap: openContainer, // 点击时触发容器变形
          );
        },
        // 展开状态：详情页
        openBuilder: (context, closeContainer) {
          return _DetailPage(
            index: index,
            onBack: () => closeContainer(returnValue: true),
          );
        },
        // 从详情页返回时的回调
        onClosed: (returnValue) {
          if (returnValue == true) {
            ScaffoldMessenger.of(context).showSnackBar(
              SnackBar(content: Text('从项目 ${index + 1} 返回')),
            );
          }
        },
      ),
    );
  }
}

4.2 Shared Axis（共享轴过渡）

Shared Axis 过渡适用于具有空间或导航关系的元素之间的切换。两个页面/元素沿着共享的 X、Y 或 Z 轴进行过渡，暗示它们处于同一导航层级。

典型使用场景：

  • 步骤前进/后退（如注册流程、向导页）
  • Tab 切换
  • 同级页面之间的导航

4.2.1 SharedAxisTransition 核心属性

属性 类型 说明
animation Animation<double> 主动画（控制进入动画）
secondaryAnimation Animation<double> 次级动画（控制退出动画）
transitionType SharedAxisTransitionType 轴方向：horizontal(X)、vertical(Y)、scaled(Z)
child Widget 当前显示的子组件
fillColor Color? 过渡期间的背景填充色

4.2.2 SharedAxisTransitionType 三种轴向

类型 方向 适用场景
horizontal 水平（X 轴） 左右导航、步骤流程
vertical 垂直（Y 轴） 上下导航、垂直列表展开
scaled 缩放（Z 轴） 层级导航、父子页面切换

4.2.3 完整代码示例：步骤向导页

import 'package:flutter/material.dart';
import 'package:animations/animations.dart';

void main() => runApp(const SharedAxisApp());

class SharedAxisApp extends StatelessWidget {
  const SharedAxisApp({super.key});

  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      theme: ThemeData(
        useMaterial3: true,
        colorSchemeSeed: Colors.teal,
      ),
      home: const SharedAxisDemo(),
    );
  }
}

class SharedAxisDemo extends StatefulWidget {
  const SharedAxisDemo({super.key});

  @override
  State<SharedAxisDemo> createState() => _SharedAxisDemoState();
}

class _SharedAxisDemoState extends State<SharedAxisDemo> {
  int _currentStep = 0;
  // 控制前进或后退方向
  bool _isForward = true;
  // 可切换的轴向类型
  SharedAxisTransitionType _transitionType =
      SharedAxisTransitionType.horizontal;

  static const _totalSteps = 4;

  // 每一步的内容
  final List<_StepContent> _steps = const [
    _StepContent(
      title: '第 1 步：基本信息',
      description: '请填写您的姓名和邮箱地址',
      icon: Icons.person_outline,
      color: Colors.blue,
    ),
    _StepContent(
      title: '第 2 步：选择偏好',
      description: '选择您感兴趣的主题类别',
      icon: Icons.category_outlined,
      color: Colors.orange,
    ),
    _StepContent(
      title: '第 3 步：设置密码',
      description: '请设置一个安全的登录密码',
      icon: Icons.lock_outline,
      color: Colors.purple,
    ),
    _StepContent(
      title: '第 4 步：完成注册',
      description: '恭喜！您的账号已创建成功',
      icon: Icons.check_circle_outline,
      color: Colors.green,
    ),
  ];

  void _goToStep(int step) {
    setState(() {
      _isForward = step > _currentStep;
      _currentStep = step;
    });
  }

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(
        title: const Text('Shared Axis 演示'),
        actions: [
          // 切换轴向类型的菜单
          PopupMenuButton<SharedAxisTransitionType>(
            onSelected: (type) => setState(() => _transitionType = type),
            itemBuilder: (context) => [
              const PopupMenuItem(
                value: SharedAxisTransitionType.horizontal,
                child: Text('X 轴（水平）'),
              ),
              const PopupMenuItem(
                value: SharedAxisTransitionType.vertical,
                child: Text('Y 轴（垂直）'),
              ),
              const PopupMenuItem(
                value: SharedAxisTransitionType.scaled,
                child: Text('Z 轴（缩放）'),
              ),
            ],
            icon: const Icon(Icons.swap_horiz),
          ),
        ],
      ),
      body: Column(
        children: [
          // 步骤指示器
          Padding(
            padding: const EdgeInsets.all(16),
            child: Row(
              children: List.generate(_totalSteps, (index) {
                final isActive = index == _currentStep;
                final isCompleted = index < _currentStep;
                return Expanded(
                  child: Row(
                    children: [
                      CircleAvatar(
                        radius: 16,
                        backgroundColor: isActive
                            ? Theme.of(context).colorScheme.primary
                            : isCompleted
                                ? Theme.of(context).colorScheme.primary.withValues(alpha: 0.5)
                                : Theme.of(context).colorScheme.surfaceContainerHighest,
                        child: isCompleted
                            ? const Icon(Icons.check, size: 16, color: Colors.white)
                            : Text(
                                '${index + 1}',
                                style: TextStyle(
                                  color: isActive ? Colors.white : null,
                                  fontWeight: FontWeight.bold,
                                  fontSize: 12,
                                ),
                              ),
                      ),
                      if (index < _totalSteps - 1)
                        Expanded(
                          child: Container(
                            height: 2,
                            color: isCompleted
                                ? Theme.of(context).colorScheme.primary.withValues(alpha: 0.5)
                                : Theme.of(context).colorScheme.outlineVariant,
                          ),
                        ),
                    ],
                  ),
                );
              }),
            ),
          ),
          // 步骤内容区域 —— 使用 PageTransitionSwitcher 驱动 SharedAxisTransition
          Expanded(
            child: PageTransitionSwitcher(
              duration: const Duration(milliseconds: 400),
              reverse: !_isForward, // 后退时反转动画方向
              transitionBuilder: (child, animation, secondaryAnimation) {
                return SharedAxisTransition(
                  animation: animation,
                  secondaryAnimation: secondaryAnimation,
                  transitionType: _transitionType,
                  child: child,
                );
              },
              child: _StepPage(
                key: ValueKey(_currentStep), // Key 变化触发过渡
                step: _steps[_currentStep],
              ),
            ),
          ),
          // 底部导航按钮
          Padding(
            padding: const EdgeInsets.all(24),
            child: Row(
              mainAxisAlignment: MainAxisAlignment.spaceBetween,
              children: [
                // 上一步按钮
                _currentStep > 0
                    ? OutlinedButton.icon(
                        onPressed: () => _goToStep(_currentStep - 1),
                        icon: const Icon(Icons.arrow_back),
                        label: const Text('上一步'),
                      )
                    : const SizedBox(width: 120),
                // 下一步 / 完成按钮
                _currentStep < _totalSteps - 1
                    ? FilledButton.icon(
                        onPressed: () => _goToStep(_currentStep + 1),
                        icon: const Icon(Icons.arrow_forward),
                        label: const Text('下一步'),
                      )
                    : FilledButton.icon(
                        onPressed: () {
                          ScaffoldMessenger.of(context).showSnackBar(
                            const SnackBar(content: Text('注册完成！')),
                          );
                        },
                        icon: const Icon(Icons.check),
                        label: const Text('完成'),
                      ),
              ],
            ),
          ),
        ],
      ),
    );
  }
}

注意事项：

  1. SharedAxisTransition 本身只是过渡动画组件，需要配合 PageTransitionSwitcher 来提供 animation 和 secondaryAnimation
  2. PageTransitionSwitcher 的 reverse 参数控制动画方向，前进时正向播放，后退时反向播放

4.3 Fade Through（淡入穿透）

Fade Through 用于没有强关联关系的两个页面或组件之间的切换。旧元素先淡出并略微缩小，然后新元素淡入并略微放大，形成一个"穿透"效果。

典型使用场景：

  • 底部导航栏切换页面
  • 抽屉导航切换页面
  • 同层级但无顺序关系的内容切换

4.3.1 FadeThroughTransition 核心属性

属性 类型 说明
animation Animation<double> 主动画
secondaryAnimation Animation<double> 次级动画
child Widget 当前显示的子组件
fillColor Color? 过渡期间的背景填充色

4.3.2 完整代码示例：底部导航页面切换

4.4 Fade（淡入淡出 / 缩放淡入淡出）

Fade 是最简单的转场模式，用于元素的出现和消失。在 animations 包中对应的是 FadeScaleTransition，它在淡入淡出的基础上增加了微妙的缩放效果。

典型使用场景：

  • 对话框的弹出/关闭
  • 菜单的出现/消失
  • 浮动按钮的显示/隐藏
  • 提示信息的出现/消失

4.4.1 FadeScaleTransition 核心属性

属性 类型 说明
animation Animation<double> 控制淡入淡出和缩放的动画
child Widget 需要应用过渡效果的子组件

5. Material 3 Easing 曲线

Easing（缓动）曲线决定了动画的加速和减速方式，是动效"手感"的关键。Material 3 定义了一套标准的 Easing 曲线，Flutter 通过 Easing 类提供。

5.1 Easing 曲线一览

曲线 说明 典型用途
Easing.emphasized 默认缓动，先快后慢，有明显的减速感 大多数过渡动画的首选
Easing.emphasizedDecelerate 强调减速进入，元素从快速运动到缓慢停下 元素进入屏幕
Easing.emphasizedAccelerate 强调加速退出，元素从静止加速离开 元素离开屏幕
Easing.standard 标准缓动，比 emphasized 更对称 屏幕内的属性变化（颜色、大小）
Easing.standardDecelerate 标准减速 屏幕内元素的进入
Easing.standardAccelerate 标准加速 屏幕内元素的退出
Easing.legacy 兼容旧版 Material 的缓动曲线 向后兼容
Easing.legacyDecelerate 旧版减速曲线 向后兼容
Easing.legacyAccelerate 旧版加速曲线 向后兼容

5.2 如何选择缓动曲线

动画类型判断流程：

1. 元素是否在屏幕上移动（位移、页面转场）？
   → 是：使用 Easing.emphasized 系列
     - 完整过渡 → Easing.emphasized
     - 仅进入 → Easing.emphasizedDecelerate
     - 仅退出 → Easing.emphasizedAccelerate

2. 元素是否在原地变化属性（颜色、大小、透明度）？
   → 是：使用 Easing.standard 系列
     - 完整变化 → Easing.standard
     - 属性渐入 → Easing.standardDecelerate
     - 属性渐出 → Easing.standardAccelerate

3. 需要兼容 Material 2？
   → 是：使用 Easing.legacy 系列

  final List<_EasingInfo> _easings = [

最佳实践：

  1. 优先使用 Easing.emphasized：它是 Material 3 的默认缓动曲线，适用于大多数场景
  2. 不要使用 Curves.linear：线性动画在 UI 中几乎总是显得机械和不自然
  3. 进入和退出使用不同曲线：进入用 Decelerate（减速），退出用 Accelerate（加速），模拟真实物理运动
  4. 新项目避免 legacy 系列：它们仅为 Material 2 向后兼容而保留

6. Material 3 Duration Tokens（时长标记）

动画时长与缓动曲线同样重要。Material 3 定义了一套标准的持续时间规范：

6.1 时长分类

分类 时长范围 适用场景 示例
短动画 < 100ms 简单的状态变化、微交互 按钮按下反馈、开关切换、图标变化
中动画 200–300ms 大多数过渡和转场 页面切换、卡片展开、菜单弹出
长动画 > 400ms 复杂的全屏转场、引导动画 Container Transform、首次引导页

6.3 选择时长的原则

// 在代码中使用标准化时长
AnimationController(
  vsync: this,
  // 推荐使用 Durations 常量，而非硬编码的 Duration
  duration: Durations.medium2, // 300ms，适合大多数过渡
);

// 根据动画类型选择时长
final fadeController = AnimationController(
  vsync: this,
  duration: Durations.short4, // 200ms，简单淡入淡出
);

final transformController = AnimationController(
  vsync: this,
  duration: Durations.long1, // 450ms，容器变形
);

最佳实践：

  1. 动画不宜过长：用户不应该"等待"动画完成。如果动画让人觉得慢，说明时长过长
  2. 一致性：同类动画使用相同的时长，避免 App 内动效风格混乱
  3. 优先使用 Durations 常量：比硬编码 Duration(milliseconds: 300) 更规范、更易维护
  4. 考虑无障碍：部分用户可能开启了"减少动画"选项，应通过 MediaQuery.disableAnimationsOf(context) 检测并跳过动画

Flutter 提供了以下内置的 PageTransitionsBuilder：

Builder 效果 说明
FadeUpwardsPageTransitionsBuilder 向上淡入 Material 2 默认（Android）
ZoomPageTransitionsBuilder 缩放过渡 Material 3 默认（Android）

OpenUpwardsPageTransitionsBuilder 从底部打开 另一种 Material 风格

7.3 自定义 PageRoute 的 transitionsBuilder

当你需要为特定路由定制转场动画时，可以使用 PageRouteBuilder：

// 自定义路由转场：Shared Axis 风格的页面跳转
Navigator.of(context).push(
  PageRouteBuilder(
    pageBuilder: (context, animation, secondaryAnimation) {
      return const TargetPage();
    },
    transitionsBuilder: (context, animation, secondaryAnimation, child) {
      // 使用 SharedAxisTransition 作为路由转场
      return SharedAxisTransition(
        animation: animation,
        secondaryAnimation: secondaryAnimation,
        transitionType: SharedAxisTransitionType.horizontal,
        child: child,
      );
    },
    transitionDuration: const Duration(milliseconds: 400),
    reverseTransitionDuration: const Duration(milliseconds: 400),
  ),
);

7.4 封装可复用的路由转场

为了避免每次跳转都写一堆样板代码，推荐封装通用的路由方法：

/// 封装 Material Motion 路由转场
class MotionPageRoute {
  /// Shared Axis 转场（水平方向）
  static Route<T> sharedAxisHorizontal<T>({
    required WidgetBuilder builder,
    Duration duration = const Duration(milliseconds: 400),
  }) {
    return PageRouteBuilder(
      pageBuilder: (context, animation, secondaryAnimation) {
        return builder(context);
      },
      transitionsBuilder: (context, animation, secondaryAnimation, child) {
        return SharedAxisTransition(
          animation: animation,
          secondaryAnimation: secondaryAnimation,
          transitionType: SharedAxisTransitionType.horizontal,
          child: child,
        );
      },
      transitionDuration: duration,
      reverseTransitionDuration: duration,
    );
  }

  /// Fade Through 转场
  static Route<T> fadeThrough<T>({
    required WidgetBuilder builder,
    Duration duration = const Duration(milliseconds: 300),
  }) {
    return PageRouteBuilder(
      pageBuilder: (context, animation, secondaryAnimation) {
        return builder(context);
      },
      transitionsBuilder: (context, animation, secondaryAnimation, child) {
        return FadeThroughTransition(
          animation: animation,
          secondaryAnimation: secondaryAnimation,
          child: child,
        );
      },
      transitionDuration: duration,
      reverseTransitionDuration: duration,
    );
  }

  /// Fade Scale 转场
  static Route<T> fadeScale<T>({
    required WidgetBuilder builder,
    Duration duration = const Duration(milliseconds: 300),
  }) {
    return PageRouteBuilder(
      pageBuilder: (context, animation, secondaryAnimation) {
        return builder(context);
      },
      transitionsBuilder: (context, animation, secondaryAnimation, child) {
        return FadeScaleTransition(
          animation: animation,
          child: child,
        );
      },
      transitionDuration: duration,
      reverseTransitionDuration: duration,
    );
  }
}

// 使用方式
Navigator.of(context).push(
  MotionPageRoute.sharedAxisHorizontal(
    builder: (context) => const TargetPage(),
  ),
);

注意事项：

  1. PageTransitionsTheme 只影响 MaterialPageRoute 的默认转场，自定义 PageRouteBuilder 不受其影响
  2. 自定义路由转场时，transitionDuration 和 reverseTransitionDuration 建议设置为相同值
  3. 使用 animations 包的 Transition 组件作为 transitionsBuilder 时，需要正确传递 animation 和 secondaryAnimation

9. 无障碍与动效

9.1 尊重用户的动效偏好

部分用户可能因为前庭障碍等原因对动画敏感，系统提供了"减少动画"的选项。你的 App 应当尊重这个设置：

@override
Widget build(BuildContext context) {
  // 检查用户是否启用了"减少动画"
  final disableAnimations = MediaQuery.disableAnimationsOf(context);

  return AnimatedContainer(
    // 如果用户禁用动画，持续时间设为 0（立即完成）
    duration: disableAnimations ? Duration.zero : Durations.medium2,
    curve: Easing.emphasized,
    color: _isSelected ? Colors.blue : Colors.grey,
    child: child,
  );
}

9.2 封装动效感知的 Duration 工具

/// 根据用户偏好返回实际的动画时长
Duration motionDuration(BuildContext context, Duration duration) {
  return MediaQuery.disableAnimationsOf(context) ? Duration.zero : duration;
}

最佳实践：

  1. 始终检查 disableAnimations 并提供零动画的回退方案
  2. 关键信息不应该只通过动画传达——即使没有动画，用户也应能理解界面变化
  3. 测试时在系统设置中开启"减少动画"，确认 App 在无动画模式下依然可用

本课小结

本课系统讲解了 Material Motion 动效规范，涵盖以下核心知识点：

  1. 设计原则：Material Motion 遵循 Informative（信息性）、Focused（聚焦性）、Expressive（表达性）三大原则，动效是设计语言的一部分而非装饰。

  2. 四大转场模式：

    • Container Transform：源元素变形为目标元素，适用于有从属关系的导航（列表→详情、FAB→新页面）
    • Shared Axis：沿共享轴过渡，适用于有顺序关系的同级导航（步骤、Tab）
    • Fade Through：淡出旧内容、淡入新内容，适用于无强关联的同级切换（底部导航）
    • Fade (FadeScale)：元素的出现和消失，适用于弹出元素（对话框、菜单）

  3. Easing 曲线：Material 3 提供 emphasized、standard、legacy 三套曲线，优先使用 Easing.emphasized 系列。进入用 Decelerate，退出用 Accelerate。

  4. Duration Tokens：短动画 < 100ms、中动画 200–300ms、长动画 > 400ms。优先使用 Durations 常量确保一致性。

  5. 路由转场定制：通过 PageTransitionsTheme 全局配置，或通过 PageRouteBuilder 为特定路由定制转场。

  6. 无障碍：通过 MediaQuery.disableAnimationsOf 尊重用户的"减少动画"偏好。
