第05课 WidgetState 与交互状态管理

课程系列：Flutter Material Design 系统学习
前置条件：已掌握按钮、输入框等交互组件的基本用法和 ButtonStyle
课程目标：深入理解 Material 交互状态体系，熟练使用 WidgetState 与 WidgetStateProperty 实现符合 Material 规范的动态样式

在 Material Design 中，用户与界面的每一次交互都应该得到即时、清晰的视觉反馈——鼠标悬停时颜色微变、按下时阴影加深、禁用时灰度化……这些看似细微的变化，背后有一套完整的交互状态体系在驱动。Flutter 提供了 WidgetState 机制来实现这一切，它是 Material 组件样式定制的核心基石。

本课将从原理出发，带你完整掌握这套机制。

1. Material 交互状态体系：7 种状态

Material Design 规范定义了 7 种交互状态，它们描述了组件在不同交互阶段的视觉表现：

状态 枚举值 含义 典型触发方式
悬停 WidgetState.hovered 鼠标指针悬停在组件上 桌面端鼠标移入
聚焦 WidgetState.focused 组件获得输入焦点 Tab 键导航、点击输入框
按下 WidgetState.pressed 用户正在按压组件 手指/鼠标按住不放
拖动 WidgetState.dragged 组件正在被拖动 拖拽手势进行中
选中 WidgetState.selected 组件处于选中状态 Checkbox 勾选、ToggleButton 激活
禁用 WidgetState.disabled 组件不可交互 onPressed: null
错误 WidgetState.error 组件处于错误状态 表单校验失败

1.1 状态的叠加与优先级

关键要理解的是：这些状态不是互斥的，它们可以同时存在。比如一个按钮可以同时处于 hovered + focused 状态（用户用 Tab 聚焦后鼠标又移了上去），一个输入框可以同时处于 focused + error 状态（获得焦点且校验失败）。

Material 规范建议的状态优先级（当多个状态同时激活时，视觉效果以高优先级为主）：

dragged > pressed > hovered > focused > selected > disabled > error > 默认

2. WidgetState 机制详解

2.1 从 MaterialState 到 WidgetState 的演变

在 Flutter 3.19 之前，这套机制使用的类名为 MaterialState 和 MaterialStateProperty。从 Flutter 3.19 开始，它们被重命名为 WidgetState 和 WidgetStateProperty，旧名称仍可用但标记为 @Deprecated。

重命名的原因是：这套状态机制不仅仅适用于 Material 组件，Cupertino 和自定义组件同样可以复用，因此使用更通用的命名更合理。

// Flutter 3.19 之前
MaterialState.hovered
MaterialStateProperty.resolveWith(...)

// Flutter 3.19+ 推荐写法
WidgetState.hovered
WidgetStateProperty.resolveWith(...)

最佳实践：新代码一律使用 WidgetState / WidgetStateProperty。如果你维护的项目有旧代码使用 MaterialState，可以逐步迁移。两者在功能上完全等价。

2.2 WidgetState 枚举

WidgetState 是一个简单的枚举，定义了上面提到的 7 种状态：

enum WidgetState {
  hovered,
  focused,
  pressed,
  dragged,
  selected,
  disabled,
  error,
}

2.3 Set<WidgetState> —— 状态集合

Flutter 用 Set<WidgetState> 来表示一个组件当前激活的所有状态。这是因为状态可以同时叠加：

// 组件同时处于 hover 和 focused 状态
final Set<WidgetState> currentStates = {
  WidgetState.hovered,
  WidgetState.focused,
};

// 判断某个状态是否激活
bool isHovered = currentStates.contains(WidgetState.hovered); // true
bool isPressed = currentStates.contains(WidgetState.pressed); // false

Flutter 还提供了一个便捷的 WidgetState.values 来获取所有可能的状态值，以及一些 Set 上的便捷方法：

import 'package:flutter/material.dart';

void main() => runApp(const MaterialApp(home: StateSetDemoPage()));

class StateSetDemoPage extends StatelessWidget {
  const StateSetDemoPage({super.key});

  @override
  Widget build(BuildContext context) {
    // 模拟一组当前状态
    final states = <WidgetState>{
      WidgetState.hovered,
      WidgetState.focused,
    };

    return Scaffold(
      appBar: AppBar(title: const Text('Set<WidgetState> 演示')),
      body: Center(
        child: Column(
          mainAxisAlignment: MainAxisAlignment.center,
          children: [
            Text('当前状态集合：$states'),
            const SizedBox(height: 16),
            Text('是否 hovered：${states.contains(WidgetState.hovered)}'),
            Text('是否 pressed：${states.contains(WidgetState.pressed)}'),
            Text('是否 disabled：${states.contains(WidgetState.disabled)}'),
            const SizedBox(height: 16),
            Text('所有 WidgetState 值：${WidgetState.values}'),
          ],
        ),
      ),
    );
  }
}

3. WidgetStateProperty<T> 详解

WidgetStateProperty<T> 是 Material 样式定制的核心接口。它的作用是：根据组件当前的状态集合，动态返回一个对应类型 T 的值。

3.1 核心原理

WidgetStateProperty<T> 本质上是一个接口，只有一个核心方法：

abstract class WidgetStateProperty<T> {
  /// 根据当前状态集合解析出最终值
  T resolve(Set<WidgetState> states);
}

当 Flutter 渲染一个 Material 组件时，框架会收集组件当前的所有状态（如 {hovered, focused}），然后将这个状态集合传给 resolve() 方法，由开发者来决定在该状态组合下返回什么样式值。

3.2 WidgetStateProperty.resolveWith

这是最常用的工厂构造函数，接受一个回调，回调参数是当前状态集合：

WidgetStateProperty.resolveWith<T>((Set<WidgetState> states) {
  // 根据 states 返回不同的值
  return someValue;
})

示例一：按钮背景色随状态变化

import 'package:flutter/material.dart';

void main() => runApp(const MaterialApp(home: ResolveWithDemoPage()));

class ResolveWithDemoPage extends StatelessWidget {
  const ResolveWithDemoPage({super.key});

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(title: const Text('resolveWith 基础示例')),
      body: Center(
        child: ElevatedButton(
          style: ButtonStyle(
            backgroundColor: WidgetStateProperty.resolveWith<Color>(
              (Set<WidgetState> states) {
                if (states.contains(WidgetState.disabled)) {
                  return Colors.grey.shade300; // 禁用：浅灰
                }
                if (states.contains(WidgetState.pressed)) {
                  return Colors.blue.shade800; // 按下：深蓝
                }
                if (states.contains(WidgetState.hovered)) {
                  return Colors.blue.shade600; // 悬停：中蓝
                }
                return Colors.blue; // 默认：蓝色
              },
            ),
            foregroundColor: WidgetStateProperty.resolveWith<Color>(
              (Set<WidgetState> states) {
                if (states.contains(WidgetState.disabled)) {
                  return Colors.grey.shade500;
                }
                return Colors.white;
              },
            ),
          ),
          onPressed: () {},
          child: const Text('状态响应按钮'),
        ),
      ),
    );
  }
}

import 'package:flutter/material.dart';

void main() => runApp(const MaterialApp(home: CombinedStateDemoPage()));

class CombinedStateDemoPage extends StatelessWidget {
  const CombinedStateDemoPage({super.key});

  @override
  Widget build(BuildContext context) {

                if (isFocused) return Colors.teal.shade300;

          ),
          onPressed: () {},

3.3 WidgetStateProperty.all

当你希望在所有状态下都使用相同的值时，使用 WidgetStateProperty.all：

ButtonStyle(
  // 不管什么状态，背景都是蓝色
  backgroundColor: WidgetStateProperty.all(Colors.blue),
  // 不管什么状态，内边距都一样
  padding: WidgetStateProperty.all(
    const EdgeInsets.symmetric(horizontal: 24, vertical: 12),
  ),
)

3.4 WidgetStatePropertyAll —— 简写形式

WidgetStatePropertyAll<T> 是 WidgetStateProperty.all 的简写类形式，效果完全相同：

ButtonStyle(
  // 以下两种写法等价
  backgroundColor: WidgetStateProperty.all(Colors.blue),
  backgroundColor: const WidgetStatePropertyAll(Colors.blue),
  
  // WidgetStatePropertyAll 的优势：可以用 const 修饰
  foregroundColor: const WidgetStatePropertyAll(Colors.white),
  padding: const WidgetStatePropertyAll(
    EdgeInsets.symmetric(horizontal: 24, vertical: 12),
  ),
)

最佳实践：当值是编译期常量时，优先使用 const WidgetStatePropertyAll(...) 而非 WidgetStateProperty.all(...)，可以获得 const 优化带来的性能收益。

3.5 WidgetStateProperty.lerp —— 状态属性插值

WidgetStateProperty.lerp 用于在两个 WidgetStateProperty 之间进行线性插值，常用于动画过渡场景（如主题切换动画）。

import 'package:flutter/material.dart';

void main() => runApp(const MaterialApp(home: LerpDemoPage()));

class LerpDemoPage extends StatefulWidget {
  const LerpDemoPage({super.key});

  @override
  State<LerpDemoPage> createState() => _LerpDemoPageState();
}

class _LerpDemoPageState extends State<LerpDemoPage> {
  double _t = 0.0; // 插值参数，0.0 到 1.0

  // 起始状态属性：蓝色系
  final WidgetStateProperty<Color?> _startColor =
      WidgetStateProperty.resolveWith((states) {
    if (states.contains(WidgetState.pressed)) return Colors.blue.shade800;
    return Colors.blue;
  });

  // 结束状态属性：红色系
  final WidgetStateProperty<Color?> _endColor =
      WidgetStateProperty.resolveWith((states) {
    if (states.contains(WidgetState.pressed)) return Colors.red.shade800;
    return Colors.red;
  });

  @override
  Widget build(BuildContext context) {
    // 在两个 WidgetStateProperty 之间插值
    final interpolated = WidgetStateProperty.lerp<Color?>(
      _startColor,
      _endColor,
      _t,
      Color.lerp, // 颜色的插值函数
    );

    return Scaffold(
      appBar: AppBar(title: const Text('WidgetStateProperty.lerp')),
      body: Center(
        child: Column(
          mainAxisAlignment: MainAxisAlignment.center,
          children: [
            ElevatedButton(
              style: ButtonStyle(
                backgroundColor: interpolated,
                foregroundColor: const WidgetStatePropertyAll(Colors.white),
              ),
              onPressed: () {},
              child: Text('插值 t = ${_t.toStringAsFixed(2)}'),
            ),
            const SizedBox(height: 32),
            Text('拖动滑块改变插值参数 t'),
            Slider(
              value: _t,
              onChanged: (v) => setState(() => _t = v),
            ),
            const SizedBox(height: 8),
            Text(
              't = 0.0 → 蓝色系   |   t = 1.0 → 红色系',
              style: Theme.of(context).textTheme.bodySmall,
            ),
          ],
        ),
      ),
    );
  }
}

4. 状态层（State Layer）

4.1 什么是状态层

状态层是 Material Design 规范中的一个核心视觉概念。简单来说，它是在组件表面叠加的一层半透明颜色，用来向用户传达当前的交互状态。

你可以把它想象为在按钮表面铺了一层"有色玻璃"：

  • 默认状态下这层玻璃完全透明（看不见）
  • 鼠标悬停时这层玻璃变为 8% 透明度的主题色
  • 按下时变为 10% 透明度
  • 拖动时变为 16% 透明度

这种设计的好处是：状态反馈与组件本身的颜色解耦。不管按钮是什么颜色，叠加同样规则的状态层都能产生自然和谐的视觉效果。

4.2 State Layer Opacity 规范值

Material Design 3 规定了不同状态下的标准透明度值：

状态 叠加透明度 说明
默认（enabled） 0%（无叠加） 组件处于静止可用状态
Hovered 8% 鼠标悬停，轻微反馈
Focused 10% 键盘聚焦，比 hover 略明显
Pressed 10% 按压反馈（通常还伴有水波纹）
Dragged 16% 拖动中，最明显的状态层
Disabled 组件整体降低对比度 通常不使用状态层，而是降低组件的整体不透明度

4.3 在 Flutter 中实现状态层

Flutter 中通过 overlayColor 来实现状态层效果：

import 'package:flutter/material.dart';

void main() => runApp(const MaterialApp(home: StateLayerDemoPage()));

class StateLayerDemoPage extends StatelessWidget {
  const StateLayerDemoPage({super.key});

  @override
  Widget build(BuildContext context) {
    final primaryColor = Theme.of(context).colorScheme.primary;

    return Scaffold(
      appBar: AppBar(title: const Text('State Layer 演示')),
      body: Center(
        child: Column(
          mainAxisAlignment: MainAxisAlignment.center,
          children: [
            const Text('符合 Material 规范的状态层按钮：'),
            const SizedBox(height: 16),
            ElevatedButton(
              style: ButtonStyle(
                // 使用 overlayColor 实现状态层
                overlayColor: WidgetStateProperty.resolveWith<Color?>(
                  (Set<WidgetState> states) {
                    if (states.contains(WidgetState.pressed)) {
                      return primaryColor.withOpacity(0.10); // 按下 10%
                    }
                    if (states.contains(WidgetState.hovered)) {
                      return primaryColor.withOpacity(0.08); // 悬停 8%
                    }
                    if (states.contains(WidgetState.focused)) {
                      return primaryColor.withOpacity(0.10); // 聚焦 10%
                    }
                    return null; // 默认无叠加
                  },
                ),
              ),
              onPressed: () {},
              child: const Text('Material 状态层按钮'),
            ),
            const SizedBox(height: 48),
            const Text('对比：自定义高对比度状态层'),
            const SizedBox(height: 16),
            ElevatedButton(
              style: ButtonStyle(
                overlayColor: WidgetStateProperty.resolveWith<Color?>(
                  (Set<WidgetState> states) {
                    // 故意设置很高的透明度以便观察效果
                    if (states.contains(WidgetState.pressed)) {
                      return Colors.black.withOpacity(0.30);
                    }
                    if (states.contains(WidgetState.hovered)) {
                      return Colors.black.withOpacity(0.15);
                    }
                    return null;
                  },
                ),
              ),
              onPressed: () {},
              child: const Text('高对比度状态层按钮'),
            ),
          ],
        ),
      ),
    );
  }
}

注意事项：Material 组件（如 ElevatedButton、FilledButton）已经内置了符合规范的状态层效果。只有在需要自定义状态层行为时才需要手动设置 overlayColor。不要在不理解默认行为的情况下盲目覆盖。

4.4 封装一个状态层工具函数

在实际项目中，你可以封装一个遵循 Material 规范的状态层颜色工具：

import 'package:flutter/material.dart';

/// 根据 Material 规范创建状态层叠加颜色
WidgetStateProperty<Color?> materialStateLayer(Color color) {
  return WidgetStateProperty.resolveWith<Color?>((states) {
    if (states.contains(WidgetState.dragged)) {
      return color.withOpacity(0.16);
    }
    if (states.contains(WidgetState.pressed)) {
      return color.withOpacity(0.10);
    }
    if (states.contains(WidgetState.hovered)) {
      return color.withOpacity(0.08);
    }
    if (states.contains(WidgetState.focused)) {
      return color.withOpacity(0.10);
    }
    return null;
  });
}

// 使用方式
void example() {
  final style = ButtonStyle(
    overlayColor: materialStateLayer(Colors.blue),
  );
}

5. 在 ButtonStyle 中运用状态属性

ButtonStyle 是状态属性使用最密集的地方。它的几乎每个属性都接受 WidgetStateProperty<T> 类型，这意味着每个样式维度都可以根据状态动态变化。

foregroundColor WidgetStateProperty<Color?> 前景色（文字、图标颜色）

shadowColor WidgetStateProperty<Color?> 阴影颜色
surfaceTintColor WidgetStateProperty<Color?> 表面色调
elevation WidgetStateProperty<double?> 阴影高度
padding WidgetStateProperty<EdgeInsetsGeometry?> 内边距
minimumSize WidgetStateProperty<Size?> 最小尺寸

side WidgetStateProperty<BorderSide?> 边框
shape WidgetStateProperty<OutlinedBorder?> 形状
textStyle WidgetStateProperty<TextStyle?> 文字样式

6. 在 InputDecoration 中运用状态属性

输入框的装饰 InputDecoration 同样支持状态属性。它的某些属性（如 labelStyle、border 等）接受 WidgetStateProperty 类型，允许根据输入框的状态（聚焦、错误、禁用等）动态调整样式。

6.1 InputDecoration 中的状态属性字段

属性 类型 说明
labelStyle WidgetStateProperty<TextStyle?>? 标签文字样式
floatingLabelStyle WidgetStateProperty<TextStyle?>? 浮动标签样式
helperStyle WidgetStateProperty<TextStyle?>? 帮助文字样式

suffixStyle WidgetStateProperty<TextStyle?>? 后缀文字样式

注意事项：InputDecoration 中支持 WidgetStateProperty 的属性随 Flutter 版本在不断增加。建议查阅你所使用 Flutter 版本的 API 文档获取最新信息。

7. WidgetStatesController：监听和控制组件状态

7.1 什么是 WidgetStatesController

前面我们学习的都是被动响应状态——组件状态改变后，样式属性自动解析出新值。但有时候你需要主动监听状态变化，或者在外部主动控制组件的状态。

WidgetStatesController 就是为此设计的。它是一个类似于 TextEditingController 的控制器，可以：

  1. 监听状态变化：通过 addListener 在状态改变时收到通知
  2. 读取当前状态：通过 value 属性获取当前的 Set<WidgetState>
  3. 修改组件状态：通过 update 方法主动添加或移除状态

7.2 基础用法：监听按钮状态

import 'package:flutter/material.dart';

void main() => runApp(const MaterialApp(home: StatesControllerDemoPage()));

class StatesControllerDemoPage extends StatefulWidget {
  const StatesControllerDemoPage({super.key});

  @override
  State<StatesControllerDemoPage> createState() =>
      _StatesControllerDemoPageState();
}

class _StatesControllerDemoPageState extends State<StatesControllerDemoPage> {
  // 创建控制器
  final WidgetStatesController _controller = WidgetStatesController();
  String _stateInfo = '尚无状态';

  @override
  void initState() {
    super.initState();
    // 监听状态变化
    _controller.addListener(_onStateChanged);
  }

  void _onStateChanged() {
    setState(() {
      final states = _controller.value;
      if (states.isEmpty) {
        _stateInfo = '默认（无激活状态）';
      } else {
        _stateInfo = states.map((s) => s.name).join(', ');
      }
    });
  }

  @override
  void dispose() {
    _controller.removeListener(_onStateChanged);
    _controller.dispose();
    super.dispose();
  }

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(title: const Text('WidgetStatesController')),
      body: Center(
        child: Column(
          mainAxisAlignment: MainAxisAlignment.center,
          children: [
            Container(
              padding: const EdgeInsets.all(16),
              decoration: BoxDecoration(
                color: Colors.grey.shade100,
                borderRadius: BorderRadius.circular(8),
              ),
              child: Text(
                '当前状态：$_stateInfo',
                style: const TextStyle(fontSize: 16),
              ),
            ),
            const SizedBox(height: 32),
            const Text('试试 hover、按下这个按钮：'),
            const SizedBox(height: 8),
            ElevatedButton(
              statesController: _controller, // 绑定控制器
              onPressed: () {},
              child: const Text('被监听的按钮'),
            ),
          ],
        ),
      ),
    );
  }
}

7.4 主动控制状态

WidgetStatesController 还允许你通过 update 方法主动添加或移除状态：

import 'package:flutter/material.dart';

void main() => runApp(const MaterialApp(home: ManualStatePage()));

class ManualStatePage extends StatefulWidget {
  const ManualStatePage({super.key});

  @override
  State<ManualStatePage> createState() => _ManualStatePageState();
}

class _ManualStatePageState extends State<ManualStatePage> {
  final WidgetStatesController _controller = WidgetStatesController();
  String _currentStates = '无';

  @override
  void initState() {
    super.initState();
    _controller.addListener(() {
      setState(() {
        final states = _controller.value;
        _currentStates = states.isEmpty
            ? '无'
            : states.map((s) => s.name).join(', ');
      });
    });
  }

  @override
  void dispose() {
    _controller.dispose();
    super.dispose();
  }

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(title: const Text('主动控制状态')),
      body: Padding(
        padding: const EdgeInsets.all(24),
        child: Column(
          crossAxisAlignment: CrossAxisAlignment.start,
          children: [
            Text('当前状态：$_currentStates',
                style: const TextStyle(fontSize: 16)),
            const SizedBox(height: 24),
            // 被控制的按钮
            ElevatedButton(
              statesController: _controller,
              style: ButtonStyle(
                backgroundColor: WidgetStateProperty.resolveWith((states) {
                  if (states.contains(WidgetState.selected)) {
                    return Colors.orange;
                  }
                  if (states.contains(WidgetState.error)) {
                    return Colors.red;
                  }
                  return Colors.blue;
                }),
                foregroundColor: const WidgetStatePropertyAll(Colors.white),
              ),
              onPressed: () {},
              child: const Text('受控按钮'),
            ),
            const SizedBox(height: 32),
            const Text('通过下方开关手动切换状态：'),
            const SizedBox(height: 8),
            // 手动切换 selected 状态
            SwitchListTile(
              title: const Text('selected'),
              value: _controller.value.contains(WidgetState.selected),
              onChanged: (value) {
                // update(state, add): add 为 true 则添加该状态，false 则移除
                _controller.update(WidgetState.selected, value);
              },
            ),
            // 手动切换 error 状态
            SwitchListTile(
              title: const Text('error'),
              value: _controller.value.contains(WidgetState.error),
              onChanged: (value) {
                _controller.update(WidgetState.error, value);
              },
            ),
          ],
        ),
      ),
    );
  }
}

最佳实践：WidgetStatesController 的使用场景包括：调试状态表现、实现自定义交互逻辑（如根据外部条件添加 error 状态）、构建组件状态调试面板等。日常开发中，大部分组件的状态由框架自动管理，不需要手动控制。

// 自定义按钮配置

class MaterialStateButtonConfig {
  final Color baseBackgroundColor;
  final Color baseForegroundColor;
  final double baseElevation;

  const MaterialStateButtonConfig({
    required this.baseBackgroundColor,
    required this.baseForegroundColor,
    this.baseElevation = 2,
  });
}

// ============================================================
// 自定义 Material 状态按钮
// ============================================================

class MaterialStateButton extends StatefulWidget {
  final String label;
  final IconData? icon;
  final VoidCallback? onPressed;
  final MaterialStateButtonConfig config;

  const MaterialStateButton({
    super.key,
    required this.label,
    this.icon,
    this.onPressed,
    required this.config,
  });

  @override
  State<MaterialStateButton> createState() => _MaterialStateButtonState();
}

class _MaterialStateButtonState extends State<MaterialStateButton> {
  final WidgetStatesController _controller = WidgetStatesController();
  String _stateLabel = '默认';

  @override
  void initState() {
    super.initState();
    _controller.addListener(_updateStateLabel);
  }

  void _updateStateLabel() {
    setState(() {
      final states = _controller.value;
      if (states.isEmpty) {
        _stateLabel = '默认';
      } else {
        _stateLabel = states.map((s) => s.name).join(' + ');
      }
    });
  }

  @override
  void dispose() {
    _controller.removeListener(_updateStateLabel);
    _controller.dispose();
    super.dispose();
  }

  /// 根据 Material 规范计算状态层叠加后的背景色
  Color _resolveBackgroundColor(Set<WidgetState> states) {
    final base = widget.config.baseBackgroundColor;
    if (states.contains(WidgetState.disabled)) {
      return base.withOpacity(0.12);
    }
    return base;
  }

  /// 前景色
  Color _resolveForegroundColor(Set<WidgetState> states) {
    final base = widget.config.baseForegroundColor;
    if (states.contains(WidgetState.disabled)) {
      return base.withOpacity(0.38);
    }
    return base;
  }

  /// 阴影高度
  double _resolveElevation(Set<WidgetState> states) {
    if (states.contains(WidgetState.disabled)) return 0;
    if (states.contains(WidgetState.pressed)) return 1;
    if (states.contains(WidgetState.hovered)) {
      return widget.config.baseElevation + 4;
    }
    return widget.config.baseElevation;
  }

  /// 边框
  BorderSide _resolveBorderSide(Set<WidgetState> states) {
    if (states.contains(WidgetState.disabled)) {
      return BorderSide(
        color: widget.config.baseBackgroundColor.withOpacity(0.12),
        width: 1,
      );
    }
    if (states.contains(WidgetState.pressed)) {
      return BorderSide(
        color: widget.config.baseBackgroundColor,
        width: 2.5,
      );
    }
    if (states.contains(WidgetState.focused)) {
      return BorderSide(
        color: widget.config.baseBackgroundColor.withOpacity(0.8),
        width: 2,
      );
    }
    if (states.contains(WidgetState.hovered)) {
      return BorderSide(
        color: widget.config.baseBackgroundColor.withOpacity(0.6),
        width: 1.5,
      );
    }
    return BorderSide(
      color: widget.config.baseBackgroundColor.withOpacity(0.3),
      width: 1,
    );
  }

  /// 状态层叠加色（遵循 Material 规范的 opacity）
  Color? _resolveOverlayColor(Set<WidgetState> states) {
    final onColor = widget.config.baseForegroundColor;
    if (states.contains(WidgetState.dragged)) {
      return onColor.withOpacity(0.16);
    }
    if (states.contains(WidgetState.pressed)) {
      return onColor.withOpacity(0.10);
    }
    if (states.contains(WidgetState.hovered)) {
      return onColor.withOpacity(0.08);
    }
    if (states.contains(WidgetState.focused)) {
      return onColor.withOpacity(0.10);
    }
    return null;
  }

  @override
  Widget build(BuildContext context) {
    return Column(
      mainAxisSize: MainAxisSize.min,
      children: [
        ElevatedButton.icon(
          statesController: _controller,
          onPressed: widget.onPressed,
          icon: widget.icon != null ? Icon(widget.icon) : const SizedBox(),
          label: Text(widget.label),
          style: ButtonStyle(
            backgroundColor: WidgetStateProperty.resolveWith(
              _resolveBackgroundColor,
            ),
            foregroundColor: WidgetStateProperty.resolveWith(
              _resolveForegroundColor,
            ),
            elevation: WidgetStateProperty.resolveWith(_resolveElevation),
            side: WidgetStateProperty.resolveWith(_resolveBorderSide),
            overlayColor: WidgetStateProperty.resolveWith(_resolveOverlayColor),
            shape: WidgetStateProperty.resolveWith((states) {
              if (states.contains(WidgetState.pressed)) {
                return RoundedRectangleBorder(
                  borderRadius: BorderRadius.circular(8),
                );
              }
              return RoundedRectangleBorder(
                borderRadius: BorderRadius.circular(12),
              );
            }),
            padding: const WidgetStatePropertyAll(
              EdgeInsets.symmetric(horizontal: 24, vertical: 14),
            ),
            textStyle: const WidgetStatePropertyAll(
              TextStyle(fontSize: 16, fontWeight: FontWeight.w500),
            ),
          ),
        ),
        const SizedBox(height: 8),
        Text(
          '状态：$_stateLabel',
          style: TextStyle(
            fontSize: 12,
            color: Colors.grey.shade600,
            fontFamily: 'monospace',
          ),
        ),
      ],
    );
  }
}

8.2 设计要点解析

  1. 将 resolve 方法提取为独立函数：每个样式维度的状态解析逻辑独立为一个方法（_resolveBackgroundColor、_resolveElevation 等），代码更清晰、易于维护。

  2. 状态层 opacity 严格遵循规范：hover 8%、focus 10%、pressed 10%、dragged 16%，确保视觉反馈符合用户的直觉预期。

  3. disabled 状态的处理：背景色降低到 12% 透明度，前景色降低到 38% 透明度，阴影设为 0——这些都是 Material 规范推荐的 disabled 处理方式。

  4. 使用 WidgetStatesController 展示实时状态：按钮下方实时显示当前的状态组合，帮助理解状态的叠加机制。

  5. 通过 config 对象实现复用：不同颜色主题的按钮只需传入不同的 config，无需重复编写状态解析逻辑。

9. 本课小结

核心知识点回顾

概念 要点
7 种交互状态 hovered、focused、pressed、dragged、selected、disabled、error
WidgetState Flutter 3.19+ 重命名自 MaterialState 的枚举
Set<WidgetState> 用集合表示当前激活的所有状态（可叠加）
WidgetStateProperty.resolveWith 根据状态集合动态返回样式值，最灵活
WidgetStateProperty.all 所有状态返回相同值
WidgetStatePropertyAll .all 的简写类形式，支持 const
WidgetStateProperty.lerp 两个状态属性之间插值，用于动画过渡
State Layer Material 规范的透明色叠加层，hover 8%、focus 10%、pressed 10%、dragged 16%
overlayColor 实现状态层的 ButtonStyle 属性
WidgetStatesController 主动监听/控制组件状态的控制器

关键最佳实践

  1. 新代码使用 WidgetState，不要再用已废弃的 MaterialState
  2. resolveWith 中的判断顺序很重要：通常按 disabled → pressed → hovered → focused → 默认的优先级排列
  3. 善用 const WidgetStatePropertyAll：对于不需要随状态变化的属性，使用 const 版本获取性能优势
  4. 遵循 Material State Layer opacity 规范：不要随意设置叠加透明度，规范值经过视觉设计验证
  5. 不要过度自定义：Material 组件已内置完善的状态响应，只在确实需要偏离默认行为时才手动覆盖
  6. WidgetStatesController 不需要滥用：大部分场景下框架自动管理状态已经足够，控制器主要用于需要外部监听或干预的高级场景

练习 1：基础状态按钮

  • 默认前景色为灰色
  • hover 时前景色变为蓝色
  • pressed 时前景色变为深蓝色
  • disabled 时前景色变为浅灰色

创建一个 TextFormField，要求：

  • 普通状态下标签为灰色
  • 聚焦时标签变为绿色加粗
  • 错误状态时标签变为红色加粗
  • 内容校验：必须是数字，否则显示错误

练习 3：多状态切换卡片

创建一个可选中的卡片组件：

  • 默认状态：白色背景，1 级阴影
  • hover 状态：浅蓝色背景，3 级阴影
  • selected 状态：蓝色边框，背景微蓝
  • 同时 hover + selected：蓝色边框加粗，背景更蓝

提示：你需要使用 WidgetStatesController 并手动管理 selected 状态。

练习 4：状态调试面板

  • 实时显示按钮当前的状态集合
  • 通过勾选 Checkbox 可以主动添加/移除按钮的状态

创建一组按钮（Primary、Secondary、Tertiary、Error），每个按钮：

  • 使用对应的 ColorScheme 颜色
  • 完全遵循 Material State Layer opacity 规范
  • 在按钮下方实时显示当前状态
